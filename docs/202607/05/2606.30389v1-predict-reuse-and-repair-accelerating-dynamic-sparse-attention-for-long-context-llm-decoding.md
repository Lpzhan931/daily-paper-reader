## 摘要
动态稀疏注意力（DSA）通过只关注与每个查询最相关的top-K KV块来加速长上下文LLM解码，但它引入了一个序列化的选择到注意力的依赖，这成为新的延迟瓶颈。我们提出PRR，一种推测-重用-修复运行时，利用DSA选择中的时间局部性来预测可能的块，在选择进行时推测对这些块的注意力，并在已知真实选择集后增量修复遗漏的块。PRR使用轻量级的基于EMA的预测器、一个基于剖析的推测预算，使推测工作不在关键路径上，以及一个基于FlashAttention的修复内核，利用在线softmax统计将遗漏块合并到部分注意力状态中。在长上下文基准测试和代表性DSA方法中，PRR将每个token的解码延迟降低高达40%，同时保持下游任务准确性。Github: https://github.com/Tianyu9748/Incremental_FlashAttention

## Abstract
Dynamic sparse attention (DSA) accelerates long-context LLM decoding by attending to only the top-K KV blocks relevant to each query, but it introduces a serialized selection-to-attention dependency that emerges as a new latency bottleneck. We present PRR, a speculate-reuse-repair runtime that exploits temporal locality in DSA selections to predict likely blocks, speculate the attention over them while selection is in flight, and incrementally repair missed blocks once the true selected set is known. PRR uses a lightweight EMA-based predictor, a profiling-guided speculation budget that keeps speculative work off the critical path, and a FlashAttention-based repair kernel that folds missed blocks into the partial attention state using online-softmax statistics. Across long-context benchmarks and representative DSA methods, PRR reduces per-token decoding latency by up to 40% while preserving downstream task accuracy. Github: https://github.com/Tianyu9748/Incremental_FlashAttention

- **核心问题**：动态稀疏注意力（Dynamic Sparse Attention, DSA）通过每步仅选择top-K关键KV块来加速长上下文LLM解码，但引入了**选择（Selection）→注意力（Attention）的串行依赖**，选择阶段（压缩注意力+top-K选取）必须完成后才能执行稀疏注意力，导致选择阶段成为新的延迟瓶颈。随着上下文长度增长，选择耗时占比从16K时的约4ms升至512K时的8ms，其与注意力的串行路径占总解码延迟的60%~71%。
- **整体含义**：打破选择-注意力依赖是进一步加速DSA解码的关键。论文发现DSA的top-K选择在相邻解码步之间具有强时间局部性（约68%的块被重复选择），且GPU资源利用率低（SM、L2带宽、DRAM带宽均低于40%），因此可以利用空闲资源**推测性**地提前计算注意力，并在真正选择完成后**增量修复**遗漏块，从而减少关键路径上的工作量。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
PRR（Predict, Reuse, and Repair）运行时包含三个协同组件：
1. **预测**：一个轻量级EMA（指数移动平均）预测器，基于历史块重要性分数预测当前步的top-K块集P。
2. **推测与重叠**：在真正的选择阶段执行期间，并行地对P执行推测性注意力和KV搬运，且通过动态推测预算（δ）确保推测工作不延长关键路径。
3. **修复**：一旦真实选择集A已知，仅对遗漏块M = A \ P进行增量注意力修复，利用在线softmax机制将M的贡献合并到已有部分注意力状态。

### 关键技术细节
- **EMA预测器**（公式2-3）：
  - 对每个块i维护两个状态：平滑水平ℓᵢ和趋势估计vᵢ。  
  - 预测分数：cISᵗᵢ = ℓᵗ⁻¹ᵢ + γ·vᵗ⁻¹ᵢ（γ控制趋势外推）。  
  - 更新：vᵗᵢ = β(ISᵗᵢ − ℓᵗ⁻¹ᵢ) + (1−β)vᵗ⁻¹ᵢ；ℓᵗᵢ = α·ISᵗᵢ + (1−α)ℓᵗ⁻¹ᵢ。  
  - 超参数(α,β,γ)在预填充阶段通过网格搜索（80组候选）在重要性分数轨迹上优化，选择使**分数加权命中率**最大的参数。
- **推测预算**：离线剖析不同上下文长度和δ下选择延迟和推测注意力延迟，生成查找表。运行时选择满足“推测注意力延迟 ≤ 选择延迟”的最大δ，确保推测不延长关键路径。

  - 计算量和内存访问仅随|M|线性增长，而非|A|。

### 流程
1. 预填充：生成重要性分数矩阵，执行超参数搜索。
2. 解码每步：预测块集P（|P| = δ·K），并行执行推测注意力；同时运行选择阶段得到真实集A；计算遗漏块M，执行增量修复；最终输出覆盖A的注意力结果，送入FFN。
