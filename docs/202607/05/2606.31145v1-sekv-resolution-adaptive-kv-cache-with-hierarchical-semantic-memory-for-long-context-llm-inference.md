## 摘要
大语言模型越来越多地处理长上下文场景，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，且需在解码过程中全程保留，导致若不加压缩，完全的GPU缓存成本过高。现有KV缓存压缩方法难以在效率与忠实保留上下文之间取得平衡。Token逐出会丢弃信息，而语义分组则在预填充阶段固定压缩决策；两者都无法在生成过程中从压缩片段中恢复出token级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织成熵引导的语义片段，并存储在GPU-CPU内存层次结构中，不丢弃任何信息。每个片段在GPU上保留轻量级摘要向量用于粗略路由，在CPU上保留低秩SVD基用于按需的token级重建。一种训练的缩放机制在解码过程中选择性地扩展与查询相关的片段，从而实现精确检索，而无需在GPU上实例化完整的KV缓存。SeKV实现了自适应的token级重建，同时保持基础LLM完全冻结，仅增加少于0.05%的可训练参数。在四个基准测试中，SeKV相比最强的语义压缩基准平均提升5.9%，同时在128K上下文下相比完整KV缓存减少53.3%的GPU内存。代码开源在https://github.com/AmirAbaskohi/SeKV。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 1. 核心问题与整体含义
**研究动机**：大语言模型（LLM）在长上下文推理（如文档分析、多轮对话、智能体规划）中，KV 缓存随着序列长度线性增长，成为主要内存瓶颈，导致完全 GPU 缓存在实际部署中成本过高。  
**现有方法的局限**：当前的 KV 缓存压缩方法（如 token 逐出、量化、合并、检索等）存在两个根本问题：
- **Token 级逐出**：会永久丢弃信息，不能恢复；
- **语义分组压缩**：在预填充阶段固定压缩决策，无法在生成过程中动态恢复被压缩片段的 token 级细节。  
尤其当早期看似边缘的证据在后续成为关键锚点时，现有方法均无法处理。  
**整体含义**：SeKV 表明，长上下文推理的 KV 缓存不仅需要更强的压缩，更需要一种不同层次的内存组织——按可变分辨率存储上下文，并根据查询相关性动态放大所需细节级别。

## 2. 方法论
### 核心思想
SeKV 将上下文组织成层次化语义内存：将输入上下文分割为语义连贯的“片段”（span），每个片段以两种分辨率同时存储：
- **GPU 端**：轻量级摘要向量（用于粗路由）和锚点 token（保留完整分辨率）
- **CPU 端**：低秩 SVD 基（用于按需 token 级重建）

### 关键技术细节
1. **熵引导语义片段分割**：利用预填充阶段模型产生的 token 惊奇度（surprisal）作为边界信号。惊奇度 $H_t = -\log p(x_t \mid x_{<t})$ 在主题转换、实体引入等语义边界处达到峰值。将 $H_t > \mu + \alpha\sigma$ 的位置作为边界，边界 token 作为锚点保留在 GPU 上，中间 token 形成片段。
2. **双分辨率表示与内存层次**：
   - **摘要向量（GPU）**：对每个片段，计算惊奇度归一化权重 $w_t = H_t / \sum_{k \in S} H_k$，通过可学习的每头每层投影矩阵 $W^{(l,h)} \in \mathbb{R}^{d' \times d_h}$ 得到压缩的键摘要 $\bar{K}_S^{(l,h)} \in \mathbb{R}^{d'}$（$d'=32$）。同时保留粗粒度均值键和值 $\bar{k}_S, \bar{v}_S \in \mathbb{R}^{d_h}$。
   - **低秩 SVD 基（CPU）**：对每个片段的键矩阵 $K_S^{(l,h)}$ 进行截断 SVD（最大秩 $R=32$），学习一个轻量级门控预测器 $g_\phi$ 为每个奇异分量分配软门控 $m_i$，从而实现自适应秩：$\hat{K}_S^{(l,h)} = \sum_{i} m_i \sigma_i u_i v_i^\top$。值矩阵做同样处理。
3. **训练 zoom-in 机制**：解码时，先对查询投影 $q^{(l,h)} = W^{(l,h)} q_t^{(l,h)}$，与片段摘要计算门控 $\tilde{\alpha}_S^{(l,h)} = \text{sig}\left( \frac{q \cdot \bar{K}_S^\top}{\sqrt{d'}} + \log|S| \right)$。若 $\tilde{\alpha}_S^{(l,h)} > \tau^{(l,h)}$（可学习阈值），则触发放大，从 CPU 异步获取 SVD 基并重建 token 级 KV 对。未放大的片段贡献粗均值（带长度校正）。所有缩放决策使用 straight-through estimator 训练。
4. **训练目标**：`L = L_distill + λ1 L_zoom + λ2 L_recon + λ3 L_budget`。蒸馏损失对齐全 KV 教师；zoom 损失监督路由概率与教师注意力覆盖目标之间的二元交叉熵；重建损失监督门控 SVD 精度；预算损失限制展开片段数和保留秩。冻结基座 LLM，仅训练投影矩阵、阈值和门控预测器（约 0.05% 参数）。

## 6. 主要结论与发现
1. **性能领先**：在 10% KV 预算下，SeKV 在所有 20 组骨干-基准组合中击败所有压缩基线，平均提升最强语义基线 SentenceKV **5.9%**。
2. **接近全 KV 性能**：LongBench 上平均差距仅 0.80 分，InfiniteBench 上 1.23 分，检索密集型任务差距稍大但显著缩小。
3. **强模型增益更大**：Qwen2.5-14B 上的提升最大，说明更强的模型更能利用分辨率自适应缓存。
4. **GPU 内存大幅节省**：128K 上下文下比全 KV 缓存减少 **53.3%** 内存（从 74.8GB 降至 34.9GB）。
5. **Zoom-in 稀疏性**：放大决策主要集中在中间和后期层中少数检索头，验证了每头每层阈值的必要性。
6. **GSM8K 多示例推理**：SeKV 即使在需要保持大量演示结构的推理任务中也能有效保存上下文。

## 7. 优点
- **无需修改基座 LLM**：仅增加极少量可训练参数（0.05%），保持冻结，易于部署。
- **无信息永久丢弃**：通过 GPU-CPU 层次结构保存所有上下文，压缩片段可动态恢复。
- **内容自适应分割**：熵引导分割无需外部工具，细化信息密集区域。
- **低秩重建高效**：SVD 基压缩比高，CPU→GPU 传输量小，可异步重叠。
- **训练高效**：仅需少量普通语言文档即可训练路由组件，无需指令数据或合成检索数据。
- **跨模型泛化**：在五个不同规模和家族的 LLM 上均有效。

## 8. 不足与局限
- **依赖熵分割质量**：噪声边界或不准确的惊奇度估计会导致子优化内存分配，尤其处理碎片化文本、表格、代码等非自然语言内容时效果可能下降。
- **额外超参数**：熵阈值 α、锚点数量、最大秩 R、局部窗口 w 等需要验证，跨模型或上下文长度迁移时可能需重新校准。
- **CPU-GPU 带宽瓶颈**：若大量远距离 span 同时被频繁放大，或 CPU→GPU 传输慢，延迟优势可能减弱；极端对抗场景下退化。
- **冻结基座模型**：虽然便于部署，但可能限制模型内部注意力行为对压缩内存结构的充分适应；联合训练路由与更表达能力强的 span 表示是未来方向。
- **实验覆盖有限**：训练只用了 RedPajama 三种子集，未在更多样化领域（如多语言、长表格、代码长上下文）测试；对比方法未包括最新如 DesireKV（2026 年，论文中提及但未纳入基线）。
