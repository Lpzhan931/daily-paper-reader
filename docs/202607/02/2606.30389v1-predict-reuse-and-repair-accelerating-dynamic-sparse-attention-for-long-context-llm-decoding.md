## 摘要
动态稀疏注意力（DSA）通过只关注与每个查询最相关的top-K KV块来加速长上下文LLM解码，但它引入了一个串行的选择到注意力的依赖关系，这成为新的延迟瓶颈。我们提出了PRR，一种推测-复用-修复运行时，利用DSA选择中的时间局部性来预测可能的块，在选择进行时对其进行推测性注意力计算，并在已知真实选择集后增量修复遗漏的块。PRR使用轻量级的基于EMA的预测器、基于性能分析的推测预算（将推测工作排除在关键路径之外），以及基于FlashAttention的修复核，该核使用在线softmax统计将遗漏块折叠到部分注意力状态中。在长上下文基准测试和代表性DSA方法上，PRR将每token解码延迟降低高达40%，同时保持下游任务准确率。Github: https://github.com/Tianyu9748/Incremental_FlashAttention

## Abstract
Dynamic sparse attention (DSA) accelerates long-context LLM decoding by attending to only the top-K KV blocks relevant to each query, but it introduces a serialized selection-to-attention dependency that emerges as a new latency bottleneck. We present PRR, a speculate-reuse-repair runtime that exploits temporal locality in DSA selections to predict likely blocks, speculate the attention over them while selection is in flight, and incrementally repair missed blocks once the true selected set is known. PRR uses a lightweight EMA-based predictor, a profiling-guided speculation budget that keeps speculative work off the critical path, and a FlashAttention-based repair kernel that folds missed blocks into the partial attention state using online-softmax statistics. Across long-context benchmarks and representative DSA methods, PRR reduces per-token decoding latency by up to 40% while preserving downstream task accuracy. Github: https://github.com/Tianyu9748/Incremental_FlashAttention

## 2. 论文提出的方法论
### 2.1 核心思想
- **预测（Predict）**：基于历史重要性分数，使用**指数移动平均（EMA）**预测器估计当前解码步各块的重要性，选出预测的top‑K块集 $P$。
- **推测（Reuse）**：在选择阶段**并行**执行对$P$的注意力计算（推测执行），从而将部分注意力工作从关键路径上移除。
- **修复（Repair）**：选择完成后获取真实top‑K块集 $A$，对于 $A \setminus P$ 中的遗漏块，通过**增量注意力修复**核将其合并到已计算的注意力状态中，保证最终输出与标准DSA完全一致。

### 2.2 关键技术细节
- **EMA预测器**：为每个KV块维护两个状态变量——平滑水平 $\ell_t^i$ 和趋势估计 $v_t^i$。预测公式：$\hat{IS}_t^i = \ell_{t-1}^i + \gamma v_{t-1}^i$，更新公式使用双指数平滑（$\alpha$、$\beta$控制平滑度）。**在线校准**：利用预填充阶段已经产生的块重要性分数轨迹，通过网格搜索（80个候选超参数组合）选择最优的 $(\alpha,\beta,\gamma)$，校准耗时极低（仅增加0.06 ms关键路径延迟）。
- **推测预算**：通过离线性能分析（profiling）获取不同上下文长度下选择阶段和推测注意力的耗时，动态选择最大安全的预算比率 $\delta$，使得推测注意力能在选择完成前结束，不扩展关键路径。
- **增量注意力修复核**：基于FlashAttention的在线softmax递推公式，修改前向核输出注意力结果 $O_{\text{spec}}$、分母 $\ell_{\text{spec}}$ 和运行最大值 $m_{\text{spec}}$。修复核只对遗漏块 $A\setminus P$ 执行递推更新（公式6-8），将其合并到已有注意力状态中，**计算量和显存流量只与遗漏块数成正比**。

### 2.3 算法流程（文字说明）
对于每个解码步的每个Transformer层：
1. 根据历史EMA状态预测当前步各块的重要性分数，选出预测top‑K集 $P$（大小由动态预算控制）。
2. 启动并行操作：
   - 执行选择阶段（压缩注意力+排序）获取真实top‑K集 $A$。
   - 同时，对 $P$ 中的KV块执行推测注意力，得到部分输出 $O_{\text{spec}}$、$\ell_{\text{spec}}$、$m_{\text{spec}}$。
3. 选择完成后，计算遗漏块 $M = A \setminus P$。
4. 调用增量修复核，将 $M$ 中的块递推合并到部分注意力状态中，得到最终输出 $O_{\text{final}}$。
5. 更新EMA状态（使用真实重要性分数）用于下一步预测。
6. 继续执行FFN。
