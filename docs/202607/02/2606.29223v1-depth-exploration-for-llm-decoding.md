## 摘要
自回归LLM解码通过完整层堆栈评估每个生成的标记，尽管许多标记在中间深度就变得可预测。现有的无损深度自适应方法通过选择单个非最终退出深度并利用最终深度模型验证其预测来利用这种冗余。然而，我们的测量表明，这种基于选择的策略留下了大量空间：选择过晚的退出会浪费计算，而选择过早的退出会触发回退并丢弃依赖的草稿。我们提出深度探索解码（DEX），一种无损解码算法，用多个候选深度上的并行探索取代单一深度选择。在每个提交位置，DEX针对最终深度参考验证候选，精确提交最终深度标记，并折叠探索网格以仅保留可重用的分支状态。这种扩展-提交-折叠过程保持了与标准自回归解码的等价性，同时降低了提交每个标记的成本。在早期退出训练和标准LLM上，DEX优于代表性深度选择基线，并在端到端吞吐量上与推测性和分布式解码方法竞争。此外，随着探索深度变得更细，DEX的性能提升，表明并行深度探索提供了一种可扩展的方式来利用LLM解码中未被充分利用的深度维度。

## Abstract
Autoregressive LLM decoding evaluates every generated token through the full layer stack, even though many tokens become predictable at intermediate depths. Existing lossless depth-adaptive methods exploit this redundancy by choosing a single non-final exit depth and verifying its prediction with the final-depth model. However, our measurements show that this selection-based strategy leaves substantial headroom: choosing an exit too late wastes computation, while choosing one too early triggers fallback and discards dependent drafts. We propose Depth Exploration Decoding (DEX), a lossless decoding algorithm that replaces single-depth selection with parallel exploration over multiple candidate depths. At each commit position, DEX validates candidates against the final-depth reference, commits exactly the final-depth token, and collapses the exploration lattice to retain only reusable branch states. This expand--commit--collapse procedure preserves equivalence to standard autoregressive decoding while reducing the cost of committing each token. Across early-exit-trained and standard LLMs, DEX outperforms representative depth-selection baselines and achieves competitive end-to-end throughput against speculative and distributed decoding methods. Moreover, DEX improves as the explored depths become finer, showing that parallel depth exploration provides a scalable way to exploit the underused depth axis of LLM decoding.

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：自回归（AR）LLM 解码时，每个生成的 token 都需要经过完整层堆栈，但研究表明许多 token 在中间层就已变得可预测，即存在深度轴上的冗余。现有无损深度自适应方法（如 LayerSkip、AdaDecode、DEL）采用**单一退出深度选择**策略：只选择一个非最终层作为退出点，用该层候选 token 与最终层输出进行验证。这种策略存在“选择瓶颈”——退出过晚会浪费计算资源，退出过早则触发回退并丢弃依赖的草稿，无法充分利用深度冗余。
- **整体含义**：本文提出 DEX（Depth Exploration Decoding），将单深度选择替换为**并行多深度探索**，通过扩展-提交-折叠（expand-commit-collapse）流程，在保持与标准 AR 解码等价的前提下，更高效地利用深度轴冗余，从而加速 LLM 解码。

### 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：在每个提交位置，DEX 并行探索多个候选深度（使用多个深度探索器），得到多个分支候选。随后用最终层（最后深度）的输出作为参考，验证这些候选：选择**最早**与最终层参考匹配的候选分支，提交最终层 token，并折叠（collapse）状态网格，仅保留与已提交前缀一致的可复用分支状态。
- **关键技术细节**：
  - **EAD（Earliest Available Depth）**：定义 token 在哪个深度首次与其最终层输出稳定的匹配。探索的开销为 ⌈EAD⌉ₓ - EAD，受探索集分辨率 Δ(X) 上界控制。
  - **并行格扩展（Lattice Expansion）**：每个深度探索器处理一个深度阶段，在每个探索回合中，token 可沿深度方向（传递给下一探索器）和位置方向（基于当前 token 作为前缀开始新位置）扩展，形成二元格结构。
  - **提交与折叠**：当锚路径达到最终深度后，用最终深度参考确定提交 token，选择最早匹配分支，折叠（丢弃）不一致的分支状态和 KV 缓存。
  - **深度耦合采样（DECS）**：在同一槽链中，较浅层已提出的 token 在较深层中被屏蔽，避免重复提案，提高命中参考的概率；最终层参考始终使用未屏蔽原始 logits 采样，保持无损。
  - **诱导适配器**：针对标准 LLM（非早期退出训练模型），在非最终候选深度附加小型残差适配器，通过自蒸馏训练使其中间状态与共享 LM head 对齐，使 EAD 提前，从而适用 DEX。
- **无损性保证**：提交 token 始终是最终层 token，因此输出序列与标准 AR 解码完全一致。

### 7. 优点：方法或实验设计上的亮点
- **理论创新**：正式定义 EAD，区分深度选择与深度探索，揭示了选择瓶颈并给出探索开销的上界（Δ(X)）。
- **算法设计**：expand-commit-collapse 流程有效管理并行分支，DECS 减少冗余提案，适应早期退出和标准模型（诱导适配器）。
- **实验全面**：覆盖多种模型类型（早期退出训练、标准）、多种任务（推理、编码、摘要）、与多类别基线对比（深度自适应、推测解码、分布式并行）。
- **可扩展性**：探索粒度越细吞吐越高，提供了利用额外 GPU 的有效方法。
- **无损性**：保持产出与 AR 解码完全一致，不改变模型分布，适用于安全要求高的场景。

### 8. 不足与局限
- **硬件依赖**：当前实现每个深度探索器需要一个独立 GPU，资源需求随探索分辨率线性增长。在单 GPU 场景下无法体现并行加速。
- **扩展开销**：在 8 个探索器时，格扩展批次可达 128 token，计算和通信开销增加，可能限制更大规模的扩展（论文指出尚未验证超过 8 个探索器的行为）。
- **纯深度探索上限**：即使所有 token 在极浅深度就 ready，加速比受限于最浅探索深度 d₀ ≈ L/K，即最多 K 倍加速，存在理论瓶颈。
- **未结合 token 轴推测解码**：DEX 仅利用深度轴，与 token 树推测解码（如 Medusa、EAGLE）互补但未结合，组合后的协同效应待研究。
- **适配器训练开销**：对标准 LLM 需要额外训练轻量适配器（约 1~3% 参数量），虽然如论文所述较小，但仍是额外步骤。
- **实验范围**：论文在最大 70B 模型和 8 GPU 上评估，未涉及更大模型（如 100B+）或更多 GPU 集群，普适性需进一步验证。此外，仅使用 greedy 解码和 batch size 1，未探索采样或更大 batch 的效果。
