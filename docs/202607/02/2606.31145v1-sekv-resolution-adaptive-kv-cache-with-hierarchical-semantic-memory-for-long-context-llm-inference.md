## 摘要
大型语言模型越来越多地处理长上下文，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，并且必须在解码过程中保留，因此如果不进行压缩，完整的GPU缓存将变得极其昂贵。现有的KV缓存压缩方法难以在效率与忠实上下文保留之间取得平衡。令牌驱逐会丢弃信息，而语义分组在预填充阶段固定压缩决策；两者都无法在生成过程中从压缩的片段中恢复令牌级别的细节。作为解决方案，我们提出了SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织成熵引导的语义片段，并存储在GPU-CPU内存层次结构中，不丢弃任何信息。每个片段在GPU上保留一个轻量级摘要向量用于粗略路由，在CPU上保留一个低秩SVD基用于按需令牌级重建。一个经过训练的缩放机制在解码过程中选择性地扩展查询相关的片段，从而在不将完整KV缓存加载到GPU上的情况下实现精确检索。SeKV实现了自适应的令牌级重建，同时保持基础LLM完全冻结，并新增少于0.05%的可训练参数。在四个基准测试中，SeKV相比最强的语义压缩基线平均改进5.9%，同时在128K上下文下与完整KV缓存相比减少了53.3%的GPU内存。代码可在https://github.com/AmirAbaskohi/SeKV上获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 一、核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）在处理长上下文时，Key-Value（KV）缓存成为主要的内存瓶颈。KV缓存大小随序列长度线性增长，且需在整个解码过程中保留，导致完整GPU缓存成本极高。现有压缩方法（如令牌驱逐、语义分组）在预填充阶段固定压缩决策，无法在生成过程中动态恢复令牌级细节，导致信息丢失或检索失败。
- **整体意义**：本文提出SeKV，一种分辨率自适应的语义KV缓存，通过将上下文组织为层次化语义记忆，在GPU-CPU内存层次中存储多分辨率表示，并动态调整缓存细节（根据查询相关性），实现高保真、低内存的长上下文推理。

## 二、方法论：核心思想、关键技术细节

### 核心思想
- **层次化语义记忆**：将输入上下文分割为语义连贯的“跨度”（semantic spans），每个跨度在GPU上保留轻量摘要向量用于粗粒度路由，在CPU上存储低秩SVD基用于按需令牌级重建。通过训练的zoom-in机制在解码时选择性扩展查询相关跨度，无需物化完整KV缓存。
- **关键设计**：
  - **熵引导的跨度分割**：利用预填充阶段令牌的负对数似然（surprisal）作为边界信号，高surprisal位置标记为语义边界并保留为完整分辨率的“锚点令牌”，中间形成语义跨度。公式：\( H_t = -\log p(x_t | x_{<t}) \)，边界条件 \( H_t > \mu + \alpha\sigma \)。
  - **双分辨率表示**：
    - GPU端：每个跨度保留一个低维摘要向量（通过投影和surprisal加权平均得到），以及原始空间的加权平均键值向量，用于粗粒度注意力。
    - CPU端：对每个跨度的键值矩阵进行截断SVD分解，存储左奇异向量\(U\)、奇异值\(\Sigma\)、右奇异向量\(V\)（秩由轻量预测器\(g_\phi\)动态分配门控）。只存储被选中的分量，节省存储。
  - **训练的zoom-in机制**：每层每头学习一个投影矩阵\(W^{(l,h)}\)和阈值\(\tau^{(l,h)}\)。解码时，计算查询与每个跨度摘要的相似度，若超过阈值则标记为“扩展”，触发从CPU异步获取SVD基，重建令牌级KV对。所有扩展跨度和锚点令牌、本地窗口令牌一起参与完整注意力。未扩展跨度使用一个粗粒度条目（加权平均）参与注意力。
  - **损失函数**：蒸馏损失（对齐全KV教师）、缩放监督（二值交叉熵，基于教师注意力覆盖）、重建损失（控制SVD门控）、预算损失（惩罚扩展数量和总秩）。

### 关键技术细节
- 路由维度\(d'=32\)，最大SVD秩\(R_{max}=32\)，每层每头独立阈值。
- 可训练参数仅约4.3M（投影矩阵、阈值、秩预测器），不足基础模型的0.05%。
- 使用直通估计器处理二进制缩放决策。

## 六、主要结论与发现

1. SeKV在所有基准和骨干上均优于所有压缩基线，平均比SentenceKV提升5.9%，同时GPU内存减少53.3%（128K上下文下）。
2. 动态缩放机制（zoom-in）是核心优势：静态压缩保留粗粒度信息，但只有按需恢复令牌级细节才能实现精确检索和多步推理。

5. 在低预算（5%KV）下SeKV依然保持高准确率（86.31% NIAH），而其他方法显著下降。

## 七、优点

- **方法创新**：首次提出层次化语义内存 + 分辨率自适应KV缓存，结合熵引导分割、低秩SVD表示、训练缩放机制。

- **跨模型通用**：在5个不同架构/规模的LLM上均有效，泛化性好。

## 八、不足与局限

- **依赖熵分割质量**：在高度碎片化文档（如代码、密集表格）中，熵边界可能不可靠，导致分割不佳，影响压缩效率。
- **引入额外超参数**：熵阈值α、最大秩R、本地窗口w、预算权重等需针对新模型或长线验证调整。
- **CPU-GPU带宽瓶颈**：在慢速传输或查询频繁激活大量跨度时，延迟可能增加（文中承认但未深入评测极限情况）。
- **训练-推断不匹配**：训练时教师利用全KV，推断时只用压缩缓存，虽然通过蒸馏和缩放监督缓解，但可能无法完全模拟真实解码行为。
- **无法完全适应内部注意力行为**：基础LLM冻结，路由模块只能进行外部缩放，不能改变模型内部注意力权重分配（作者提及可作为未来方向）。
- **实验覆盖有限**：仅使用英文数据（RedPajama训练，英文基准评估），未测试多语言或特殊领域（如医疗、法律）的泛化性。
- **未与其他高效架构（如稀疏注意力）联合评测**：当前对比仅针对KV缓存压缩方法，未与全局稀疏注意力方法（如Sparse Transformers）比较——尽管SeKV可视为一种混合稀疏方法。
