## 摘要
长上下文语言模型推理日益受到存储键值缓存所需的内存带宽和容量的限制，然而现有的压缩方法通常在各层或令牌上应用统一的预算，并在词汇线索和语义状态需要不同保留时降低检索质量。我们引入了DepthWeave-KV，一种令牌自适应缓存压缩方法，该方法使用共享的低秩通道基础对相邻Transformer层之间的键值状态进行分解，同时在注意力行为敏感的令牌处保留轻量级的令牌特定残差。DepthWeave-KV结合跨层残差分解与一个令牌条件深度路由器，为承载指令和检索关键的令牌分配更高的重建秩，并使用注意力输出探针的无校准在线误差跟踪，在生成过程中自适应压缩，无需重新训练基础模型。一个融合的CUDA实现联合执行基础查找、残差反量化和注意力投影，以减少解码时的内存流量。在LongBench、Needle-in-a-Haystack、L-Eval以及长格式问答和摘要基准测试中，DepthWeave-KV在显著降低内存使用的同时实现了接近全缓存的任务质量，在先前压缩缓存的基础上提高了平均得分和检索准确率，同时在64K上下文中实现了8.3倍的KV内存压缩和每秒72.8个令牌的处理速度。

## Abstract
Long-context language model inference is increasingly limited by the memory bandwidth and capacity required to store key-value caches, yet existing compression methods often apply uniform budgets across layers or tokens and degrade retrieval when lexical cues and semantic states require different preservation. We introduce DepthWeave-KV, a token-adaptive cache compression method that factorizes key and value states across neighboring transformer layers using shared low-rank channel bases while retaining lightweight token-specific residuals where attention behavior is sensitive. DepthWeave-KV combines cross-depth residual factorization with a token-conditional depth router that allocates higher reconstruction rank to instruction-bearing and retrieval-critical tokens, and uses calibration-free online error tracking from attention-output probes to adapt compression during generation without retraining the base model. A fused CUDA implementation jointly performs basis lookup, residual dequantization, and attention projection to reduce decode-time memory traffic. Across LongBench, Needle-in-a-Haystack, L-Eval, and long-form QA and summarization benchmarks, DepthWeave-KV achieves near-full-cache task quality with substantially lower memory use, improving average score and retrieval accuracy over prior compressed caches while reaching 8.3x KV memory reduction and 72.8 tokens per second at 64K context.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：长上下文语言模型推理时，Key-Value（KV）缓存成为内存带宽与容量的主要瓶颈，限制了batch size、上下文长度和服务吞吐量。
- **现有方法的不足**：已有压缩方法（token丢弃、合并、量化、低秩注意力、层间缓存共享）通常对所有层或所有token应用统一的压缩预算，忽略了以下问题：
  - 不同层（浅层词汇锚点 vs. 深层语义抽象）所需缓存保真度不同；
  - 不同token（指令分隔符、实体提及、检索关键跨度 vs. 流畅性续写token）需要差异化重建精度；
  - 平均困惑度或短上下文基准无法反映检索和推理的退化。
- **本文目标**：提出一种同时感知深度、token自适应、且能在线跟踪注意力行为误差的压缩机制，在不牺牲检索质量的前提下大幅降低内存占用。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **跨层残差分解**：将相邻Transformer层的KV状态分解为共享低秩通道基 + token特定的轻量级残差。
- **token自适应路由**：根据token的注意力行为、指令边界、重建误差等特征动态分配残差秩。
- **在线误差追踪**：使用注意力输出探针在解码过程中监控压缩引入的误差，并自适应调整压缩比。

### 关键技术细节
- **深度窗口划分**：将L层模型划分为重叠的深度窗口 \(W_m\)（窗口大小w），每个窗口内为每个注意力头维护共享基张量 \(B_{m,h}^K, B_{m,h}^V \in \mathbb{R}^{r_b \times d}\)。
- **状态重建公式**：
  \[
  \hat{X}_{\ell,h,t}^Z = A_{\ell,h,t}^Z B_{m,h}^Z + g_{\ell,h,t}^Z R_{\ell,h,t}^Z, \quad Z \in \{K,V\}
  \]
  其中 \(A\) 为紧凑系数向量（8-bit块量化），\(R\) 为量化残差（4-bit组量化），\(g\) 为可学习残差门控。
- **token条件深度路由器**：为每个token计算四维特征（累积注意力质量、最大注意力尖峰、分隔符/指令边界指示、前一探针区间的注意力输出重建误差），通过一个浅层路由器生成残差秩 \(\rho_{m,h,t}\)，分为三级：0/2（续写token）、4（结构token）、8（检索关键token）。
- **在线注意力输出探针**：每p个解码步，对少量头和最近查询位置用压缩缓存和高保真缓存分别计算注意力输出，测量相对误差 \(e_{\ell,h}\)。若误差超过阈值则降低路由器阈值 \(\tau_m\)，提高该窗口内更多token的残差秩；否则增加 \(\tau_m\) 以减少存储。该机制无需校准数据，无需重训练基座模型。
- **融合CUDA内核**：将基查找、残差反量化、系数乘法、注意力投影融合，避免将重建的KV张量写回全局内存，减少解码时内存流量。
