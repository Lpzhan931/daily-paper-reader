## 摘要
本报告描述了我们针对高效Qwen竞赛的方法，其目标是在资源受限的NVIDIA A10G GPU上实现Qwen3.5-4B的低延迟服务。我们的系统结合了量化目标模型与推测解码。为了恢复精度，我们对目标模型应用了量化感知蒸馏，同时保留原始量化网格。为了加速解码，我们使用两阶段过程训练了一个专门针对量化目标模型的块扩散草稿模型：首先从高精度目标学习，然后适应低精度目标。由于草稿模型在每个推测解码步骤都会被调用，我们进一步通过量化和滑动窗口注意力减少其开销，在提高长上下文解码延迟的同时保持草稿令牌接受率。因此，我们的提交方案在满足所需质量阈值的情况下，实现了相比基线平均6.978倍的加速，总体排名第三。我们希望这些结果为实际的大型语言模型推理提供有用的见解。代码和资源可在https://github.com/nota-github/adaptfm-quant-dflash获取。

## Abstract
This report describes our approach to the Efficient Qwen Competition, where the goal is to enable low-latency serving of Qwen3.5-4B on a resource-constrained NVIDIA A10G GPU. Our system combines a quantized target model with speculative decoding. To recover accuracy, we apply quantization-aware distillation to the target model while retaining the original quantization grid. To speed up decoding, a block-diffusion drafter specialized for the quantized target model is trained using a two-stage procedure: first learning from the high-precision target and then adapting to the low-precision target. Because the drafter is invoked at every speculative decoding step, we further reduce its overhead with quantization and sliding-window attention, preserving draft-token acceptance while improving long-context decoding latency. As a result, our submission achieves a 6.978$\times$ average speedup over the baseline while satisfying the required quality thresholds, ranking 3rd overall. We hope these results provide useful insights for practical LLM inference. The code and resources are available at https://github.com/nota-github/adaptfm-quant-dflash

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 使用 **量化目标模型** 进行推理，同时配合 **推测解码（speculative decoding）** 加速生成过程。
- 对目标模型进行 **量化感知蒸馏（QAD）** 恢复精度，并设计一个 **块扩散草稿模型（DFlash）**，专门适配量化后的目标模型。
- 进一步对草稿模型进行 **量化（PTQ）** 和 **滑动窗口注意力（SWA）**，降低其每次推测步骤的开销。

### 关键技术细节
- **目标模型优化（QAD）**：
  - 从已有的AWQ INT4检查点出发，初始化学生模型（去量化后的权重），冻结对称的每组缩放因子（per-group scales）。
  - 以原高精度BF16模型为教师，通过 **逐令牌前向KL散度** 进行蒸馏：损失函数为 \( \mathbb{E}_x\left[ \sum_t \text{KL}(p_{\text{tea}}(\cdot|x_{<t}) \| p_{\text{stu}}(\cdot|x_{<t}; Q_s(W))) \right] \)。
  - 利用直通估计器（STE）更新去量化权重 \(W\)，通过固定网格的伪量化器 \(Q_s(\cdot)\)，最终将BF16权重四舍五入回原AWQ量化网格，得到新的INT4码本。
  - 推理时以W4A16压缩张量格式提供。
- **推测解码与DFlash草稿模型**：
  - 使用DFlash（Chen et al., 2026a）：一个基于扩散的块草稿模型，利用目标模型的隐藏状态一次性预测多个未来令牌。
  - 从目标模型的多层提取隐藏表示，投影为紧凑的上下文特征，注入草稿模型的KV缓存。
  - 训练目标：预测草稿块中被掩码的令牌（一个锚点令牌后跟掩码未来位置）。
  - **两阶段训练**：
    - Stage 1：使用原始BF16目标模型及其重新生成的220K对话数据，从随机初始化训练草稿模型。
    - Stage 2：使用已应用QAD的INT4目标模型及其重新生成的400K对话数据，微调Stage 1的草稿模型。
- **草稿模型优化（PTQ + SWA）**：
  - 对训练后的草稿模型应用 **GPTQ** 进行INT4量化（组大小为128），校准数据为256个样本（来自Stage 2的训练集）。
  - 应用 **滑动窗口注意力（SWA）**：仅在草稿模型的注意力核上使用对称窗口（如1024或2048 tokens），KV缓存保持完整，不进行缓存淘汰。短序列时回退到全注意力。

### 算法流程（文字说明）
1. 使用AWQ INT4检查点作为目标模型起点。
2. 对目标模型执行QAD（教师为BF16原始模型），保持原量化网格。
3. 冻结QAD后的INT4目标模型。
4. 两阶段训练DFlash草稿模型：
   - Stage 1：用BF16目标模型生成数据，训练随机初始化的草稿模型。
   - Stage 2：用QAD后的INT4目标模型生成数据，微调Stage 1草稿模型。
5. 对训练好的草稿模型进行GPTQ量化（INT4）和SWA配置。
6. 推理阶段：草稿模型每一步生成一组候选令牌，量化目标模型并行验证，加速整体生成。

## 7. 优点
- **创新的两阶段草稿训练**：先学习高精度目标分布，再适应量化目标，这种课程学习策略提高了草稿质量。
- **固定网格量化蒸馏**：在QAD过程中冻结原始AWQ量化网格，避免了重新定义量化参数，简化部署。
- **草稿模型量化+SWA的联合优化**：既减少每次推测步骤的计算（量化），又减轻长上下文注意力开销（SWA），且不显著影响接受长度。
- **完整的消融实验**：逐步累加每个组件，清晰显示各自贡献（尤其是Long场景下SWA的增益）。
- **开源代码和资源**：提供GitHub仓库，便于复现和社区应用。

## 8. 不足与局限
- **实验覆盖不全面**：

- **偏差风险**：
