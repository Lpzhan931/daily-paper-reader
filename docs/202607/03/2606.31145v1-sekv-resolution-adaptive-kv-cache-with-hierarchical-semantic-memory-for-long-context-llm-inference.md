## 摘要
大型语言模型越来越多地处理长上下文，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长且需在整个解码过程中保留，因此若不进行压缩，完全在GPU上缓存的成本高得令人望而却步。现有的KV缓存压缩方法难以在效率与忠实保留上下文之间取得平衡。词元驱逐会丢弃信息，而语义分组在预填充阶段固定压缩决策；两者都无法在生成过程中某个压缩片段变得相关时恢复其中的词元级细节。作为解决方案，我们提出了SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织为熵引导的语义片段，并在GPU-CPU内存层级中存储它们而不丢弃信息。每个片段在GPU上保留一个轻量级摘要向量用于粗略路由，在CPU上保留一个低秩SVD基用于按需恢复词元级别细节。一个经过训练的缩放机制在解码过程中选择性地扩展与查询相关的片段，从而在不将完整KV缓存物化到GPU上的情况下实现精确检索。SeKV实现了自适应词元级重建，同时保持基础LLM完全冻结，并且仅增加不到0.05%的可训练参数。在四个基准测试中，相对于最强的语义压缩基线，SeKV平均提升了5.9%，同时在128K上下文下，相比完整KV缓存减少了53.3%的GPU内存。代码可在https://github.com/AmirAbaskohi/SeKV获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 2. 方法论（SeKV）
- **核心思想**：将输入上下文分割为语义连贯的“跨度”（span），建立 GPU–CPU 层次化记忆：GPU 上保留轻量摘要向量用于粗略路由；CPU 上存储低秩 SVD 基用于按需重建；通过训练的 **zoom-in 机制** 在解码时动态展开与查询相关的跨度，获得词元级精度。
- **关键技术细节**：
  1. **熵引导跨度分割**：利用预填充时的词元惊讶度 $H_t = -\log p(x_t|x_{<t})$，检测语义边界（$H_t > \mu + \alpha\sigma$），边界词元作为“锚点”以完整分辨率留在 GPU 上，中间词元构成跨度。
  2. **双重分辨率表示**：
     - GPU：每个跨度的摘要向量 $\bar{K}^{(l,h)}_S$（投影到 32 维空间），由惊讶度加权平均获得；同时保留原始空间的均值 $\bar{k}^{(l,h)}_S, \bar{v}^{(l,h)}_S$ 用于非展开跨度的粗略贡献。
     - CPU：对每个跨度键/值矩阵进行截断 SVD（最大秩 $R=32$），通过轻量门控预测器 $g_\phi$ 学习每个分量的保留门 $m_i$，动态决定有效秩 $r$。
  3. **训练 zoom-in 机制**：每个注意力头/层学习可独立投影和阈值 $\tau^{(l,h)}$；路由门控 $\tilde{\alpha}^{(l,h)}_S$ 通过 sigmoid 计算（含 $\log|S|$ 大小先验），若超过阈值则触发异步 CPU→GPU 获取 SVD 基并重建。
  4. **损失函数**：蒸馏损失（对齐全 KV 教师）、zoom 监督（基于教师注意力 mass 的二分目标）、重建损失（控制 SVD 门）、预算成本（惩罚展开跨度数和总保留秩），通过直通估计器训练二进制决策。
- **可训练参数**：仅约 4.3M（占基础模型的 0.05%），基础 LLM 冻结。

## 6. 主要结论与发现
- SeKV **在所有 20 个压缩设置中均优于现有方法**，对比最强语义基线 SentenceKV 平均提升 **5.9%**，同时 GPU 内存减少 **53.3%**（128K 上下文）。
- 检索密集型任务（NIAH, RULER）提升最大（+5.68, +3.63），表明自适应 zoom-in 对召回细微证据至关重要。
- 即使在 10% KV 预算下，SeKV 仍接近完整 KV 性能（LongBench 差 0.80 点，InfiniteBench 差 1.23 点）。
- zoom-in 行为 **稀疏且集中在特定头/层**（尤其中间层），验证了逐头逐层阈值设计的合理性。
- 在低预算（5%）下优势更明显（NIAH 比 SentenceKV 高 7.69 点），说明 SeKV 能充分利用少量 GPU 内存。

## 7. 优点
- **分辨率自适应**：首次实现变粒度缓存，在不丢弃信息的前提下动态展开相关跨度。
- **高效性**：仅增加 0.05% 参数，训练轻量（数小时），推理内存近似常增长（8K→128K 仅 3.7GB 增加）。
- **强泛化性**：在 5 种不同骨干模型上一致领先，训练数据无关。
- **检索保真性**：NIAH 热图显示几乎全绿，优于其他方法（出现大面积失败区域）。
- **理论清晰**：熵引导分割、SVD 重建、路由门控等模块均有明确动机和公式支撑。

## 8. 不足与局限
- **依赖分割质量**：熵引导边界可能在高碎片文档（表格、代码、非自然语言）中失效，导致次优内存分配。
- **参数敏感**：熵阈值 $\alpha$、锚点数量、有效秩、预算等需要验证调参，跨新模型或长度范围时可能需重调。
- **带宽瓶颈**：依赖 CPU–GPU 传输，在慢速设备或对抗性查询（反复激活大量跨度）时延迟优势可能减弱。
- **冻结 LLM**：仅训练路由模块可能限制模型完全适应压缩后的记忆结构，未来工作可探索更 expressive 的联合训练。
- **评估覆盖**：未测试 >128K 上下文（如 1M）、多轮对话、非英语文本；GPU 内存比较未包含量化等其他压缩正交技术。
