## 摘要
无状态推理服务器（vLLM、SGLang、TensorRT-LLM）在请求之间空闲，而加速器等待；有状态会话则利用这段空闲时间。推测性预置使用目标模型自身的前向传播（无需草稿模型）将会话向前解码至下一个决策点，从而将跨请求的预填充和入口解码移出关键路径：下一个请求从其差量的预填充入口恢复，或者，当置信门触发时，通过一次接近恒定词汇量的扫描从缓存分布中直接回答，无需解码，代价仅为能源消耗和罕见的有界误接受。其收益取决于模型能力：一个能力强的模型以接近全覆盖和约87%的精度触发置信门（较小的模型则永远无法触发），在约1.0毫秒内返回第一个token，而前缀缓存仍需要39毫秒的解码时间。

## Abstract
A stateless inference server (vLLM, SGLang, TensorRT-LLM) idles between requests while the accelerator waits; a stateful session reclaims that idle time. Speculative pre-positioning decodes the session forward to its next decision point with the target model's own forward pass and no draft model, moving the cross-request prefill and entry-decode off the critical path: the next request resumes from a pre-paid entry on its delta, or, when a confidence gate fires, is answered from a cached distribution in one near-constant vocabulary scan with no decode, at a cost only of energy and a rare, bounded false accept. The payoff is conditional on capability: a capable model fires the gate at near-full coverage and about 87% precision (a smaller one never clears it), returning the first token in about 1.0 ms versus the 39 ms decode a prefix cache still pays.

## 1. 核心问题与整体含义（研究动机和背景）
无状态推理服务器（如 vLLM、SGLang、TensorRT‑LLM）在请求之间完全空闲，加速器处于等待状态，造成计算资源浪费。而**有状态会话**保持 KV 缓存跨请求存活，将等待时间转化为可利用的资源——例如流式数据更新到下一个查询之间的间隔、或在代理场景中工具调用到结果返回之间的空闲窗口。论文提出**推测性预定位（Speculative Pre‑Positioning）**，正是为了利用这些空闲窗口，将下一个请求的解码入口工作（预填充入口）从请求的关键路径上提前到空闲时间内完成，从而大幅降低首令牌延迟。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
在会话状态更新后（例如数据提交或工具调用发出），系统在加速器空闲期间执行一次目标模型前向传播，将会话解码到**决策点**——即下一个请求恢复生成的位置（流式场景：查询前缀 + 助手头；代理场景：工具调用后的信封）。该前向传播的输出分布被缓存为**预备分布（ready distribution）**，下一个请求到达时：
- 若置信门控触发，直接扫描缓存分布返回单个令牌（**快速路径**），无需任何解码；
- 否则走**回退路径**，但入口预填充已经在空闲窗口内完成，只需处理增量部分。

### 关键技术细节
- **决策点定位**：决策点位置 p = n_s + L_entry，其中 n_s 是会话已累积的 token 数，L_entry 是入口 token 长度。
- **预备分布缓存**：不仅缓存 KV 状态，还缓存决策点处的输出分布（或仅缓存 top‑1 token 及其 logit 差），使快速路径能跳过解码。
- **置信门控**：计算 top‑1 与 top‑2 logit 差值 γ = ℓ(1) − ℓ(2)，若 γ > τ 且预测 token 在快速答案集合 A 中，则触发快速路径。τ 通过校准确定。这里引入**选择性预测**框架（El‑Yaniv & Wiener, 2010），覆盖率和误接受率随 τ 单调变化。
- **失效纪律**：任何导致会话状态变更的操作（滑动窗口、回滚、新数据追加）都会原子性地清空缓存标志，确保永远不会服务于过期分布。
- **最佳努力**：若内存预算不足，跳过预定位，不影响正确性和容量。
- **两个算法**：Algorithm 1 描述空闲窗口中的预定位（背景运行）；Algorithm 2 描述请求到达后的置信门控服务。

## 6. 主要结论与发现
- 在 70B 类目标模型上，置信门控在 τ=2.0 时覆盖约 94.5% 的决策点，误接受率约 13.2%（即快速路径与完整路径一致率约 87%）。
- 快速路径首令牌延迟 P50 = 1.01 ms（小样本：128 次触发），P99 = 1.35 ms；而冷路径 P50 = 53.1 ms，P99 = 66.9 ms；快速路径比冷路径快 **50 倍以上**。
- 相比前缀缓存（仍需约 39 ms 解码），本方法去除了解码步骤，实现约 **40 倍**的提速。
- 8B 模型的门控间隙约 0.65，从未超过阈值，因此快速路径对该模型是“无操作” – pre-positioning 只在模型对任务自信时才有收益。
- 命中率随查询‑更新比率 ρ 增大而提高：查询密集型会话几乎消费所有预定位，更新密集型则浪费能量。
- 预定位的主要成本是能量（约 63.8 J/次）和误接受风险，而非延迟。

## 7. 优点
- **关键路径消除**：将入口预填充和解码从请求关键路径移到空闲时间，首令牌延迟极大降低。
- **无需草稿模型**：所有工作均使用目标模型自身，无需额外训练或部署小模型。
- **预备分布缓存创新**：不仅缓存 KV，还缓存输出分布，使得快速路径可零解码回答。
- **置信门控制**：基于选择性预测框架，通过校准 τ 可调风险‑覆盖权衡，且误接受率有界。
- **统一机制**：同时适用于流式会话和代理会话。
- **失效纪律**：原子性清除标志，确保永远不会使用过期分布。
- **最佳努力设计**：内存不足时不预定位，不影响正确性和容量。

## 8. 不足与局限
- **模型能力依赖性**：仅对高能力模型（如 70B）在自信任务上有效；低能力模型（如 8B）门控从不触发，机制退化为普通路径。
- **单一任务验证**：实验仅在一个二值市场趋势任务上进行，缺乏多领域、多任务泛化验证。
- **快速答案集合限制**：仅支持属于集合 A 的特定 token，许多查询需回退。
- **能量开销**：在查询稀疏（低 ρ）的会话中，大量预定位被浪费，能量效率低。
- **缺少公平对比**：未与主流推理系统（如 vLLM、TensorRT‑LLM 的前缀缓存）进行同一硬件下端到端延迟对比，仅给出了自身基线。
