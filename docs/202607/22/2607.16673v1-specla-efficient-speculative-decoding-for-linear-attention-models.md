## 摘要
线性注意力模型用循环状态替代不断增长的KV缓存，但自回归解码仍然每次只读取、更新和写入一个状态。推测解码可以通过一次目标前向验证多个草稿令牌来降低这一成本，然而现有的推测系统是为Transformer KV缓存设计的。对于有状态的线性注意力目标，验证必须遵循跨链和分支的循环依赖关系，接受操作必须仅更新已接受的状态轨迹，且草稿模型必须避免提交浪费有状态验证工作的候选令牌。本文提出SpecLA，一个面向有状态线性注意力模型的推测解码运行时。SpecLA使用拓扑感知的内核验证链和树结构，存储验证过程中产生的紧凑因子以恢复已接受状态，并采用置信度剪枝加上目标对齐的EAGLE风格草稿模型，为验证器提供有用的候选令牌。在NVIDIA H100上使用公共GDN-1.3B目标模型，SpecLA相比自回归解码实现了高达1.70倍的端到端加速。

## Abstract
Linear-attention models replace the growing KV cache with recurrent states, but autoregressive decoding still reads, updates, and writes these states one token at a time. Speculative decoding can reduce this cost by verifying several draft tokens in one target pass, yet existing speculative systems are designed for Transformer KV caches. For stateful linear-attention targets, verification must follow recurrent dependencies across chains and branches, acceptance must update only the accepted state trajectory, and the drafter must avoid submitting candidates that waste stateful verification work. This paper presents SpecLA, a speculative decoding runtime for stateful linear-attention models. SpecLA verifies chains and trees with topology-aware kernels, stores compact factors produced during verification to recover accepted states, and uses confidence pruning plus a target-aligned EAGLE-style drafter to feed useful candidates to the verifier. On an NVIDIA H100 with a public GDN-1.3B target, SpecLA achieves up to 1.70x end-to-end speedup over autoregressive decoding.

## 2. 方法论：核心思想与关键技术  

- **核心思想**：保持外部推测循环（草稿→验证→接受）不变，但将目标侧的KV缓存后缀操作替换为三种目标感知机制。  
- **关键技术细节**：  
  - **拓扑感知验证**：根据候选拓扑选择执行路径。  
    - 链式候选：采用层主序（layer-major）执行，并利用V维度分块使循环状态驻留SRAM，避免HBM往返。  
    - 树形候选：  
      - 全并行路径：将树依赖掩码插入GDN的UT变换，实现一次性并行验证。  
      - 链分解混合路径：用重轻分解（heavy-light decomposition）将树拆分为依赖链，每条链使用序列核串行验证，就绪链之间并行执行。  
  - **接受因子状态管理**：  
    - 因子缓冲：验证时缓存每个候选token的紧凑因子（key，value，衰减系数，Delta规则系数），而非完整状态或token标识。接受后仅对这些因子做轻量Delta构造更新状态，避免全状态快照或token重放。  
    - 延迟状态更新：将接受因子的应用推迟到下一轮验证核内部，与验证融合，消除独立状态更新导致的HBM往返。  
  - **目标对齐草稿集成**：  
    - 置信度剪枝：基于累积路径对数概率进行树剪枝，保留接近最优路径的节点。  
    - EAGLE风格草稿层：使用目标自身的循环状态特征训练草稿层，使剪枝分数反映目标动态。  

- **算法流程（文字说明）**：  
  1. 草稿阶段：从当前前缀出发，草稿模型生成候选token树；通过置信度剪枝保留高概率路径。  
  2. 验证阶段：目标运行时接收候选拓扑，选择适当的验证核（链/全并行/混合），从当前持久状态开始并行计算所有候选token的对数概率和因子。  
  3. 接受阶段：应用后验选择规则确定接受路径；仅收集接受路径上的因子缓冲，拒绝其他候选。  
  4. 状态提交：将接受因子放入待处理缓冲，下一轮验证核开始前自动融合应用，更新持久状态。
