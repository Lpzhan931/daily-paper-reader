## 摘要
多模态大语言模型（MLLMs）处理长视觉标记序列的计算量日益增加，导致整体推理计算开销上升。现有加速方法通常移除视觉标记或跳过整个层中的视觉标记更新，但这些粗粒度策略可能丢弃细粒度证据，或将有用算子与冗余算子一同抑制。本文从答案可观测视角研究视觉标记计算，发现后期视觉标记更新虽然计算量庞大，但对答案标记表示影响甚微。受这种答案静默冗余的启发，我们将每个Transformer层分解为注意力算子和FFN算子，并表明有用的视觉计算通常具有算子主导性和层依赖性。我们提出了一种算子级视觉标记跳跃框架，该框架保留完整的视觉标记序列，同时选择性地绕过冗余的注意力、FFN或两者。在三种MLLM架构和10个VQA基准上的实验表明，我们的方法实现了强效率-准确率权衡，在Qwen3-VL上减少了33.7%的TFLOPs，同时保留了原始模型99.5%的性能。

## Abstract
Multimodal large language models (MLLMs) increasingly process long visual-token sequences, increasing the overall inference computation. Existing acceleration methods usually remove visual tokens or skip visual-token updates in entire layers, but these coarse strategies may discard fine-grained evidence or suppress useful operators together with redundant ones. In this paper, we study visual-token computation from an answer-observable perspective and find that late visual-token updates can remain large while having little effect on answer-token representations. Motivated by this answer-silent redundancy, we decompose each Transformer layer into attention and FFN operators and show that useful visual computation is often operator-dominant and layer-dependent. We propose an operator-level visual-token skipping framework that preserves the full visual-token sequence while selectively bypassing redundant attention, FFN, or both. Experiments across three MLLM architectures and 10 VQA benchmarks show that our method achieves strong efficiency-accuracy trade-offs, reducing \textbf{33.7\%} TFLOPs on Qwen3-VL while retaining \textbf{99.5\%} of the vanilla model performance.

- **核心问题**：能否在保留完整视觉 token 序列的前提下，以更细的粒度识别冗余计算，从而在保持答案质量的同时显著降低推理计算量？
- **整体含义**：本文揭示了视觉 token 的冗余是“答案可观测”（answer-observable）且“算子依赖”（operator-dependent）的，即后期层的视觉更新虽然变化大但对最终答案影响极小，且不同层中注意力（Attention）和前馈网络（FFN）对答案的贡献不同。据此提出算子级视觉跳过策略，实现高效推理。

## 2. 论文提出的方法论
### 2.1 核心思想
- 将每个 Transformer 层分解为 **Attention 算子**和 **FFN 算子**，分别评估它们对最终 prompt token 隐状态的贡献（即“答案可观测影响”）。
- 根据贡献大小，识别每层的“主导算子”和“冗余算子”，然后在推理时**仅保留主导算子**，跳过冗余算子，甚至当两个算子都冗余时直接冻结该层视觉分支。
- 整个视觉 token 序列保持不变，避免证据丢失。

### 2.2 关键技术细节
- **答案可观测诊断**：
  - 定义 **视觉更新幅度（VUM）**：相邻层视觉隐状态变化量的 Frobenius 范数。
  - 定义 **答案可观测影响（AOI）**：视觉更新通过雅可比矩阵传递到最终 prompt token 隐状态的变化量。实际用随机投影估计。
  - 定义 **答案可观测效率（AOE）**：AOI 除以 VUM，衡量每单位视觉更新的实际贡献。
- **算子风险分析**：
  - 在小型校准集上，分别计算跳过视觉 Attention（RA(l)）、跳过视觉 FFN（RF(l)）、冻结整个视觉分支（RAF(l)）对输出分布（相对于原始模型）的 KL 散度。
  - 定义算子主导比 Γ(l) = log(RA(l) / RF(l))：正数表示注意力主导，负数表示 FFN 主导。
  - 当 RA、RF、RAF 均很小时，该层可视为“冻结层”。
- **推理策略分配**：
  - 给定计算预算 B，选择风险总和 RA(l)+RF(l) 最低的 B 层进行简化。
  - 对每层分配策略：**FFN-ONLY**（跳过注意，保留 FFN）、**ATTENTION-ONLY**（跳过 FFN，保留注意）、**FREEZE-UPDATE**（跳过两者）。
  - 文本 token 的计算不受影响。

### 2.3 公式/算法流程（文字说明）
1. 对原始模型进行前向，缓存每层视觉隐状态和最终 prompt token 隐状态。
2. 用随机投影估计每层 AOI，并计算 AOE。
3. 在小型校准集上，通过反向传播计算 RA(l)、RF(l)、RAF(l)。
4. 根据 Γ(l) 和风险值确定每层的主导算子。
5. 按预算选择低风险层，分配对应策略。
6. 推理时按策略跳过指定算子，保持视觉 token 序列完整。

## 6. 论文的主要结论与发现
- **发现 1**：后期层的视觉 token 更新幅度大，但对答案的影响（AOI）和效率（AOE）持续降低，即存在“答案静默冗余”。冻结这些更新仅带来微小性能损失（表 1）。
- **发现 2**：视觉计算在 Transformer 层内呈现算子依赖：不同层的主导算子不同，有的层注意力重要，有的层 FFN 重要，有的两者皆冗余。
- **核心结论**：算子级跳过（保留主导算子、跳过冗余算子）能在几乎不损失性能的前提下大幅降低计算量。在 Qwen3-VL 上减少 33.7% TFLOPs，保留 99.5% 性能。
- **辅助结论**：与 token 剪枝互补，在低 token 保留率下使用算子跳过比进一步剪枝更优。

## 7. 优点
- **粒度细**：首次在算子级别（Attention vs FFN）分析视觉冗余，比层跳过和 token 剪枝更精准。
- **保留视觉序列完整性**：不删除任何 token，避免不可逆证据丢失，尤其有利于需要细粒度空间线索的任务（如 OCR、图表理解）。
- **训练无关**：仅需少量校准样本（无标签）获取风险分布，不修改模型参数，即插即用。
- **泛化性强**：在多种 MLLM 架构（LLaVA、Qwen 系列）和 10 个基准上均有效，且与 token 剪枝等正交方法兼容。
- **可视化分析清晰**：通过 AOI、AOE、算子风险等指标直观揭示冗余本质，具有较强的可解释性。

## 8. 不足与局限
- **校准依赖**：需要一小批校准样本（约 56 张）来计算算子风险，虽然量少但引入了额外的前向-反向开销。对于某些新任务可能需要重新校准。
- **仅覆盖 prefill 阶段**：方法聚焦于 prefill 阶段视觉 token 的计算，未涉及解码阶段的视觉-文本交互加速。
- **未试验更大模型**：实验限于 7B/8B 规模，未在 13B、70B 或更大 MLLM 上验证，其算子冗余模式可能不同。
