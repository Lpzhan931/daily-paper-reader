## 摘要
大型语言模型越来越多地处理长上下文场景，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，且在整个解码过程中必须保留，若不进行压缩，完全在GPU上缓存的成本将高得令人望而却步。现有的KV缓存压缩方法难以平衡效率与对上下文的忠实保留。令牌驱逐会丢弃信息，而语义分组在预填充阶段固定压缩决策；一旦在生成过程中某个压缩片段变得相关，这两者都无法从中恢复令牌级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织成熵引导的语义片段，并跨GPU-CPU内存层次结构存储，不丢弃任何信息。每个片段在GPU上保留一个轻量级摘要向量用于粗粒度路由，在CPU上保留一个低秩SVD基用于按需令牌级重建。一个经过训练的缩放机制在解码过程中选择性扩展与查询相关的片段，实现精确检索而无需在GPU上物化完整的KV缓存。SeKV实现了自适应的令牌级重建，同时保持基础LLM完全冻结，仅增加少于0.05%的可训练参数。在四个基准测试中，SeKV相比最强的语义压缩基线平均提升5.9%，同时在128K上下文中相比完全KV缓存减少53.3%的GPU内存。代码可在 https://github.com/AmirAbaskohi/SeKV 获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 1. 论文的核心问题与整体含义（研究动机和背景）

大型语言模型在处理长上下文（如文档分析、多轮对话、智能体规划）时，KV 缓存（Key-Value cache）的大小随序列长度线性增长，并需在整个解码过程中保留，成为主要内存瓶颈。完全在 GPU 上缓存会导致内存爆炸性增长，难以部署。

现有 KV 缓存压缩方法存在两难：  
- **令牌驱逐**（如 StreamingLLM、H2O）永久丢弃信息，无法恢复被压缩的细节。  
- **语义分组**（如 ChunkKV、SemantiCache、SentenceKV）在预填充阶段固定压缩决策，无法在生成过程中动态恢复令牌级细节，尤其当早期看似外围的片段在后期变为关键证据时。

因此，论文提出：长上下文 KV 缓存需要一种**分辨率自适应的组织方式**，能够按查询相关性动态调整缓存粒度，而非一次性固定压缩。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想：层次化语义记忆（Hierarchical Semantic Memory）
SeKV 将上下文组织为**熵引导的语义片段**，每个片段同时存储两种分辨率表示：
- **GPU 端**：轻量级摘要向量，用于粗粒度路由。
- **CPU 端**：低秩 SVD 基，用于按需令牌级重建。

解码时，一个**训练过的缩放（zoom-in）机制**动态选择查询最相关的片段，异步从 CPU 拉取其 SVD 基并在 GPU 上重建令牌级 KV，从而避免永久性信息丢弃，且不要求 GPU 全量缓存。

### 2.2 关键技术细节

#### （1）熵引导片段分割（Entropy-Guided Span Segmentation）
- 预填充时，计算每个 token 的**惊奇度（surprisal）**：  
  \( H_t = -\log p(x_t \mid x_{<t}) \)  
- 惊奇度高于阈值 \(\mu + \alpha \sigma\) 的位置被视为边界，产生语义片段；高惊奇度的 token 保留为**锚点（anchor tokens）**，始终以全分辨率驻留 GPU。
- 优点：内容自适应、无需外部分词器或额外模型。

#### （2）双分辨率片段表示与存储层级
- **摘要向量（GPU）**：  
  对每个片段，用每头每层的可学习投影矩阵 \(W^{(l,h)} \in \mathbb{R}^{d' \times d_h}\) 将 token 键投影到低维路由空间（\(d'=32\)），再按惊奇度加权平均得到紧凑摘要 \(\bar{K}^{(l,h)}_S\)。同时保留原始空间内的均值键和值（\(\bar{k}_S, \bar{v}_S\)）用于非扩展片段的大致贡献。
- **低秩 SVD 基（CPU）**：  
  对片段内键矩阵 \(K_S^{(l,h)}\) 做截断 SVD，保留最大秩 \(R=32\)，并通过一个轻量级预测器 \(g_\phi\) 学习每成分的软门控 \(m_i\)，从而得到自适应有效秩。门控重建公式：  
  \(\hat{K}_S = \sum_{i=1}^R m_i \sigma_i u_i v_i^\top\)  
  对值矩阵同样处理。存储因子 \(U, \Sigma, V\)，传输量远小于完整 KV 对。

- 短于 \(L_{\min}=16\) 的片段保持全分辨率在 GPU。

#### （3）训练过的缩放机制（Trained Zoom-In Mechanism）
- 每解码步，先基于查询投影 \(q^{(l,h)} \in \mathbb{R}^{d'}\) 与各片段摘要计算路由分数：  
  \(\tilde{\alpha}^{(l,h)}_S = \text{sig}\left( \frac{q \cdot \bar{K}_S^\top}{\sqrt{d'}} + \log|S| \right)\)  
- 与可学习阈值 \(\tau^{(l,h)}\) 比较决定是否扩展（二值决策 \(z^{(l,h)}_S\)）。
- 扩展的片段从 CPU 取回 SVD 基，重建令牌级 KV，进入注意力计算；未扩展的片段用均值键值贡献一个粗粒度 entry。
- 锚点 token 始终参与细粒度注意力。

#### （4）训练目标（仅约 4.3M 参数，占基础模型的 0.05%）
- 蒸馏损失：对齐完整 KV 教师与 SeKV 的注意力输出。
- 缩放监督：基于教师注意力分布的二元交叉熵（固定目标，不依赖阈值）。
- 重建损失：以奇异值加权的门控近似误差，训练秩预测器 \(g_\phi\)。
- 预算损失：惩罚扩展片段数及保留的总秩，平衡精度与开销。

训练使用直通估计器处理二值决策。

## 6. 论文的主要结论与发现

1. SeKV 在所有压缩方法中表现最佳，在 10% KV 预算下平均提升 5.9%，GPU 内存节省 53.3%（128K 上下文）。
2. **动态分辨率恢复**是关键：去除训练缩放或 SVD 重建导致性能大幅下降（NIAH 从 91.17 降至 83.47/85.96）。
3. 熵引导分割优于固定分块，锚点 token 和非均匀加权摘要均贡献提升。
4. 缩放行为稀疏、集中于中间及后期层，支持每头每层独立阈值的必要性。
5. SeKV 在紧预算（5%）下增益最大，随预算增大边际收益递减。

## 7. 优点

- **信息无损的层次存储**：不永久丢弃 token，通过 GPU-CPU 层级和低秩基实现按需重建。
- **极低额外参数**：仅增加 0.05% 参数，基础模型保持冻结，易于部署。
- **内容自适应**：熵引导分割适应文本内在语义结构，无需外部工具。
- **训练高效**：在通用文档语料上训练，无需合成检索或指令数据，3K 步即可收敛。
- **跨模型通用性**：在五种不同规模和家族的模型上验证，增益稳定。

## 8. 不足与局限

- **依赖语义分割质量**：熵估计对噪声或格式异常（如密集表格、代码）敏感，可能导致边界不准确。
