## 摘要
推测解码通过将草稿生成与目标验证解耦，加速了大语言模型（LLM）的推理。虽然最近的并行草稿模型能在单次前向传播中高效生成较长的token序列，但由于缺乏token间的依赖关系，其接受率会迅速衰减。此外，不加区分地验证这些扩展块会浪费关键的批处理容量在具有高拒绝风险的token上，严重降低了高并发服务系统的吞吐量。我们提出了DSpark，一种统一了高吞吐量并行生成与自适应、负载感知验证的推测解码框架。为了保持草稿质量，DSpark采用了半自回归架构，将并行主干与轻量级顺序模块相结合，引入了块内依赖建模并缓解了后缀衰减。为了优化系统效率，DSpark采用了基于置信度调度的验证，根据估计的前缀存活概率和引擎特定的吞吐量配置文件，动态调整每个请求的验证长度。在跨多个领域的离线基准测试中，DSpark相对于最先进的自回归和并行草稿模型显著提高了接受长度。当在实时用户流量下部署于DeepSeek-V4服务系统中时，DSpark成功缓解了验证浪费。与已建立的生产基线（MTP-1）相比，在匹配的吞吐量水平下，DSpark将每个用户的生成速度提升了60%至85%。更重要的是，通过防止在严格的交互性约束下出现严重的吞吐量下降，它实现了以前无法达到的性能层级，移动了我们服务系统的帕累托前沿。

## Abstract
Speculative decoding accelerates Large Language Model (LLM) inference by decoupling draft generation from target verification. While recent parallel drafters efficiently propose long token sequences in a single forward pass, they suffer from rapid acceptance decay due to a lack of inter-token dependencies. Furthermore, indiscriminately verifying these extended blocks wastes critical batch capacity on tokens with high rejection risks, severely degrading throughput in high-concurrency serving systems. We introduce DSpark, a speculative decoding framework that unifies high-throughput parallel generation with adaptive, load-aware verification. To maintain draft quality, DSpark utilizes a semi-autoregressive architecture, coupling a parallel backbone with a lightweight sequential module, to introduce intra-block dependency modeling and mitigate suffix decay. To optimize system efficiency, DSpark employs confidence-scheduled verification, dynamically tailoring the verification length for each request based on estimated prefix survival probabilities and engine-specific throughput profiles. On offline benchmarks across diverse domains, DSpark substantially improves the accepted length over state-of-the-art autoregressive and parallel drafters. When deployed within the DeepSeek-V4 serving system under live user traffic, DSpark successfully mitigates verification waste. Compared to the established production baseline (MTP-1), DSpark accelerates per-user generation speeds by 60 to 85 percent at matched throughput levels. More importantly, by preventing severe throughput degradation under strict interactivity constraints, it enables performance tiers that were previously unattainable, shifting the Pareto frontier of our serving system.

## 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：推测解码（Speculative Decoding）通过轻量草稿模型生成候选token，再由目标模型批量验证来加速LLM推理。然而，现有方法面临两大瓶颈：
  - **生成质量瓶颈**：并行草稿模型（如DFlash）虽然一次前向传播可生成多个候选，但由于缺乏块内token间的依赖关系，后续位置常出现多模态冲突（如“of problem”），导致接受率快速衰减（suffix decay）。
  - **系统效率瓶颈**：不加区分地验证长草稿块会浪费关键批处理容量给高拒绝风险的token，尤其在高并发场景下严重降低吞吐量。传统静态阈值策略无法适应负载变化。
- **整体含义**：需要一种既能保持并行生成高效率又能恢复块内依赖、同时自适应控制验证开销的统一框架，以突破LLM服务系统的性能极限。

## 2. 论文提出的方法论

### 2.1 核心思想
DSpark将草稿生成分为两阶段，并通过置信度调度动态裁剪验证长度，实现“生成更快、更准，验证更聪明”：

- **半自回归生成**：保留并行骨干的高吞吐特性（单次前向传播），在其输出上附加一个轻量级顺序模块，注入局部依赖信息，缓解后缀衰减。
- **置信度调度验证**：训练一个置信度头预测每个位置的条件接受概率，再结合硬件吞吐量配置文件，全局优化各请求的验证长度，将计算资源分配给预期收益最高的token。

### 2.2 关键技术细节

#### 半自回归架构
- **并行骨干**：基于DFlash的块结构，但修改输入（锚点token + \(\gamma-1\)个掩码token），一次前向传播产生\(\gamma\)个草案logits \(U_1,\dots,U_\gamma\)。
- **顺序模块**（两种实例化）：
  - **Markov头**：仅依赖前一个token，使用低秩分解 \(B = W_1W_2\)（\(r=256\)）计算转移偏置，保持轻量。
  - **RNN头**：维护循环状态捕捉更长历史，通过门控单元更新状态，产生偏置。
- 最终草案分布：\(p_k(v|x_0, x_{<k}) = \frac{\exp(U_k(v) + B_k(x_0,x_{<k},v))}{\sum_u \exp(U_k(u) + B_k(\dots))}\)，顺序采样。

#### 置信度头
- 输入：并行骨干隐藏状态 \(h_k\) 与前一个token的马尔可夫嵌入 \(W_1[x_{k-1}]\)，通过线性投影+sigmoid输出 \(c_k \in (0,1)\)。
- 训练目标：监督真实步骤接受率 \(c_k^* = 1 - \frac12 \|p_k^d - p_k^t\|_1\)，使用BCE损失。
- **后处理校准**（STS）：对累积乘积 \(\prod_{i\le k} c_i\) 按位置顺序进行温度缩放，减小期望校准误差（ECE）。

#### 硬件感知前缀调度器
- **问题建模**：最大化全局系统token吞吐量 \(\Theta = \tau \cdot \text{SPS}(B)\)，其中 \(B = R + \sum \ell_r\) 是总验证批大小，\(\tau = R + \sum_{r}\sum_{j\le \ell_r} a_{r,j}\) 是期望接受数，\(a_{r,j} = \prod_{i\le j} c_{r,i}\) 是前缀存活概率。
- **贪心求解**：由于 \(a_{r,j}\) 随 \(j\) 单调非增，按全局降序排列所有候选扩展，依次添加，若当前吞吐量下降则停止（早期停止机制保证无选择偏差）。
- **生产适配**：异步运行（利用两步前的置信度预测决定截断长度），避免GPU流水线停滞，兼容CUDA图和零开销调度（ZOS）。

#### 训练目标
- 三项损失加权：交叉熵 \(\mathcal{L}_{\text{ce}}\)、总变差距离 \(\mathcal{L}_{\text{tv}}\)、置信度BCE \(\mathcal{L}_{\text{conf}}\)。所有损失按指数衰减权重 \(\exp(-(k-1)/\gamma)\) 对各位置加权。
