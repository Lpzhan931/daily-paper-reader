## 摘要
大语言模型越来越多地处理长上下文，其中键值缓存成为主要的内存瓶颈：其大小随序列长度线性增长，并且必须在整个解码过程中保留，这使得在没有压缩的情况下进行完整的GPU缓存成本过高。现有的键值缓存压缩方法难以在效率与忠实上下文保留之间取得平衡。令牌驱逐会丢弃信息，而语义分组在预填充阶段固定压缩决策；两者都无法在生成过程中一旦某个压缩片段变得相关时恢复令牌级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应语义键值缓存，它将上下文组织成熵引导的语义片段，并跨GPU-CPU内存层次结构存储它们，而不丢弃任何信息。每个片段在GPU上保留一个轻量级摘要向量用于粗粒度路由，在CPU上保留一个低秩SVD基用于按需令牌级重建。一个经过训练的放大机制在解码过程中选择性地扩展与查询相关的片段，从而实现精确检索而无需在GPU上物化完整的键值缓存。SeKV在保持基础大语言模型完全冻结且添加少于0.05%可训练参数的情况下，实现了自适应令牌级重建。在四个基准测试中，SeKV相比最强的语义压缩基线平均提升5.9%，同时相比128K上下文下的完整键值缓存，GPU内存减少53.3%。代码可在 https://github.com/AmirAbaskohi/SeKV 获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

# 论文总结：SeKV (Resolution-Adaptive KV Cache with Hierarchical Semantic Memory for Long-Context LLM Inference)

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：长上下文大语言模型推理时，KV Cache（键值缓存）是主要显存瓶颈——其大小随序列长度线性增长，且需在整个解码过程中保留。完整GPU缓存在不压缩的情况下成本过高。
- **现有方法局限**：
  - 令牌驱逐（如StreamingLLM、H2O、SnapKV）永久丢弃信息，无法恢复。
  - 语义分组（如ChunkKV、SemantiCache、SentenceKV）在预填充（prefill）阶段固定压缩决策，无法在生成时动态恢复已压缩片段的令牌级细节。
  - 当初始看似边缘的上下文在后续解码中成为关键证据时，这些方法会失败。
- **论文目标**：提出一种分辨率自适应的语义KV缓存，既能高效压缩，又能在需要时无损地恢复细粒度信息。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：将KV Cache组织成分层语义记忆（Hierarchical Semantic Memory），以多分辨率存储上下文，并根据查询相关性在解码时动态调整缓存细节。
- **关键技术细节**：
  - **(1) 熵引导的跨度分割（Entropy-Guided Span Segmentation）**：
    - 利用预填充阶段令牌的惊奇度（surprisal）作为零成本边界信号：\( H_t = -\log p(x_t \mid x_{<t}) \)。
    - 将 \( H_t > \mu + \alpha\sigma \) 的位置作为边界，形成语义跨度 \( S \)。
    - 高惊奇度的锚点令牌（anchor tokens）以全分辨率保留在GPU上，便于后续路由和重建。
  - **(2) 双分辨率跨度表示与内存层次（Dual-Resolution Span Representation）**：
    - **GPU端**：每个跨度存储一个轻量级摘要向量（summary vector，维度 \( d'=32 \)），用于粗粒度路由；以及保留原始空间的惊奇度加权平均键/值向量（\(\bar{k},\bar{v}\)）作为未扩展跨度的粗粒度贡献。
    - **CPU端**：存储低秩SVD基（low-rank SVD basis），支持按需令牌级重建。通过可学习的门控预测器 \( g_\phi \) 为每个组件分配软门控，实现自适应有效秩（rank budget）。
    - 锚点令牌始终全分辨率在GPU；短于 \( L_{\min} \) 的跨度保持全分辨率在GPU。
  - **(3) 训练的放大机制（Trained Zoom-In Mechanism）**：
    - 每层、每头使用可学习的路由投影 \( W^{(l,h)} \) 和阈值 \( \tau^{(l,h)} \)。
    - 路由分数：\( \tilde{\alpha}_S^{(l,h)} = \text{sig}\left( \frac{\bar{q}^{(l,h)} \cdot \bar{K}_S^{(l,h)\top}}{\sqrt{d'}} + \log|S| \right) \)。
    - 若 \( \tilde{\alpha}_S^{(l,h)} > \tau^{(l,h)} \)，则触发放大：异步从CPU取回SVD基，重建令牌级KV对，参与细粒度注意力。否则仅用粗粒度摘要向量。
    - 软共分割束（budget）：若超出预算，按 \( \tilde{\alpha} \) 排序选前K个。
  - **训练**：基础LLM冻结，仅训练路由投影（~4.19M）、阈值（1K+）和门控预测器（<0.1M），总计约4.3M参数（<0.05%）。目标函数包含蒸馏损失、zoom监督（BCE）、重建损失、预算惩罚。使用直通估计器（straight-through estimator）处理二值决策。

## 6. 主要结论与发现
- SeKV在所有20个模型-基准组合中均取得最佳压缩结果。相比最强基线SentenceKV，在10% KV预算下平均提升**5.9%**。

## 7. 优点
- **分辨率自适应**：不固定压缩决策，可根据查询动态调整缓存细节，避免永久性的信息丢失。
- **熵引导分割**：无需外部工具或额外模型，利用预填充阶段自带的惊奇度信号，内容自适应地划分语义块。
- **轻量训练**：仅增加<0.05%可训练参数，基础LLM冻结，易于部署。
- **内存层次设计**：GPU存摘要用于快速路由，CPU存低秩基用于按需重建，有效平衡速度与精度。
- **管道式异步取回**：放大时与GPU计算重叠，减少延迟。
- **实验全面**：覆盖多种模型、基准、预算和超参数，且消融充分，结果可靠。
- **代码开源**：促进可复现性。

## 8. 不足与局限
- **依赖分割质量**：熵引导分割在高度碎片化文本（如密集表格、代码、格式异常文档）中可能产生次优边界，导致内存分配低效。
- **CPU-GPU带宽瓶颈**：当查询频繁激活大量远处跨度时，异步取回可能受限于传输带宽，削弱延迟优势。
- **多项设计参数需调整**：包括惊奇度阈值α、锚点数量、低秩最大秩R、局部窗口w等，跨模型或任务迁移时可能需重新验证。
- **冻结基础LLM**：虽然有利于部署，但可能限制了模型完全适应压缩记忆结构的潜力；联合训练路由与更表达丰富的跨度表示是未来方向。
- **实验覆盖局限**：未涉及实时对话或持续流式场景；主要基于英文文本，多语言泛化尚未验证。
- **近似重建而非无损**：低秩SVD是近似重建，尽管门控控制误差，但极端情况下可能影响精度。
