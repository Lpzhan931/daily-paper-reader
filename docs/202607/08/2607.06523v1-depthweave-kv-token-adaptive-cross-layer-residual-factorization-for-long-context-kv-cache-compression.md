## 摘要
长上下文语言模型推理日益受限于存储键值缓存所需的内存带宽和容量，然而现有压缩方法通常在各层或各词元上施加统一预算，并在需要不同保留程度的词汇线索和语义状态时降低检索性能。我们提出DepthWeave-KV，一种词元自适应缓存压缩方法，该方法利用共享低秩通道基对相邻Transformer层的键值状态进行分解，同时在对注意力行为敏感的词元上保留轻量级词元特定残差。DepthWeave-KV将跨深度残差分解与词元条件深度路由器相结合，为承载指令和检索关键型词元分配更高的重建秩，并利用注意力输出探针进行免校准在线误差追踪，从而在生成过程中自适应压缩而无需重新训练基础模型。一种融合CUDA实现联合执行基查询、残差反量化和注意力投影，以减少解码时的内存流量。在LongBench、Needle-in-a-Haystack、L-Eval以及长文档问答和摘要基准测试中，DepthWeave-KV在显著降低内存使用的情况下实现了接近全缓存的任务质量，相比先前压缩缓存提高了平均得分和检索准确率，同时在64K上下文下达到8.3倍的KV内存压缩和每秒72.8词元。

## Abstract
Long-context language model inference is increasingly limited by the memory bandwidth and capacity required to store key-value caches, yet existing compression methods often apply uniform budgets across layers or tokens and degrade retrieval when lexical cues and semantic states require different preservation. We introduce DepthWeave-KV, a token-adaptive cache compression method that factorizes key and value states across neighboring transformer layers using shared low-rank channel bases while retaining lightweight token-specific residuals where attention behavior is sensitive. DepthWeave-KV combines cross-depth residual factorization with a token-conditional depth router that allocates higher reconstruction rank to instruction-bearing and retrieval-critical tokens, and uses calibration-free online error tracking from attention-output probes to adapt compression during generation without retraining the base model. A fused CUDA implementation jointly performs basis lookup, residual dequantization, and attention projection to reduce decode-time memory traffic. Across LongBench, Needle-in-a-Haystack, L-Eval, and long-form QA and summarization benchmarks, DepthWeave-KV achieves near-full-cache task quality with substantially lower memory use, improving average score and retrieval accuracy over prior compressed caches while reaching 8.3x KV memory reduction and 72.8 tokens per second at 64K context.

# 深度总结：DepthWeave-KV

## 1. 论文的核心问题与整体含义（研究动机和背景）

长上下文语言模型（Long-context LLM）在推理过程中，key-value（KV）缓存的内存占用成为主要瓶颈，限制了批处理大小、上下文长度和推理吞吐量。现有KV缓存压缩方法通常对**所有层**和**所有词元**施加统一的压缩预算，忽略了不同层（浅层词汇锚点 vs. 深层语义抽象）和不同词元（指令/检索关键词元 vs. 流畅续写词元）对缓存保真度的不同需求。这种均匀压缩策略在**检索型任务**（如Needle-in-a-Haystack、多跳问答）中容易导致严重退化，因为一些看似不重要的词元可能在后续解码中变得关键。因此，需要一种**同时感知深度和词元重要性、并能在线自适应调整**的压缩机制。

## 2. 论文提出的方法论

### 核心思想
DepthWeave-KV 用**跨层低秩共享基 + 词元特定残差**的分解代替独立存储每层的KV缓存，并利用 **词元条件深度路由器** 和 **在线注意力输出误差跟踪** 动态分配残差秩（rank），无需重训基模型。

### 关键技术细节

- **跨深度残差分解**：将相邻层（重叠深度窗口 \(W_m\)）的KV状态分解为：
  - **共享低秩基** \(B_{m,h}^K, B_{m,h}^V \in \mathbb{R}^{r_b \times d}\)（每个注意力头独立，且键/值分开）。
  - **层特定混合系数** \(A_{\ell,h,t}^Z \in \mathbb{R}^{1 \times r_b}\)（8-bit块量化存储）。
  - **词元特定残差** \(R_{\ell,h,t}^Z \in \mathbb{R}^{1 \times d}\)（4-bit群量化，仅在路由器分配的秩下存储）。
  - **残差门控** \(g_{\ell,h,t}^Z \in [0,1]\)。
  
  重建公式：\(\hat{X}_{\ell,h,t}^Z = A_{\ell,h,t}^Z B_{m,h}^Z + g_{\ell,h,t}^Z R_{\ell,h,t}^Z\)（\(Z \in \{K,V\}\)）。这与直接共享缓存（如MiniCache）不同，相邻层共享的是低秩子空间，而非完全相同的缓存向量。

- **词元条件深度路由器**：为每个词元 \(t\) 和窗口 \(W_m\) 计算一个路由分数，依据四个在线特征：累积注意力质量、最大注意力尖峰、分隔符/指令指示、近期注意力输出重建误差。路由输出三个等级的残差秩（\(\rho = 0,2,4,8\)），使得续写词元使用极低秩，结构词元中等，检索关键词元高秩。路由器参数 \(u\) 小而浅，仅需一次前向传播。

- **免校准在线误差跟踪**：每隔 \(p\) 个解码步骤，对少量头与查询位置执行一次双重注意力计算：一次用压缩缓存，一次用临时高保真缓存。计算**注意力输出相对误差** \(e_{\ell,h}\)（见公式3），若超过窗口特定容忍度，则降低路由器阈值 \(\tau_m\)，提升残差秩；若持续低于容忍度，则增加 \(\tau_m\) 以更激进压缩。该反馈直接针对影响下一隐藏状态的量，比直接最小化KV向量误差更可靠。

- **融合CUDA实现**：将基查询、残差反量化、系数乘法与注意力得分计算融合在一个核中，避免将重建的KV张量写回全局内存，显著减少内存流量。

- **唯一需学习的组件**：残差门控和路由器参数，在冻结的教师缓存目标下预先训练一次（使用无标注长上下文文本），部署时无需任务特定校准。

## 7. 优点

- **方法创新性**：将跨层共享基与词元自适应残差结合，避免硬驱逐和均匀压缩，同时保留深度冗余利用和词元级差异化保真度。
- **免校准在线适应**：无需任务特定数据或重训练，利用注意力输出误差实时调整压缩，降低部署门槛。
