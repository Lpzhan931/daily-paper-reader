## 摘要
大型语言模型越来越多地处理长上下文，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，并且必须在解码过程中一直保留，这使得在没有压缩的情况下进行完整的GPU缓存代价高昂。现有的KV缓存压缩方法难以在效率与忠实上下文保留之间取得平衡。令牌驱逐会丢弃信息，而语义分组在预填充阶段固定压缩决策；一旦生成过程中某个压缩片段变得相关，两者都无法恢复令牌级别的细节。作为解决方案，我们提出了SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织成熵引导的语义片段，并跨GPU-CPU内存层次结构存储这些片段而不丢弃信息。每个片段在GPU上保留一个轻量级摘要向量用于粗粒度路由，在CPU上保留一个低秩SVD基用于按需令牌级重建。一个经过训练的缩放机制在解码过程中选择性地扩展与查询相关的片段，从而无需在GPU上物化完整的KV缓存即可实现精确检索。SeKV实现了自适应的令牌级重建，同时保持基础LLM完全冻结，并且添加的可训练参数少于0.05%。在四个基准测试中，与最强的语义压缩基线相比，SeKV平均提高了5.9%，同时在128K上下文中与完整KV缓存相比减少了53.3%的GPU内存。代码可在https://github.com/AmirAbaskohi/SeKV获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型（LLM）在长上下文推理时，KV缓存大小随序列长度线性增长，成为GPU内存的瓶颈。现有压缩方法要么通过令牌逐出永久丢弃信息，要么在预填充阶段固定压缩决策，两者都无法在解码过程中动态恢复被压缩片段的令牌级细节，导致长上下文推理时关键证据丢失或注意力分散。
- **整体含义**：为了在有限GPU内存下高效支持长上下文LLM推理，需要一种**分辨率自适应**的KV缓存组织方式：能够根据查询相关性动态调整缓存细节，避免信息不可逆丢失，同时保持低GPU显存占用。

## 2. 论文提出的方法论

### 核心思想
提出**SeKV（Semantic KV Cache）**，一种分辨率自适应的语义KV缓存，将上下文组织为**分层语义记忆**：通过熵引导的语义分段、GPU/CPU内存层次存储（GPU存轻量摘要，CPU存低秩SVD基）、以及训练好的**缩放机制（zoom-in）**，在解码时只按需展开与查询相关的片段到令牌级分辨率。

### 关键技术细节

1. **熵引导语义分段**  
   利用预填充阶段的令牌惊异度（surprisal） $H_t = -\log p(x_t|x_{<t})$，当 $H_t > \mu + \alpha\sigma$ 时标记为边界。边界令牌作为**锚点令牌**全分辨率保留在GPU上，相邻锚点之间的令牌构成一个语义跨度（span）。

2. **双分辨率跨度表示与内存层次**  
   - **GPU摘要向量**：对每个跨度，用惊异度加权平均投影后的键向量得到紧凑摘要 $\bar{K}_S^{(l,h)} \in \mathbb{R}^{d'}$（$d'=32$），用于粗粒度路由。同时保留原始空间内的加权平均键和值 $\bar{k}_S^{(l,h)}, \bar{v}_S^{(l,h)}$ 用于未展开跨度的粗略注意力贡献。
   - **CPU低秩SVD基**：对每个跨度的键矩阵和值矩阵进行截断SVD，存储左奇异向量、奇异值和右奇异向量。通过轻量预测器 $g_\phi$ 为每个奇异分量分配软门控 $m_{S,i}$，实现每跨度的自适应有效秩 $r_S = \sum_i m_{S,i}$。仅当跨度被选中展开时才将SVD基从CPU异步传输到GPU重建令牌级KV对。

3. **训练好的缩放机制**  
   在每个解码步骤，先通过路由门控计算每个跨度的相关性概率 $\tilde{\alpha}_S^{(l,h)} = \text{sig}\big((\bar{q} \cdot \bar{K}_S)/\sqrt{d'} + \log|S|\big)$，然后与每头每层学习阈值 $\tau^{(l,h)}$ 比较，决定是否展开该跨度。展开后的跨度和未展开跨度（以及局部窗口令牌）一起参与混合分辨率注意力计算。训练损失包含四项：蒸馏损失（对齐全KV教师）、缩放监督（二分类交叉熵）、重建损失（控制SVD近似误差）和预算损失（惩罚展开跨度和保留秩）。

### 公式或算法流程（文字说明）
- **预填充阶段**：计算每令牌惊异度，确定边界和锚点；对每个跨度计算GPU摘要向量；对每个跨度进行SVD分解并存储因子；训练轻量组件（路由投影、阈值、门控预测器）。
- **解码阶段**：①路由阶段：用当前查询投影后与每个GPU摘要计算相关性概率，结合阈值和预算决定展开集；②异步从CPU预取被选中跨度的SVD基；③重建令牌级KV；④合并所有贡献（锚点、局部窗口、未展开跨度单个条目、展开跨度全令牌）进行单次注意力计算。

## 6. 论文的主要结论与发现

- SeKV在所有压缩方法中取得最优结果，平均比最强语义基线SentenceKV高**5.9%**（10%预算下）。

## 7. 优点

- **方法创新性**：首次提出分辨率自适应KV缓存层次结构，结合熵引导分段、GPU/CPU内存层次和训练缩放机制，实现“按需恢复”而非常规“丢弃或固定压缩”。

- **实验完整且公平**：覆盖多种基准、模型、预算，消融充分，效率分析详细。

## 8. 不足与局限

- **依赖熵分割质量**：如果文档具有高碎片化话语、密集表格、代码或异常格式，熵信号可能产生噪声边界，导致次优内存分配。
- **超参数众多**：熵阈值、锚点数量、局部窗口大小、最大SVD秩、预算参数等需调优，跨模型或上下文长度转移时可能需要重新验证。
- **CPU-GPU带宽依赖**：在慢速主机-设备传输或恶意查询频繁激活多个远程跨度时，延迟优势可能下降。
- **冻结基础LLM限制**：路由模块无法引导基础LLM内部注意力适应压缩结构，联合训练路由与更丰富跨度表示可能进一步提升性能。
- **实验覆盖**：未在更大规模模型（如70B+）或生产级部署场景下验证，仅至14B；未评估量化变体下的表现。
