## 摘要
推测解码通过将草稿生成与目标验证解耦，加速了大语言模型（LLM）的推理。虽然最近的并行草稿生成器能够在单次前向传递中高效生成较长的令牌序列，但由于缺乏令牌间依赖关系，它们面临接受率快速衰减的问题。此外，不加区分地验证这些扩展的块会浪费关键批次容量在高拒绝风险的令牌上，严重降低高并发服务系统的吞吐量。我们提出了DSpark，一个统一高吞吐量并行生成与自适应、负载感知验证的推测解码框架。为了保持草稿质量，DSpark采用半自回归架构，将并行主干与轻量级顺序模块相结合，引入块内依赖建模以缓解后缀衰减。为了优化系统效率，DSpark采用置信度调度的验证，基于估计的前缀生存概率和引擎特定的吞吐量特征，动态调整每个请求的验证长度。在跨多个领域的离线基准测试中，DSpark相对于最先进的自回归和并行草稿生成器显著提高了接受长度。当部署在实时用户流量下的DeepSeek-V4服务系统中时，DSpark成功缓解了验证浪费。与已建立的生产基线（MTP-1）相比，DSpark在匹配吞吐量水平下将每用户生成速度提高了60%至85%。更重要的是，通过在严格的交互性约束下防止吞吐量严重下降，它实现了以前无法达到的性能水平，从而改变了我们服务系统的帕累托前沿。

## Abstract
Speculative decoding accelerates Large Language Model (LLM) inference by decoupling draft generation from target verification. While recent parallel drafters efficiently propose long token sequences in a single forward pass, they suffer from rapid acceptance decay due to a lack of inter-token dependencies. Furthermore, indiscriminately verifying these extended blocks wastes critical batch capacity on tokens with high rejection risks, severely degrading throughput in high-concurrency serving systems. We introduce DSpark, a speculative decoding framework that unifies high-throughput parallel generation with adaptive, load-aware verification. To maintain draft quality, DSpark utilizes a semi-autoregressive architecture, coupling a parallel backbone with a lightweight sequential module, to introduce intra-block dependency modeling and mitigate suffix decay. To optimize system efficiency, DSpark employs confidence-scheduled verification, dynamically tailoring the verification length for each request based on estimated prefix survival probabilities and engine-specific throughput profiles. On offline benchmarks across diverse domains, DSpark substantially improves the accepted length over state-of-the-art autoregressive and parallel drafters. When deployed within the DeepSeek-V4 serving system under live user traffic, DSpark successfully mitigates verification waste. Compared to the established production baseline (MTP-1), DSpark accelerates per-user generation speeds by 60 to 85 percent at matched throughput levels. More importantly, by preventing severe throughput degradation under strict interactivity constraints, it enables performance tiers that were previously unattainable, shifting the Pareto frontier of our serving system.

### 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

#### 核心思想
DSpark提出两种互补机制：
1. **半自回归生成**：保留并行骨干（计算密集）进行单次前向生成块，附加轻量级顺序头注入局部转移信息，缓解后缀衰减。
2. **置信度调度验证**：通过置信度头估计每位置前缀存活概率，结合硬件感知调度器动态修剪低置信后缀，减少不必要的验证计算。

#### 关键技术细节
- **半自归架构**：
  - 并行阶段：使用DFlash变体，将锚点令牌作为第一个预测位置，产生γ个草案logits。
  - 顺序阶段（两种实现）：
    - **马尔可夫头**：一阶转移偏置B = W1*W2（低秩分解，r=256），仅依赖前一个令牌。
    - **RNN头**：维护递归状态sk累积块内历史，输入为前状态、前令牌嵌入和骨干隐藏状态，使用门控更新。
  - 公式：每个位置k的草案分布 pk(v|x0,x<k) = exp(Uk(v)+Bk)/sum(exp(...))，采样从左到右。

- **置信度调度验证**：
  - **置信度头**：输出标量ck = σ(w^T[hk; W1[xk-1]])，监督使用理论接受率 ck* = 1 - 0.5*||p_d - p_t||_1。
  - **后处理校准**：顺序温度缩放（STS），最小化累积乘积的期望校准误差（ECE），仅调整绝对值不改变排序。
  - **硬件感知前缀调度器（Algorithm 1）**：
    - 对每个请求计算前缀生存概率 a_r,j = Π_i≤j c_r,i。
    - 全局排序所有候选(a_r,j)，贪心逐轮加入验证，维护当前批次大小B和期望接受τ*，通过profile的SPS(B)（步/秒）计算吞吐Θ = τ* · SPS(B)。
    - 当吞吐下降时早停，保证因果性（无未来信息泄露），实现无损推测解码。
  - **生产适应性**：采用异步调度（使用两步前的置信度近似容量），适应CUDA图重放和零开销调度（ZOS）。

### 论文的主要结论与发现

1. **离线性能**：DSpark在所有目标模型和域上一致优于Eagle3和DFlash（Qwen3-4B/8B/14B宏观平均接受长度相对提升30.9%/26.7%/30.0% 和 16.3%/18.4%/18.3%）。
2. **位置分析**：并行草案器初始接受率更高（深层骨干），但后缀衰减快；DSpark半自归有效结合两者长处，保持稳定高接受率。
3. **轻量顺序头高效**：2层DSpark超越5层DFlash；马尔可夫头几乎与RNN头相当，但实现简单。
4. **置信度调度有效性**：静态阈值扫描显示置信度头可有效修剪低置信令牌；校准后ECE降至~1%。
5. **在线部署**：DSpark vs MTP-1在DeepSeek-V4-Flash/Pro下：
   - 同等吞吐下用户生成速度提升60-85%（Flash 60-85%，Pro 57-78%）。
   - 在严格SLA（如120 TPS Flash）下使吞吐提升406-661%（名义值），实质扩展了可行交互性前沿。
   - 动态调度在轻负载时分配更长的验证预算（4-6令牌），重负载时自动收缩。

### 优点

1. **算法创新**：半自归生成巧妙结合并行效率与自回归质量，轻量顺序头几乎无额外延迟。
2. **系统感知**：将验证长度选择形式化为全局吞吐最大化问题，硬件感知调度器实时适应负载。

4. **广泛验证**：离线+在线、多模型、多领域、消融全面，开源模型和代码库DeepSpec促进复现。
5. **实际影响**：在DeepSeek-V4生产中上线并提升效率，论文展示了从算法到系统部署的完整链路。

### 不足与局限

1. **固定草案计算开销**：即使调度器减少验证，草案生成仍需要整个并行骨干前向（5个MoE层）。对于低接受率请求，草案计算不可回收。
2. **资源细节缺失**：训练和推理的具体GPU型号、数量、时间未报道，影响可重复性。
3. **非理想SPS曲线处理**：调度器原本假设平滑下降，生产中使用异步近似和历史预测，可能存在次优性。
4. **评估偏差**：离线评估关闭调度器，块长固定；在线部署只与MTP-1比较（未与Eagle3/DFlash在生产中对比）。此外，MTP-1是单令牌草稿器，优势对比可能夸大。
5. **泛化性**：主要在DeepSeek-V4和Qwen3系列上测试，架构依赖性（MoE、多模态注入）是否通用需进一步验证。
6. **潜在选择偏差**：异步调度使用两步前预测决定截止长度，虽然保持无损，但可能因历史信息过时而无法捕捉突发负载变化。
