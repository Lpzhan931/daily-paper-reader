## 摘要
线性注意力模型用循环状态取代了不断增长的KV缓存，但自回归解码仍每次只读取、更新和写入一个token的状态。推测解码可通过在一次目标验证中验证多个候选token来降低这一成本，然而现有的推测系统是为Transformer KV缓存设计的。对于有状态的线性注意力目标，验证必须遵循跨链和分支的循环依赖，接受操作只能更新被接受的状态轨迹，并且草稿模型必须避免提交浪费有状态验证工作的候选。本文提出了SpecLA，一种面向有状态线性注意力模型的推测解码运行时。SpecLA采用拓扑感知的内核对链和树进行验证，存储验证过程中产生的紧凑因子以恢复被接受的状态，并使用置信度剪枝加上与目标对齐的EAGLE风格草稿模型，为验证器提供有用的候选。在配备NVIDIA H100和公开GDN-1.3B目标的测试中，SpecLA相较于自回归解码实现了高达1.70倍的端到端加速。

## Abstract
Linear-attention models replace the growing KV cache with recurrent states, but autoregressive decoding still reads, updates, and writes these states one token at a time. Speculative decoding can reduce this cost by verifying several draft tokens in one target pass, yet existing speculative systems are designed for Transformer KV caches. For stateful linear-attention targets, verification must follow recurrent dependencies across chains and branches, acceptance must update only the accepted state trajectory, and the drafter must avoid submitting candidates that waste stateful verification work. This paper presents SpecLA, a speculative decoding runtime for stateful linear-attention models. SpecLA verifies chains and trees with topology-aware kernels, stores compact factors produced during verification to recover accepted states, and uses confidence pruning plus a target-aligned EAGLE-style drafter to feed useful candidates to the verifier. On an NVIDIA H100 with a public GDN-1.3B target, SpecLA achieves up to 1.70x end-to-end speedup over autoregressive decoding.

### 论文《SpecLA: 面向线性注意力模型的高效推测解码》详细总结

1.  **核心问题与整体含义 (研究动机和背景)**
    *   **核心问题**：线性注意力模型（如Gated DeltaNet， GDN）通过循环状态代替了Transformer不断增长的KV缓存，理论上能解决长文本推理的内存瓶颈。然而，其自回归解码过程仍然逐token地读写、更新这个庞大的密集循环状态，这导致了新的性能瓶颈。现有针对Transformer的推测解码系统，由于依赖对KV缓存后缀的操作，无法直接有效地应用于这种有状态的线性注意力模型。
    *   **研究背景**：推测解码通过一个快速的草稿模型生成多个候选token，再由目标模型并行验证，从而分摊昂贵的模型调用成本。但在线性注意力模型中，验证过程需要遵循严格的循环依赖，候选序列的接受与拒绝无法像Transformer那样通过截断KV缓存实现，这带来了验证效率低、状态管理复杂和草稿模型不匹配三大挑战。

2.  **方法论：核心思想、关键技术细节**
    *   **核心思想**：SpecLA 提出了一套专为有状态线性注意力模型设计的推测解码运行时，通过**拓扑感知的验证、紧凑因子状态管理**和**目标对齐的草稿集成**，在保持外部“草稿-验证-接受”循环不变的前提下，解决了三大挑战。
    *   **关键技术细节**：包含三个层级的设计。
        *   **层级1: 拓扑感知验证内核**：该层级负责高效验证草稿。
            *   **状态驻留串行验证**：针对链式草稿，将执行顺序从“逐token处理所有层”改为“逐层处理所有token”，保持循环状态驻留在片上SRAM，并对状态进行V维度分块，避免重复的HBM读写。
            *   **树掩码并行验证**：针对树状草稿，构建一个树掩码矩阵，将其插入GDN的Delta-rule UT分解中。这允许在单个并行前向传播中计算所有候选节点的输出，同时正确维护祖先-后代依赖关系，避免分支间的干扰。
            *   **链分解混合验证**：结合上述两者的优点，将树状草稿分解为若干条依赖正确的链。每条链使用低开销的状态驻留串行内核验证，而不同链之间在边界状态准备好后可以并行执行，既利用了串行验证的高效性，又挖掘了树间的并行性。
        *   **层级2: 接受因子状态管理**：该层级负责在验证后安全地更新循环状态。
            *   **因子缓冲**：在验证过程中，每个token都会生成用于更新状态的“因子”（如key， value， 衰减系数等），但这些因子**远小于**完整的状态。SpecLA 缓存这些因子，在接受阶段只对接受路径上的因子进行轻量级的Delta构造（向量叉积和逐元素操作），从而避免了全状态快照的高内存开销和token重放的大量冗余计算。
            *   **延迟状态更新**：为避免在每次接受后独立执行一次状态更新引发的额外HBM读写，SpecLA将接受后的状态更新与下一次验证融合。新的“融合更新-验证”内核在加载状态图块时，先应用上次的缓冲因子更新状态，然后立即用于当前的验证，消除了冗余的读写往返。
        *   **层级3: 目标对齐的草稿集成**：该层级确保草稿模型生成有用的候选。
            *   **置信度引导的草稿剪枝**：基于累积路径对数概率对草稿树进行剪枝，只保留概率高的路径，避免为目标验证提交大量无用候选，浪费有状态的验证工作。
            *   **目标对齐的EAGLE风格草稿**：训练一个EAGLE风格的草稿层，其特征提取器直接使用目标线性注意力模型的循环状态，使其能更好地预测目标模型的生成动态，提高草稿质量和剪枝信号的有效性。

3.  **实验设计：数据集、基准与方法**
    *   **目标模型与运行时**：使用公开的GDN-1.3B检查点作为目标模型，以及一个EAGLE风格的GDN草稿层。
    *   **数据集**：在三个场景上评估：
        *   **EAGLE风格混合提示集**：包含480个涵盖对话、指令遵循、数学推理、代码生成、问答和摘要的提示。
        *   **GSM8K**：包含1319个数学推理问题。
        *   **HumanEval**：包含164个代码生成问题。
    *   **对比方法 (Baselines)**：
        *   **Autoregressive (自回归)**：每次只解码一个token。
        *   **FLA-SD**：使用相同的GDN草稿器，但通过原生的FLA/GDN代码路径进行验证（即朴素地调用现有内核）。

        *   **SpecLA (SpecLA-Tree)**：SpecLA的完整树状验证路由（包含链分解混合验证）。
    *   **主要指标**：端到端速度提升 (Speedup over autoregressive target decoding)、平均接受长度 (Average accepted length)、首次token匹配率 (First-token match)。

6.  **主要结论与发现**
    *   SpecLA在所有三个评估集上均优于自回归解码和朴素适配的推测解码方法（FLA-SD），实现了高达**1.70倍**的端到端速度提升（在GSM8K上）。
