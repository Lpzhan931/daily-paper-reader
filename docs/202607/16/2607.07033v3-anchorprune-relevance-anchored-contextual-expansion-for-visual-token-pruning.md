## 摘要
大型视觉-语言模型由于高分辨率输入引入成千上万的视觉令牌而产生大量推理成本，其中许多令牌对于给定查询是冗余的。现有的剪枝方法通常结合查询相关性和令牌多样性，但这些目标在激进压缩下可能冲突：相关性驱动选择可能过度集中于相关局部证据，而多样性驱动选择可能抑制不可或缺的令牌或保留独特但无信息区域。我们提出AnchorPrune，一种无需训练的框架，首先构建受保护的相关性锚点，然后用互补的视觉上下文扩展它。AnchorPrune根据相关性排序令牌的新颖性分布自适应确定锚点大小，保留一组紧凑的查询关键证据，并通过重要性加权的多样性分配剩余预算，以恢复相对于锚点的信息丰富且非冗余的上下文。这种有序设计防止上下文扩展取代不可或缺的查询线索，同时改善整体视觉覆盖。AnchorPrune轻量级、架构感知，既不需要重新训练也不需要修改模型。在图像和视频视觉-语言模型及基准测试中，它始终优于无需训练的基线方法，尤其在严重压缩下。在LLaVA-NeXT-7B上，AnchorPrune仅使用2880个视觉令牌中的160个就保持了全令牌性能的97.6%。这些结果确立了基于相关性锚定的上下文扩展作为高效多模态推理的有效原则。代码见https://github.com/MULTI-cau/AnchorPrune。

## Abstract
Large vision-language models incur substantial inference costs because high-resolution inputs introduce thousands of visual tokens, many of which are redundant for a given query. Existing pruning methods often combine query relevance and token diversity, yet these objectives can conflict under aggressive compression: relevance-driven selection may overconcentrate the budget on correlated local evidence, while diversity-driven selection may suppress indispensable tokens or retain distinct but uninformative regions. We introduce AnchorPrune, a training-free framework that first constructs a protected relevance anchor and then expands it with complementary visual context. AnchorPrune adaptively determines the anchor size from the novelty profile of relevance-ranked tokens, preserving a compact set of query-critical evidence, and allocates the remaining budget through importance-weighted novelty to recover informative, non-redundant context relative to the anchor. This ordered design prevents contextual expansion from displacing indispensable query cues while improving overall visual coverage. AnchorPrune is lightweight, architecture-aware, and requires neither retraining nor model modification. Across image and video vision-language models and benchmarks, it consistently improves the accuracy-efficiency trade-off over training-free baselines, particularly under severe compression. On LLaVA-NeXT-7B, AnchorPrune preserves 97.6% of full-token performance using only 160 of 2,880 visual tokens. These results establish relevance-anchored contextual expansion as an effective principle for efficient multimodal inference. Code is available at https://github.com/MULTI-cau/AnchorPrune.

# 论文总结：AnchorPrune: 基于相关性锚定的上下文扩展用于视觉令牌剪枝

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：大型视觉语言模型（VLMs）处理高分辨率图像或视频时会产生大量视觉令牌（例如 LLaVA-NeXT-7B 的 2880 个令牌），大部分令牌对于给定查询是冗余的，导致推理成本高昂（尤其 prefilling 阶段）。
- **核心问题**：现有视觉令牌剪枝方法在激进压缩下存在目标冲突：
  - 基于**相关性**的选择（如 VisionZip）容易过度集中到局部相关区域，忽略分散的查询关键证据；
  - 基于**多样性**的选择（如 DivPrune）可能保留视觉独特但无关的区域，或打散局部连贯证据；
  - 同时优化相关性和多样性的统一方法（如 CDPruner）在高压下无法为不可或缺的查询证据提供显式保护，多样性可能取代关键令牌。
- **整体含义**：作者主张查询关键证据和互补上下文具有非对称角色，应按照固定顺序选择——先保护不可替代的相关证据，再扩展可替代的上下文。此原则有望在不增加推理成本的前提下提升压缩下的任务性能。

## 2. 提出的方法论

### 2.1 核心思想
- 提出 **AnchorPrune**，一种**无需训练**的框架，分为两阶段：
  - **Stage 1：受保护的相关性锚定（Protected Relevance Anchoring）**：利用架构适配的查询条件优先级分数对视觉令牌排序，从新颖性曲线中自适应确定锚点大小，保留一组紧凑的查询关键证据。
  - **Stage 2：上下文扩展（Contextual Expansion）**：用剩余预算通过**重要性加权的新颖性**（importance-weighted novelty）选择令牌，要求所选令牌同时具有全局信息量且与当前保留集互补。

### 2.2 关键技术细节
#### Stage 1：相关性锚定
- **优先级分数计算**（分两种架构）：
  - **CLIP 对齐模型**（LLaVA-1.5/NeXT）：利用预投影器的 CLIP 特征，计算**负的**令牌-文本相似度（negated patch–text similarity），避免 CLIP 原始相似度偏向背景。
  - **非 CLIP 模型**（Qwen2.5-VL）：使用后投影器嵌入，取视觉令牌与指令令牌的最大余弦相似度（token-wise max matching）。
- **自适应锚点大小**：不是固定比例，而是基于新颖性阈值检测。
  - 定义最小锚点大小 \(K_{\min}\)，最大锚点 \(K_{\max} = \lfloor K/2 \rfloor\)。
  - 从排序序列中依次检查后续令牌相对于初始锚点的**余弦距离新颖性** \(\Delta(i; S) = \min_{j \in S} (1 - \cos(h_i, h_j))\)。
  - 当累计出现 \(P\) 个新颖令牌（新颖性 > 阈值 \(\tau\)）时停止扩展锚点，否则取 \(K_{\max}\)。默认 \(\tau=0.2, P=3\)。

#### Stage 2：上下文扩展
- **全局重要性先验**：使用视觉编码器的 [CLS] 注意力或平均注意力质量作为先验 \(p_i\)。
- **锚定条件新颖性**：迭代选择使 \(p_i \cdot \Delta(i; S)\) 最大的令牌，其中 \(S\) 初始化为锚点 \(S_{\text{rel}}\)。
- **意义**：此乘积抑制了视觉独特但无关（低 \(p_i\)）以及重要但冗余（低 \(\Delta\)）的令牌，保证所选上下文既有信息量又互补。

## 3. 实验设计

- **评测模型与数据集**：

## 6. 主要结论与发现

1. **相关性锚定上下文扩展原则有效**：在几乎所有预算和架构上，AnchorPrune 取得最高相对保留性能，尤其在高压缩下（例如 LLaVA-NeXT-7B 保留 160 令牌时 97.6% Rel，优于第二的 92.9%）。
2. **自适应锚点优于固定比例**：通过新颖性曲线动态调整锚点大小，在集中和分散证据场景均表现稳健。
3. **重要性加权扩展优于纯多样性或统一目标**：保护锚点后，选择既重要又互补的令牌比最大化多样性或联合优化相关性-多样性更高效。
4. **跨架构通用性**：在无 CLIP 对齐的 Qwen2.5-VL 上仍取得最佳结果，说明 Anchoring 思想可迁移。
5. **计算开销轻量**：预填延迟和内存开销与最优基线持平，但性能显著更高。

## 7. 优点

- **无需训练、无需修改模型**：直接基于现有表示，即插即用。
- **架构感知**：根据 VLM 视觉编码器类型（CLIP 对齐、非 CLIP）分别设计优先级信号和重要性先验，适应性强。
- **自适应锚点大小**：避免了固定比例的不合理假设，能根据查询证据分布动态调整，减少超参数调优。
- **两阶段顺序设计**：明确区分不可替代的查询关键证据和可替换的上下文，解决了统一优化目标中的保护缺失问题。
- **全面的实验验证**：覆盖主流架构、多预算、多基准，消融实验严谨，分析充分。
- **开源代码**：提供可复现的 GitHub 仓库。

## 8. 不足与局限

- **对优先级信号的依赖**：Stage 1 需要有效的查询-令牌相关性信号，对于某些弱视觉文本对齐或无指令嵌入的模型可能不适用。虽然给出了 CLIP 负相似度、后投影最大匹配等变体，但未测试更极端的架构（如仅用 SigLIP 或纯视觉模型）。
- **视频实验仅 16 帧**：在更长视频（如 32/64 帧）上的有效性未验证，跨帧锚定策略可能需调整。
- **未与基于训练的方法对比**：虽然定位是 training-free，但近期有训练式剪枝（如 LLaVA-PruMerge+ 的变体也无需训练？实际 PruMerge+ 也无需训练，已对比）。一些更先进的训练式方法（如 Token Merging with training）未覆盖。
- **超参数通用性风险**：阈值 τ=0.2 和 P=3 虽然跨模型稳定，但未在高维视觉特征（如 DINOv2）上验证，可能不够鲁棒。
- **缺乏对幻觉或安全性的专门评估**：仅使用 POPE 衡量幻觉，其他基准（如 MM-Vet）可能未覆盖全部安全风险；在极端压缩下模型输出质量可能下降。
- **代码复现依赖特定库**：使用 lmms-eval，但某些基准未支持，可能影响完全复现。
