## 摘要
投机解码通过让轻量级草稿模型提出多个令牌，并由更大的目标模型验证，从而加速自回归生成。尽管对纯文本大语言模型有效，但在视觉语言模型中，投机解码的收益有限，因为草稿模型通常会在视觉关键内容上出现分歧，而现有的多模态加速方法并不能直接解决不相关的视觉证据问题，也无法优化决定加速效果的验证器接受前缀长度。我们提出TIGER，一种用于多模态投机解码的文本条件视觉门控路由框架。TIGER根据草稿模型的当前文本状态，动态选择一组稀疏的上下文相关视觉令牌，而不是暴露全部视觉令牌集或固定的压缩接口。为了更好地将训练与推理效率对齐，我们基于接受对齐的组策略训练优化草稿模型，使用基于接受前缀长度的验证器衍生奖励，并在KL锚定的蒸馏热启动基础上构建。这鼓励草稿模型不仅模仿目标模型，还能产生在验证中存活更长的投机延续。实验表明，TIGER在精确验证器侧投机解码下，在接受的令牌长度和投机加速方面取得一致提升，同时在视觉路由分析中实现了有利的质量-延迟权衡，下游精度相当。

## Abstract
Speculative decoding accelerates autoregressive generation by letting a lightweight drafter propose multiple tokens that are verified by a larger target model. Although effective for text-only LLMs, speculative decoding yields limited gains in VLMs because drafters often diverge on vision-critical content, while existing multimodal acceleration methods do not directly address irrelevant visual evidence or optimize the verifier-accepted prefix length that governs speedup. We propose TIGER, a Text-conditioned vIsual GatEd Routing framework for multimodal speculative decoding. TIGER dynamically selects a sparse set of context-relevant visual tokens based on the drafter's current textual state, rather than expose the full visual token set or a fixed compressed interface. To better align training with inference-time efficiency, we optimize the drafter with acceptance-aligned group-based policy training using verifier-derived rewards based on accepted prefix length, built on top of distillation warm start with KL anchoring. This encourages the drafter not only to imitate the target model, but also to produce speculative continuations that survive verification for longer prefixes. Experiments show that TIGER yields consistent gains in accepted prefix length and speculative speedup under exact verifier-side speculative decoding, while achieving favorable quality-latency trade-offs with comparable downstream accuracy in visual-routing analyses.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：投机解码（Speculative Decoding）通过轻量级草稿模型（drafter）提出多个候选令牌，再由大型目标模型（verifier）验证接受，可加速自回归生成。然而，在多模态视觉语言模型（VLMs）中，该方法收益有限，核心原因有二：
  - **视觉不匹配**：草稿模型通常暴露于全部视觉token或固定压缩接口，其中包含与当前解码上下文无关的噪声信息，导致在OCR、计数等视觉关键内容上过早与验证器产生分歧。
  - **目标不匹配**：现有训练策略仅优化令牌级模仿损失，而非直接最大化验证器接受的连续前缀长度——后者才是决定投机解码加速效果的实际运行指标。
- **整体含义**：本文旨在通过动态选择上下文相关的视觉证据，并直接优化接受前缀长度，弥补草稿-验证器间的双重不匹配，从而显著提升多模态投机解码的效率。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：TIGER 框架包含两大部分——文本条件视觉门控路由（Text-conditioned Visual Gated Routing）和接受对齐训练（Acceptance-Aligned Training）。
- **文本条件视觉门控路由**：
  - 在每个投机回合，根据草稿模型当前文本前缀的隐藏状态 \(h_j\)，计算一个路由查询 \(q_j = W_q h_j\)。
  - 对视觉token库中的每个token \(v_i\)（包含一个可选的全局池化token \(v_g\)），计算相关性得分 \(s_i = q_j^\top W_v v_i / \sqrt{d}\)。
  - 选择得分最高的 top-k 个token作为路由视觉上下文 \(V_j^{\text{route}}\)，代替全量视觉输入，供草稿模型用于生成下一投机块。
  - 路由机制只在每个投机回合计算一次，视觉投影可缓存，引入的额外开销较小。
- **接受对齐训练**：
  - 先通过知识蒸馏（KD）热启动草稿模型，获得稳定参考策略 \(q_{\text{ref}}\)。
  - 第二阶段：对每个训练样本，基于路由上下文采样 \(K\) 个候选投机块，由验证器运行标准接受测试，每个候选块的奖励定义为验证器接受的连续前缀长度 \(R^{(k)} = A^{(k)}\)。
  - 使用组内归一化奖励 \(\hat{g}^{(k)}\)，以类似 GRPO 的目标优化草稿模型：\(\mathcal{L}_{\text{accept}} = -\sum_k \hat{g}^{(k)} \log q_\theta(\hat{Y}^{(k)}|X_j) + \beta \cdot \text{KL}(q_\theta \| q_{\text{ref}})\)。
  - 该目标直接鼓励草稿模型产生能被验证器接受更长的续文，而非仅模仿令牌分布。
- **算法流程**：见 Algorithm 1（文本描述为：热启动KD → 对每个样本构建路由上下文 → 采样K个候选块 → 验证器计算接受前缀长度作为奖励 → 归一化 → 更新草稿模型）。

- TIGER 在几乎所有设置下取得最佳/最佳的接受前缀长度和加速比，且提升一致但适中（相对于强基线如ViSpec、EAGLE-3等），但在某些高要求场景如长视频中增益较微。

- 接受前缀长度奖励优于二元接受或令牌级一致性奖励。
- 方法在较广的模型家族（LLaVA、Qwen-VL）和任务类型上有效。

## 7. 优点

- **方法创新**：同时解决两个特定于多模态投机解码的瓶颈——视觉证据选择和训练目标不匹配，具有针对性和实用性。
- **路由机制轻量高效**：仅每个投机回合一次查询，可缓存视觉投影，延迟开销可控。
- **训练目标直接**：以验证器接受前缀长度为奖励，直接优化决定加速效果的核心指标，而非间接指标。
- **实验设计严谨**：提供详尽消融、稀疏性、奖励形式、替代组件等分析，并与多个基线作公平比较，结果可信。

## 8. 不足与局限

- **依赖基模型质量**：TIGER 无法完全解决基模型的幻觉或推理错误，整体性能仍受限于草稿和验证器的能力。
