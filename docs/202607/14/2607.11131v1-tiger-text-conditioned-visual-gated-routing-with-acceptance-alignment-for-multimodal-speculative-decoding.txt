# TIGER: Text-conditioned Visual Gated Routing with Acceptance Alignment for Multimodal Speculative Decoding 

Quynh Vo Cong-Duy Nguyen Ponhvoan Srey Luu Anh Tuan Thong Nguyen *

National University of Singapore Nanyang Technological University Center of AI Research, VinUniversity thong.nguyen@u.nus.edu 

Abstract 

Speculative decoding accelerates autoregres-sive generation by letting a lightweight drafter propose multiple tokens that are verified by a larger target model. Although effective for text-only LLMs, speculative decoding yields limited gains in VLMs because drafters often diverge on vision-critical content, while exist-ing multimodal acceleration methods do not directly address irrelevant visual evidence or optimize the verifier-accepted prefix length that governs speedup. We propose TIGER, a Text-conditioned vIsual GatEd Routing framework for multimodal speculative decoding. TIGER dynamically selects a sparse set of context-relevant visual tokens based on the drafter’s current textual state, rather than expose the full visual token set or a fixed compressed inter-face. To better align training with inference-time efficiency, we optimize the drafter with acceptance-aligned group-based policy train-ing using verifier-derived rewards based on ac-cepted prefix length, built on top of distillation warm start with KL anchoring. This encourages the drafter not only to imitate the target model, but also to produce speculative continuations that survive verification for longer prefixes. Ex-periments show that TIGER yields consistent gains in accepted prefix length and speculative speedup under exact verifier-side speculative decoding, while achieving favorable quality-latency trade-offs with comparable downstream accuracy in visual-routing analyses. 

1 Introduction 

Vision-language models (VLMs) have rapidly be-come the backbone of multimodal assistants (Liu et al., 2023; Li et al., 2023a), enabling instruc-tion following over images and videos for tasks such as visual question answering, document un-derstanding, chart reasoning, and grounded dia-logue (Laurençon et al., 2024). Despite steady

progress in model quality, real-world deployment remains constrained by autoregressive decoding 

cost: generating outputs token-by-token requires repeated passes through large decoders and exten-sive key–value (KV) caching (Sadhukhan et al., 2025). As VLMs scale to higher resolutions and richer visual contexts, decoding latency and serving cost increasingly dominate practical deployment. 

Speculative decoding offers a principled, loss-less route to accelerate generation. A lightweight drafter proposes multiple tokens, and a large ver-ifier checks and accepts a prefix of these pro-posals, thereby reducing the number of expen-sive verifier decoding steps while preserving the target distribution through a modified rejection-sampling test (Wang et al., 2025b). In text-only LLMs, speculative decoding can provide substan-tial speedups when the drafter closely matches the target model, and recent work further improves efficiency through better drafting structures such as dynamic draft trees (Li et al., 2024). However, directly transferring this paradigm to VLMs often yields limited gains. The main challenge is that multimodal specula-tive decoding is highly sensitive to vision-critical tokens—for example, OCR strings in TextVQA-style queries (Singh et al., 2019), numerical an-swers in chart or science reasoning (Lu et al., 2022), compositional relations in GQA (Hudson and Manning, 2019), and grounded object descrip-tions where hallucination must be avoided (Li et al., 2023b). On such tokens, a small drafter often di-verges early from the verifier, causing short ac-cepted prefixes and sharply reducing speculative speedup. We argue that this failure stems from two coupled mismatches. First, the drafter is of-ten exposed to either the full visual token set or a fixed compressed visual interface, which may still contain irrelevant or noisy visual evidence for the current decoding context. Second, most exist-ing training strategies optimize imitation quality 1arXiv:2607.11131v1 [cs.CL] 13 Jul 2026 rather than the runtime quantity that actually gov-erns speculative efficiency: the verifier-accepted prefix length. These observations motivate a different perspec-tive on multimodal speculative decoding. Instead of only asking how to compress visual inputs for a small drafter, we ask: what visual evidence does the drafter actually need for the next speculative block, and how should it be trained so that the verifier accepts that block for as long as possible? Based on this view, we propose TIGER (Text-conditioned v Isual Gat Ed Routing), a framework for multimodal speculative decoding built on two key ideas. First, rather than exposing the drafter to the full image token set or a fixed compressed adaptor, TIGER dynamically selects a sparse sub-set of context-relevant visual evidence conditioned on the drafter’s current textual state. This yields a lightweight, adaptive visual interface tailored to the next speculative block. Second, rather than training the drafter only to imitate the target model, TIGER directly aligns the drafter with the verifier’s acceptance mechanism. Concretely, we first warm-start the drafter with knowledge distillation to provide a stable behav-ioral initialization. We then formulate drafter learning as an acceptance alignment problem: for each input, we sample multiple draft continuations, run verifier-based acceptance tests, and assign rewards based on verifier-accepted prefix length. This enables group-based policy optimization us-ing verifier-derived supervision, without human preference annotations or learned reward models. In effect, the drafter is encouraged not merely to match the target distribution in aggregate, but to generate speculative blocks that the verifier can ac-cept for longer prefixes. To ensure stable training, we employ KL anchoring to prevent policy drift, and we design the routing and verification pipeline to remain compatible with efficient speculative de-coding at inference time. We evaluate TIGER on a diverse suite of multi-modal benchmarks spanning OCR-heavy, counting-intensive, and visually grounded reasoning tasks. Experiments show that TIGER consistently length-ens verifier-accepted prefixes, reduces target-model decoding work, and improves end-to-end specula-tive decoding speedup over strong multimodal spec-ulative baselines, while preserving downstream task accuracy. Our contributions are summarized as follows: • We propose TIGER, a text-conditioned visual gated routing mechanism that dynamically se-lects context-relevant visual evidence for the drafter in each speculative block. • We introduce an acceptance-aligned training objective based on verifier-accepted prefix length, directly targeting the runtime quantity that governs speculative decoding speedup. • We conduct experiments that explicitly dis-tinguish exact verifier-side speculative decod-ing from auxiliary visual-routing analyses, showing improved accepted prefix length and speedup in the former and favorable quality– latency trade-offs in the latter. 

2 Related Works 

2.1 Inference-Time Acceleration via Visual Token Pruning 

A major efficiency bottleneck in VLMs is the large number of visual tokens, which increases prefill computation and KV-cache traffic. Prior work therefore accelerates VLM inference by pruning or compressing visual tokens. Xing et al. (2024) progressively removes image tokens across layers, while S PARSE VLM performs text-guided token sparsification with adaptive sparsity (Zhang et al., 2025b). V IS PRUNER instead selects salient and diverse visual tokens using visual cues, avoiding reliance on potentially unstable text–visual atten-tion signals (Zhang et al., 2025a). Related pruning ideas have also been extended to specialized multi-modal tasks such as reasoning segmentation (Chen et al., 2025). These methods primarily reduce the amount of visual computation processed by the model. TIGER is complementary: rather than globally pruning or compressing visual tokens, it performs 

text-conditioned visual routing for speculative de-coding, exposing the drafter to a sparse subset of context-relevant visual evidence for each draft block. This targets drafter–verifier mismatch on vision-critical continuations, a bottleneck not di-rectly addressed by standard visual-token pruning methods. 

2.2 Speculative Decoding for Multimodal Models 

Speculative decoding accelerates generation by us-ing a lightweight drafter to propose multiple tokens 2that are verified by a larger target model, yield-ing lossless sampling under the verifier distribu-tion (Wang et al., 2025b). Prior work improves drafting and verification through token trees, dy-namic draft structures, direct token prediction, and serving-oriented optimizations, including S PEC IN-

> FER

, EAGLE-2, EAGLE-3, RACER, T OKEN -RECYCLING , and VLLM (Miao et al., 2024; Li et al., 2024, 2026; Zhang et al., 2026; Luo et al., 2025; Kwon et al., 2023). However, these methods are mostly developed for text-only or general LLM serving and do not directly address multimodal draft–verifier divergence on vision-critical tokens. In VLMs, draft tokens can be rejected early when the drafter diverges on OCR strings, counts, chart values, or visually grounded entities, leading to short accepted prefixes and limited speedup. Re-cent multimodal methods such as S PEC FLASH and V ISPEC improve draft–verifier coherence through latent-aware drafting or vision-aware adap-tation (Wang et al., 2026; Kang et al., 2025). TIGER differs by dynamically routing context-relevant visual evidence to the drafter and by opti-mizing a verifier-derived accepted-prefix objective, aligning both visual access and training with veri-fier acceptance rather than token imitation alone.

TIGER addresses two coupled bottlenecks in mul-timodal speculative decoding. The first is visual mismatch : a small drafter may condition on ex-cessive or irrelevant visual evidence, causing early divergence on vision-critical tokens. The second is objective mismatch : distillation improves aver-age imitation quality, whereas speculative decod-ing speed is governed more directly by verifier-accepted prefix length. Let Ai denote the accepted prefix length at spec-ulative round i, and let ¯A = E[Ai]. To generate a response of N tokens, the expected number of verification rounds is approximately 

E[M ] ≈ N

¯A . (16) Hence, increasing ¯A directly reduces the num-ber of expensive verifier decoding steps. TIGER improves this quantity from both sides: text-conditioned routing increases the likelihood that the drafter accesses the right visual evidence for the next block, while acceptance-aligned optimization explicitly favors continuations that survive verifier checking longer. As a result, TIGER targets the runtime quantity that most directly governs specula-tive decoding efficiency in VLMs, namely verifier-accepted prefix length. 

Routing overhead and cache reuse. TIGER does not re-encode the image at every speculative round. The visual token bank and projected visual rep-resentations are computed once and cached. At each round, TIGER computes only the current text-conditioned query, scores the cached visual tokens, 5and performs Top-k selection. The routed visual context is reused for the entire draft block. Impor-tantly, routing affects only the drafter-side visual interface: the verifier remains unchanged, uses the full visual context, and follows the standard specu-lative decoding acceptance protocol. Thus, verifier-side KV-cache reuse is the same as in standard speculative decoding, while the reported end-to-end latency includes routing overhead.

Datasets. We evaluate TIGER on a diverse set of multimodal benchmarks covering visual question answering, reasoning, OCR understanding, and captioning, including MMBench (Liu et al., 2024), ScienceQA (Lu et al., 2022), TextVQA (Singh et al., 2019), MM-Vet (Yu et al., 2024), and COCO Captions (Chen et al., 2015). These benchmarks assess perception, knowledge grounding, diagram-based reasoning, text understanding in natural im-ages, and open-ended generation. To further ex-amine context-heavy settings, we include addi-tional evaluations on Video-MME (Fu et al., 2025), NLVR2 (Suhr et al., 2019), and MuirBench (Wang et al., 2025a), covering video understanding and multi-image reasoning. Detailed dataset splits, sam-pling protocols, and evaluation details are provided in Appendix C. 

Evaluation Metrics. We evaluate both task per-formance and decoding efficiency. For specula-tive decoding, we report Accepted Prefix Length 

(AccLen ), Verifier Rounds (VerRds ), End-to-End Latency (Lat. ), and Speedup (Spd. ). AccLen measures the average number of drafted tokens ac-cepted per round and is the key efficiency metric, since longer accepted prefixes reduce verifier de-coding rounds. Lat. includes routing overhead, and Spd. is measured relative to autoregressive verifier-only decoding. For non-speculative visual-token efficiency baselines, AccLen and VerRds are not defined, so we report task performance and latency-based efficiency instead.

5 Conclusion 

We study speculative decoding for vision-language models and show that draft quality alone is not suf-ficient; decoding efficiency depends directly on ver-ifier acceptance. We propose TIGER , which com-bines text-conditioned visual gated routing with acceptance-aligned training. TIGER routes context-relevant visual tokens to the drafter and optimizes continuations that yield longer verifier-accepted prefixes. Experiments show improved accepted-prefix efficiency and decoding speed while main-taining competitive task performance, highlighting verifier-acceptance alignment as an effective direc-tion for efficient multimodal generation. 8Limitations 

Despite the promising results, our work has sev-eral limitations. First, TIGER focuses primarily on improving speculative decoding efficiency rather than fundamentally modifying the architecture of vision-language models. As a result, the overall performance of the system still depends on the ca-pabilities of the underlying drafter and verifier mod-els. If the base models exhibit strong hallucination or reasoning errors, TIGER may not fully resolve these issues. Second, the proposed acceptance-aligned training requires sampling multiple specu-lative blocks and evaluating them with the verifier during training. Although we employ block-level verification to reduce computational overhead, this training procedure can still be more expensive than standard distillation-based approaches. Third, our routing mechanism selects a sparse subset of visual tokens based on the current textual prefix. While this design improves efficiency in practice, it may occasionally omit useful visual evidence, especially in complex scenes requiring global visual reason-ing. Finally, our experiments focus on widely used multimodal benchmarks and model families such as LLaVA and Qwen-VL. Future work could ex-plore the generality of TIGER across a broader range of architectures, larger-scale models, and more diverse multimodal tasks.

Routing complexity. For a visual bank of size

This is precisely the regime targeted by our design: sparse, text-conditioned routing for the drafter, combined with direct optimization of verifier-accepted prefix length.

Because the official checkpoints do not match our exact VLM backbone pairs, we retrain the EAGLE-2 drafting module on top of LLaVA-v1.6-Vicuna-7B and Qwen3-VL-4B-Instruct, with model-dependent hidden size, vocabulary size, at-tention heads, and RoPE settings matched to each backbone. We follow the official EAGLE-2 train-ing configuration, including learning rate 3 × 10 −5,batch size 4, 20 epochs, maximum length 2048, uni-form noise with standard deviation 0.2, bf16 mixed precision, and loss weights pw = 0 .1, vw = 1 .0,and head w = 0 .1. At inference, we use the offi-cial tree-based decoding configuration with total draft tokens 60, depth 5, top-k = 10 , and the cor-responding EAGLE tree structure, while matching TIGER’s verifier, prompt format, visual preprocess-ing, and decoding budget. 

EAGLE-3. EAGLE-3 extends EAGLE-style speculative decoding with direct token predic-tion, multi-layer feature fusion, and a compact draft vocabulary. We adapt EAGLE-3 to both LLaVA/Vicuna-style and Qwen-style VLMs by constructing its draft module on top of the mul-timodal hidden states from the corresponding drafter backbone. For LLaVA-v1.6, we use the Vicuna/LLaMA-compatible tokenizer and configu-ration; for Qwen3-VL, we use the Qwen-specific tokenizer, chat template, and model configuration. The verifier is kept unchanged and always uses the full visual context for verification. Following the official EAGLE-3 setup, the draft module is a lightweight one-layer transformer-style head, with hidden size, attention configuration, vocabulary size, RoPE settings, and tokenizer-dependent IDs matched to each backbone rather than hard-coded from a single LLM family. When supported by the backbone tokenizer, we retain the compact draft-vocabulary design with draft_vocab_size =32000 . We train the EAGLE-3 draft module using the official training recipe: learning rate 5 × 10 −5,batch size 1 per GPU with gradient accumulation 2, 40 epochs, maximum length 2048, gradient check-pointing, fp16 mixed precision, and DeepSpeed ZeRO-2. At inference, we enable EAGLE-3 mode and use the same EAGLE-style tree-decoding pa-rameters, including total draft tokens 60, depth 5, and top-k = 10 . All evaluation settings, including verifier model, prompt format, visual preprocess-ing, maximum generation length, and acceptance protocol, are matched with TIGER. 

RACER. RACER is a training-free speculative decoding method that constructs draft candidates using retrieval-augmented future-token prediction. It does not train a separate drafter; instead, it builds an automaton over recent token histories and com-bines retrieved exact patterns with logit-driven can-didate expansion. In our VLM adaptation, the multimodal input is processed by the target VLM with the original visual context, and RACER op-erates over the generated-token stream while pre-serving the same verifier and visual preprocess-ing as TIGER. For Qwen3-VL, we use the native Qwen-style chat template and supported KV-model path; for LLaVA/Vicuna, we use the corresponding Vicuna-style prompt template and KV-compatible model wrapper. We use the official default decod-ing configuration, including max_num_draft = 64 ,

max_breadth = 8 , ngram = 10 , max_nodes =10000 , greedy decoding with temperature 0, and the same maximum generation budget as the other baselines. Since RACER is training-free and sup-ports batch-size-1 decoding in its native imple-mentation, we evaluate it under the same latency-14 oriented setting and report results with the adapted multimodal protocol. 

TokenRecycling. TokenRecycling is a plug-and-play speculative decoding method that accelerates inference without training an external draft model or using a datastore. It reuses candidate tokens pro-duced during decoding and organizes them into a fixed draft tree for verification by the target model. To adapt it to VLMs, we keep the target VLM and visual encoder unchanged, process each mul-timodal input with the full visual context, and ap-ply token recycling only to the language decoding stream. We follow the official static-tree configura-tion, using matrix_top_k = 8 and the hardcoded draft tree of depth 6. No additional multimodal drafter is trained for this baseline. All candidate tokens are verified by the same target VLM under the matched prompt format, visual preprocessing, decoding budget, and latency measurement proto-col. 

ViSpec. ViSpec is a multimodal speculative de-coding method whose drafter explicitly incorpo-rates visual information through a lightweight vision-aware adaptor. Its main design compresses visual tokens and injects a global image feature into the drafting process before verification by the target model. We follow this design as closely as possible and use the official implementation when compatible. However, because released weights are not uniformly available for all backbones con-sidered in our experiments, we reproduce ViSpec on our evaluation backbones by retaining its vision-adaptor-based architecture and retraining the adap-tor and speculative drafting components on the cor-responding LLaVA-v1.6 and Qwen3-VL drafters. The verifier remains unchanged and uses the full visual context. 

SpecFLASH. SpecFLASH is reproduced using its early-exit speculative design, where an auxiliary draft head is attached to an intermediate layer of the base VLM to predict speculative tokens more cheaply before full verification. We follow the orig-inal training pipeline as closely as possible, includ-ing the construction of hidden-state supervision for the early-exit head. Because the released setup does not directly cover every model pair in our eval-uation, we train the method-specific early-exit head on top of the exact LLaVA-v1.6 and Qwen3-VL drafters used in our experiments, while keeping the verifier unchanged. 

SparseVLM. SparseVLM is not a speculative de-coding method but a prompt-conditioned visual to-ken sparsification baseline. Its key idea is to retain only a subset of visual tokens based on text–vision interactions, thereby reducing visual computation during generation. The official implementation is primarily centered on LLaVA-style backbones, so for LLaVA-v1.6 we follow the original sparsifica-tion design directly. For Qwen3-VL, we adapt the same retention principle to the corresponding vi-sual token interface of the backbone and tune the retained-token budget under matched evaluation settings. Because SparseVLM does not include a verifier-based drafting stage, accepted-prefix met-rics are not applicable. 

VisPruner. VisPruner is another non-speculative efficiency baseline that accelerates VLM inference through visual token pruning. Its pruning proce-dure first identifies important visual tokens using visual attention signals and then removes redun-dant tokens based on token similarity so that the retained subset remains both informative and di-verse. The original method is primarily developed for LLaVA-style backbones, and thus we reproduce its two-stage important-and-diverse pruning mecha-nism on the model families used in our experiments rather than comparing against unmatched released checkpoints. As with SparseVLM, this baseline is evaluated as a visual-token reduction method rather than as a speculative decoder, so accepted-prefix-based metrics are not reported. Across all baselines, we use the same evalua-tion benchmarks, prompt format, decoding bud-get, and verifier backbone corresponding to each model family whenever applicable. For specula-tive decoding methods with a matched draft–verify protocol, we report accepted prefix length, veri-fier rounds, end-to-end speedup, latency, and task performance. For training-free or plug-and-play speculative decoding methods, we use the adapted multimodal protocol above and report accepted-prefix metrics when their draft–verify interface is matched; otherwise, we additionally report task per-formance and latency-oriented metrics. C.1 Training Data KD Training Data. As the first stage of training, we warm-start the drafter using knowledge distil-lation on a mixed multimodal corpus constructed from six public vision–language datasets: LLaVA-1.5 COCO, TextVQA, ScienceQA, DocVQA, Tal-lyQA, and ChartQA. The resulting mixture covers a broad range of multimodal capabilities, includ-ing general VQA, OCR, document understanding, chart reasoning, counting, and science-oriented vi-sual QA. The goal of this stage is to provide the drafter with a strong and diverse multimodal initialization before acceptance-aligned optimization. Rather than optimizing for a single benchmark, we expose the model to heterogeneous image–text interaction patterns so that it can learn to mimic the target model across different visual contexts and response formats. This diversity improves the drafter’s gen-erality and makes it a more reliable starting point for the later speculative decoding objective. Each example in the KD pool consists of an im-age I, a textual instruction or question X, and a reference target response Y . The drafter is trained to imitate the target model on this mixed corpus using standard distillation-based supervision. Un-less otherwise stated, all datasets are merged into a single pool and uniformly sampled during KD train-ing. The final KD mixture contains approximately 

92 .5K multimodal examples; detailed dataset statis-tics are reported in Table 7. 

Acceptance-Aligned Training Data For the acceptance-aligned stage, we construct a dedi-cated multimodal training pool of approximately 

80 K examples from six vision–language data sources: TextVQA, DocVQA, ChartQA, TallyQA, ScienceQA, and LLaVA-1.5 COCO. Compared with the KD warm-start corpus, this pool is de-signed specifically for reward-based optimization, where the goal is not merely to imitate the target model, but to expose the drafter to prompts that can induce meaningful variation in verifier acceptance outcomes. To support stable preference- or reward-based training, we apply a lightweight heuristic filter to retain medium-difficulty prompts. In particu-lar, we remove trivially short prompts (fewer than five words), excessively long prompts (more than sixty words), and short pure yes/no questions, since these cases often lead to degenerate or low-variance reward signals. The remaining prompts are as-signed coarse capability tags, including count , ocr ,

reasoning , description , and visual_qa , which provide a simple characterization of the training mixture. This filtering strategy is intended to en-courage speculative samples with diverse accep-tance lengths, which is important for effective GRPO-style optimization. Each example in the GRPO pool consists of an image I, a prompt X, and an optional reference an-swer Y stored as ground_truth . During training, the drafter generates multiple speculative continua-tions conditioned on the routed multimodal context, and the verifier evaluates these continuations to pro-duce acceptance-based rewards. Thus, supervision is derived directly from verifier feedback rather than from manually curated preference annotations or external reward models. We build the accepted-alignment pool by merg-ing all retained examples into a single training set and sampling uniformly from the resulting mixture. Because the publicly available Hugging Face versions of ChartQA and DocVQA provide only limited test/validation splits, their contribution is capped by actual availability, and the LLaVA-COCO portion is correspondingly increased to pre-serve the target pool size. The final GRPO training pool contains approximately 80 K multimodal ex-amples; detailed dataset statistics are reported in Table 8.

E Additional Experiments 

E.1 Accepted Prefix Distribution Analysis 

We further analyze the distribution of verifier-accepted prefix lengths on the COCO Caption benchmark using LLaVA-v1.6-7B as the drafter. Although the average accepted prefix length (Ac-cLen) is a useful summary metric, it does not fully reveal how often a method achieves long accepted continuations, which are the main source of prac-tical speculative decoding gains. A method may obtain a similar mean accepted length while ex-hibiting very different behavior in the tail of the distribution, especially in the frequency of long accepted prefixes. Table 10 reports both the average accepted pre-fix length and the probability that the verifier ac-cepts at least 2, 4, or 8 tokens in a speculative round. Compared with baseline speculative decod-ing methods, TIGER not only achieves the highest mean accepted prefix length, but also consistently shifts the distribution toward longer accepted pre-fixes. In particular, TIGER yields the highest prob-abilities at all reported thresholds, indicating that it produces verifier-compatible continuations more reliably across speculative rounds. This distributional improvement is important be-cause speculative decoding speedup is driven dis-proportionately by long accepted prefixes: when the verifier accepts a larger block, more tokens are generated per expensive verification step, thereby reducing the overall number of verifier rounds. The results therefore support our central claim that TIGER improves efficiency not only on average, but also by increasing the frequency of high-value speculative rounds that contribute most to end-to-end acceleration. 

E.2 Temperature Sensitivity 

We analyze the sensitivity of TIGER to decoding temperature on the COCO Caption benchmark us-ing LLaVA-v1.6-7B. Speculative decoding relies on agreement between the drafter and the verifier, and this agreement typically becomes weaker under higher-temperature sampling. As the temperature increases, the drafter produces more diverse con-tinuations, but these continuations are also more likely to deviate from the verifier, leading to shorter accepted prefixes and reduced speculative gains. Table 11 reports the effect of temperature on ac-cepted prefix length, speedup, verifier rounds, and latency. We observe a clear monotonic degradation in speculative decoding efficiency as temperature increases. Specifically, the average accepted prefix length drops from 3.37 at temperature 0 to 2.02 at temperature 1.0, while the corresponding speedup decreases from 3.24 × to 1.96 ×. At the same time, verifier rounds increase from 0.37 to 0.494 , and latency rises from 0.389 to 0.432 , indicating that weaker drafter–verifier alignment causes the ver-ifier to intervene more frequently. Overall, these results confirm that TIGER is most effective in lower-temperature settings, where speculative con-tinuations remain better aligned with verifier pref-erences.

E.8 Fine-Grained Visual Routing Ablations 

We further ablate the internal design of TIGER’s visual gated routing module. The goal is to test whether the gains come merely from reducing the number of visual tokens, or from the proposed text-conditioned and visual-conditioned routing design. We compare TIGER with several routing variants: random Top-k selection, static image-only Top-k

selection, a text-only query variant, TIGER routing without acceptance-aligned training, and the full TIGER model. The results show that token reduction alone is not sufficient: random Top-k and static image-only Top-k underperform text-conditioned vari-ants. The text-only query improves over non-conditioned selection, but still lags behind TIGER’s visual-conditioned query, suggesting that multi-modal query construction is useful for selecting evidence that supports drafter–verifier agreement. TIGER full performs best overall, indicating that visual-conditioned routing and acceptance-aligned training are complementary.

E.10 Routing Overhead, Caching, and KV-Cache Reuse 

TIGER recomputes the routed visual context once per speculative round because the routing query depends on the current textual prefix. However, this does not require re-encoding the image at ev-ery round. The vision encoder is executed once per input to obtain the visual token bank V , and the visual-side projections used for routing can be cached because the image remains fixed throughout decoding. Concretely, for each input image, TIGER first computes and stores the visual token bank V and the projected visual tokens WvV . At speculative round j, TIGER only computes the current text-conditioned query qj = Wqhj , scores the cached 23 visual tokens, and performs Top-k selection:

This design adds a lightweight routing cost while reducing the drafter-side visual interaction. If r is the number of full visual tokens and k ≪ r is the routing sparsity, TIGER introduces a routing term Croute (r), but replaces the drafter-side visual cost Cvis (r) with Cvis (k). The routing cost con-sists mainly of relevance scoring and Top-k selec-tion, which are much cheaper than re-encoding the image or attending to the full visual token bank throughout drafting. Importantly, dynamic routing does not change the verifier. The verifier still operates with the full visual token bank V and follows the standard spec-ulative decoding acceptance protocol. Therefore, verifier-side KV-cache reuse is unchanged from standard speculative decoding. Routing only af-fects the drafter-side visual interface. On the drafter side, the routed context is fixed within each spec-ulative block and can be reused while generating that block. The reported end-to-end latency includes the routing step. For longer decoding trajectories, rout-ing overhead scales with the number of speculative rounds rather than the number of generated tokens. Since TIGER increases the accepted prefix length, it reduces the expected number of verifier rounds, which helps amortize the additional routing cost.
