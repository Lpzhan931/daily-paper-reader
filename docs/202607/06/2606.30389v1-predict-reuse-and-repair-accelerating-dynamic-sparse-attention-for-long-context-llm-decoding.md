## 摘要
动态稀疏注意力（DSA）通过仅关注每个查询相关的前K个KV块来加速长上下文LLM解码，但这引入了一个串行的选择到注意力依赖关系，成为一个新的延迟瓶颈。我们提出了PRR，一种推测-重用-修复运行时，利用DSA选择中的时间局部性来预测可能的块，在选择进行时推测性地对它们进行注意力计算，并在已知真实选择集后增量式地修复遗漏的块。PRR使用轻量级的基于EMA的预测器、一个基于性能分析的推测预算（使推测工作远离关键路径），以及一个基于FlashAttention的修复内核，利用在线softmax统计将遗漏块合并到部分注意力状态中。在长上下文基准测试和代表性DSA方法上，PRR将每个token的解码延迟降低了高达40%，同时保持了下游任务的准确性。GitHub: https://github.com/Tianyu9748/Incremental_FlashAttention

## Abstract
Dynamic sparse attention (DSA) accelerates long-context LLM decoding by attending to only the top-K KV blocks relevant to each query, but it introduces a serialized selection-to-attention dependency that emerges as a new latency bottleneck. We present PRR, a speculate-reuse-repair runtime that exploits temporal locality in DSA selections to predict likely blocks, speculate the attention over them while selection is in flight, and incrementally repair missed blocks once the true selected set is known. PRR uses a lightweight EMA-based predictor, a profiling-guided speculation budget that keeps speculative work off the critical path, and a FlashAttention-based repair kernel that folds missed blocks into the partial attention state using online-softmax statistics. Across long-context benchmarks and representative DSA methods, PRR reduces per-token decoding latency by up to 40% while preserving downstream task accuracy. ## 1. 核心问题与整体含义

*   **研究动机**：长上下文大型语言模型（LLM）解码过程昂贵，因为需访问并计算不断增长的KV缓存。动态稀疏注意力（DSA）通过仅关注每个查询最相关的前K个KV块来缓解此问题，是高效解码的有效方法。
*   **核心问题**：DSA虽然减少了注意力计算量，但其“块选择”与“注意力计算”是严格串行的。块选择进行前，注意力无法开始。这种**串行化的“选择-到-注意力”依赖关系**成为了新的延迟瓶颈。测量表明，选择阶段可占总生成延迟的41%。
*   **整体含义**：本文旨在打破这一串行依赖，通过利用DSA选择中的**时间局部性**（相邻解码步骤的块选择具有高度相似性），设计一种**投机执行**机制来加速DSA。

## 2. 论文提出的方法论

*   **核心思想**：提出**PRR**（Predict, Reuse, and Repair）运行时系统。其核心是**推测-重用-修复**：在真实的块选择完成前，预测可能在当前步骤中被选中的KV块，并**投机地**执行对这些预测块的注意力计算（Overlap）。一旦真实选择集确定，仅需对**遗漏的块**进行增量式修复，而非重新计算全部注意力。

*   **关键技术细节**：
    1.  **轻量级EMA预测器 (Lightweight EMA-based Predictor)**：
        *   **目的**：高效且准确地预测当前解码步骤可能被选中的前K个KV块，超越简单的“重用上一步选择”。
        *   **机制**：对于每个KV块，维护其重要性得分的平滑水平（level, ℓ）和趋势估计（trend, v）。通过指数移动平均（EMA）公式（见公式2和3）进行预测和更新。具体地，在解码开始时，利用Prefill阶段产生的块重要性分数矩阵，通过**网格搜索**在线校准EMA的三个超参数（α, β, γ）。此过程在CPU上并行，增加延迟仅约0.06ms。
    2.  **增量式在线Softmax修复 (Incremental Repair via Online Softmax)**：
        *   **目的**：在预测出错（遗漏块）后，高效地将遗漏块的贡献合并到已计算的投机注意力结果中，无需完全重算。
        *   **机制**：修改FlashAttention的前向核，使其不仅输出注意力结果O，还输出必要的在线softmax统计量（分母ℓ和运行最大值m）。专门实现一个“修复核”，它利用这些统计量和遗漏块的KV条目，通过FlashAttention的在线softmax递推公式（公式6-8）进行增量合并。修复成本仅与遗漏块的数量 |M| 成正比，而非整个选择集 |A|。
    3.  **关键路径感知的投机预算管理 (Critical-Path-Aware Speculation Budget)**：
        *   **目的**：动态调整投机计算的块数量（预算比δ），确保投机注意力计算能在块选择完成前结束，不会延长关键路径。
        *   **机制**：在离线阶段，针对特定模型、硬件和DSA方法，性能剖析不同上下文长度和预算比下的选择与投机注意力延迟。运行时据此查表，选择**最大安全预算δ**，使投机计算的耗时能被选择阶段完全覆盖。

*   **算法流程**：
    1.  当前解码步骤开始，EMA预测器根据历史数据预测当前步骤的Top-K块集P。
    2.  **并行执行**：GPU同时运行DSA的“块选择”阶段（计算真实选择集A）和PRR的“投机注意力”阶段（对预测集P进行计算）。
    3.  块选择结束后，得到真实集A。若预测完全正确（P=A），则直接将投机结果传递给FFN层。
    4.  若预测不完美（A\P不为空），则将未命中的块M=A\P输入“增量修复核”，修补投机结果，获得最终基于真实集A的注意力输出。
    5.  修复完成后，将最终结果传递给FFN层。
