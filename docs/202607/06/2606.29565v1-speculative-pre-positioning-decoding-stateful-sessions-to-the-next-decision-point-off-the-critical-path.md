## 摘要
无状态推理服务器（vLLM、SGLang、TensorRT-LLM）在请求间空闲，而加速器在等待；有状态会话则利用这段空闲时间。投机预定位利用目标模型自身的前向传播（无需草稿模型）将会话向前解码至其下一个决策点，从而将跨请求的预填充和入口解码移出关键路径：下一个请求从其增量的预付费入口恢复，或者当置信门控触发时，通过一次近恒定的词汇扫描从不包含解码的缓存分布中回答，仅消耗能量和极少数有界误接受。其回报取决于模型能力：有能力的模型以接近全覆盖和约87%的精度触发门控（较小的模型则始终无法触发），第一个token返回时间约为1.0毫秒，而前缀缓存仍需支付39毫秒的解码时间。

## Abstract
A stateless inference server (vLLM, SGLang, TensorRT-LLM) idles between requests while the accelerator waits; a stateful session reclaims that idle time. Speculative pre-positioning decodes the session forward to its next decision point with the target model's own forward pass and no draft model, moving the cross-request prefill and entry-decode off the critical path: the next request resumes from a pre-paid entry on its delta, or, when a confidence gate fires, is answered from a cached distribution in one near-constant vocabulary scan with no decode, at a cost only of energy and a rare, bounded false accept. The payoff is conditional on capability: a capable model fires the gate at near-full coverage and about 87% precision (a smaller one never clears it), returning the first token in about 1.0 ms versus the 39 ms decode a prefix cache still pays.

### 论文总结：投机预定位：将有状态会话的解码移至关键路径外的下一个决策点

#### 1. 核心问题与整体含义
-   **研究动机**：传统的无状态推理服务器（如vLLM、SGLang）在请求间处于空闲状态，导致了加速器计算资源的浪费。更重要的是，每个新请求都必须在**关键路径**上处理预填充和解码，导致首次token延迟很高。
-   **核心问题**：如何利用有状态会话中请求间的空闲时间（Idle Window），将昂贵的计算任务（如预填充）从对延迟敏感的请求关键路径上移除，从而显著降低响应延迟。
-   **整体含义**：论文提出一种名为“投机预定位”的机制，将目标模型自身的前向传播操作安排在空闲窗口中提前执行。这使得后续请求要么从已预付的增量状态快速恢复，要么在模型高度自信时，直接从缓存的输出分布中返回一个token，从而实现**近恒定的、约1毫秒**的首token延迟，代价是额外的能量消耗和可控的、极低的误接受率。这本质上是用计算资源换延迟。

#### 2. 方法论：核心技术思想
-   **核心思想**：
    1.  **空闲窗口利用**：在有状态会话中，两次请求之间（如流式数据更新后、工具调用发出后）存在空闲窗口。利用这个窗口执行本应由下一个请求承担的预填充工作。
    2.  **决策点（Decision Point）**：定义为会话状态加上下一个请求的固定入口结构（如查询前缀+助手头部）之后的位置。在这个点上，模型即将开始生成下一个回答。
    3.  **就绪分布缓存（Ready-Distribution Cache）**：在空闲窗口内，用目标模型进行一次前向传播，计算并缓存决策点处的输出分布（logits）。这不仅是KV缓存，而是包含了“模型的下一个回答”的信息。
    4.  **置信门控（Confidence Gate）**：基于输出分布中最高logit和第二高logit的差距（γ=ℓ(1)-ℓ(2)）和一个阈值τ，决定是否使用缓存的快路径。只有当差距足够大且预测token属于领域内快速答案集（Fast-answer Set A）时，才触发快路径。这是一个选择性预测（Selective Prediction）框架。

-   **关键技术细节与算法流程**：
    -   **快路径（Fast Path）**：置信门控触发时，直接扫描缓存的输出分布，返回logits最大的token。关键路径延迟仅为词汇扫描时间（L_fast ≈ 1 ms），**无需预填充和解码**，实现了约40倍于前缀缓存的加速。
    -   **降级路径（Fallback Path）**：当置信门控未触发时，请求进入降级路径。由于入口的预填充已在空闲时间完成，该路径只需恢复已缓存的状态并处理请求独有的增量（Δ），延迟（L_fall）显著低于基线。
    -   **无效化规则（Invalidation Discipline）**：一旦会话状态发生任何改变（如新数据输入、回滚），立即清除缓存的就绪分布和状态，确保永远不会提供过时的预测。
    -   **公式化模型**：论文定义了空闲窗口（I_idle）、入口长度（L_entry）、各种延迟（T_prefill, T_decode, T_scan）以及命中率（h）、误接受率（r）等指标，并给出了期望延迟（E[L_{pp}]）的公式，清晰展示了加速、收益和风险。

#### 8. 不足与局限
1.  **强烈的模型能力依赖性**：这是最主要的局限。该方法不适用于在所有任务上能力不足的模型或模型处理不熟悉、不确定的任务时。论文仅对比了70B和8B模型，没有探讨不同能力等级模型（如13B, 34B）和不同任务的表现，结论的普适性有待验证。
2.  **严格的空闲窗口假设**：方法成立的前提是存在一个足够长的、**真实空闲**的窗口。如果系统负载高（空闲窗口被其他任务填满），预定位工作将无法按计划进行，收益为零。
3.  **实验场景单一**：实验仅在一个简单的二进制市场趋势分析任务上进行。该任务天生适合单token快路径回答。对于需要多token生成或复杂的、非结构化的单token回答的任务，快路径的有效性存疑。通用性未经验证。

6.  **依赖有状态会话**：该机制完全建立在作者前期工作的有状态推理引擎之上，无法直接应用于主流的无状态推理系统，通用性受限。
