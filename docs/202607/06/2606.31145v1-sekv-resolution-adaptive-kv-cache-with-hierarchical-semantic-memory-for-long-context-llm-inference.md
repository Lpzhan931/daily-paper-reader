## 摘要
大语言模型越来越多地处理长上下文场景，其中KV缓存成为主要内存瓶颈：其大小随序列长度线性增长，并且必须在解码过程中全程保留，这使得未经压缩的完整GPU缓存成本过高而不可行。现有的KV缓存压缩方法难以在效率与忠实保留上下文之间取得平衡。词元驱逐会丢弃信息，而语义分组在预填充阶段就固定了压缩决策；一旦在生成过程中某个压缩片段变得相关，这两种方法都无法从中恢复词元级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织成熵引导的语义片段，并跨GPU-CPU内存层次结构存储它们，而不丢弃信息。每个片段在GPU上保留轻量级摘要向量用于粗略路由，在CPU上保留低秩SVD基用于按需词元级别重建。一种训练好的放大机制在解码过程中选择性扩展与查询相关的片段，从而在无需在GPU上实例化完整KV缓存的情况下实现精确检索。SeKV实现了自适应的词元级别重建，同时保持基础LLM完全冻结，并且只需增加不到0.05%的可训练参数。在四个基准测试中，SeKV相较于最强的语义压缩基线平均提升5.9%，同时相比128K上下文下的完整KV缓存，GPU内存减少53.3%。代码可在 https://github.com/AmirAbaskohi/SeKV 获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型（LLM）在长上下文推理中，KV缓存（Key-Value Cache）的内存占用随序列长度线性增长，解码时必须全程保留，导致完整GPU缓存在长序列下成本过高。现有压缩方法（如词元驱逐Token Eviction、语义分组Semantic Grouping）存在根本性缺陷：
  - 词元驱逐永久丢弃信息，无法恢复被删除的关键证据。
  - 语义分组在预填充阶段固定压缩决策，后续无法从压缩片段中恢复词元级别的细节。
- **研究动机**：长上下文推理需要的不只是更强的压缩，而是一种不同方式组织的**可变分辨率记忆**，能够根据查询相关性动态调整缓存细节，在保留所有信息的同时实现高效推理。
- **整体含义**：提出SeKV（Semantic KV Cache），一种分辨率自适应的语义KV缓存，通过将上下文组织为层次化语义记忆，在不丢弃信息的前提下实现GPU内存节省，同时保持对细粒度证据的可恢复性。

## 2. 论文提出的方法论

### 核心思想
- 将输入上下文分割成语义连贯的**跨度（spans）**，每个跨度在两种分辨率下表示：
  - **GPU端**：紧凑的摘要向量，用于粗略路由（coarse routing）。
  - **CPU端**：低秩SVD基，用于按需的词元级重建（on-demand token-level reconstruction）。
- 训练一个**zoomin机制**，在解码时动态选择与查询相关的跨度并将其扩展为全分辨率，同时通过异步CPU→GPU传输避免阻塞。

### 关键技术细节
1. **熵引导的跨度分割（Entropy-Guided Span Segmentation）**
   - 利用预填充阶段模型产生的token surprisal（Ht = -log p(xt|x<t)）作为零成本边界信号。
   - 高surprisal位置（Ht > μ+ασ）标记为跨度边界，保留为**锚点词元（anchor tokens）**，全分辨率保存在GPU上。
   - 两个锚点之间的词元形成一个语义跨度S。

2. **双分辨率跨度表示与内存层级**
   - **摘要向量（GPU）**：对跨度内非锚点词元，使用归一化surprisal权重加权平均，经学习到的per-head per-layer投影W(l,h)映射到低维空间d′=32，得到紧凑摘要向量\(\bar{K}_S^{(l,h)}\)。
   - **低秩SVD基（CPU）**：对跨度的键矩阵\(K_S^{(l,h)}\)进行截断SVD（最大秩Rmax=32），并通过轻量门控预测器gφ学习每个分量的保留门（mi），实现自适应有效秩。
   - 同样对值矩阵进行类似SVD分解并存储。

3. **训练好的Zoom-In机制**
   - 路由：将当前查询投影到相同路由空间，计算与跨度摘要的点积加对数长度偏置后经sigmoid得到相关性概率\(\tilde{\alpha}_S^{(l,h)}\)。
   - 阈值判断：每个头每层有可学习的阈值τ(l,h)，当概率超过阈值时触发zoomin，异步获取CPU上SVD基并重建词元级KV对。
   - 非放大贡献：未放大的跨度用其保留的均值键值\(\bar{k}_S,\bar{v}_S\)加log|S|修正作为一项参与softmax，确保没有跨度被丢弃。
   - 训练目标：联合蒸馏损失（对齐全KV教师）、zoom监督（正加权二值交叉熵，目标由教师注意质量分布决定）、重建损失（门控近似误差）、预算损失（控制放大跨度数量和保留秩）。
   - 可训练参数：仅路由投影W(l,h)、阈值τ(l,h)和门控预测器gφ，共约4.3M（<0.05%基础模型参数），基础模型冻结。

## 6. 论文的主要结论与发现

- SeKV在所有压缩方法中达到最佳性能，在四个基准上相对于最强基线SentenceKV平均提升**5.9%**，特别在检索密集型任务（NIAH +5.68, RULER +3.63）上增益显著。

- 消融表明，去除训练好的zoom-in或SVD重建导致最大性能下降（NIAH从91.17降至85.96和83.47），说明动态放大和可恢复的词元级细节对性能至关重要。

- 在低预算设置（5%）下，SeKV（86.31 NIAH）显著优于SentenceKV（78.62）和ChunkKV（74.31），说明方法在极端压缩条件下尤为有效。

- 延迟与吞吐量介于StreamingLLM（最快但最差精度）和SentenceKV之间，提供了最佳精度-效率折中。

## 8. 不足与局限

- **依赖语义跨度质量**：熵引导分割质量取决于token surprisal的可靠性，对碎片化文档（如密集表格、代码、不规则格式）可能产生噪声边界，导致次优内存分配。
- **参数敏感度**：引入多个超参数（熵阈值α、局部窗口w、最大SVD秩、预算权重等），迁移到新模型或上下文长度时需要重新验证。
- **CPU-GPU带宽依赖**：zoom-in时异步传输SVD基，若设备间带宽慢或查询反复激活大量远程跨度，延迟优势可能减弱。
- **有限的联合优化**：路由模块是轻量的，但并未与更表达力的跨度表示或自适应重构策略联合训练；模型不能完全适应压缩内存结构。
- **评估范围**：基准均为英文；未测试多语言或多模态场景；仅检测单个推理路径（贪心解码），未见随机解码或温度效应的分析。
- **不可逆性**：虽然声称不丢弃信息，但低秩SVD重建是近似的，不是精确无损；高压缩下仍有重建误差。
- **计算开销**：预填充阶段的随机SVD计算（每个跨度每层每头）可能带来额外时间，尤其对于极长序列。
