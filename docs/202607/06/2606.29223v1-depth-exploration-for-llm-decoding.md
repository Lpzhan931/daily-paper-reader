## 摘要
自回归大语言模型解码时，每个生成的token都经过完整的层堆栈处理，尽管许多token在中间深度就已变得可预测。现有无损深度自适应方法通过选择一个非最终退出深度并用最终深度模型验证其预测来利用这种冗余。然而，我们的测量表明，这种基于选择的策略仍有很大改进空间：选择退出过晚会浪费计算，而选择退出过早则会触发回退并丢弃依赖的草稿。我们提出深度探索解码（DEX），这是一种无损解码算法，它用对多个候选深度的并行探索取代了单深度选择。在每个提交位置，DEX根据最终深度参考验证候选，精确提交最终深度token，并折叠探索网格以仅保留可重复使用的分支状态。这种扩展-提交-折叠过程保持与标准自回归解码等效，同时降低了每个token的提交成本。在早期退出训练和标准大语言模型上，DEX优于有代表性的深度选择基线，并在端到端吞吐量上与投机性和分布式解码方法竞争。此外，随着探索深度的细化，DEX的性能进一步提升，表明并行深度探索为利用大语言模型解码中未被充分利用的深度维度提供了一种可扩展的方式。

## Abstract
Autoregressive LLM decoding evaluates every generated token through the full layer stack, even though many tokens become predictable at intermediate depths. Existing lossless depth-adaptive methods exploit this redundancy by choosing a single non-final exit depth and verifying its prediction with the final-depth model. However, our measurements show that this selection-based strategy leaves substantial headroom: choosing an exit too late wastes computation, while choosing one too early triggers fallback and discards dependent drafts. We propose Depth Exploration Decoding (DEX), a lossless decoding algorithm that replaces single-depth selection with parallel exploration over multiple candidate depths. At each commit position, DEX validates candidates against the final-depth reference, commits exactly the final-depth token, and collapses the exploration lattice to retain only reusable branch states. This expand--commit--collapse procedure preserves equivalence to standard autoregressive decoding while reducing the cost of committing each token. Across early-exit-trained and standard LLMs, DEX outperforms representative depth-selection baselines and achieves competitive end-to-end throughput against speculative and distributed decoding methods. Moreover, DEX improves as the explored depths become finer, showing that parallel depth exploration provides a scalable way to exploit the underused depth axis of LLM decoding.

### 论文中文总结

#### 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题**：自回归（AR）大语言模型解码时，每个生成的 token 都必须经过完整的模型层堆栈（forward pass），但许多 token 在中间层就已经变得可预测。现有深度自适应方法（如 LayerSkip、AdaDecode、DEL）通过**选择单一非最终退出深度**来减少计算，然后使用最终深度模型验证预测。然而，这种单深度选择策略存在**瓶颈**：选择退出过晚会浪费计算，选择退出过早会触发回退（fallback）并丢弃依赖的草稿（descendant drafts），导致额外的计算开销。
- **背景**：论文指出，深度侧（depth axis）冗余尚未被充分利用。早期工作（如 speculative decoding）主要关注 token 轴（预测未来多个 token），而深度轴仍有较大优化空间。作者希望设计一种**无损**的并行深度探索方法，避免单深度选择的风险，从而更高效地利用深度冗余。

#### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程
- **核心思想**：用**并行探索**（parallel exploration）替代单深度选择。在每次提交（commit）位置，同时探索多个候选深度（candidate depths），使用最终深度输出作为参考，选择**最早匹配**的探索分支，提交最终深度 token，并**折叠**（collapse）探索格子（lattice）以保留可重用状态。
- **关键技术细节**：
  - **Earliest Available Depth (EAD)**：定义为 token 首次在中间层达到稳定预测（与最终深度一致）的最小深度。这是衡量 token 何时“就绪”的量化指标。
  - **Expand–Commit–Collapse 过程**：
    - **Expand**：在深度–位置格子上并行展开候选分支，每个深度探索器（depth explorer）负责一段层（stage），并生成候选 token。
    - **Commit**：当 anchor path（从当前提交位置出发的路径）到达最终深度后，使用最终深度 token 作为参考。如果某一非最终探索分支的候选与最终深度 token 匹配，则选择最早匹配的分支，但提交的 token 永远是最终深度 token，保证无损。
    - **Collapse**：根据选中的分支，丢弃所有不一致的格子节点和 KV 缓存，保留可重用的分支状态。
  - **Depth-Coupled Sampling (DECS)**：在同一 slot chain（同一位置不同深度的节点）中，后探索的深度会屏蔽先前深度已提出的 token，从而减少重复提议，多样化候选，提高匹配机会，同时不影响最终深度的采样分布。
  - **Inducing Adapter**：对于标准 LLM（非 early-exit 训练模型），在中间候选深度上附加轻量残差适配器，通过自蒸馏（self-distillation）训练，使中间层表示更接近最终深度输出，从而提升 EAD 分布的浅层性。
- **算法流程**（文字描述）：
  1. 初始化深度探索器（每个探索器负责一段层）和 KV 缓存。
  2. 在每个探索周期中，并行执行扩张（expand），生成深度–位置格子中的候选 token。
  3. 当 anchor path 到达最终深度后，执行提交（commit）：验证各非最终深度的候选，提交最终深度 token。
  4. 根据选中的最早匹配分支，执行折叠（collapse），保留该分支的状态，丢弃其他分支。
  5. 重复直至生成完成。

- **DECS 的有效性**：深度耦合采样在保持无损的同时提升了吞吐量。

#### 7. 优点
- **方法优点**：
  - 突破单深度选择瓶颈，通过并行覆盖多个深度降低风险，将深度适应误差从分类问题转化为分辨率控制问题。
  - 无损等价：始终提交最终深度 token，保证分布不变，可无缝替换标准解码。
  - 可扩展性：探索粒度可调，与可用硬件资源正相关。
- **实验亮点**：
  - 同时评估 early-exit 模型和标准模型，验证方法的普适性。
  - 使用 EAD oracle 作为理论上限，清晰展示差距和进展。
  - 提供详细的开销分析（附录 B），表明 commit–collapse 开销很小。

#### 8. 不足与局限
- **资源依赖**：DEX 需要多个 GPU 来实现并行深度探索，单设备上无法获得加速。资源消耗与探索粒度成正比（K 个 GPU 对应 1/K 粒度）。
- **理论上限**：纯深度探索的加速上限受限于最浅探索深度（d0），约 L/d0 ≤ K。当 K 增大时，边际收益可能因格子扩张开销而饱和（实验中仅测试到 8 个探索器，未外推更大规模）。
- **未结合 token 轴加速**：DEX 只优化深度轴，与投机解码（token 轴）正交但未结合。作者提到这需要协调 token 树验证与深度格子折叠，是未来工作。
- **标准 LLM 的适配代价**：需要额外训练 inducing adapter，虽然参数占比小（1-3%），但仍需额外算力和数据。
- **局限性讨论**：论文在附录 E 中坦诚讨论了上述局限，包括资源依赖、扩展性饱和、正交加速等，并指出实际部署需考虑安全机制（由于无损，风险继承自基础模型）。
