## 摘要
线性注意力模型用循环状态取代日益增长的KV缓存，但自回归解码仍逐一读取、更新和写入这些状态。推测解码通过在一次目标前向传递中验证多个草稿令牌来降低此成本，然而现有推测系统专为Transformer KV缓存设计。对于有状态的线性注意力目标，验证必须遵循跨链和分支的循环依赖，接受仅更新已接受的状态轨迹，且草稿模型必须避免提交浪费有状态验证工作的候选。本文提出SpecLA，一个面向有状态线性注意力模型的推测解码运行时。SpecLA利用拓扑感知的内核验证链和树，存储验证过程中产生的紧凑因子以恢复已接受状态，并采用置信度剪枝加上目标对齐的EAGLE风格草稿模型，向验证器提供有用的候选。在配备GDN-1.3B目标的NVIDIA H100上，SpecLA相较自回归解码实现了高达1.70倍的端到端加速。

## Abstract
Linear-attention models replace the growing KV cache with recurrent states, but autoregressive decoding still reads, updates, and writes these states one token at a time. Speculative decoding can reduce this cost by verifying several draft tokens in one target pass, yet existing speculative systems are designed for Transformer KV caches. For stateful linear-attention targets, verification must follow recurrent dependencies across chains and branches, acceptance must update only the accepted state trajectory, and the drafter must avoid submitting candidates that waste stateful verification work. This paper presents SpecLA, a speculative decoding runtime for stateful linear-attention models. SpecLA verifies chains and trees with topology-aware kernels, stores compact factors produced during verification to recover accepted states, and uses confidence pruning plus a target-aligned EAGLE-style drafter to feed useful candidates to the verifier. On an NVIDIA H100 with a public GDN-1.3B target, SpecLA achieves up to 1.70x end-to-end speedup over autoregressive decoding.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究背景**：线性注意力模型（如Gated DeltaNet, GDN）使用循环状态替代Transformer中不断增长的KV缓存，但自回归解码仍需要逐token读取、更新和写入该密集循环状态，导致推理瓶颈（内存受限、状态移动开销大）。
- **核心问题**：现有推测解码（Speculative Decoding）系统专为Transformer的KV缓存设计，无法直接迁移到有状态的线性注意力模型上：
    - 验证阶段需要处理链式和分支候选的递归依赖；
    - 接受阶段无法通过截断缓存后缀来回滚，必须精确更新已接受状态路径；
    - 草稿模型需要适应目标模型的递归特性，避免提交无效候选。
- **研究含义**：提出专用运行时SpecLA，实现拓扑感知验证、紧凑因子状态管理、目标对齐草稿集成，首次高效地将推测解码应用于线性注意力模型。

## 2. 方法论

### 核心思想
SpecLA保持外部草稿-验证-接受循环，但替换了Transformer的KV缓存后缀操作，引入三个核心机制：

### 关键技术细节

#### 2.1 拓扑感知验证内核
- **状态驻留串行验证（State-Resident Serial）**：将执行顺序从token主序改为层主序（layer-major），同一层内相邻候选token连续执行，使循环状态瓦片（tile）保持在SRAM中，避免HBM往返。采用V维度瓦片化（V-dimension tiling）解决SRAM容量限制。
- **树掩码并行验证（Tree-Masked Parallel）**：针对分支草稿，构建树依赖掩码M_tree，插入到GDN的Delta-rule UT分解中，一次性并行计算所有候选节点的输出，避免根到叶的串行重放。
- **链分解混合验证（Chain-Decomposed Hybrid）**：将草稿树分解为若干依赖链，每条链使用低开销的串行内核验证，就绪链并行执行，平衡串行与并行方案的优缺点。

#### 2.2 推测状态管理
- **因子缓冲（Factor Buffering）**：验证过程中缓存每个候选token的紧凑因子（如k、v、衰减系数α、Delta系数β），而非完整状态快照或仅存储token ID。因子大小仅约128.5 KiB/层（L=8），远小于完整状态2 MiB/层，恢复已接受状态时只需轻量Delta构造操作。
- **延迟状态更新（Delayed State Update）**：将已接受因子缓冲区延迟到下一轮验证时融合应用，与验证内核合并，消除单独的状态更新传递，减少一次HBM状态读写往返。

#### 2.3 目标对齐草稿集成
- **置信度引导剪枝（Confidence-Guided Pruning）**：基于每条路径累积对数概率进行树剪枝，保留高于阈值τ_tree的路径，并受token预算T限制，确保提交的候选有价值。
- **目标对齐EAGLE风格草稿（Target-Aligned Drafting）**：使用从目标模型递归状态特征训练得到的EAGLE风格草稿层，使草稿评分反映目标动态，剪枝更准确。草稿保持Transformer风格（小窗口内二次注意力成本低），避免引入额外递归状态管理。

## 6. 主要结论与发现

- **SpecLA在所有任务上均优于自回归解码**：混合套件1.42×，GSM8K 1.70×，HumanEval 1.06×（HumanEval上接受长度仅2.34，提升有限）。

- **混合验证内核效率最高**：相比根到叶重放，混合内核加速可达1.80–7.11×；串行内核其次，并行内核因短窗口内摊销不足反而更慢。
