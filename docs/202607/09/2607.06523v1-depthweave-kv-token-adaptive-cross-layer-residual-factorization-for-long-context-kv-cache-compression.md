## 摘要
长上下文语言模型推理日益受到存储键值缓存所需内存带宽和容量的限制，然而现有压缩方法通常在各层或各令牌之间应用统一预算，当词汇线索和语义状态需要不同的保留时，检索性能会下降。我们提出了DepthWeave-KV，一种令牌自适应的缓存压缩方法，它使用共享的低秩通道基矩阵分解相邻Transformer层的键和值状态，同时在注意力行为敏感的令牌上保留轻量级的令牌特定残差。DepthWeave-KV结合了跨深度残差分解与一个令牌条件深度路由器，该路由器为承载指令和检索关键的令牌分配更高的重建秩，并利用来自注意力输出探针的无校准在线误差跟踪，在生成过程中自适应压缩而无需重新训练基模型。一个融合的CUDA实现联合执行基查找、残差反量化和注意力投影，以减少解码时的内存流量。在LongBench、针锋相对、L-Eval以及长形式问答和摘要基准测试中，DepthWeave-KV以显著降低的内存使用实现了接近全缓存的任务质量，在平均得分和检索准确率上优于先前的压缩缓存，同时在64K上下文下实现了8.3倍的KV内存减少和每秒72.8个令牌的处理速度。

## Abstract
Long-context language model inference is increasingly limited by the memory bandwidth and capacity required to store key-value caches, yet existing compression methods often apply uniform budgets across layers or tokens and degrade retrieval when lexical cues and semantic states require different preservation. We introduce DepthWeave-KV, a token-adaptive cache compression method that factorizes key and value states across neighboring transformer layers using shared low-rank channel bases while retaining lightweight token-specific residuals where attention behavior is sensitive. DepthWeave-KV combines cross-depth residual factorization with a token-conditional depth router that allocates higher reconstruction rank to instruction-bearing and retrieval-critical tokens, and uses calibration-free online error tracking from attention-output probes to adapt compression during generation without retraining the base model. A fused CUDA implementation jointly performs basis lookup, residual dequantization, and attention projection to reduce decode-time memory traffic. Across LongBench, Needle-in-a-Haystack, L-Eval, and long-form QA and summarization benchmarks, DepthWeave-KV achieves near-full-cache task quality with substantially lower memory use, improving average score and retrieval accuracy over prior compressed caches while reaching 8.3x KV memory reduction and 72.8 tokens per second at 64K context.

# 论文深度总结：DepthWeave-KV

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：长上下文语言模型推理时，KV（键值）缓存成为主要的内存瓶颈，限制了批量大小、上下文长度和服务吞吐量。现有的压缩方法（如令牌驱逐、合并、量化、低秩注意力、逐层缓存共享）通常对所有层或所有令牌施加统一的压缩预算，忽略了不同层（浅层编码词汇线索、深层编码语义抽象）和不同令牌（如指令分隔符、实体提及、答案承载句 vs. 流畅延续令牌）对缓存保真度的不同需求。

- **后果**：统一压缩会导致在“大海捞针”（Needle-in-a-Haystack）检索、多跳问答、长文档摘要等任务上出现脆弱的失败——看似不重要的令牌后来可能变得关键，而平均困惑度或短上下文基准无法揭示这种退化。

- **研究含义**：需要一种同时考虑**深度差**和**令牌适应性**、并且能够在线根据注意力行为动态调整压缩的机制，无需重新训练基础模型。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
**跨深度残差分解 + 令牌条件深度路由器 + 在线误差追踪**。用共享的低秩通道基捕获相邻层KV状态的共同成分，同时对重要令牌保留高保真残差，其余令牌仅由共享基表示。

### 关键技术细节

1. **跨深度残差分解**：
   - 将层划分为重叠深度窗口 \( W_m \)（窗口大小 \( w \)）。每个窗口内，为每个注意力头维护共享的键基 \( BK_{m,h} \in \mathbb{R}^{r_b \times d} \) 和值基 \( BV_{m,h} \in \mathbb{R}^{r_b \times d} \)。
   - 重建形式：  
     \[
     \hat{K}_{\ell,h,t} = A_{\ell,h,t} B_{m,h}^{K} + g_{\ell,h,t} R_{\ell,h,t}^{K}
     \]
     其中 \( A \) 是系数向量（8位块量化），\( R \) 是残差（4位组量化），\( g \) 是残差门控（0-1）。残差仅在路由器分配的非零秩时才存储。

2. **令牌条件深度路由器**：
   - 为每个令牌 \( t \) 和深度窗口 \( W_m \) 计算分数，基于四个在线特征：累积注意力质量、最大注意力尖峰、分隔符/指令边界指示符、注意力输出重建误差。
   - 路由到的残差秩为：
     \[
     \rho_{m,h,t} = \rho_{\min} + (\rho_{\max} - \rho_{\min}) \cdot \mathbb{1}[\sigma(u^\top \phi_{m,h,t}) > \tau_m]
     \]
     实际上使用三级秩：延续令牌通常 \( \rho=0 \) 或 \( \rho=2 \)，结构令牌 \( \rho=4 \)，检索关键跨度 \( \rho=8 \)。

3. **无校准在线误差追踪**：
   - 每 \( p \) 个解码步骤，用压缩缓存和临时高保真缓存分别计算一小部分头的注意力输出，测量相对误差：
     \[
     e_{\ell,h} = \frac{\|\text{Attn}(Q, K, V) - \text{Attn}(Q, \hat{K}, \hat{V})\|_2}{\|\text{Attn}(Q, K, V)\|_2 + \epsilon}
     \]
   - 若误差超过窗口特定容差，则降低阈值 \( \tau_m \) 并为更多令牌分配更高秩；若误差持续低于容差，则提高阈值减少残差存储。

4. **融合CUDA实现**：
   - 将基查找、残差反量化、系数乘法和注意力投影融合到单个核中，避免将重建的KV张量写回全局内存，减少解码时的内存流量。

### 流程概要
- **预填充阶段**：形成深度窗口基、投影KV到共享系数空间、根据路由器存储量化的残差、记录探测统计。
- **解码阶段**：融合核加载共享基、反量化路由的残差、重建需要的KV片段并直接用于注意力计算。

- **任务质量**：DepthWeave-KV平均任务分数62.9%，仅比全缓存低0.9分，同时实现8.3倍KV内存减少。在所有压缩基线中最高（TailorKV 61.4%）。
