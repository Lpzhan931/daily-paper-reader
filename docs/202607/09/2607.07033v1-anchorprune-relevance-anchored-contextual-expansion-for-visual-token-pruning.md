## 摘要
大型视觉-语言模型因高分辨率输入引入数千个视觉令牌而招致巨额推理开销，其中许多令牌对于给定查询是冗余的。现有剪枝方法常结合查询相关性与令牌多样性，但这两个目标在激进压缩下可能冲突：相关性驱动的选择可能将预算过度集中于相关的局部证据，而多样性驱动的选择可能抑制不可或缺的令牌或保留独特但信息量少的区域。我们提出AnchorPrune，一种无需训练的框架，首先构建受保护的相关性锚点，然后利用互补的视觉上下文扩展该锚点。AnchorPrune根据相关性排序令牌的新颖性分布自适应确定锚点大小，保留一组紧凑的查询关键证据，并通过重要性加权的新颖性分配剩余预算，以恢复相对于锚点信息丰富且非冗余的上下文。这种有序设计防止上下文扩展取代不可或缺的查询线索，同时改善整体视觉覆盖。AnchorPrune轻量、架构感知，且无需重新训练或修改模型。在图像和视频视觉-语言模型及基准测试中，它持续改善了无训练基线的准确率-效率权衡，尤其在强压缩下。在LLaVA-NeXT-7B上，AnchorPrune仅使用2,880个视觉令牌中的160个即可保持全令牌性能的97.6%。这些结果确立了基于相关锚点的上下文扩展作为高效多模态推理的一项有效原则。代码见https://github.com/MULTI-cau/AnchorPrune。

## Abstract
Large vision-language models incur substantial inference costs because high-resolution inputs introduce thousands of visual tokens, many of which are redundant for a given query. Existing pruning methods often combine query relevance and token diversity, yet these objectives can conflict under aggressive compression: relevance-driven selection may overconcentrate the budget on correlated local evidence, while diversity-driven selection may suppress indispensable tokens or retain distinct but uninformative regions. We introduce AnchorPrune, a training-free framework that first constructs a protected relevance anchor and then expands it with complementary visual context. AnchorPrune adaptively determines the anchor size from the novelty profile of relevance-ranked tokens, preserving a compact set of query-critical evidence, and allocates the remaining budget through importance-weighted novelty to recover informative, non-redundant context relative to the anchor. This ordered design prevents contextual expansion from displacing indispensable query cues while improving overall visual coverage. AnchorPrune is lightweight, architecture-aware, and requires neither retraining nor model modification. Across image and video vision-language models and benchmarks, it consistently improves the accuracy-efficiency trade-off over training-free baselines, particularly under severe compression. On LLaVA-NeXT-7B, AnchorPrune preserves 97.6% of full-token performance using only 160 of 2,880 visual tokens. These results establish relevance-anchored contextual expansion as an effective principle for efficient multimodal inference. Code is available at https://github.com/MULTI-cau/AnchorPrune.

# AnchorPrune: 基于相关锚点的上下文扩展视觉令牌剪枝——论文详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

大视觉语言模型（VLMs）在处理高分辨率、多裁剪或多帧输入时，会产生数千个视觉令牌，导致巨大的推理开销（尤其是预填充阶段）。这些令牌中有许多对给定查询是冗余的。现有的视觉令牌剪枝方法主要分为几类：
- **基于显著性/相关性**：根据与查询的相关性选择令牌，但容易过度集中于局部相关区域，遗漏分散的查询证据。
- **基于多样性**：通过最大化特征空间覆盖来分散选择，但可能保留视觉独特但查询无关的令牌，或打碎局部连贯证据。
- **联合相关性与多样性**：如CDPruner通过行列式点过程统一优化，但在激进压缩下，多样性可能挤占关键令牌，且弱相关性无法通过后续补偿。

论文指出**查询关键证据与互补上下文扮演非对称角色**：前者不可替代，一旦丢失无法恢复；后者可替代，多个信息子集都能提供支持。因此应该先保护关键证据（相关性锚点），再扩展互补上下文。

## 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：构建一个受保护的“相关性锚点”，然后通过重要性加权的新颖性（importance-weighted novelty）扩展上下文。两个阶段顺序进行，确保锚点不被后续选择挤占。

**阶段1：Protected Relevance Anchoring（受保护相关性锚定）**
- 对每个视觉令牌计算架构特定的查询条件优先级分数 $s_i$。
  - 对于CLIP对齐的VLM（如LLaVA）：使用**负的**CLIP局部相似度（negated patch–text similarity）作为优先级，以避免原始相似度偏向背景。
  - 对于非CLIP架构（如Qwen2.5-VL）：在后投影空间中使用令牌与指令令牌的最大余弦相似度。
- 按优先级降序排序，选择前 $K_{\text{rel}}$ 个作为相关性锚点。
- **自适应锚点大小**：并非固定比例，而是从优先级排序序列的新颖性分布中确定。设置最小锚点大小 $K_{\min}$ 和最大 $K_{\max} = \lfloor K/2 \rfloor$。从 $K_{\min}$ 开始，扫描后续令牌，计算相对于初始锚点的新颖性 $\Delta$，当遇到第 $P$ 个新颖性超过阈值 $\tau$ 的令牌时，将该索引作为最终锚点大小；否则保留 $K_{\max}$。超参数 $\tau=0.2$, $P=3$ 固定。

**阶段2：Contextual Expansion（上下文扩展）**
- 初始化保留集 $S = S_{\text{rel}}$（锚点）。
- 分配剩余预算 $K_{\text{ctx}} = K - K_{\text{rel}}$。
- 每个候选令牌 $i\notin S$，计算其全局重要性先验 $p_i$（基于CLS注意力或平均注意力质量）和相对于当前 $S$ 的新颖性 $\Delta(i;S)$（最小余弦距离）。
- 迭代选择：$i^\star = \arg\max_{i\notin S} p_i \cdot \Delta(i;S)$，加入 $S$，直到达到总预算 $K$。
- 乘性目标确保每个被选令牌既全局重要又互补于已有证据，避免单纯追求多样性或相关性。

**最后**：将选中的令牌恢复原始空间/时间顺序（栅格扫描/帧主序）送入语言模型，无需修改模型或重新训练。

**无需训练、架构感知**：可适应CLIP对齐和非CLIP模型，视频模型可逐帧构建锚点后全局扩展。

## 6. 论文的主要结论与发现

1. **相关性锚定上下文扩展优于现有联合优化方法**：在所有模型和预算下，AnchorPrune的保留性能最高，尤其是在激进压缩时优势更大。
2. **自适应锚点大小的重要性**：固定比例可能不适用于不同查询分布，自适应机制（基于新颖性剖面）更鲁棒。
3. **重要性加权新颖性优于纯多样性或简单相关-多样性组合**：消融实验显示，Stage-2中使用重要性×新颖性乘法比加法/乘法相关-多样性或纯多样性高3-4个百分点相对性能。
4. **跨架构迁移性**：非CLIP模型（Qwen2.5-VL）和视频模型同样有效，证明该方法不依赖于特定表示空间。
5. **性能-效率权衡**：AnchorPrune在实现接近全令牌性能的同时，大幅降低FLOPs（81.3%）和预填充延迟（5.13×加速），且GPU内存接近基线，优于其他方法。
6. **负CLIP相似度作为锚定优先级的有效性**：在CLIP对齐模型中，负相似度对前景/查询相关区域的定位更好，带来显著改进（如MME从1245提升到1394）。

## 7. 优点

- **无需训练，即插即用**：直接应用于现有预训练VLM，无需微调或架构修改。
- **原理清晰，补救现方法的“顺序缺陷”**：明确区分不可替代的相关性证据和可替代的上下文，避免联合优化中关键证据被挤占。
- **自适应锚点大小**：根据查询的新颖性分布动态调整，比固定比例更合理。
- **重要性加权新颖性**：在扩展时兼顾全局重要性和互补性，而非单纯最大化特征分散。
- **架构感知性强**：分别为CLIP对齐和非CLIP模型设计不同的优先级计算方式；并支持多帧视频的多阶段处理。
- **实验完整公平**：覆盖多种模型、基准、压缩比率，严格控制变量；消融实验充分验证各组件贡献。
- **效率出色**：剪枝选择本身轻量（仅需一次CLIP推理或注意力提取），且实现了最佳准确率-效率权衡。

## 8. 不足与局限

- **需要特定架构信号**：对CLIP模型依赖CLIP文本编码器和[CLS]注意力；对非CLIP模型需后投影空间和注意力分布。若模型不公开这些中间表示（如某些商业API），则无法应用。
- **超参数对模型性能有轻微影响**：虽论文表明 $\tau$ 和 $P$ 在多数设置下稳定，但发现LLaVA-NeXT上 $\tau=0.3$ 略优于0.2，说明可能存在模型特定最优设置。
- **视频场景仅测试16帧输入**：未验证长视频（>16帧）或动态帧采样场景；视频锚点按帧独立构建再全局扩展，对帧间强冗余可能不够优化。
- **未与需要微调的方法对比**：论文聚焦训练无关方法，未与ToMe、TokenMerge等需要训练或自适应合并的方法比较，虽然这是设计选择，但可能遗漏部分竞争力信息。
- **效率测量不够详尽**：仅对LLaVA-1.5-7B在32令牌下进行详细效率分析，未覆盖其他模型和预算，也未报告相比训练无关方法的总推理时间节省。
- **对细粒度空间推理基准的影响**：文本发现保留令牌顺序对OCR和文本VQA有显著影响（栅格扫描优于原始顺序），说明AnchorPrune本身不保留空间结构，依赖后处理，可能弱于保持拓扑的结构保留方法。
- **未在更大模型（如70B）上验证**：所有实验基于7B模型，扩展性尚待验证。
- **对指令长度敏感**：若指令超长（如视频长文本描述），分段后平均相似度可能稀释信号，最大匹配可能更好但未系统评估。
