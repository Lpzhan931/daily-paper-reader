## 摘要
投机解码通过缓解内存瓶颈显著加速了大语言模型（LLM）推理。然而，传统的投机解码通常依赖于辅助草稿模块，导致显著的训练和通信开销。尽管最近的方法尝试在目标模型内部生成草稿，但由于缺乏结构协调，它们往往无法充分利用其潜在的并行能力。在本文中，我们提出了渐进式树草稿（PTD），它采用一种结构化的、引导式的并行草稿策略来利用模型的并行潜力。通过将渐进式树结构与逐步剪枝机制相结合，PTD主动引导LLM在单次前向传播中探索多个语义路径，确保草稿的多样性和连贯性。实验表明，PTD在各种基准测试中实现了高达2倍的解码加速，同时无需训练且与模型无关。我们的代码可在 https://github.com/MINE-USTC/PTD 获取。

## Abstract
Speculative decoding has significantly accelerated Large Language Model (LLM) inference by alleviating memory-bound bottlenecks. However, traditional speculative decoding typically relies on auxiliary draft modules, incurring significant training and communication overhead. Although recent methods attempt to generate drafts within the target model itself, they often fail to fully exploit its latent parallel capacity due to a lack of structural coordination. In this paper, we propose \textbf{Progressive Tree Drafting (PTD)}, which employs a structured, guided parallel drafting strategy to harness the model's parallel potential. By coupling a progressive tree structure with a stepwise pruning mechanism, PTD actively guides the LLM to explore multiple semantic paths in a single forward pass, ensuring both draft diversity and coherence. Experiments demonstrate that PTD achieves up to $2\times$ decoding speedup across various benchmarks while remaining training-free and model-agnostic. Our code is available at: https://github.com/MINE-USTC/PTD.

## 2. 方法论

### 核心思想
- 利用Transformer对输入内容和注意力掩码的敏感性，通过重配置可让模型从单一路径转移到多条并发推理轨迹。
- 构建**草稿树 (Draft Tree)**，以共享前缀压缩冗余，并通过逐步扩展和剪枝动态演化。

### 关键技术细节
1. **树结构定义**：树 T=(V,E)。每个节点 v 可观测其祖先节点集 π(v) = {v} ∪ π(P(v))，位置编码由 π(v) 的基数决定，兼容因果注意力。
2. **渐进扩展**：每步前向传播后，将所有节点生成的下一个 d_v 作为子节点添加到树中：  
   V_t = V_{t-1} ∪ {d_v}，E_t = E_{t-1} ∪ {(v, d_v)}。
3. **约束控制**：
   - **宽度控制**：限制每个父节点的最大子节点数（默认4）。
   - **步进深度控制**：当子树深度超过阈值（默认6）时，保留最早添加的子节点及其后代作为新子树，剪除其他分支。
4. **候选树验证**：从草稿池中检索候选树，使用相同注意力掩码和位置编码，每节点生成验证令牌。采用贪婪（Top-1）或采样（无替换重采样）策略确定可接纳边（eligible edges），最终接收路径为从根节点出发的连续可接纳边序列。
5. **算法流程**（见附录D Algorithm 1）：
   - 检索候选树 → 前向传播生成草稿和验证令牌 → 扩展草稿树并修剪 → 合并子树到候选池 → 确定可接纳边 → 追加路径到生成序列。

## 6. 主要结论与发现

1. **PTD在所有模型和任务上取得最高加速**，尤其数学（GSM-100高达2.30×）和代码（MBPP-100达2.08×）等推理密集型任务优势明显。
2. **树结构带来更长接受长度**：虽然PTD的Hit Rate略低于Self-Draft，但Accept Length显著更高，说明草稿具有更好的上下文连贯性。
3. **开销可控**：PTD的计算开销与Self-Draft相当，低于LADE，证明树结构未过度增加延迟。
4. **超参数鲁棒**：在宽度=4、深度=6附近达到性能峰值，调整范围较宽时仍保持良好加速（图6）。
5. **初始化策略影响**：对于结构化任务（编码、数学），NER初始化优于随机初始化；对开放式任务差异较小。
