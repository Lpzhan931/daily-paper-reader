## 摘要
条件计算可以将语言模型质量与每个token的推理成本解耦，但领先的技术孤立地作用于单一轴：混合专家（MoE）稀疏化FFN，混合深度（MoD）跳过整个Transformer块，KV缓存量化压缩注意力内存。我们认为这三个决策（注意力分辨率、专家选择和缓存位宽）是强耦合的，应该联合做出：一个足够罕见、值得完全关注的token，无论由哪个专家处理，也可能需要高精度缓存。我们引入TriRoute，这是一个在所有三个轴上共享的轻量级控制器，对于每一层的每个token，它发出一个协调策略：（i）注意力模式（跳过/局部/全局），（ii）FFN专家的稀疏集（具有恢复MoD的空专家），以及（iii）KV缓存位宽。该控制器通过异质松弛（用于分类决策的Gumbel-Softmax与直通估计，以及用于专家的负载均衡top-k门控）在拉格朗日预算约束下进行端到端训练，该约束将平均计算和内存成本变为可控旋钮。我们在天真的联合训练中识别出跨轴路由坍缩级联，其中一轴的坍缩传播到其他轴，并通过每轴归一化和耦合感知平衡损失来解决它。在从160M到1.3B参数的计算最优token计数的解码器-only模型上，TriRoute在匹配推理FLOPs和内存的情况下帕累托支配了最佳的独立MoD+MoE+KV量化组合，同时更好地保留了纯困惑度优化所侵蚀的罕见实体、代码和算术上的尾例鲁棒性。事后分析揭示了可解释的结构：控制器将完全注意力和高精度缓存分配给句首位置、罕见子词和命名实体，而廉价路由功能词。

## Abstract
Conditional computation can decouple language model quality from per-token inference cost, yet leading techniques act on a single axis in isolation: Mixture-of-Experts (MoE) sparsifies the FFN, Mixture-of-Depths (MoD) skips whole transformer blocks, and KV-cache quantization compresses attention memory. We argue these three decisions (attention resolution, expert selection, and cache bit-width) are strongly coupled and should be made jointly: a token rare enough to warrant full attention may also need high-precision caching regardless of which expert processes it. We introduce TriRoute, a single lightweight controller shared across all three axes that, for every token at every layer, emits a coordinated policy: (i) an attention mode (skip/local/full), (ii) a sparse set of FFN experts (with a null expert recovering MoD), and (iii) a KV-cache bit-width. The controller trains end-to-end via a heterogeneous relaxation (Gumbel-Softmax with straight-through estimation for categorical decisions and load-balanced top-k gating for experts) under a Lagrangian budget constraint that turns the average compute and memory cost into a controllable knob. We identify a cross-axis routing-collapse cascade in naive joint training, where collapse on one axis propagates to the others, and address it with per-axis normalization and a coupling-aware balancing loss. On decoder-only models from 160M to 1.3B parameters at compute-optimal token counts, TriRoute Pareto-dominates the best independent MoD+MoE+KV-quantization combination at matched inference FLOPs and memory, while better preserving tail-case robustness on rare entities, code, and arithmetic that pure perplexity optimization erodes. Post-hoc analysis reveals interpretable structure: the controller allocates full attention and high-precision cache to sentence-initial positions, rare subwords, and named entities, while cheaply routing function words.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前大语言模型推理中，三种主流的条件计算技术——混合专家（MoE）稀疏化前馈网络、混合深度（MoD）跳过整个 Transformer 块、KV 缓存量化压缩注意力内存——**各自独立发展**，仅作用于单一轴（FFN 专家、块深度、缓存精度）。然而，这三个决策（注意力分辨率、专家选择、缓存位宽）在实际中是**强耦合**的：一个罕见实体 token 可能需要完全的注意力分辨率和高精度缓存，但可以跳过 FFN 变换；而一个功能词可能三方面都可以廉价处理。孤立优化导致资源分配次优，且无法表达真实的协同策略。

- **研究意义**：本文旨在用一个**统一的轻量级控制器**，为每个 token 的每一层联合决定上述三个轴上的资源分配，在单一可调预算下实现更优的推理效率与质量折中，并保持对稀有输入的鲁棒性。

## 2. 方法论

### 2.1 核心思想

- 设计一个**共享的控制器**（两层的 MLP，宽度 128），输入为归一化残差状态和廉价侧特征（相对位置、边界、预测熵、上一层的决策），输出三个独立的头：
  - **注意力头**：选择模式 a∈{skip, local-window, full causal}，控制 token 如何读取过去。
  - **专家头**：从 E 个 FFN 专家（含一个空专家 f₀=0）中选 top-k（k=2，null 可占一个），实现可跳过的 FFN。
  - **KV 位宽头**：选择 b∈{2,4,8,16}，控制 token 的 KV 值如何写入缓存供未来查询。

- 三个头的输出经对应层执行后，产生协调的推理路径，通过**可微成本模型**（FLOPs 和内存期望）与语言模型损失一起反向传播。

### 2.2 关键技术细节

- **异构松弛**：
  - 注意力模式和位宽采用**直通 Gumbel-Softmax**（温度从 2.0 退火到 0.5）。
  - 专家选择保持标准可微 top-k softmax 门控（配合负载均衡损失）。
- **跨轴梯度平衡**：对每个头施加梯度范数的运行指数移动平均（EMA），对 logits 进行归一化，防止注意力头梯度主导共享主干，使位回头得以学习。
- **耦合感知平衡损失**（解决跨轴坍缩级联）：
  - 每轴特征白化（减去均值除以标准差）。
  - 每轴边际熵下限损失，当使用熵低于阈值 ζ·log|O| 时激活惩罚。
  - 标准负载均衡损失（Switch 风格）和 z-loss 用于每轴。
- **可微多资源成本模型**：每个决策都赋予相应 FLOPs 和内存成本（注意力：0 / 2d min(t,w) / 2dt；专家：3dd_f * 激活数；缓存：2 d_kv b）。总成本归一化为密集模型的比值。
- **在线拉格朗日约束**：对 FLOPs 和内存分别设置目标预算 C*_r，通过投影对偶更新 λ_r，将三个轴的资源压力融入单一可控旋钮。

- **推理**：采用硬 argmax 决策，仅执行选中的计算，支持左到右单遍推理，兼容批处理。

## 6. 主要结论与发现

1. **联合路由优于独立组合**：在 160M–1.3B 三个规模下，TriRoute 在所有预算水平上帕累托支配独立的 MoD+MoE+KV-quant 组合，尤其在计算紧张时（35–55% FLOPs）差距最大。
2. **恢复密度模型质量**：仅用 ~55% FLOPs 和 ~40% 缓存即达到密集模型困惑度，同时保留 96–99% 的下游准确率。
3. **保持尾案例鲁棒性**：独立组合在稀有实体、代码、数学上退化严重（+2.1, +0.4, +1.7 困惑度），而 TriRoute 退化很小（<0.4），GSM8K 从 3.6% 提升至 4.5%（接近密集 4.8%）。
4. **控制器学习可解释结构**：
   - 完全注意力和高精度缓存分配给句首、罕见子词和命名实体。
   - FFN 计算与注意力解耦：罕见实体常跳过 FFN 但保留完全注意力（即“读取/写入” vs “转换”解耦）。
   - 早层注意力高，晚层选择性高；专家激活深度增加趋势。
5. **跨轴耦合不对称**：注意力与位宽强耦合（互信息 ~0.31 nats），FFN 更独立（<0.08 nats），解释了共享主干+分离头的正确性。

## 7. 优点

- **创新性**：首次将三个异构决策（注意力模式、专家选择、缓存位宽）统一在一个控制器和单一预算下，提出跨轴坍缩级联问题并给出实用解决方案。
- **方法完整性**：异构松弛、梯度平衡、耦合感知平衡、可微成本模型、在线拉格朗日约束等一整套设计，训练稳定且可迁移。
- **实验扎实**：多模型规模、完整消融、尾案例分析、可解释性分析，且提供了参考实现。
- **实用价值**：真实解码加速（1.7× 吞吐提升），且尾案例保持好，对于部署中公平性至关重要。
- **可解释性**：路由模式聚类揭示语言学结构，便于理解和调试。

## 8. 不足与局限

- **规模局限**：仅验证到 1.3B，未在 ≥10B 或极端稀疏度（如 8% 活跃 FLOPs）下测试，坍缩级联是否随规模扩大更容易或更难控制未知。
- **实际加速低于理论**：由于不规则注意力（每头不同模式）、混合精度缓存、路由器同步点，实现速度（1.7×）低于 FLOP 比例（1/0.55≈1.8×），需专用内核优化。

- **跨领域公平性未测试**：尾案例桶是代理，未研究低资源语言、方言等；存在下沉退化风险（虽然优于独立组合）。
- **计算资源未量化**：未报告训练具体 GPU 型号、数量、总时长，难以复现算力消耗。
