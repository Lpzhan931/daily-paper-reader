## 摘要
仅解码器（decoder-only）语言模型中的长上下文推理成本高昂，因为长提示词在预填充（Prefill）阶段被处理，在每一层都被缓存，并在自回归解码（Decode）阶段被反复关注。我们提出了“浅层预填充，深层解码”（SPEED），这是一种阶段不对称的 KV 可见性策略，它仅在较低层实例化非锚点提示词标记（non-anchor prompt-token）的 KV 状态，同时保持解码阶段的标记为全深度。与以往降低高层提示词 KV 状态存储或构建成本的方法不同，SPEED 完全从高层解码可见性集合中移除了预填充标记。通过使用极小的 BoS 锚点，这种简单的改变在降低长上下文成本的同时，保持了广泛的基准测试质量。在一项受控的 Llama-3.1-8B 指令微调研究中，仅对 75% 的层使用预填充标记的 SPEED 在 OLMES 风格的基准测试中达到了 51.2 的平均分（全深度基准模型为 51.4），同时在 128K 上下文下将首字延迟（TTFT）提升了 33%，逐字延迟（TPOT）提升了 22%，并减少了 25.0% 的活跃 KV 显存占用。逐层诊断表明，这种截断保留了全深度模型中主要的提示词选择和表示稳定区域。这些结果表明，当解码阶段的标记保持全深度时，长上下文提示词标记并不总是需要作为全深度的 KV 缓存对象存在。

## Abstract
Long-context inference in decoder-only language models is costly because long prompts are processed during Prefill, cached at every layer, and repeatedly attended to during autoregressive Decode. We introduce \emph{Shallow Prefill, dEEp Decode} (SPEED), a phase-asymmetric KV-visibility policy that materializes non-anchor prompt-token KV states only in lower layers while keeping Decode-phase tokens full-depth. Unlike previous approaches that make upper-layer prompt KV states cheaper to store or construct, SPEED removes prefill tokens from the upper-layer Decode visibility set altogether. With a minimal BoS anchor, this simple change preserves broad benchmark quality while reducing long-context cost. In a controlled Llama-3.1-8B instruction-tuning study, SPEED using only 75\% of layers for prefill tokens reaches 51.2 average score on OLMES-style benchmarks, compared with 51.4 for the full-depth baseline, while improving TTFT by 33\%, TPOT by 22\%, and reducing active KV memory by 25.0\% at 128K context. Layer-wise diagnostics suggest that this cutoff retains the main prompt-selection and representation-stabilization regions of the full-depth model. These results show that long-context prompt tokens need not always persist as full-depth KV-cache objects when Decode-phase tokens remain full-depth.

### 1. 核心问题与整体含义
*   **研究背景**：在 Decoder-only 架构的大语言模型（LLM）中，长上下文推理面临三大瓶颈：预填充（Prefill）阶段的高延迟（TTFT）、解码（Decode）阶段受内存带宽限制的逐字延迟（TPOT），以及随上下文长度线性增长的 KV 缓存显存占用。
*   **核心动机**：作者观察到，模型高层对预填充 Token 的注意力权重逐渐减弱，信息处理趋于冗余。因此提出：是否可以只在低层保留长提示词的 KV 状态，而在高层将其移除，从而实现推理加速和显存节省？

### 2. 方法论：SPEED 策略
*   **核心思想**：采用**阶段不对称（Phase-asymmetric）**的 KV 可见性策略。将模型分为低层（1 到 $K$ 层）和高层（$K+1$ 到 $L$ 层）。
*   **关键技术细节**：
    *   **浅层预填充（Shallow Prefill）**：非锚点预填充 Token 仅在低层（前 $K$ 层）进行计算并存储 KV 缓存。
    *   **深层解码（Deep Decoding）**：解码阶段生成的 Token 依然经过所有 $L$ 层计算，并生成全深度的 KV 缓存。
    *   **层不对称可见性**：在高层（$l > K$）中，解码 Token 的注意力机制不再关注非锚点的预填充 Token，仅关注：1) 之前的解码 Token；2) 极少数预定义的“锚点”Token。
    *   **BoS 锚点（BoS Anchoring）**：使用现有的 BoS（序列起始）Token 作为唯一的全深度锚点。实验发现，仅保留这一个 Token 就能起到稳定生成、防止模型在高层因失去参考而崩溃的作用。
    *   **SPEED-aware 训练**：为了消除训练与推理时的可见性不匹配，在指令微调阶段就引入这种不对称可见性约束。

### 6. 主要结论与发现
*   **性能平衡点**：在 Llama-3.1-8B 上，$K=24$（即仅 75% 的层处理预填充 Token）是最佳平衡点。其平均分（51.2）与全深度基准（51.4）几乎持平。
*   **效率提升**：在 128K 上下文下，SPEED-24 相比全深度模型：
    *   **TTFT（首字延迟）降低 33%**。
    *   **TPOT（逐字延迟）降低 22%**。
    *   **活跃 KV 显存减少 25%**。
*   **锚点作用**：BoS 锚点能有效抑制“后缀重复循环”故障，是浅层预填充方案成功的关键。
*   **任务敏感度**：代码任务对预填充深度最不敏感，而数学和复杂指令遵循任务则需要更深的预填充可见性。

### 7. 优点
*   **全方位优化**：不同于以往只优化预填充或只优化解码的方法，SPEED 同时降低了延迟并节省了显存。
*   **实现简单**：不需要复杂的动态 Token 选择算法或额外的压缩模块，仅通过修改注意力掩码和缓存逻辑即可实现。
*   **兼容性强**：证明了该方法可以通过轻量级的 LoRA 适配应用于现有的现成（Off-the-shelf）指令模型。

### 8. 不足与局限
*   **模型架构局限**：实验主要集中在 Llama-3.1-8B 上，尚未在更大规模（如 70B+）或其他非 Transformer 变体架构上验证。
*   **适配成本**：虽然支持 LoRA，但为了达到最佳性能，仍建议进行 SPEED-aware 的微调，这对于完全不想重新训练的用户存在一定门槛。
*   **硬件依赖**：实际的加速效果高度依赖于推理后端（如内核实现、内存带宽），在不同的推理框架下表现可能存在差异。
