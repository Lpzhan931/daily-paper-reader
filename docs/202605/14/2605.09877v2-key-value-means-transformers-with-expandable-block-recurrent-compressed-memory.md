## 摘要
我们提出了 Key-Value Means（“KVM”），这是一种针对注意力机制的新型块循环（block-recurrence）方法，可以容纳固定大小或增长的状态。为强大的 Transformer 基准模型配备固定大小的 KVM 注意力层，可以产生一个强大的 $O(N)$ 分块 RNN，且仅增加极少量的参数。我们训练了一个带有可增长 KVM 缓存的 Transformer，并证明它在长上下文测试中表现出色，且仅需次二次（subquadratic）预填充时间和次线性（sublinear）状态增长。KVM 可以使用标准操作实现，无需自定义内核，并支持分块并行的训练和预填充。它在一个统一的框架中集成了传统 Transformer（可扩展的上下文内存、分块并行的训练和预填充）和线性 RNN 的许多优点。它可以应用于每一层，节省 KV 缓存内存，并允许在 $O(N)$ 到 $O(N^2)$ 之间连续选择预填充时间复杂度。它还可以作为混合解决方案与 LRNN 层协同实现，以取代传统的注意力机制，通过改进的次线性内存增长上下文长度使用和长上下文解码来补充 LRNN。我们在 Apache 2.0 许可下发布了代码（https://github.com/recursal/KVM-paper）和训练好的模型（https://huggingface.co/collections/recursal/key-value-means）。

## Abstract
We present Key-Value Means ("KVM"), a novel block-recurrence for attention that can accommodate either fixed-size or growing state. Equipping a strong transformer baseline with fixed-size KVM attention layers yields a strong $O(N)$ chunked RNN, while adding only an insignificant number of new parameters. We train a transformer with a growable KVM cache and show it performs competitively on long-context tests with only subquadratic prefill time and sublinear state growth. KVM is implementable with standard operations and without custom kernels, and supports chunk-wise parallelizable training and prefill. It provides many of the benefits of both traditional transformers (expandable context memory, chunk-wise parallelizable training and prefill) and linear RNNs in a single unified package. It can be used on every layer, saving KV-cache memory, and allowing a continuous range of choices of prefill time complexity between $O(N)$ and $O(N^2)$. It can also be implemented in a hybrid solution in tandem with LRNN layers in place of traditional attention, to supplement the LRNN with improved sublinear memory growth context length usage and long context decoding. We release our code at https://github.com/recursal/KVM-paper and trained models at https://huggingface.co/collections/recursal/key-value-means under the Apache 2.0 license.

传统的 Transformer 在处理长序列时面临显存和计算量的二次方增长（$O(N^2)$），而线性 RNN（LRNN）虽然推理成本恒定，但在长上下文记忆和检索（Recall）方面表现较弱。
**研究动机**：开发一种能够根据需求动态扩展状态（State）的架构，在保持 $O(N)$ 或亚二次方复杂度的同时，提供接近全注意力机制的记忆质量。KVM 旨在填补“固定状态 RNN”与“全缓存 Transformer”之间的空白。

### 2. 方法论：核心思想与关键技术
KVM 的核心是将**分块滑动窗口注意力（BSWA）**与**可扩展的压缩循环内存**相结合：
*   **状态压缩（Merge）**：当一个 Token 块从滑动窗口溢出时，系统计算溢出 Token 与当前状态（State）的相似度。采用“胜者全得（Winner-take-all）”策略，将溢出 Token 合并到最相似的状态槽位中。
*   **状态扩展（Append）**：如果溢出 Token 的特征非常新颖（冗余度低），系统会动态增加状态槽位，直接将其追加到内存中。这实现了状态大小的亚线性增长（如 $\sqrt{N}$）。
*   **即时重归一化（JIT Renormalization）**：为了防止向量在多次合并后模长塌缩，KVM 在注意力计算前对 Key 和 Value 进行即时归一化，并保留了“汇聚 Token（Sink Tokens）”的原始模长特性。
*   **位置编码兼容性**：通过“部分 RoPE（Partial RoPE）”方案，在滑动窗口部分保留旋转位置编码，而在压缩状态部分将其置零，解决了压缩内存与位置敏感性之间的冲突。
*   **算法流程**：输入序列被分为块，每块处理完后，溢出部分通过相似度检索决定是“合并”还是“新增”到状态中，随后下一块 Query 同时对滑动窗口和压缩状态进行注意力检索。

*   **状态增长优势**：通过亚线性增长策略（如 $\sqrt{N}$），KVM 能够以极小的内存代价处理极长上下文，解决了传统 RNN 容易遗忘早期信息的问题。

### 7. 优点
*   **灵活性**：提供了一个连续的权衡空间，用户可以根据算力预算自由选择状态增长曲线（从 $O(1)$ 到 $O(N)$）。
*   **易于实现**：不需要编写复杂的自定义 CUDA 内核，使用标准深度学习算子即可高效运行。
*   **统一性**：将压缩、检索、滑动窗口集成在单个 Softmax 注意力层中，架构简洁。

### 8. 不足与局限
*   **模型规模限制**：实验仅在 350M 以下规模进行，尚未在大规模参数（如 7B+）模型上验证其扩展性（Scaling Laws）。
*   **增长策略简单**：目前的状态增长（Append）采用的是预设的固定时间表（如 $\sqrt{N}$），而非完全由数据驱动的自适应增长。
*   **RoPE 损失**：为了兼容压缩状态，牺牲了部分维度的位置表达能力，可能在极精细的位置感知任务中存在瓶颈。
