## 摘要
大语言模型（LLMs）在多种任务中表现出强大的性能，但其在长输入上下文下的推理受到内存容量和带宽的瓶颈限制。键值（KV）缓存的大小随序列长度线性增长，且在每个解码步骤中都需要从片外高带宽内存（HBM）重新读取到片内内存，导致推理过程受内存限制。现有方法通过剔除或量化来减小缓存，但通常将两者孤立对待。在本文中，我们将 KV 缓存压缩建模为一个率失真问题，在此框架下，剔除和量化是同一种比特分配方案的两个极端。这揭示了联合优化两者的必要性，从而促使我们提出了 RDKV（率失真 KV 缓存压缩）方法。RDKV 根据压缩对注意力计算造成的失真来推导每个 token 或通道的权重。基于这些权重，它在预填充阶段后应用一次反向注水算法，为每个 token 或通道分配从全精度到零比特不等的位宽。在 LongBench、RULER 和 InfiniteBench 上的实验表明，RDKV 的表现平均优于最佳评估基准 9.1%。在 LongBench 上，它仅需 2.48% 的缓存保留率即可恢复 97.81% 的全缓存准确率。与全缓存 FlashAttention-2 解码相比，在 128K 上下文长度下，它在保持相当性能的同时，实现了 4.5 倍的解码加速和 1.9 倍的峰值内存降低。

## Abstract
Large language models (LLMs) have shown strong performance across diverse tasks, but their inference with long input contexts is bottlenecked by memory size and bandwidth. The Key-Value (KV) cache size grows linearly with sequence length and needs to be re-read from off-chip high-bandwidth memory (HBM) to on-chip memory at every decoding step, resulting in memory-bound inference. Existing methods reduce the cache by either eviction or quantization, but typically treat the two in isolation. In this paper, we cast KV cache compression as a rate-distortion problem, under which eviction and quantization are two end-points of the same bit allocation scheme. This exposes the need to optimize them jointly, motivating our method, RDKV (Rate-Distortion KV cache compression). RDKV derives the weight of each token or channel from the distortion that compression induces on the attention computation. Based on these weights, it assigns each token or channel a bit-width ranging from full precision down to zero bits guided by reverse water-filling, applied once after the prefilling stage. Experiments on LongBench, RULER, and InfiniteBench show that RDKV outperforms the best evaluated baseline by 9.1% on average. On LongBench it recovers 97.81% of full-cache accuracy with only 2.48% cache retention. Compared with full-cache FlashAttention-2 decoding, it achieves 4.5x decode speedup and 1.9x peak memory reduction with 128K context length, while maintaining comparable performance.

*   **核心问题**：大语言模型（LLM）在处理长文本推理时，键值（KV）缓存随序列长度线性增长，导致内存容量和带宽成为严重瓶颈（Memory-bound）。
*   **研究动机**：
    *   现有压缩方法主要分为**剔除（Eviction）**和**量化（Quantization）**两类，但通常被孤立对待。
    *   剔除是“全有或全无”的二元决策，忽略了中间精度的可能性；量化则往往保留了所有 token，包括贡献极小的冗余信息。
    *   作者认为剔除和量化本质上是同一比特分配问题的两个极端，应在统一框架下进行联合优化。

### 2. 方法论：RDKV 框架
*   **核心思想**：将 KV 缓存压缩建模为**率失真（Rate-Distortion, R-D）**优化问题。在给定总比特预算的前提下，为每个缓存单元分配最优位宽，以最小化注意力计算的加权失真。
*   **关键技术细节**：
    *   **失真权重推导**：
        *   **V 缓存（Token 维度）**：权重 $w_t$ 定义为该 token 在所有查询位置上的累积注意力分数。
        *   **K 缓存（通道维度）**：权重 $w_c$ 基于查询（Q）和键（K）矩阵对应列向量范数的乘积，捕捉对 logit 矩阵的影响。
    *   **比特分配算法**：
        *   将问题转化为**多重选择背包问题（MCKP）**。
        *   使用**反向注水算法（Reverse Water-filling）**和拉格朗日松弛法求解。
        *   可选位宽集合为 $\{0, 2, 4, 8, 16\}$ 比特，其中 0 比特等同于剔除。
    *   **硬件实现（TriZone 布局）**：
        *   为了将理论节省转化为实际速度提升，设计了 TriZone 缓存布局。
        *   将缓存分为三个区域：Zone A（打包的量化数据）、Zone B（FP16 保留数据）、Zone C（新生成的解码数据）。
        *   将反量化操作融合进注意力算子（Kernel Fusion），避免了额外的内存读写开销。

*   **理论完备**：首次将 KV 缓存压缩严谨地建模为率失真问题，消除了剔除和量化之间的人为界限。
*   **动态分配**：无需预设量化比例或剔除阈值，算法根据数据重要性自动决定哪些 token 该丢弃、哪些该低比特保留。
*   **硬件友好**：通过 TriZone 布局和算子融合，解决了混合位宽带来的计算分支和内存对齐难题，实现了真正的端到端加速。

### 8. 不足与局限
*   **静态分配限制**：目前的比特分配是在预填充阶段后一次性完成的（Frozen），无法根据解码过程中注意力模式的动态漂移进行实时调整。
*   **校准依赖**：算法依赖于预先计算的经验失真表（Calibration），虽然开销小，但仍需针对不同模型进行一次性离线校准。
*   **流式处理**：对于无限流式生成的场景，目前的策略可能需要进一步扩展以支持滑动窗口或动态重分配。
