## 摘要
视觉自回归（VAR）模型近期在保持低延迟的同时，展现出了令人印象深刻的图像生成质量。然而，它们面临严重的 KV 缓存（KV-cache）内存限制，每生成一张图像通常需要数 GB 的内存。我们提出了 HeatKV，这是一种新颖的压缩方法，它根据每个注意力头对先前生成尺度的注意力权重来调整其缓存分配。利用一个较小的离线校准集，根据注意力头在先前尺度上的注意力得分对其进行排名。基于该排名，我们针对给定的内存预算构建了一个静态剪枝方案。在 Infinity-2B 模型上的应用表明，与现有方法相比，HeatKV 在 KV 缓存内存分配方面实现了 2 倍更高的压缩率，同时保持了相似或更好的图像保真度、提示词对齐度以及人类感知评分。我们的方法在 VAR 模型 KV 缓存压缩方面达到了新的最先进水平（SOTA），展示了细粒度、特定于头部的缓存分配的有效性。

## Abstract
Visual Autoregressive (VAR) models have recently demonstrated impressive image generation quality while maintaining low latency. However, they suffer from severe KV-cache memory constraints, often requiring gigabytes of memory per generated image. We introduce HeatKV, a novel compression method that adapts cache allocation in each head based on its attention to previously generated scales. Using a small offline calibration set, the attention heads are ranked according to their attention scores over prior scales. Based on this ranking, we construct a static pruning schedule tailored to a given memory budget. Applied to the Infinity-2B model, HeatKV achieves $2 \times$ higher compression ratio in memory allocation for KV cache compared to existing methods, while maintaining similar or better image fidelity, prompt alignment and human perception score. Our method achieves a new state-of-the-art (SOTA) for VAR model KV-cache compression, showcasing the effectiveness of fine-grained, head-specific cache allocation.

### 1. 核心问题与研究动机
*   **研究背景**：视觉自回归（VAR）模型通过逐尺度（scale-by-scale）预测图像标记（tokens），在生成质量和速度上表现优异。
*   **核心问题**：由于 VAR 模型在每个生成步骤中会产生大量标记，其 KV 缓存的增长呈超线性。生成一张高分辨率图像（如 1024x1024）可能需要高达 10GB 的显存，这严重限制了并发用户数和批处理大小（Batch Size）。
*   **现有局限**：现有的压缩方法（如 ScaleKV 或 HACK）通常采用粗粒度的策略，例如将整个层或整个注意力头简单分类为“重要”或“不重要”，导致内存分配不够精细，存在浪费。

### 2. 方法论
HeatKV 的核心思想是**细粒度的头部微调分配**，即为每个注意力头在每个图像尺度上独立分配内存预算。
*   **S-CAS（尺度相关累积注意力得分）**：利用少量离线校准图像，计算每个注意力头（Head）对过去不同尺度（Scale）的平均注意力权重。通过 S-CAS 对“层-头-尺度”三元组进行重要性排序。
*   **静态剪枝方案**：根据预设的内存预算（如原始大小的 10%），依据 S-CAS 排名确定哪些头部的哪些尺度应该被保留或剔除。
*   **贪婪早期剪枝算法（Greedy Early Pruning）**：为了防止在生成过程中出现瞬时显存峰值（Transient Over-allocation），算法会在每一层执行时动态检查并提前剔除低贡献的缓存，确保显存占用始终低于预算。
*   **保留策略**：始终保留前几个尺度作为“注意力锚点”（Attention Sinks），因为这些初始尺度对全局结构至关重要。

### 3. 实验设计
*   **基准模型**：Infinity-2B 和 Infinity-8B（目前最先进的 VAR 模型）。
*   **对比方法**：StreamingLLM、SnapKV、HACK、ScaleKV（当前的 SOTA 方法）。
*   **数据集与指标**：
    *   **保真度（Fidelity）**：在 MS-COCO 2017 验证集上测量 FID、LPIPS 和 PSNR。
    *   **提示词对齐（Alignment）**：使用 GenEval 评估对象计数、位置、颜色等属性。
    *   **人类感知**：使用 HPSv2.1（人类偏好得分）评估视觉美感。
*   **压缩梯度**：测试了 20%、10% 和 4% 三种不同的内存预算水平。

### 4. 资源与算力
*   **硬件设备**：Infinity-2B 的实验和延迟测量在 **NVIDIA H100 GPU** 上完成；Infinity-8B 的实验在 **NVIDIA B200 GPU** 上完成。

*   **推理延迟**：在 Batch Size 为 8 或 16 时，HeatKV 相比全缓存模式能显著降低延迟（因为避免了 OOM 并减少了数据搬运）。

### 5. 实验数量与充分性
*   **实验规模**：论文涵盖了从 2B 到 8B 不同参数规模的模型，并在多个维度（保真度、对齐度、感知质量）进行了详尽对比。
*   **消融实验**：作者对校准提示词的数量（1 到 10 个）、锚点尺度（Sink Scales）的数量以及剪枝算法的有效性进行了消融研究。
*   **客观性**：实验对比了多种主流的 LLM 和 VAR 专用压缩技术，结果显示 HeatKV 在极低预算（4%）下依然能保持较高的图像质量，实验设计较为全面且客观。

### 6. 主要结论与发现
*   **性能突破**：在相同图像质量下，HeatKV 实现的压缩率比之前的 SOTA 方法（ScaleKV）高出 **2 倍**。
*   **稳定性**：注意力模式在不同提示词之间具有高度稳定性，这意味着少量的离线校准即可泛化到各种生成场景。
*   **显存效益**：在 10% 的预算下，Infinity-8B 的 KV 缓存从 42GB 骤降至 2.1GB，使得在消费级显卡上运行大规模 VAR 模型成为可能。

### 7. 优点
*   **高精细度**：打破了以往“层级”或“头级”的二元分类，实现了“头-尺度”级别的精细控制。
*   **即插即用**：无需微调模型参数，仅需简单的离线分析即可应用。
*   **显存管理优化**：提出的贪婪早期剪枝算法解决了 VAR 模型特有的超线性增长导致的瞬时显存溢出问题。

### 8. 不足与局限
*   **注意力稳定性假设**：虽然实验证明了稳定性，但在极端或分布外（OOD）的提示词下，预设的静态剪枝方案可能不是最优的。
*   **尺度内均匀性假设**：HeatKV 假设同一尺度内的所有标记重要性相同，未考虑尺度内部的局部稀疏性（例如图像中背景区域的标记可能比前景物体更不重要）。
*   **应用范围**：目前仅在 VAR 模型上验证，尚未探讨其在视频生成或其他多模态自回归模型中的表现。
