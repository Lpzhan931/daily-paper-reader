## 摘要
长上下文推理越来越多地在驻留 CPU 的 KV 缓存上运行，这要么是因为解码时的 KV 状态超过了 GPU 显存容量，要么是因为解耦的预填充-解码系统将 KV 数据放置在主机内存中。尽管在这种情况下块稀疏注意力（block-sparse attention）降低了注意力计算开销，但仅靠稀疏性不足以实现端到端的效率。仅 GPU 的设计仍受限于 PCIe 带宽和元数据内存开销，而 CPU-GPU 混合设计仍面临显著的 GPU 空闲时间以及 CPU 端 top-k 选择和稀疏注意力计算的瓶颈。Fluxion 基于三个关键见解：输出感知的 KV 预算分配、针对特定注意力头且感知粒度的稀疏配置，以及针对驻留 CPU 的 KV 缓存的跨设备协同执行。在这些见解的指导下，Fluxion 结合了轻量级注意力头属性预测器、粒度-预算选择器和基于优先级的调度器，以共同优化预算分配、稀疏配置和 CPU-GPU 执行重叠。这种协同设计使混合稀疏注意力在长上下文推理中能够兼顾准确性和系统效率。在 2 个模型、3 个基准测试和 40 个任务中，Fluxion 很好地保持了质量——相对于全量注意力（FULL），最差的平均性能下降仅为 -0.26，同时相比于 KV 预算仅为 0.05 的最强固定稀疏混合基准，实现了 1.5 倍至 3.7 倍的加速。

## Abstract
Long-context inference increasingly operates over CPU-resident KV caches, either because decoding-time KV states exceed GPU memory capacity or because disaggregated prefill-decode systems place KV data in host memory. Although block-sparse attention reduces attention cost in this setting, sparsity alone is insufficient for end-to-end efficiency. GPU-only designs remain constrained by PCIe bandwidth and metadata memory overhead, while CPU-GPU hybrid designs still suffer from substantial GPU idle time and bottlenecks in CPU-side top-k selection and sparse attention computation.   Fluxion is built on three key insights: output-aware KV budgeting, head-specific and granularity-aware sparse configuration, and cross-device coordinated execution for sparse attention over CPU-resident KV caches. Guided by these insights, Fluxion combines a lightweight head-property predictor, a granularity-budget selector, and a priority-based scheduler to jointly optimize budget allocation, sparse configuration, and CPU-GPU execution overlap. This co-design enables hybrid sparse attention to achieve both accuracy and system efficiency in long-context inference. Across 2 models, 3 benchmarks, and 40 tasks, Fluxion preserves quality well -- the worst average degradation is only -0.26 relative to FULL, while delivering 1.5$\times$-3.7$\times$ speedup over the strongest fixed sparse hybrid baseline, whose KV budget is only 0.05.

### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）处理的上下文长度不断增加，KV 缓存（KV Cache）的大小迅速超过了 GPU 显存容量。现有的解决方案通常将 KV 缓存卸载（Offloading）到 CPU 内存中。然而，现有的稀疏注意力机制在 CPU-GPU 异构架构下存在严重瓶颈：
*   **GPU-only 设计**：受限于 PCIe 带宽，且在 GPU 上存储元数据会带来额外的显存开销。
*   **CPU-GPU 混合设计**：虽然减少了数据传输，但由于 CPU 端 Top-K 选择和稀疏计算速度慢，导致 GPU 出现大量空闲等待时间（Idle Time）。
*   **核心挑战**：如何在保证推理精度的前提下，消除 CPU-GPU 之间的执行瓶颈，实现真正的端到端加速。

### 2. 方法论：核心思想与关键技术
论文提出了 **Fluxion** 框架，其核心思想是通过“软硬件协同设计”优化预算分配、配置选择和任务调度。

*   **输出感知的 KV 预算分配（Output-aware Budgeting）**：
    *   **核心发现**：传统的基于注意力分数（Score-based）的预算分配不准确，因为“注意力汇”（Attention Sink）标记分数虽高但对输出贡献小（Value 向量范数小）。
    *   **技术细节**：Fluxion 以最小化注意力输出偏差（Output Deviation）为目标分配预算，并考虑不同注意力头对最终表示的相对贡献。
*   **头特定且感知粒度的稀疏配置**：
    *   **分类处理**：将注意力头分为“流式头”（Streaming heads，仅需局部缓存）和“检索头”（Retrieval heads，需全局 Top-K 检索）。
    *   **耦合模型**：建立预算（$bgt$）与块粒度（$blk$）的线性关系模型：$bgt(blk) = bgt_0 + k \cdot \log_2(blk)$，动态选择最优的块大小以平衡 CPU 计算开销和精度。
*   **跨设备协同执行与优先级调度**：
    *   **并行架构**：将稀疏注意力任务抽象为独立单元，由 CPU 和 GPU 作为独立工人从共享队列中提取任务。
    *   **优先级策略**：基于数据访问量 $V(blk)$ 计算优先级，优先执行开销大的任务，利用 CPU-GPU 的执行重叠（Overlap）掩盖延迟。
