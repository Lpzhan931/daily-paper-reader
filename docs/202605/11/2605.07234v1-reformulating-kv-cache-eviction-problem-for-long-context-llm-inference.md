## 摘要
大语言模型（LLM）支持长上下文推理，但由于键值（KV）缓存的增长，面临巨大的内存和运行开销。现有的 KV 缓存驱逐方法主要依赖于局部注意力权重，忽略了值表示、输出投影以及多头间交互的影响。在这项工作中，我们将 KV 缓存驱逐从传统的基于多头、权重平均的方法重构为一种输出感知、层级的矩阵乘法近似问题。我们提出了 LaProx，这是一种新颖的驱逐策略，它显式地建模了注意力图与投影后的值状态之间的乘法交互，从而在考虑多头依赖关系的同时，准确量化 Token 的贡献。基于这一指标，我们提出了首个统一的驱逐策略，为 Token 分配全局可比的重要性分数，从而实现全模型范围的选择，而非局部的多头决策。在长上下文基准测试 LongBench 和 Needle-In-A-Haystack 的 19 个数据集上的实验结果表明，我们的方法在仅保留 5% KV 缓存的情况下仍能维持模型性能，并在所有配置下始终优于先前的工作。值得注意的是，与现有的最先进基准相比，我们的方法在极端压缩场景下实现了高达 2 倍的准确率损失降低，且开销极小。

## Abstract
Large language models (LLMs) support long-context inference but suffer from substantial memory and runtime overhead due to Key-Value (KV) Cache growth. Existing KV Cache eviction methods primarily rely on local attention weights, neglecting the influence of value representations, output projection, and inter-head interactions. In this work, we reformulate KV Cache eviction from a conventional head-wise, weight-averaging approach into an output-aware, layer-wise matrix multiplication approximation problem. We introduce LaProx, a novel eviction strategy that explicitly models the multiplicative interaction between attention maps and projected value states to accurately quantify token contributions while accounting for inter-head dependencies. Building on this metric, we propose the first unified eviction strategy that assigns globally comparable importance scores to tokens, enabling model-wide selection instead of local, head-wise decisions. Experimental results across 19 datasets on long-context benchmarks LongBench and Needle-In-A-Haystack demonstrate that our approach maintains model performance with only 5\% of the KV cache and consistently outperforms prior works across all configurations. Notably, our method achieves up to 2$\times$ accuracy loss reduction under extreme compression scenarios compared to existing state-of-the-art baselines with minimal overhead.

### 论文总结：重构长上下文 LLM 推理中的 KV 缓存驱逐问题

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）处理的上下文长度不断增加，键值（KV）缓存的大小随序列长度线性增长，成为内存占用和推理延迟的主要瓶颈。现有的 KV 缓存驱逐（Eviction）方法（如 H2O, SnapKV 等）主要依赖**局部注意力权重**来判断 Token 的重要性。
**研究动机：** 作者发现仅靠注意力权重是不够的，因为：
*   注意力权重（$A$）是归一化的概率，范围有限；而值表示（$V$）和输出投影矩阵（$W_O$）的量级变化巨大，对最终输出贡献影响更深。
*   现有的方法通常在每个“头”（Head）内部独立进行驱逐，忽略了多头之间和层级之间的交互。
本文旨在将驱逐问题从“局部权重平均”重构为“全局矩阵乘法近似”问题。

#### 2. 方法论：LaProx 核心思想与技术细节
作者提出了 **LaProx (Layer Approximated Cache)**，其核心思想是将 KV 缓存驱逐视为一个**矩阵乘法近似优化问题**，即寻找一个 Token 子集，使得近似后的注意力层输出与原始输出的误差最小。

*   **核心指标（Eviction Indicator）：** 基于蒙特卡洛矩阵乘法理论，Token $i$ 的重要性得分 $p_i$ 定义为注意力权重范数与投影后值状态范数的乘积：
    $$p_i \propto \|A[:, i]\|_2 \cdot \|V W_O[i, :]\|_2$$
    这种方法显式地考虑了 $V$ 和 $W_O$ 对输出的直接贡献。
*   **全局驱逐决策（Global Selection）：** 
    *   **打破头级限制：** 不再为每个头分配固定预算，而是将同一层内所有头的得分展平。
    *   **层级归一化：** 为了解决不同层之间得分量级不可比的问题，对每层的得分进行归一化处理。
    *   **模型级选择：** 在全模型或全层范围内根据归一化得分统一选择 Top-K 个 Token。这允许模型动态地为重要的头或层分配更多缓存。
*   **算法流程：** 在预填充（Prefill）阶段后计算得分，进行一次性驱逐，随后在解码阶段维持固定大小的缓存。

#### 3. 实验设计
*   **数据集/场景：**
    *   **LongBench：** 包含 16 个子数据集，涵盖单/多文档问答、摘要、少样本学习、合成任务和代码任务。
    *   **Needle-In-A-Haystack (NIAH)：** 测试大海捞针式的检索能力（包括单针和多针变体）。
    *   **RULER & InfiniteBench：** 用于测试极长上下文（高达 128K）和复杂推理任务。
*   **对比方法：** FullKV（全缓存）、StreamingLLM、SnapKV、AdaKV、CriticalKV、CAKE。
*   **测试模型：** Llama-3.1-8B-Instruct、Mistral-7B-Instruct-v0.3、Qwen3-8B。

#### 4. 资源与算力
*   **硬件：** 实验主要在单张 **NVIDIA H100 (80GB)** GPU 上进行。
*   **效率表现：** 在 128K 上下文下，LaProx 相比 FullKV 实现了约 2.3 倍的解码加速，并将峰值内存从 63.3GB 降至 47.5GB。
*   **训练时长：** 该方法属于 **Training-free（无需训练）** 和 **Calibration-free（无需校准）** 方法，因此没有训练时长，推理时的额外计算开销极小。

#### 5. 实验数量与充分性
*   **实验规模：** 论文在 3 个主流模型上测试了 19 个以上的数据集，涵盖了从 128 到 2048 不等的多种缓存预算配置。
*   **消融实验：** 详细对比了“仅注意力”、“注意力+V”、“注意力+V+W_O”的效果，并验证了全局分配策略（展平+归一化）的必要性。
*   **充分性评价：** 实验设计非常充分且客观。作者不仅对比了最终精度，还通过余弦相似度分析了近似输出与原始输出的保真度，并可视化了不同头/层之间缓存分配的动态变化。

#### 6. 主要结论与发现
*   **性能卓越：** LaProx 在仅保留 **5%** 缓存的情况下，性能显著优于所有基准方法。在极端压缩场景下，其精度损失比现有 SOTA 方法降低了多达 2 倍。
*   **全局视角的重要性：** 实验证明，Token 的重要性在不同头和层之间分布极不均匀。传统的“头级独立驱逐”会浪费空间给那些在全局贡献较小的 Token。
*   **输出感知：** 引入 $W_O$（输出投影）是准确评估 Token 贡献的关键，忽略它会导致重要性排序出现偏差。

#### 7. 优点
*   **理论完备：** 将启发式的驱逐规则提升到了矩阵近似的数学高度。
*   **全局优化：** 首次提出了模型级的统一驱逐策略，实现了自动的跨头/跨层资源分配。
*   **易于部署：** 无需微调、无需离线校准、超参数极少（几乎只有预算大小），且与 FlashAttention-2 等加速库兼容。

#### 8. 不足与局限
*   **归一化方案简单：** 目前采用的层级归一化虽然有效，但可能仍有改进空间（如更复杂的跨层权重平衡）。
*   **静态驱逐：** 驱逐主要发生在预填充之后，虽然解码时保持固定预算，但对于极长序列的持续生成，可能需要更动态的实时更新机制。
*   **模型覆盖：** 虽然测试了 MoE 模型（Qwen3），但主要实验集中在 7B-8B 规模的模型上，更大规模模型（如 70B+）的表现有待进一步验证。
