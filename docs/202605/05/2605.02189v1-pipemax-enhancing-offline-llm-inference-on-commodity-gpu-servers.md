## 摘要
离线大语言模型（LLM）推理旨在固定预算下最大化请求处理量，这使得通用 GPU 服务器成为一个极具前景的选择。然而，先前的工作通常孤立地考虑卸载（offloading）和并行化，导致性能不佳。在本文中，我们提出了 PipeMax，这是一个高吞吐量的 LLM 推理系统，它将流水线并行（pipeline parallelism）与卸载相结合，以克服 GPU 服务器上的互连和内存限制。特别是，流水线并行自然地产生较低的通信开销，并且每次在每个 GPU 上仅保持一个批次（batch）处于活跃状态，从而能够卸载非活跃批次的 KV 缓存。通过协调计算与卸载数据移动，PipeMax 有效地扩展了 GPU 内存容量并支持大批次执行。实验表明，在 8-GPU 节点上，PipeMax 的吞吐量比 vLLM 高出多达 2.51 倍，分别比最先进的高吞吐量 LLM 系统高出多达 1.42 倍和 1.38 倍。

## Abstract
Offline LLM inference seeks to maximize request processing under fixed budgets, making commodity GPU servers a promising choice. However, prior work typically considers offloading and parallelism in isolation, resulting in suboptimal performance. In this paper, we propose PipeMax, a high-throughput LLM inference system that integrates pipeline parallelism with offloading to overcome interconnect and memory constraints on GPU servers. Particularly, pipeline parallelism naturally incurs low communication overhead and keeps only one batch active on each GPU at a time, which enables offloading the KV cache of inactive batches. By coordinating computation with offloading data movement, PipeMax effectively expands GPU memory capacity and sustains large-batch execution. Experiments show that PipeMax achieves up to 2.51x higher throughput than vLLM, and up to 1.42x and 1.38x higher throughput than state-of-the-art high-throughput LLM systems, respectively, on an 8-GPU node.

*   **核心问题**：如何在预算有限的通用 GPU 服务器（如 RTX 5090、L20 等缺乏高带宽 NVLink 互连的设备）上，实现离线大语言模型（LLM）推理的高吞吐量。
*   **研究背景**：
    *   离线推理（如数据库处理、信息提取）优先考虑吞吐量而非延迟。
    *   **显存瓶颈**：LLM 推理受限于显存容量，尤其是解码阶段（Decode）庞大的 KV 缓存。
    *   **现有方案局限**：纯卸载（Offloading）受限于 PCIe 带宽；张量并行（TP）在无 NVLink 的服务器上通信开销巨大；流水线并行（PP）虽然通信开销低，但在解码阶段显存利用率低，因为非活跃批次的 KV 缓存长期占用显存。

#### 2. 方法论
PipeMax 的核心思想是将**流水线并行（PP）与 KV 缓存卸载**深度集成，利用 PP 的结构特性来掩盖卸载带来的延迟。
*   **核心机制**：
    *   **非活跃批次卸载**：在 PP 中，每个 GPU 在同一时刻只处理一个活跃批次。PipeMax 将其他非活跃批次的 KV 缓存卸载到 CPU 内存，仅在需要时预取（Prefetch），从而释放显存以支持更大的批次。
    *   **预取感知调度器（Prefetch-aware Scheduler）**：通过一个执行时间估计模型（公式：$Time = \alpha \cdot b + \beta \cdot L + \delta$，其中 $b$ 是批大小，$L$ 是总前缀长度），动态计算预取预算，确保数据传输能被计算完全掩盖。
    *   **块优先布局（Block-First Layout）**：针对 PagedAttention 进行了优化，将同一块的所有层 KV 缓存连续存储，使 PCIe 传输从碎片化变为连续化，带宽利用率从 30% 提升至 90%。
    *   **异步与优先级管理**：采用异步卸载技术，并对流水线激活值传输（高优先级）和 KV 缓存迁移（低优先级）进行优先级编排，减少互连竞争。

*   **场景限制**：该系统专为**离线吞吐量**设计，频繁的 KV 缓存换入换出可能会增加首字延迟（TTFT）或单请求延迟，不适用于对延迟极其敏感的在线对话场景。

*   **预取误差风险**：尽管估计器准确率高，但在极端不规律的生成任务中，若预测失误可能导致流水线气泡（Stall）。
