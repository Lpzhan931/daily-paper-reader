## 摘要
我们提出了 Key-Value Means（“KVM”），这是一种针对注意力机制的新型块循环方法，能够适应固定大小或增长的状态。为强大的 Transformer 基准模型配备固定大小的 KVM 注意力层，可以产生一个强大的 $O(N)$ 分块 RNN，且仅增加微不足道的新参数。我们训练了一个带有可增长 KVM 缓存的 Transformer，并证明它在长上下文测试中表现出色，且仅需次二次（subquadratic）的预填充时间以及次线性（sublinear）的状态增长。KVM 可以通过标准操作实现，无需自定义算子，并支持分块并行的训练和预填充。它在一个统一的框架中提供了传统 Transformer（可扩展的上下文内存、分块并行的训练和预填充）和线性 RNN 的许多优点。它可以应用于每一层，节省 KV 缓存内存，并允许在 $O(N)$ 到 $O(N^2)$ 之间连续选择预填充时间复杂度。它还可以作为混合方案的一部分，与 LRNN 层协同工作以取代传统注意力机制，通过改进的次线性内存增长上下文长度使用和长上下文解码来补充 LRNN。我们在 Apache 2.0 许可下发布了代码（https://github.com/featherless-ai/KVM-paper）和训练好的模型（https://huggingface.co/collections/featherless-ai/kvm-paper）。

## Abstract
We present Key-Value Means ("KVM"), a novel block-recurrence for attention that can accommodate either fixed-size or growing state. Equipping a strong transformer baseline with fixed-size KVM attention layers yields a strong $O(N)$ chunked RNN, while adding only an insignificant number of new parameters. We train a transformer with a growable KVM cache and show it performs competitively on long-context tests with only subquadratic prefill time and sublinear state growth. KVM is implementable with standard operations and without custom kernels, and supports chunk-wise parallelizable training and prefill. It provides many of the benefits of both traditional transformers (expandable context memory, chunk-wise parallelizable training and prefill) and linear RNNs in a single unified package. It can be used on every layer, saving KV-cache memory, and allowing a continuous range of choices of prefill time complexity between $O(N)$ and $O(N^2)$. It can also be implemented in a hybrid solution in tandem with LRNN layers in place of traditional attention, to supplement the LRNN with improved sublinear memory growth context length usage and long context decoding. We release our code at https://github.com/featherless-ai/KVM-paper and trained models at https://huggingface.co/collections/featherless-ai/kvm-paper under the Apache 2.0 license.

#### 1. 核心问题与整体含义（研究动机和背景）
Transformer 模型在处理长文本时面临两大瓶颈：**计算复杂度**随序列长度呈二次方增长，且 **KV 缓存（KV-cache）内存**随长度线性增长。相比之下，线性 RNN（LRNN）虽然内存固定，但在长程记忆和检索能力上通常较弱。
本文提出了 **Key-Value Means (KVM)** 架构，旨在打破这种“二选一”的局面。其核心动机是创建一个既能像 Transformer 一样并行训练和扩展内存，又能像 RNN 一样高效压缩状态的统一框架，实现在计算效率、内存占用和长文本召回率之间的连续权衡。

#### 2. 方法论：核心思想与关键技术
KVM 的核心是将**块滑动窗口注意力（BSWA）**与**可扩展的循环压缩状态**相结合：
*   **块循环机制**：模型按块（Chunk）处理输入。当一个块从滑动窗口中“溢出”时，其信息会被压缩进一个动态更新的状态（State）中。
*   **胜者为王（Winner-Take-All）合并规则**：溢出令牌（Tokens）不会简单地平均，而是根据余弦相似度找到状态中最匹配的条目进行合并。这种稀疏更新有助于保持状态条目的可区分性。
*   **状态扩展策略**：KVM 支持动态增加状态大小。它会识别最“新颖”（即与当前状态相似度最低）的溢出令牌并将其作为新条目追加到状态中，从而实现次线性（如 $\sqrt{N}$）的内存增长。
*   **即时归一化（JIT Renormalization）**：在注意力计算前对状态的 Key 和 Value 进行即时归一化，防止因多次合并导致的向量模长萎缩或爆炸。
*   **部分 RoPE 处理**：为了兼容旋转位置编码（RoPE），KVM 在压缩状态区域将 RoPE 相关的维度清零，使其具有位置无关性，而滑动窗口部分保留 RoPE 以捕捉局部位置信息。

#### 7. 优点
*   **无需自定义算子**：完全基于标准深度学习操作实现，易于在现有框架中部署。
*   **统一性**：将 Transformer 的可扩展性和 RNN 的高效性整合进单一的 Softmax 注意力层。
*   **内存节省**：显著降低了 KV 缓存的压力，尤其适合显存受限的推理场景。
*   **训练友好**：支持分块并行化训练，保持了 Transformer 的训练效率。

#### 8. 不足与局限
*   **模型规模有限**：实验主要集中在 350M 以下的小规模模型，尚未在大参数量（如 7B+）模型上验证其扩展性。
*   **RoPE 损失**：为了兼容状态压缩，牺牲了部分维度的位置表达能力（Partial RoPE），可能影响某些对位置极其敏感的任务。
*   **超参数依赖**：虽然减少了运行时优化器，但状态增长计划（Schedule）仍需预先设定或手动调优。
*   **预填充延迟**：虽然是次二次复杂度，但在极长序列下的预填充速度仍慢于纯线性模型。
