## 摘要
本研究展示了一款基于 55nm 工艺和投机解码的大语言模型（LLM）加速器，采用了基于凸点的面对面 ReRAM-on-logic 堆叠技术。其特点包括：用于无离群值低比特量化的局部旋转单元；与块级向量量化协同设计的感知堆叠近存储计算（PNM）架构，以减少权重外部存储访问（EMA）开销；以及一种带有乱序调度器的自适应并行投机解码方案，以实现高资源和带宽利用率。我们的芯片实现了 14.08 至 135.69 token/s 的吞吐量，相比原生投机解码实现了 4.46 至 7.17 倍的加速。

## Abstract
This work presents a 55nm speculative decoding-based LLM accelerator with bumping-based face-to-face ReRAM-on-logic stacking technology. It features a local rotation unit for outlier-free low-bit quantization, a stacking-aware PNM architecture co-designed with blockwise vector quantization to reduce weight EMA overheads, and an adaptive parallel speculative decoding scheme with an out-of-order scheduler for high resource and bandwidth utilization. Our chip achieves 14.08-to-135.69token/s and 4.46-to-7.17x speedup over vanilla speculative decoding.

### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLM）的自回归解码范式受限于频繁的权重外部存储访问（EMA），导致严重的延迟。虽然投机解码（SD）通过“小模型（DLM）预测、大模型（TLM）验证”的机制缓解了这一问题，但在资源受限的边缘加速器上仍面临三大挑战：
*   **量化精度损失：** 低比特量化受激活值离群值（Outliers）影响，导致精度大幅下降。
*   **片上存储不足：** 即使是较小的 DLM，其权重也难以完全放入片上 SRAM，导致频繁的外部带宽竞争。
*   **投机效率低下：** 在长草稿长度（Draft Length）下，超过 90% 的 token 会被拒绝，导致计算资源浪费。

### 2. 方法论：核心思想与关键技术
论文提出了一种基于 55nm 工艺、采用 ReRAM-on-logic 面对面堆叠技术的 LLM 加速器，核心技术包括：
*   **局部旋转单元 (LRU)：** 针对 TLM，设计了分解式快速沃尔什-哈达玛变换（FWHT）。它将深层全局旋转分解为重叠的上下两阶段局部旋转，以极小的面积开销（仅为全局旋转的 7.3%）消除激活离群值，支持稳定的 W4A8 量化。
*   **ReRAM 堆叠近存架构 (RS-PNM) 与块向量量化 (BVQ)：** 
    *   利用 3D 堆叠技术将 4 层 ReRAM 芯片通过 2048 个凸点（Bumps）直接连接在逻辑芯片上，提供 8MB 高带宽片上存储。
    *   采用 BVQ 算法将 DLM 权重聚类为块级码本（Codebooks），存储在 ReRAM 中。
    *   设计 Tile 融合单元（TFU），通过融合共享码本的 Token Tile，将码本读取延迟减半。
*   **自适应并行投机解码 (APSD) 与乱序调度器 (WDOS)：**
    *   **APSD：** 根据 TLM 的反馈动态切换“短草稿非并行”和“长草稿并行验证”模式，降低拒绝率。
    *   **WDOS：** 将工作负载解耦为 4 个并行指令队列（收发、计算、ReRAM 加载、EMA 控制），通过依赖感知同步机制实现乱序执行，最大化资源利用率。
