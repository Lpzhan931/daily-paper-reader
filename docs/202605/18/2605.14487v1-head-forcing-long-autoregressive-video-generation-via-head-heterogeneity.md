## 摘要
自回归视频扩散模型支持实时合成，但在长时程生成中面临误差累积和上下文丢失的问题。我们发现，自回归视频扩散 Transformer 中的注意力头在功能上扮演着不同的角色：用于细节细化的局部头（local heads）、用于结构稳定的锚点头（anchor heads）以及用于长程上下文聚合的记忆头（memory heads）。然而，现有方法对它们进行统一处理，导致 KV 缓存分配并非最优。我们提出了 Head Forcing，这是一个无需训练的框架，为每种类型的注意力头分配定制的 KV 缓存策略：局部头和锚点头仅保留必要的 token，而记忆头则采用具有动态情节更新的分层记忆系统，以确保长程一致性。一种按头进行的 RoPE 重编码方案进一步确保了位置编码保持在预训练范围内。在无需额外训练的情况下，Head Forcing 将生成时长从 5 秒延长至分钟级，支持多提示词交互式合成，并持续优于现有的基准方法。项目主页：https://jiahaotian-sjtu.github.io/headforcing.github.io/。

## Abstract
Autoregressive video diffusion models support real-time synthesis but suffer from error accumulation and context loss over long horizons. We discover that attention heads in AR video diffusion transformers serve functionally distinct roles as local heads for detail refinement, anchor heads for structural stabilization, and memory heads for long-range context aggregation, yet existing methods treat them uniformly, leading to suboptimal KV cache allocation. We propose Head Forcing, a training-free framework that assigns each head type a tailored KV cache strategy: local and anchor heads retain only essential tokens, while memory heads employ a hierarchical memory system with dynamic episodic updates for long-range consistency. A head-wise RoPE re-encoding scheme further ensures positional encodings remain within the pretrained range. Without additional training, Head Forcing extends generation from 5 seconds to minute-level duration, supports multi-prompt interactive synthesis, and consistently outperforms existing baselines. *   **核心问题**：自回归（AR）视频扩散模型虽然在实时合成和长视频生成方面具有潜力，但在长程生成时面临两大挑战：**误差累积**（导致颜色漂移和细节丢失）和**上下文丢失**（由于 KV 缓存容量限制导致身份漂移和场景遗忘）。
*   **研究动机**：作者发现现有的长视频生成方法（无论是基于训练还是免训练）都对 Transformer 中的注意力头（Attention Heads）采取统一的 KV 缓存策略。然而，通过分析发现，不同的注意力头在功能上具有显著的**异构性**。统一处理会导致缓存分配效率低下：不需要历史信息的头浪费了带宽，而需要长程记忆的头却缺乏足够的证据。

### 2. 方法论
核心思想是根据注意力头的功能角色，实施差异化的 KV 缓存管理框架 **Head Forcing**。
*   **注意力头画像（Profiling）**：通过离线分析，将头分为三类：
    *   **局部头（Local Heads）**：关注当前块及邻近帧，负责细节细化和运动连续性。
    *   **锚点头（Anchor Heads）**：对第一帧（Sink Frame）表现出极高的注意力，用于稳定全局结构。
    *   **记忆头（Memory Heads）**：注意力广泛分布于整个历史上下文，负责长程一致性。
*   **定制化 KV 缓存策略**：
    *   **局部/锚点头**：进行大幅度剪枝，仅保留必要的邻近帧或初始帧，释放空间并减少噪声传播。
    *   **记忆头（分层记忆系统）**：采用“快速记忆”（邻近场景）+“情节记忆”（存储代表性、干净的历史帧）。
*   **动态更新与压缩**：
    *   **场景新颖度（Scene Novelty）**：基于 Key 嵌入的余弦相似度动态决定是否将新帧存入情节记忆。
    *   **提示词引导压缩**：当情节记忆满时，根据 token 与当前文本提示词的相关性进行重要性筛选和压缩。
*   **头级 RoPE 重编码**：针对不同头维护的不同长度缓存，重新计算帧级相对位置索引，确保位置编码始终处于预训练范围内，避免分布外（O.O.D.）错误。
