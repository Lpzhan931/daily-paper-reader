## 摘要
在移动设备上部署大语言模型（LLMs）是实现 LLM 应用数据隐私和离线访问的新兴趋势。现代移动神经网络处理器（NPUs）使得这种部署变得日益可行。然而，现有的移动端 LLM 推理框架由于不可避免的冷启动（即在模型未驻留在设备内存时启动 LLM 推理）而面临高启动延迟的问题。在本文中，我们发现移动端 LLM 冷启动的关键瓶颈在于不重要的模型参数浪费了闪存带宽。我们设计了 EdgeFlow，这是一个通过自适应调整 LLM 参数精度来缓解冷启动问题的移动端 LLM 推理框架。具体而言，EdgeFlow 利用了：1）一种感知 NPU 的自适应量化算法，根据权重的重要性和 NPU 约束，以更细的粒度为权重分配不同的精度；2）一种 SIMD 友好的打包格式，可加速将各种精度的权重转换为固定大小的 NPU 原生数据类型；3）一种协同的粒度流水线，以细粒度和动态的方式协调 CPU 和 NPU 的计算。实验结果表明，在模型准确率相当的情况下，与 llama.cpp、MNN 和 llm.npu 这三种最先进的移动端 LLM 推理框架相比，EdgeFlow 将冷启动延迟降低了高达 4.07 倍。

## Abstract
Deploying large language models (LLMs) on mobile devices is an emerging trend to enable data privacy and offline accessibility of LLM applications. Modern mobile neural processing units (NPUs) make such deployment increasingly feasible. However, existing mobile LLM inference frameworks suffer from high start-up latency due to their inevitable cold starts, i.e., launching LLM inferences when the model is not hosted in device memory. In this paper, we identify the key bottleneck of mobile LLM cold starts as the waste of flash bandwidth on unimportant model parameters. We design EdgeFlow, a mobile LLM inference framework that mitigates the cold start issue by adaptively adjusting the precisions of LLM parameters. Specifically, EdgeFlow leverages 1) an NPU-aware adaptive quantization algorithm that assigns different precisions to weights in a finer granularity according to their importance and NPU constraints, 2) an SIMD-friendly packing format that accelerates the transformation of various-precision weights into fixed-sized NPU-native data types, and 3) a synergistic granular pipeline that coordinates CPU and NPU computation in a fine-grained and dynamic manner. Experimental results show that EdgeFlow reduces cold-start latency by up to 4.07x compared with three state-of-the-art mobile LLM inference frameworks, i.e., llama.cpp, MNN, and llm.npu, under comparable model accuracy.

### EdgeFlow：移动设备上大语言模型的快速冷启动

#### 1. 核心问题与整体含义
随着大语言模型（LLM）在移动端的普及，**冷启动延迟**（即模型未驻留内存时，从闪存加载并启动推理的过程）成为了用户体验的主要瓶颈。现有框架（如 llama.cpp, MNN）在冷启动时需加载数 GB 的权重，导致首字生成时间（TTFT）长达数秒甚至十秒以上。论文指出，核心瓶颈在于**闪存带宽的浪费**：现有框架对所有权重进行均匀量化，忽略了不同参数对模型精度贡献的差异，导致带宽被大量不重要的参数占用。

#### 2. 方法论
EdgeFlow 通过软硬件协同设计，提出了三个关键技术：
*   **NPU 感知的自适应量化（NPU-Aware Adaptive Quantization）：**
    *   **核心思想：** 根据权重的重要性分配不同的位宽（1-8 bit）。
    *   **技术细节：** 引入“相对误差（Relative Error, RE）”指标，通过建模期望平方误差来估算量化损失。采用**贪心位宽分配算法**，在满足 NPU 约束（如仅支持每输出通道量化）和带宽预算的前提下，最小化总 RE。
    *   **NPU 平滑：** 静态地将激活值的通道间方差转移到权重上，以适应 NPU 粗粒度的每张量（per-tensor）量化约束。
*   **SIMD 友好的打包格式（SIMD-Friendly Packing Format）：**
    *   将变长位宽权重分解为 1、2、4-bit 的“权重微元（weightlets）”，并采用交错布局存储。
    *   利用 Arm Neon 指令集实现高效的并行解包，将压缩权重快速转换为 NPU 原生的 INT8 格式，避免了传统格式（如 K-Quant）在高压缩率下的计算开销。
*   **协同粒度流水线（Synergistic Granular Pipeline）：**
    *   **细粒度算子放置：** 将矩阵乘法（Matmul）放在 NPU 上，将低算术强度的算子（如 Norm, Add）放在 CPU 上。
    *   **动态调度：** 引入“位置引导优先级”，优先处理 Prompt 前部的 Chunk 以尽早解锁下游 NPU 任务；支持“任务窃取”机制，当 CPU 空闲时主动承担 NPU 队列中的任务。

#### 3. 实验设计
*   **数据集/场景：** 使用了 LAMBADA, WinoGrande, OpenBookQA, MMLU, HellaSwag 等 5 个主流数据集，涵盖了从短到长的不同 Prompt 长度。
*   **模型：** Llama3 8B, Mistral 7B, Phi3 3.8B, Qwen1.5 1.8B。
*   **Benchmark（对比方法）：**
    *   **llama.cpp：** 基于 CPU 的主流推理框架。
    *   **MNN：** 阿里巴巴开源的移动端推理引擎。
    *   **llm.npu：** 最先进的基于 NPU 的移动端 LLM 框架（实验中对其进行了物化和重叠优化增强）。

#### 5. 实验数量与充分性
*   **实验规模：** 涵盖了 4 种模型在 5 个数据集上的端到端 TTFT 测试、困惑度（Perplexity）测试、消融实验（针对量化、打包、流水线三个组件）、内存占用及功耗分析。

#### 6. 主要结论与发现
*   **延迟降低：** 在相同精度下，EdgeFlow 的冷启动延迟比 llama.cpp 降低了 **4.07 倍**，比 MNN 降低了 **2.40 倍**，比 llm.npu 降低了 **1.55 倍**。
*   **精度保持：** 在平均 5-7 bit 的配置下，EdgeFlow 能够保持与 INT8 量化相当的准确率，显著优于传统的均匀量化方法。
*   **能效提升：** 尽管 CPU 参与更多计算导致瞬时功率上升，但由于冷启动时间大幅缩短，总能耗降低了 **12.9%**。

#### 7. 优点
*   **切中痛点：** 首次深入探讨并解决了移动端 LLM 部署中被忽视的“闪存带宽瓶颈”问题。
*   **软硬协同：** 算法设计充分考虑了移动端 NPU 的硬件约束（如对称量化、特定粒度要求），具有很强的工程落地价值。
*   **动态优化：** 任务窃取和优先级调度机制有效解决了异构计算中的负载不均衡问题。

#### 8. 不足与局限
*   **硬件依赖：** 目前的实现高度依赖高通 Hexagon NPU 及其 QNN SDK，对于 Apple A 系列芯片或其他厂商 SoC 的适配性尚未验证。
*   **低比特精度挑战：** 当平均位宽降至 4-bit 时，模型精度仍有较明显的下降，对于极度资源受限场景的适用性受限。
*   **工程复杂度：** 需要对 NPU 的私有执行图格式进行逆向工程以实现动态权重加载，这增加了在不同系统版本上的维护难度。
