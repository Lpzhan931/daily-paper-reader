## 摘要
蒙特卡洛树搜索 (MCTS) 是一种有效的测试时计算缩放 (TTCS) 方法，用于提升大语言模型的推理性能，但其高度可变的执行时间在实践中导致了严重的长尾延迟。现有的优化方法（如正向提前退出）在有利情况下可以降低延迟，但在搜索持续进行却无实质性进展时效果较差。我们引入了“负向提前退出”（negative early exit），用于剪除无成效的 MCTS 轨迹，以及一种“自适应增强机制”（adaptive boosting mechanism），通过重新分配回收的计算资源来减少并发搜索之间的资源竞争。这些技术集成到 vLLM 中后，在保持推理准确性的同时，显著降低了 p99 端到端延迟并提高了吞吐量。

## Abstract
Monte Carlo Tree Search (MCTS) is an effective test-time compute scaling (TTCS) method for improving the reasoning performance of large language models, but its highly variable execution time leads to severe long-tail latency in practice. Existing optimizations such as positive early exit, reduce latency in favorable cases but are less effective when search continues without meaningful progress. We introduce {\it negative early exit}, which prunes unproductive MCTS trajectories, and an {\it adaptive boosting mechanism} that reallocates reclaimed computation to reduce resource contention among concurrent searches. Integrated into vLLM, these techniques substantially reduce p99 end-to-end latency while improving throughput and maintaining reasoning accuracy.

### 1. 论文的核心问题与整体含义
**研究动机与背景：**
大语言模型（LLM）在处理复杂推理任务时，通过“测试时计算缩放”（TTCS）增加推理资源（如搜索和验证）能显著提升准确率。蒙特卡洛树搜索（MCTS）因其平衡探索与利用的能力，被认为是比并行采样（Best-of-N）更有效的搜索策略。然而，MCTS 存在严重的**系统性缺陷**：
*   **高延迟与长尾效应：** MCTS 本质上是顺序执行的，随着搜索深度增加，执行时间剧增。
*   **资源浪费：** 现有的“正向提前退出”（找到好答案就停止）无法处理那些搜索很久却始终无法改善结果的“死胡同”请求，导致这些请求长时间占用系统资源，造成严重的排队延迟和长尾延迟（p99 latency）。

### 2. 论文提出的方法论
论文提出了一种自适应并行 MCTS 框架，核心思想是将 MCTS 视为一个需要主动资源管理的系统，而非单纯的算法。
*   **负向提前退出（Negative Early Exit, NE）：** 
    *   **核心逻辑：** 基于过程奖励模型（PRM）的特性，如果当前搜索树中所有叶节点的得分都低于某个阈值（被称为“徒劳节点”），则判定该请求无法在预算内找到高质量解，立即终止搜索。
    *   **选择性徒劳检查：** 利用第一步奖励与最终得分的相关性，提前排除低潜力分支，进一步提高剪枝效率。
*   **并行 MCTS (WU-PUCT)：** 
    *   借鉴 WU-UCT 算法，引入“未观测计数”（unobserved counts）来修正 PUCT 选择规则，允许在统计数据尚未回传时启动多个并行 Rollout，避免探索坍缩。
*   **自适应增强机制（Adaptive Boosting）：** 
    *   **动态并行度控制：** 根据任务的“资历”（等待时间）和“紧迫性”（当前最高分接近阈值的程度）计算优先级得分。
    *   **资源重分配：** 将“提前退出”释放的计算资源动态分配给最有希望快速完成的任务，通过提高其并行度（DOP）来加速清理队列。
