## 摘要
大语言模型（LLM）中的长上下文解码受限于 I/O：每个 token 都要重新读取不断增长的 KV 缓存。先前的加速方法通过压缩（会降低保真度）或选择/剔除（会限制可访问的内容）来减少字节数，这两者都可能损害延迟召回和长文本生成能力。我们提出了 MAC-Attention，这是一种保持保真度和访问能力的替代方案，它通过复用语义相似的近期查询的先前注意力计算结果来加速解码。它包含三个阶段：匹配（match）阶段，在较短的局部窗口内执行 RoPE 前的 L2 匹配；修正（amend）阶段，通过重新计算匹配边界附近的一个窄带来修正复用的注意力；以及补全（complete）阶段，通过数值稳定的合并操作，将修正后的结果与在 KV 尾部计算的新注意力进行融合。在匹配命中时，计算和带宽复杂度与上下文长度无关，保持为常数。该方法与模型无关，且可与 I/O 感知算子、分页 KV 管理器以及 MQA/GQA 结合使用。在 LongBench v2 (120K)、RULER (120K) 和 LongGenBench (16K 连续生成) 的测试中，与最新的 FlashInfer 库相比，MAC-Attention 将 KV 访问量减少了高达 99%，在 128K 长度下将 token 生成延迟降低了 60% 以上，并实现了超过 14.3 倍的注意力阶段加速，端到端加速高达 2.6 倍，同时保持了全注意力的质量。通过复用计算，MAC-Attention 实现了既快速又可靠的长上下文推理。代码可在以下地址获取：https://github.com/YJHMITWEB/MAC-Attention.git

## Abstract
Long-context decoding in LLMs is IO-bound: each token re-reads an ever-growing KV cache. Prior accelerations cut bytes via compression, which lowers fidelity, or selection/eviction, which restricts what remains accessible, and both can degrade delayed recall and long-form generation. We introduce MAC-Attention, a fidelity- and access-preserving alternative that accelerates decoding by reusing prior attention computations for semantically similar recent queries. It starts with a match stage that performs pre-RoPE L2 matching over a short local window; an amend stage rectifies the reused attention by recomputing a small band near the match boundary; and a complete stage fuses the rectified results with fresh attention computed on the KV tail through a numerically stable merge. On a match hit, the compute and bandwidth complexity is constant regardless of context length. The method is model-agnostic and composes with IO-aware kernels, paged-KV managers, and MQA/GQA. Across LongBench v2 (120K), RULER (120K), and LongGenBench (16K continuous generation), compared to the latest FlashInfer library, MAC-Attention reduces KV accesses by up to 99%, cuts token generation latency by over 60% at 128K, and achieves over 14.3x attention-phase speedups, up to 2.6x end-to-end, while maintaining full-attention quality. By reusing computation, MAC-Attention delivers long-context inference that is both fast and faithful. Code is available here: https://github.com/YJHMITWEB/MAC-Attention.git

### 1. 核心问题与研究动机
*   **核心问题**：大语言模型（LLM）在处理长文本时，解码阶段（Decoding）面临严重的 **I/O 瓶颈**。每生成一个新 token，都需要从显存（HBM）中重新读取不断增长的整个 KV Cache。
*   **研究动机**：
    *   现有的加速方案如 **KV 压缩**（降低精度）或 **Token 剔除/选择**（限制访问范围）虽然能减少 I/O，但往往会导致模型在长程召回、复杂推理或长文本生成任务中表现下降。
    *   作者观察到解码过程存在**时间冗余性**：相邻或语义相关的查询（Query）往往具有相似的注意力分布。
    *   **目标**：开发一种既能显著加速（减少 I/O），又能保持全注意力（Full Attention）精度的方案。

### 2. 方法论：MAC-Attention 方案
MAC-Attention 采用“匹配-修正-补全”（Match-Amend-Complete）三阶段流程，核心思想是**复用相似查询的计算结果**：

1.  **匹配 (Match)**：
    *   在预设的滑动窗口 $K$ 内，寻找与当前查询 $\tilde{Q}_m$ 语义最接近的历史查询 $\tilde{Q}_p$。
    *   **关键技术**：在 **RoPE（旋转位置编码）之前**进行 L2 距离匹配。因为 RoPE 会引入位置偏差，掩盖语义相似性；预位置编码匹配能更准确地捕捉语义重复。
2.  **修正 (Amend)**：
    *   直接复用历史注意力结果会产生误差，尤其是在匹配点附近的 token（注意力权重通常较高）。
    *   **关键技术**：MAC 丢弃匹配点 $p$ 附近宽度为 $r$ 的一小段“带状区域”（Band），并针对当前查询重新计算该区域的注意力，以消除主要近似误差。
3.  **补全 (Complete)**：
    *   将复用的历史前缀摘要（已修正）与当前新生成的 KV 尾部（Tail）进行融合。
    *   **关键技术**：使用数值稳定的 **Log 域合并算法**（类似于 FlashAttention 的在线 Softmax 合并），确保最终输出在数学上高度接近全注意力结果。
*   **系统实现**：维护两个小型环形缓冲区（Query Ring 和 Summary Ring），支持常数级 $O(1)$ 的插入与查找。
