## 摘要
为了减少多头自注意力（MHA）中自注意力机制的内存占用，研究者们提出了一系列丰富的方法，例如分组查询注意力（GQA）和多头潜在注意力（MLA）。这些方法利用了跨嵌入维度或注意力头的专门低秩分解。从经典低秩逼近的角度来看，这些方法是非传统的，并引发了关于它们究竟在逼近哪些对象，以及如何解释所得表示的低秩行为的问题。为了回答这些问题，本研究提出了自注意力层中权重对象的泛化视角以及一种分解策略，使我们能够构建一种参数高效的方案，称为 Tucker Attention。在大语言模型（LLM）和视觉 Transformer（ViT）测试用例的评估中，与 GQA 和 MLA 相比，Tucker Attention 在达到同等验证指标的情况下，所需的参数量减少了一个数量级。此外，Tucker Attention 将 GQA、MLA、MHA 视为特例，并且与 Flash-Attention 和旋转位置嵌入（RoPE）完全兼容。这种泛化策略揭示了 MHA、GQA 和 MLA 所实现的实际秩，并进一步实现了对 MLA 的简化。

## Abstract
The pursuit of reducing the memory footprint of the self-attention mechanism in multi-headed self attention (MHA) spawned a rich portfolio of methods, e.g., group-query attention (GQA) and multi-head latent attention (MLA). The methods leverage specialized low-rank factorizations across embedding dimensions or attention heads. From the point of view of classical low-rank approximation, these methods are unconventional and raise questions of which objects they really approximate and how to interpret the low-rank behavior of the resulting representations. To answer these questions, this work proposes a generalized view on the weight objects in the self-attention layer and a factorization strategy, which allows us to construct a parameter efficient scheme, called Tucker Attention. Tucker Attention requires an order of magnitude fewer parameters for comparable validation metrics, compared to GQA and MLA, as evaluated in LLM and ViT test cases. Additionally, Tucker Attention~encompasses GQA, MLA, MHA as special cases and is fully compatible with flash-attention and rotary position embeddings (RoPE). This generalization strategy yields insights of the actual ranks achieved by MHA, GQA, and MLA, and further enables simplifications for MLA.

### 1. 核心问题与研究动机
*   **核心问题**：多头自注意力机制（MHA）在 Transformer 架构中带来了巨大的内存和计算开销，主要体现在：训练时的参数存储（优化器状态和梯度）、推理时的 KV Cache 占用，以及计算过程中的 GPU 缓存压力。
*   **研究动机**：现有的改进方法如分组查询注意力（GQA）和多头潜在注意力（MLA）虽然通过低秩分解减少了开销，但它们通常只针对特定维度进行操作。作者认为，这些方法在“注意力头”维度和“输出投影”维度上仍存在未被挖掘的冗余。本文旨在通过张量分解的视角，提出一种更具普适性、参数效率更高的注意力机制。

### 2. 方法论：Tucker Attention
*   **核心思想**：将注意力机制中的权重重新建模为两个三阶张量：一个是 **Pre-softmax 张量**（融合了 Query 和 Key 权重），另一个是 **Post-softmax 张量**（融合了 Value 和 Output 权重）。
*   **关键技术细节**：
    *   **张量建模**：将 $n_H$ 个头的权重堆叠，形成维度为 $(n_H, d_{model}, d_{model})$ 的张量。
    *   **Tucker 分解**：对上述张量应用 Tucker 分解，将其分解为一个核心张量（Core Tensor）和三个基矩阵（分别对应头维度、查询/输出维度、键/值维度）。
    *   **参数控制**：通过设置三个独立的 Tucker 秩 $(r_1, r_2, r_3)$，可以独立控制不同维度的压缩率。
    *   **Latent RoPE**：针对 Tucker 分解无法直接应用旋转位置编码（RoPE）的问题，提出在潜在空间（Latent Space）应用旋转，并证明了其相对位置不变性。
*   **统一性证明**：论文在数学上证明了 MHA、MQA、GQA 和 MLA 都是 Tucker Attention 在特定秩设定下的特例。

### 6. 主要结论与发现
*   **极致的参数效率**：在保持相同验证精度的情况下，Tucker Attention 所需的参数量比 GQA 和 MLA 减少了约 **一个数量级（约 9 倍）**。
*   **KV Cache 优化**：Tucker Attention 结合了 MQA 的共享机制和 MLA 的潜在维度压缩，显著降低了推理时的内存压力。
*   **MLA 的简化**：通过 Latent RoPE，证明了 MLA 可以在不使用复杂的“解耦 RoPE”情况下实现位置编码，简化了架构实现。

### 7. 优点
*   **理论优雅**：提供了一个统一的数学框架来解释各种近似注意力机制。
*   **多维度压缩**：首次实现了对“头维度（Head Mode）”的显式低秩压缩，挖掘了现有方法忽略的冗余。
*   **工程兼容性**：完全兼容 Flash-Attention 2，无需编写复杂的自定义 CUDA 内核即可获得加速。

### 8. 不足与局限
*   **秩的选择复杂性**：相比 MLA 只需要调整一个潜在维度，Tucker Attention 需要调整三个秩 $(r_1, r_2, r_3)$，增加了超参数调优的难度。
*   **算子优化**：虽然兼容 Flash-Attention，但目前尚未开发专门针对 Tucker 核心张量收缩的深度优化内核，目前的实现可能未达到理论上的最高效率。
*   **模型规模限制**：实验主要集中在 1B 参数规模及以下，尚未在超大规模模型（如 70B+）上验证其长文本处理的稳定性。
