## 摘要
随着大语言模型（LLMs）扩展到支持超过一百万个 token 的上下文窗口，键值（KV）缓存的线性增长带来了严重的内存容量和带宽瓶颈，限制了长上下文推理的效率。现有的压缩方法通常根据局部显著性指标对 token 进行优先级排序，以将预填充计算与解码内存解耦。然而，这些方法往往依赖于特定层的局部显著性快照，从而系统性地丢弃了那些在整个网络深度中充当全局信息枢纽、但在选定的剪枝层中暂时处于休眠状态的 token。为了解决这一局限性，我们提出了 StructKV，这是一个结构感知的 KV 缓存压缩框架，它引入了三项核心创新：首先，全局入度中心性（Global In-Degree Centrality）聚合了整个网络深度的注意力模式，以识别全局信息枢纽；其次，动态枢轴检测（Dynamic Pivot Detection）利用信息论指标自适应地定位压缩的最佳层；最后，结构传播与解耦（Structural Propagation and Decoupling）将计算预算与内存存储预算分离。在 LongBench 和 RULER 基准测试上的实验结果表明，StructKV 有效地保留了长程依赖关系和检索鲁棒性。

## Abstract
As Large Language Models (LLMs) scale to support context windows exceeding one million tokens, the linear growth of Key-Value (KV) cache imposes severe memory capacity and bandwidth bottlenecks, constraining the efficiency of long-context inference. Existing compression approaches typically prioritize tokens based on local saliency metrics to decouple prefill computation from decoding memory. However, these methods often rely on local saliency snapshots at a specific layer, thereby systematically discarding tokens that act as global information hubs across the network depth but appear temporarily dormant at the specific layer selected for pruning. To address this limitation, we propose StructKV, a structure-aware KV cache compression framework that introduces three core innovations: First, Global In-Degree Centrality aggregates attention patterns across the network depth to identify global information hubs. Second, Dynamic Pivot Detection utilizes information-theoretic metrics to adaptively locate the optimal layer for compression. Finally, Structural Propagation and Decoupling separates the computational budget from the memory storage budget. Experimental results on the LongBench and RULER benchmarks demonstrate that StructKV effectively preserves long-range dependencies and retrieval robustness.

随着大语言模型（LLM）支持的上下文窗口突破百万级别，**KV 缓存（Key-Value Cache）** 的线性增长成为了推理效率的致命瓶颈，导致显存占用过高和显存带宽受限。
*   **现有方法的缺陷：** 现有的压缩技术（如 SnapKV, FastKV）通常依赖于特定层的“局部显著性”（Local Saliency）快照来决定保留哪些 token。
*   **研究动机：** 作者发现这种局部视角是“近视”的。某些 token 在特定层可能看起来不重要（处于休眠状态），但在整个网络深度中却充当着全局信息枢纽。如果仅凭单层快照进行剪枝，会系统性地丢弃这些对全局结构至关重要的信息，导致长文本推理能力下降。

### 2. 方法论：StructKV 框架
StructKV 旨在识别并保留上下文的“结构骨架”，包含三大核心创新：
*   **全局入度中心性（Global In-Degree Centrality）：** 不再依赖单层注意力分数，而是通过递归累加从输入层到“枢轴层”（Pivot Layer）的所有注意力模式。引入衰减因子 $\lambda$ 来平衡历史层与深层语义的重要性，从而识别出在全局范围内具有高连接度的 token 枢纽。
*   **动态枢轴检测（Dynamic Pivot Detection）：** 放弃了固定剪枝层的做法。该机制在线监控注意力熵（Entropy）、稀疏度（Sparsity）和方差（Variance）的梯度变化，自动定位模型从“广泛探索”转向“聚焦提取”的相位临界点（即最佳压缩层 $L^*$）。
*   **结构化传播与解耦（Structural Propagation & Decoupling）：** 
    *   **计算与存储解耦：** 在枢轴层之后，模型仅传播少量的结构化 token 以加速深层计算（减少 Prefill 阶段的计算量）。
    *   **独立预算：** 允许计算预算（$R_{struct}$）和解码存储预算（$R_{KV}$）独立设置，确保在极低显存占用下仍能维持高质量的生成。

### 6. 主要结论与发现
*   **性能领先：** 在 10% KV 缓存预算下，StructKV 在 LongBench 上的表现显著优于 FastKV 和 SnapKV，甚至在减少 40% 预填充计算的情况下接近全上下文（Full-Context）的准确度。

*   **效率提升：** 在 A800 上实现了约 **1.87x** 的预填充阶段加速。

### 7. 优点与亮点
*   **全局视角：** 突破了局部显著性的局限，通过累加注意力流捕捉长程依赖。
*   **自适应性：** 动态枢轴检测解决了不同模型架构需要手动调参（固定剪枝层）的痛点。
*   **解耦策略：** 灵活平衡了预填充速度（计算）与解码容量（显存），在极端压缩比下表现稳健。
*   **低开销：** 算法实现高效，对推理延迟的额外负担极小。

*   **长度验证上限：** 虽然动机是百万级上下文，但受限于算力，实验验证主要集中在 128K 长度。
