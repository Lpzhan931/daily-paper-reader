## 摘要
自回归（AR）语言模型一次生成一个标记（token），即使在给定先前上下文的情况下连续标记具有高度可预测性时也是如此。我们介绍了 MARS（Mask AutoRegreSsion），这是一种轻量级的微调方法，旨在教导经过指令微调的 AR 模型在单次前向传播中预测多个标记。MARS 不增加任何架构修改或额外参数，并生成一个单一模型，该模型仍可像原始 AR 模型一样被调用，且性能无损。与维护独立草稿模型的目标模型投机解码，或如 Medusa 等附加额外预测头的多头方法不同，MARS 仅需在现有指令数据上进行持续训练。在单次前向传播生成一个标记时，MARS 在六个标准基准测试中达到或超过了 AR 基准。当允许每步接受多个标记时，它在保持基准水平准确度的同时，实现了 1.5-1.7 倍的吞吐量。我们进一步开发了一种用于批量推理的块级 KV 缓存策略，在 Qwen2.5-7B 上相比带有 KV 缓存的 AR 实现了高达 1.71 倍的实际运行时间加速。最后，MARS 支持通过置信度阈值进行实时速度调整：在高请求负载下，服务系统可以在不更换模型或重启的情况下即时提高吞吐量，为部署提供了一个实用的延迟-质量调节旋钮。

## Abstract
Autoregressive (AR) language models generate text one token at a time, even when consecutive tokens are highly predictable given earlier context. We introduce MARS (Mask AutoRegreSsion), a lightweight fine-tuning method that teaches an instruction-tuned AR model to predict multiple tokens per forward pass. MARS adds no architectural modifications, no extra parameters, and produces a single model that can still be called exactly like the original AR model with no performance degradation. Unlike speculative decoding, which maintains a separate draft model alongside the target, or multi-head approaches such as Medusa, which attach additional prediction heads, MARS requires only continued training on existing instruction data. When generating one token per forward pass, MARS matches or exceeds the AR baseline on six standard benchmarks. When allowed to accept multiple tokens per step, it maintains baseline-level accuracy while achieving 1.5-1.7x throughput. We further develop a block-level KV caching strategy for batch inference, achieving up to 1.71x wall-clock speedup over AR with KV cache on Qwen2.5-7B. Finally, MARS supports real-time speed adjustment via confidence thresholding: under high request load, the serving system can increase throughput on the fly without swapping models or restarting, providing a practical latency-quality knob for deployment.

### 1. 核心问题与整体含义（研究动机）
*   **核心问题**：传统的自回归模型（如 GPT 系列）在生成文本时，无论后续标记（Token）是否容易预测，都必须逐一进行前向传播，这造成了巨大的计算浪费。
*   **研究动机**：现有的加速方案（如投机解码、Medusa 等）通常需要额外的草稿模型或预测头，增加了架构复杂度和内存占用。MARS 旨在探索：**能否仅通过微调，让原始 AR 模型在不改变架构、不增加参数的前提下，具备单次前向传播生成多个 Token 的能力？**

### 2. 方法论
*   **核心思想**：将“块掩码预测”（Block-masked prediction）引入 AR 模型，同时消除传统扩散模型与 AR 模型之间的不兼容性。
*   **关键技术细节**：
    *   **消除四个差距**：MARS 坚持使用**因果注意力（Causal Attention）**、**右移 Logits 对齐**和**从左到右的生成顺序**，仅保留“Token 掩码”这一必要差异。
    *   **双流训练架构**：在训练时，将原始序列 $x$（干净流）和带有掩码的序列 $\tilde{x}$（噪声流）拼接。通过特殊的注意力掩码，使噪声流中的位置能看到之前的干净前缀，并以因果方式预测当前块内的掩码。
    *   **SFT 辅助损失函数**：这是 MARS 的关键。总损失 $L = L_{mask} + L_{AR}$。通过在干净流上保留标准的自回归损失（SFT Loss），确保模型在学习多 Token 预测的同时，不会丧失原始的自回归能力。
    *   **滑动窗口推理**：推理时在当前前缀后附加 $B$ 个 [MASK] 标记。模型根据置信度阈值 $\tau$ 决定接受多少个预测结果。若不自信，则退化为标准的逐个生成模式。
    *   **块级 KV 缓存**：设计了一种同步策略，允许在批量推理中高效复用键值缓存，减少重复计算。

### 6. 主要结论与发现
*   **质量无损**：在单 Token 模式下，MARS 的表现甚至略优于原始 AR 模型（如 7B 模型平均分从 56.6 提升至 58.1）。

### 7. 优点

*   **严格超集**：模型保留了完整的 AR 能力，可作为原始模型的直接替代品。
*   **训练高效**：仅需在 SFT 数据上进行少量持续训练，无需大规模预训练。
*   **实用性强**：提出的块级 KV 缓存解决了多 Token 预测在实际推理中的瓶颈。

### 8. 不足与局限
*   **训练成本翻倍**：由于采用了序列拼接（干净流+噪声流），单次训练的前向传播计算量是标准 SFT 的两倍。

*   **规模限制**：论文主要验证到 7B 规模，更大规模模型（如 70B+）的效果尚待验证。
