## 摘要
视频大语言模型 (VideoLLMs) 在视频理解方面展现出了卓越的能力，但海量的输入视频 Token 为模型部署带来了沉重的计算负担。现有方法主要在输入层面进行视频 Token 剪枝，忽略了视频和语言大模型 (LLMs) 内部固有的信息结构。为此，我们提出了 HieraVid，一个能够逐步且动态地减少视觉冗余的层级化剪枝框架。基于视频具有“片段-帧”结构以及 LLM 内部单向传播多模态信息这两个观察，我们将剪枝分解为三个层面：1) 片段级，首先对视频 Token 进行时间分段和空间合并；2) 帧级，对同一片段内的相似帧进行联合剪枝以保留多样性；3) 层级，随着 LLM 层数的增加，在不损害性能的前提下逐渐减少冗余。我们在四个广泛使用的视频理解基准测试上进行了大量实验，全面评估了 HieraVid 的有效性。值得注意的是，在仅保留 30% Token 的情况下，HieraVid 达到了新的最先进性能，同时分别保持了 LLaVA-Video-7B 和 LLaVA-OneVision-7B 98% 和 99% 以上的性能。

## Abstract
Video Large Language Models (VideoLLMs) have demonstrated impressive capabilities in video understanding, yet the massive number of input video tokens incurs a significant computational burden for deployment. Existing methods mainly prune video tokens at input level while neglecting the inherent information structure embedded in videos and large language models (LLMs). To address this, we propose HieraVid, a hierarchical pruning framework that progressively and dynamically reduces visual redundancy. Based on two observations that videos possess the segment-frame structure and LLMs internally propagate multi-modal information unidirectionally, we decompose pruning into three levels: 1) segment-level, where video tokens are first temporally segmented and spatially merged; 2) frame-level, where similar frames within the same segment are jointly pruned to preserve diversity; 3) layer-level, redundancy gradually shrinks as LLM layer increases w/o compromising performance. We conduct extensive experiments on four widely used video understanding benchmarks to comprehensively evaluate the effectiveness of HieraVid. Remarkably, with only 30% of tokens retained, HieraVid achieves new state-of-the-art performance, while maintaining over 98% and 99% of the performance of LLaVA-Video-7B and LLaVA-OneVision-7B, respectively.

### 1. 核心问题与研究动机
*   **核心问题**：视频大语言模型（VideoLLMs）在处理视频时，由于密集的帧采样会产生海量的视觉 Token。由于 LLM 自注意力机制的计算复杂度随序列长度呈平方级增长，过长的 Token 序列导致了巨大的计算负担、内存消耗和推理延迟。
*   **研究动机**：
    *   **视觉冗余性**：视频中存在大量空间（背景）和时间（相邻帧）上的重复信息。
    *   **现有方法缺陷**：目前的剪枝方法多在输入端进行简单过滤，或在 LLM 内部固定层进行剪枝，忽略了视频固有的“片段-帧”语义结构，也未考虑 LLM 内部多模态信息流随层数加深的动态变化特性。

### 2. 方法论：HieraVid 框架
HieraVid 提出了一种三层级的层级化剪枝方案，旨在实现“训练无关（Training-free）”的即插即用加速：
*   **片段级合并（Segment-level Merging）**：
    *   **合并率引导的分段**：计算相邻帧之间的空间相似度图，定义“重叠率（Overlap Ratio）”。当重叠率低于阈值 $\beta$ 时划定语义边界，将视频切分为多个片段。
    *   **空间合并**：在片段内部，对时序上静态的 Token 进行空间合并，减少背景冗余。
*   **帧级 DPP 剪枝（Frame-level DPP Pruning）**：
    *   **多样性最大化**：引入行列式点过程（DPP）算法，通过最大化核矩阵的行列式来选择最具代表性且冗余度最低的帧子集。
    *   **指令相关性集成**：将 LLM 最后一个指令 Token 的注意力分数融入 DPP 核矩阵，确保保留的 Token 既具有视觉多样性，又与用户问题高度相关。
    *   **动态预算分配**：根据片段间的差异性动态分配剪枝比例，内容丰富的片段保留更多 Token。
*   **层级平衡（Layer-level Balancing）**：
    *   **多阶段剪枝**：基于 LLM 内部信息流向（视觉信息逐渐向文本 Token 聚合）的观察，将 LLM 划分为多个阶段（如 4 个阶段），随着层数加深逐步加大剪枝力度，平衡信息损失与推理效率。

### 7. 优点
*   **结构化感知**：不同于随机或简单的注意力剪枝，它利用了视频天然的语义分段特性。
*   **多样性与相关性平衡**：通过改进的 DPP 算法，解决了传统剪枝容易导致信息丢失（Over-pruning）的问题。
*   **无需微调**：作为一种即插即用的方法，极大地降低了部署成本。
*   **层级化视角**：结合了 LLM 内部信息传播的物理特性，使剪枝过程更符合模型的工作原理。

### 8. 不足与局限
*   **超参数依赖**：分段阈值 $\beta$ 和平衡系数 $\lambda$ 等超参数可能需要针对不同风格的视频（如快节奏动作 vs. 静态监控）进行微调。

*   **任务局限**：主要针对视频问答和理解任务，对于需要极高空间细节的任务（如视频目标检测或细粒度分割）的适用性尚未深入探讨。
