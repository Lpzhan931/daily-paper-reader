## 摘要
在 K=16 个 Token（4K 上下文的 0.4%）时，现有的所有 KV 缓存压缩方法在凭据检索任务上的准确率均为 0%。其失败模式在于“休眠 Token”：即凭据、API 密钥和配置值等，它们在处理过程中接收到的注意力几乎为零，但在生成阶段却至关重要。由于这些 Token 缺乏逐出策略所依赖的统计信号，任何基于注意力分数、重构损失或学习保留门控的方法都无法保留它们。我们提出了事务性注意力（Transactional Attention, TA），这是一种赞助机制，通过结构化锚点模式（例如 “key:”、“password:”）保护相邻的承载值的 Token 免遭逐出。在 K=16 时，TA 实现了 100% 的凭据检索准确率，而六个基准模型（H2O、TOVA、SnapKV、StreamingLLM、PyramidKV、DynamicKV）均为 0%，且在 200 次函数调用测试中保持了 100% 的准确率。TA-Fast 是一种无注意力变体，可减少 52% 的内存开销，并与 SDPA 和 FlashAttention 兼容。TA 与现有的压缩方法正交，且增加的延迟开销不到 1%。

## Abstract
At K=16 tokens (0.4% of a 4K context), every existing KV-cache compression method achieves 0% on credential retrieval. The failure mode is dormant tokens: credentials, API keys, and configuration values that receive near-zero attention but become essential at generation time. Because these tokens lack the statistical signals that eviction policies rely on, no method based on attention scores, reconstruction loss, or learned retention gates retains them. We introduce Transactional Attention (TA), a sponsorship mechanism in which structural anchor patterns (e.g., "key:", "password:") protect adjacent value-bearing tokens from eviction. TA achieves 100% credential retrieval at K=16 where six baselines (H2O, TOVA, SnapKV, StreamingLLM, PyramidKV, DynamicKV) achieve 0%, and sustains 100% accuracy across 200 function-calling trials. TA-Fast, an attention-free variant, reduces memory overhead by 52% and is compatible with SDPA and FlashAttention. TA is orthogonal to existing compression methods and adds less than 1% latency overhead.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现有的 KV 缓存压缩方法（如 H2O, TOVA, SnapKV 等）主要依赖注意力分数、重构损失或学习到的门控来决定保留哪些 Token。然而，在实际应用中（如 API 调用、配置读取），存在一类**“休眠标记”**（如 API 密钥、密码、会话 ID）。
*   **失效模式**：这些标记在长文本中出现后，在后续数千个 Token 的处理中可能接收到近乎零的注意力，但在生成阶段（如调用工具时）却是必不可少的。在极高压缩比下（如 4K 上下文仅保留 16 个 Token），现有方法会因这些标记“统计上不显著”而将其逐出，导致模型无法提取关键凭证。
*   **研究意义**：随着 Agent 架构和多轮工具调用成为主流，如何在高压缩比下精准保留这些“语义关键但统计稀疏”的信息成为提升长文本模型实用性的关键。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：引入**“语义赞助”（Semantic Sponsorship）**机制。利用结构化模式作为“锚点”（Anchors），主动保护其相邻的价值承载标记（Values）。
*   **关键技术细节**：
    *   **锚点检测（Anchor Detection）**：通过正则匹配（Regex）、语义识别（如“remember”、“said”）或嵌入向量相似度（Embedding-based）识别关键前缀（如 `key:`）。
    *   **赞助凭证分配（Voucher Allocation）**：一旦检测到锚点，会为其后长度为 $L$ 的 Token 序列分配一个随距离指数衰减的“赞助分值” $V_j$。
    *   **综合效用评分（Utility Scoring）**：Token 的最终保留优先级由注意力分数、新近度、锚点状态、惩罚项以及**赞助分值**共同决定。公式为：$u_i = \alpha A_i + \beta R_i + \gamma S_i + \delta P_i - \lambda F_i + V_i$。
    *   **TA-Fast 变体**：一种无注意力（Attention-free）版本，完全舍弃注意力权重项，仅依赖位置和语义赞助，从而兼容 FlashAttention 等硬件加速库。
    *   **预算分区**：将总预算 $K$ 分为主要池和受保护池，确保赞助标记不会被高注意力标记完全挤占。

### 6. 主要结论与发现
*   **性能飞跃**：在极低预算（K=16）下，所有基线方法准确率均为 0%，而 TA 维持了 **100%** 的准确率。
*   **效率优势**：TA 增加的延迟开销小于 **1%**，且 TA-Fast 变体在长文本下避免了 OOM（显存溢出）。
*   **正交性**：TA 并不取代现有的压缩策略，而是作为一个附加信号注入评分函数，可与 H2O 或 SnapKV 等方法结合使用。

### 7. 优点
*   **直击痛点**：敏锐捕捉到了注意力机制在处理“休眠标记”时的本质缺陷。
*   **架构兼容**：TA-Fast 的设计使其能够无缝集成到现代高性能推理框架中。
*   **低成本**：无需对大模型进行微调，仅需在推理侧修改 KV 缓存管理逻辑。
*   **理论支撑**：论文提供了形式化证明，解释了为何基于注意力的策略在长文本下必然失效。

### 8. 不足与局限
*   **依赖锚点模式**：如果关键信息没有明显的结构化前缀（如非结构化的随口提及），准确率会下降（从 100% 降至 32%）。
*   **对抗风险**：恶意输入可以通过注入大量伪造锚点来耗尽“赞助预算”，导致真实关键信息被逐出。
*   **分词器敏感**：不同模型的分词方式（Tokenizer）会影响赞助跨度 $L$ 的设定，需要针对特定模型微调预算。
*   **通用理解无提升**：TA 专注于“精准保留”，并不能提升模型本身对长文本的逻辑推理能力。
