## 摘要
推测解码通过允许草稿标记（draft tokens）绕过完整验证来加速自回归生成，但传统框架常面临频繁的错误拒绝问题，特别是当草稿模型生成的输出在语义上正确但在词汇上存在差异时。在本文中，我们提出了校准推测解码（CSD），这是一种无需训练的框架，能够恢复被标准验证丢弃的有效标记。在“频率引导的候选选择与概率守护的接受”原则指导下，CSD 包含两个轻量级模块：在线修正记忆（Online Correction Memory），用于聚合历史拒绝记录并将反复出现的差异模式作为补救候选提出；以及语义一致性门控（Semantic Consistency Gating），利用概率比率而非精确标记匹配来验证候选的可接受性。我们在多种大语言模型上的评估表明，CSD 优于现有方法，实现了高达 2.33 倍的峰值吞吐量加速。CSD 在所有任务中均保持了模型准确性，并进一步提升了在复杂推理数据集上的性能。这些结果证明 CSD 是实际大语言模型部署中一种高效且轻量级的解决方案。

## Abstract
Speculative decoding accelerates autoregressive generation by letting draft tokens bypass full verification, but conventional frameworks suffer from frequent false rejections, particularly when draft models produce semantically correct but lexically divergent outputs. In this paper, we present Calibrated Speculative Decoding (CSD), a training-free framework that recovers valid tokens discarded by standard verification. Guided by the principle of "Frequency-Guided Candidate Selection and Probability-Guarded Acceptance," CSD incorporates two lightweight modules: Online Correction Memory, which aggregates historical rejections to propose recurring divergence patterns as rescue candidates, and Semantic Consistency Gating, which verifies candidate admissibility using probability ratios instead of exact token matching. Our evaluation across diverse large language models demonstrates that CSD outperforms existing methods, achieving a peak throughput speedup of 2.33x. CSD preserves model accuracy across all tasks while further boosting performance on complex reasoning datasets. These results establish CSD as a highly effective, lightweight solution for practical LLM deployments.

*   **核心问题**：传统的推测解码依赖于草稿模型（Draft Model）与目标模型（Target Model）之间的**精确词汇匹配**或严格的拒绝采样。然而，现代小模型（如 Llama-3.2-1B）已具备较强的推理能力，它们生成的 Token 往往在语义上是正确的，但可能因为词汇选择（如“*”与“x”）或风格差异被目标模型拒绝。
*   **研究动机**：这种“错误拒绝（False Rejections）”导致了计算资源的浪费。论文旨在打破这种僵化的验证逻辑，通过识别并恢复那些“语义正确但词汇不同”的 Token，在不损失（甚至提升）精度的前提下，进一步提升推理吞吐量。

### 2. 论文提出的方法论：校准推测解码（CSD）
CSD 遵循“**频率引导候选选择，概率守护接受**”的原则，包含两个核心模块：
*   **在线修正记忆（Online Correction Memory, OCM）**：
    *   **核心思想**：利用拒绝模式的长尾分布特性（前 20% 的模式占拒绝总数的 69%）。
    *   **技术细节**：维护一个轻量级的内存表，记录草稿模型与目标模型之间频繁出现的差异对（如草稿输出 `A`，目标输出 `B`）。在推理时，如果发生拒绝，OCM 会检查该差异是否属于高频模式（频率 $\ge \lambda$），若是，则将其作为“补救候选”。
*   **语义一致性门控（Semantic Consistency Gating, SCG）**：
    *   **核心思想**：利用目标模型的置信度而非精确匹配来验证候选词。
    *   **技术细节**：通过计算目标模型对草稿 Token 的 Logit 值与目标 Token 的 Logit 值之差（即概率比率）来判断。如果比率超过阈值 $\tau$，则认为该 Token 在语义上是可接受的。
*   **算法流程**：草稿模型生成序列 $\rightarrow$ 目标模型并行验证 $\rightarrow$ 若某位置不匹配，触发 CSD 逻辑 $\rightarrow$ 检查 OCM 频率并进行 SCG 概率校验 $\rightarrow$ 若均通过，则“救回”该 Token 并继续后续验证。

### 7. 优点：亮点与创新
*   **无需训练（Training-free）**：不需要对模型进行微调或添加额外的辅助头，部署成本极低。
*   **轻量高效**：通过 Logit 空间的简单减法实现验证，几乎不增加推理延迟。
*   **自适应性**：具备在线更新机制，能够随推理过程动态捕捉特定领域的词汇模式。
*   **即插即用**：可以无缝集成到现有的推测解码框架中。

### 8. 不足与局限
*   **偏离分布一致性**：与标准 SD 不同，CSD 是一种启发式宽松验证，不再严格保证输出分布与目标模型完全一致（虽然实验证明精度未受损）。
*   **依赖草稿质量**：如果草稿模型产生严重的幻觉或错误，CSD 的加速效果会退化至接近自回归水平。
*   **高并发同步挑战**：论文提到在超大规模并发环境下，OCM 的实时频率更新可能存在同步开销，这一点尚未在生产级引擎（如 vLLM）中完全验证。
*   **校准依赖**：虽然有通用校准集，但最佳性能仍依赖于一定量的离线校准数据。
