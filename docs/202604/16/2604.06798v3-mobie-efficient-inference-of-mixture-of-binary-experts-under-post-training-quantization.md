## 摘要
基于专家混合（MoE）的大语言模型（LLM）虽然性能强大，但面临高昂的内存和计算开销。权重二值化提供了极高的效率，然而现有的针对稠密 LLM 设计的二值化方法在处理 MoE 特有议题时面临挑战，包括跨专家冗余、任务无关的重要性估计以及量化引起的路由偏移。为此，我们提出了 MoBiE，这是首个专为基于 MoE 的 LLM 定制的二值化框架。MoBiE 基于三项核心创新：1. 使用联合 SVD 分解来减少跨专家冗余；2. 将全局损失梯度集成到局部 Hessian 指标中，以增强权重重要性估计；3. 引入由输入零空间引导的误差约束，以减轻路由失真。值得注意的是，MoBiE 在实现这些优化的同时不产生额外的存储开销，在效率和模型性能之间取得了平衡。广泛的实验表明，MoBiE 在多个基于 MoE 的 LLM 和基准测试中始终优于最先进的二值化方法。例如，在 Qwen3-30B-A3B 上，MoBiE 将困惑度降低了 52.2%，平均零样本性能提升了 43.4%，实现了超过 2 倍的推理加速，并进一步缩短了量化时间。代码可在 https://github.com/Kishon-zzx/MoBiE 获取。

## Abstract
Mixture-of-Experts (MoE) based large language models (LLMs) offer strong performance but suffer from high memory and computation costs. Weight binarization provides extreme efficiency, yet existing binary methods designed for dense LLMs struggle with MoE-specific issues, including cross-expert redundancy, task-agnostic importance estimation, and quantization-induced routing shifts. To this end, we propose MoBiE, the first binarization framework tailored for MoE-based LLMs. MoBiE is built on three core innovations: 1. using joint SVD decomposition to reduce cross-expert redundancy; 2. integrating global loss gradients into local Hessian metrics to enhance weight importance estimation; 3. introducing an error constraint guided by the input null space to mitigate routing distortion. Notably, MoBiE achieves these optimizations while incurring no additional storage overhead, striking a balance between efficiency and model performance. Extensive experiments demonstrate that MoBiE consistently outperforms state-of-the-art binary methods across multiple MoE-based LLMs and benchmarks. For example, on Qwen3-30B-A3B, MoBiE reduces perplexity by 52.2$\%$, improves average zero-shot performance by 43.4$\%$, achieves over 2 $\times$ inference speedup, and further shortens quantization time. The code is available at https://github.com/Kishon-zzx/MoBiE.

# 论文总结：MoBiE——训练后量化下二值专家混合模型的高效推理

## 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）规模的扩大，**专家混合架构（MoE）**因其“计算量小但性能强”的特性成为主流。然而，MoE 模型面临严峻的**显存瓶颈**：虽然推理时仅激活少数专家，但所有专家权重必须常驻显存（例如 120B 模型需占用超过 240GB 显存）。

现有的**权重二值化（1-bit 量化）**技术虽能极大压缩模型，但在 MoE 架构上表现极差。研究团队发现三个核心痛点：
*   **专家冗余**：现有方法独立处理每个专家，忽略了专家间存在的结构相似性。
*   **重要性评估脱节**：传统的 Hessian 矩阵仅基于局部重建误差，未考虑下游任务的全局损失。
*   **路由偏移（Expert-shift）**：极低比特量化带来的扰动会破坏路由器的 Top-k 选择，导致 Token 被分发给错误的专家，引发性能崩溃。

## 2. 方法论：核心思想与关键技术
MoBiE 提出了一个三阶段的训练后量化（PTQ）框架：

*   **跨专家联合分解（CEJD）**：
    *   **核心思想**：利用专家间的相似性。
    *   **技术细节**：将同一层的所有专家权重矩阵水平拼接，进行奇异值分解（SVD）。提取出一个共享的、高精度的“主干”矩阵（$U\Sigma$），而仅对专家特有的投影矩阵（$V$）进行二值化。这在几乎不增加存储的情况下，利用高精度主干保护了核心语义信息。
*   **全局损失对齐显著性（GLAS）**：
    *   **核心思想**：让量化过程“感知”任务目标。
    *   **技术细节**：在构建 Hessian 矩阵时，显式地注入全局损失函数的梯度信息。通过梯度加权的残差目标，识别出对最终输出影响最大的权重，从而在二值化过程中对这些关键权重进行特殊保护。
*   **零空间引导的路由偏移抑制（NGES）**：
    *   **核心思想**：将量化噪声限制在“不敏感”区域。
    *   **技术细节**：利用输入激活值的“零空间”（Null Space）特性（即在该空间内的扰动对输出几乎无影响）。通过交替最小二乘法（ALS）优化二值化缩放因子（行/列缩放），将量化误差约束在路由不敏感的方向上，从而稳定路由器的决策。

## 3. 实验设计
*   **实验对象**：涵盖了多种规模和架构的 MoE 模型，包括 OLMoE-7B、DeepSeek-V2-Lite、Qwen1.5-MoE、Qwen3-30B/80B 以及 GPT-OSS-20B。
*   **数据集**：
    *   **语言建模**：WikiText2（评估困惑度 PPL）。
    *   **零样本推理**：Arc-Challenge/Easy、HellaSwag、LAMBADA、PIQA、WinoGrande。
    *   **复杂任务**：MMLU（知识）、GSM8K（数学）、HumanEval（代码）。
*   **对比方法（Baselines）**：
    *   通用量化：AWQ、GPTQ。
    *   MoE 专用量化：MoEQuant、NoWag。
    *   先进二值化方法：BiLLM、ARB-LLM。

## 4. 资源与算力
*   **硬件环境**：实验主要在 **NVIDIA RTX A6000 GPU** 上进行。
*   **量化耗时**：作为 PTQ 方法，其量化过程相对高效。例如，Qwen3-30B 的量化时间约为 11 小时左右，比需要复杂搜索的 ARB-LLM 更快。
*   **推理效率**：在 A6000 上实现了 **2.33 倍** 的吞吐量加速，并将专家层的显存占用降低了 **90% 以上**。

## 5. 实验数量与充分性
*   **实验规模**：论文在 6 种不同的 MoE 模型上进行了测试，涵盖了从 1B 到 80B 的参数规模。
*   **消融实验**：详细验证了 CEJD、GLAS 和 NGES 三个组件的各自贡献，并分析了不同位宽设置（如共享主干的比特数）对性能的影响。
*   **客观性**：实验不仅对比了 PPL，还对比了多个下游任务的准确率，并提供了定性的对话示例。对比基准涵盖了从 2-bit 到 1-bit 的最先进方法，实验设计较为全面、客观。

## 6. 主要结论与发现
*   **性能卓越**：MoBiE 在 1-bit 级别上显著优于所有现有的二值化方法。在 Qwen3-30B 上，其平均零样本准确率比之前的 SOTA 提升了 43.4%。
*   **超越高比特**：在某些模型上，MoBiE 的 1-bit 表现甚至优于 2-bit 甚至 3-bit 的 GPTQ/AWQ 方法，同时显存占用减半。
*   **路由稳定性**：NGES 成功将路由相似度保持在 0.9 以上，证明了稳定路由机制是 MoE 量化成功的关键。
*   **高效部署**：通过将专家权重压缩至 1-bit，使得原本需要多卡运行的大模型可以在单张消费级显卡上高效推理。

## 7. 优点
*   **架构针对性强**：不同于通用的二值化方法，MoBiE 深入挖掘了 MoE 架构的数学特性（专家冗余和路由机制）。
*   **零额外开销**：通过复用二值化必备的缩放因子来实现误差约束，没有引入额外的存储负担。
*   **即插即用**：GLAS 和 NGES 组件可以轻松集成到现有的量化算法（如 BiLLM）中并提升其效果。

## 8. 不足与局限
*   **校准数据依赖**：作为 PTQ 方法，其性能高度依赖于校准数据集的质量和分布，若部署环境与校准数据差异过大，可能存在性能退化风险。
*   **实现复杂度**：相比于简单的均匀量化，涉及 SVD 分解和 ALS 迭代优化，量化流程相对复杂。
*   **极端情况下的路由失效**：虽然 NGES 缓解了路由偏移，但在极少数极端噪声下，仍可能出现专家误分配的情况。
