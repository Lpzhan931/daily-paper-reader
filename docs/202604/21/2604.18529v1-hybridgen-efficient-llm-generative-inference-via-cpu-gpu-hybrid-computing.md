## 摘要
随着现代大语言模型（LLM）支持数千到数百万个 token，KV 缓存增长到数百 GB，给内存容量和带宽带来了巨大压力。现有的解决方案（如 KV 缓存剪枝和卸载）虽然缓解了这些问题，但由于仅依赖 GPU 或 CPU 进行注意力计算，且仅考虑有限的 CPU 本地内存用于 KV 缓存存储，导致硬件利用率不足。我们提出了 HybridGen，一种用于长上下文 LLM 推理的高效混合注意力框架。HybridGen 在具有扩展分层内存（如 CXL 内存）的系统上实现了 CPU-GPU 协同注意力，解决了三个关键挑战：(1) 多维注意力依赖关系，(2) 随序列变长而加剧的 CPU-GPU 负载不平衡，以及 (3) 分层内存的 NUMA 惩罚。HybridGen 通过引入注意力 logit 并行、反馈驱动的调度器以及语义感知的 KV 缓存映射来解决这些问题。在三个具有 CXL 扩展内存的 GPU 平台上，对三种不同模型、十一种不同规模的 LLM 进行的实验表明，HybridGen 在保持卓越准确性的同时，平均性能优于六种最先进的 KV 缓存管理方法 1.41 倍至 3.2 倍。

## Abstract
As modern LLMs support thousands to millions of tokens, KV caches grow to hundreds of gigabytes, stressing memory capacity and bandwidth. Existing solutions, such as KV cache pruning and offloading, alleviate these but underutilize hardware by relying solely on either GPU or CPU for attention computing, and considering yet limited CPU local memory for KV cache storage. We propose HybridGen, an efficient hybrid attention framework for long-context LLM inference. HybridGen enables CPU-GPU collaborative attention on systems with expanded tiered memory (e.g., CXL memory), addressing three key challenges: (1) multi-dimensional attention dependencies, (2) intensifying CPU-GPU load imbalance with longer sequences, and (3) NUMA penalty of tiered memories. HybridGen tackles these by introducing attention logit parallelism, a feedback-driven scheduler, and semantic-aware KV cache mapping. Experiments with three LLM models with eleven different sizes on three GPU platforms with a CXL-expanded memory show that HybridGen outperforms six state-of-the-art KV cache management methods by 1.41x--3.2x on average while maintaining superior accuracy.

*   **核心问题**：随着大语言模型（LLM）上下文长度迈向百万级，KV Cache（键值缓存）的大小呈线性增长，远超单体 GPU 的显存容量（例如 70B 模型处理 1M token 需要数百 GB 缓存）。
*   **研究动机**：
    *   **现有方案局限**：现有的“剪枝”方案会损失精度；“卸载（Offloading）”方案（如 FlexGen）将缓存存入 CPU 内存，但在推理时需频繁将数据传回 GPU，导致严重的 I/O 瓶颈。
    *   **资源利用不均**：目前的方案要么只用 GPU 计算，要么完全切换到 CPU，导致 CPU 计算能力被闲置，或 GPU 并行性无法发挥。
    *   **新兴硬件挑战**：CXL 扩展内存虽然提供了大容量，但其 NUMA 延迟（比本地 DRAM 慢）尚未在 KV 缓存管理中得到优化利用。

### 2. 方法论
HybridGen 提出了一个高效的混合注意力框架，核心思想是**让 CPU 和 GPU 协同处理注意力计算**，并利用分层内存。
*   **注意力 Logit 并行化（Attention Logit Parallelism）**：
    *   **解耦计算**：将注意力机制分为 Logit 计算（$Q \cdot K^T$）和 Softmax/Value 聚合两步。CPU 负责计算其本地内存中旧 token 的 Logit，GPU 负责计算新 token 的 Logit。
    *   **跨层流水线**：利用相邻 Transformer 层输入高度相似的特性，让 CPU 在 GPU 处理当前层时，提前利用当前层输入预计算下一层的 Logit，实现计算重叠。
*   **反馈驱动调度器（Feedback-driven Scheduler）**：
    *   动态调整 CPU 处理的 token 数量（$K$ 值）以及选择策略（Post-QK 或 Pre-QK）。
    *   如果 CPU 成为瓶颈，则减少 $K$ 或切换到更轻量但精度略低的预预测模式；如果 GPU 较慢，则增加 $K$ 以提升精度。
*   **语义感知 KV 缓存映射（Semantic-aware KV Mapping）**：
    *   **K 向量存 DRAM**：由于 CPU 计算 Logit 需频繁访问 Key，将其放在低延迟的本地 DRAM。
    *   **V 向量存 CXL**：Value 向量仅在最后一步聚合时通过 DMA 传给 GPU，对延迟不敏感，因此存放在高容量的 CXL 内存中。
