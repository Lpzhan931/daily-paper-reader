## 摘要
我们提出了 Open-TQ-Metal，这是在 Apple 芯片上首次实现的融合压缩域注意力，使得在单台 64GB 消费级 Mac 上实现 Llama 3.1 70B 的 128K 上下文推理成为可能——这一配置在所有现有推理框架下都是无法实现的。Open-TQ-Metal 将 KV 缓存实时量化为 int4，并通过自定义 Metal 计算着色器直接在压缩表示上计算注意力，从而消除了所有中间反量化矩阵。在涵盖两个模型系列（Gemma 4 31B 和 Llama 3.1 70B）的 330 次实验中，融合的 sdpa_int4 内核在 128K 上下文下相比“先反量化再计算注意力”的基准实现了 48 倍的注意力加速，将 KV 缓存内存从 40 GB 减少到 12.5 GB（3.2 倍压缩），并保持了与 FP16 推理完全一致的 top-1 token 预测。我们还提供了首个 KV 缓存量化方法的跨架构分析，揭示了注意力缩放因子（而非模型大小）决定了像 PolarQuant 这样的角量化方案的成败；其中 Gemma 4 的 attn_scale=1.0 对方向误差的放大作用比 Llama 标准的 1/sqrt(d) 缩放高出 25-100 倍。

## Abstract
We present Open-TQ-Metal, the first implementation of fused compressed-domain attention on Apple Silicon, enabling 128K-context inference for Llama 3.1 70B on a single 64GB consumer Mac -- a configuration impossible with all existing inference frameworks. Open-TQ-Metal quantizes the KV cache to int4 on the fly and computes attention directly on the compressed representation via custom Metal compute shaders, eliminating all intermediate dequantization matrices. Across 330 experiments spanning two model families (Gemma 4 31B and Llama 3.1 70B), the fused sdpa_int4 kernel achieves 48x attention speedup at 128K context over the dequantize-then-attend baseline, reduces KV cache memory from 40 GB to 12.5 GB (3.2x compression), and maintains identical top-1 token predictions to FP16 inference. We further provide the first cross-architecture analysis of KV cache quantization methods, revealing that the attention scale factor -- not model size -- determines whether angular quantization schemes like PolarQuant succeed or fail, with Gemma 4's attn_scale=1.0 amplifying directional error 25-100x more than Llama's standard 1/sqrt(d) scaling.

*   **研究动机**：在消费级硬件（如 Apple Silicon Mac）上运行超长上下文（128K+）的大语言模型（LLM）面临严重的“内存墙”限制。以 Llama 3.1 70B 为例，128K 上下文的 FP16 KV 缓存需 40GB，加上 4-bit 权重（39.1GB），总需求接近 80GB，远超 64GB 的消费级内存上限。
*   **核心问题**：现有的推理框架（如 mlx-lm, llama.cpp）在处理 KV 缓存量化时，通常采用“先反量化再计算”的模式，这会产生巨大的中间矩阵，消耗带宽并导致内存溢出。
*   **整体含义**：本研究开发了 **Open-TQ-Metal**，这是首个在 Apple 芯片上实现“融合压缩域注意力”的系统，通过在 Metal 自定义着色器中直接对压缩数据进行计算，打破了内存限制，实现了在 64GB 设备上运行 128K 上下文 70B 模型。

### 2. 方法论
*   **核心思想**：**融合 int4 SDPA 内核**。将 KV 缓存以 int4 格式存储，在执行注意力计算（Scaled Dot-Product Attention）时，直接在 GPU 寄存器中完成解包、反量化和点积运算，不写回任何中间 FP16 矩阵。
*   **关键技术细节**：
    *   **寄存器级反量化**：利用位掩码（Bitwise masks）和移位操作，在 GPU 寄存器中实时重建 4-bit 尼布尔（nibbles）为仿射表示。
    *   **Split-K 并行化**：针对长序列，将 KV 序列划分为多个块（Chunks）并行处理，通过 MLX Primitive 链式调用解决 Metal 调度的竞态问题，实现延迟的平坦化缩放。
    *   **在线 Softmax（Online Softmax）**：在单次传递中计算注意力，避免了存储完整的注意力得分矩阵。
    *   **混合预填充/解码**：预填充阶段使用标准 FP16 计算并量化存储；解码阶段（Sq=1）则直接调用融合的 int4 内核。
    *   **内存锁定（Wired Memory）**：通过 `set_wired_limit` 锁定权重，防止 macOS 将内存交换至 SSD，从而恢复 10 倍的吞吐量。

*   **实验规模**：共进行了 **330 组实验**，涵盖了两个模型系列的不同配置。

### 6. 主要结论与发现
*   **性能突破**：在 128K 上下文下，融合内核比基准方法快 **48 倍**；KV 缓存压缩比达 **3.2 倍**（40GB 降至 12.5GB）。
*   **精度保持**：int4 per-group 量化在 Greedy Decode 下产生的 Top-1 token 与 FP16 完全一致。
*   **架构敏感性发现**：**注意力缩放因子（$\alpha$）**是决定角量化（如 PolarQuant）成败的关键。Gemma 4 因 $\alpha=1.0$ 导致角度误差被放大 25-100 倍，输出变为乱码；而 Llama 因 $\alpha \approx 0.088$ 能够抑制误差。
*   **深度失效**：1-bit QJL 算法在 8B 模型上可行，但在 70B（80层）模型上因误差累积导致信号完全丢失。

### 7. 优点
*   **极致优化**：针对 Apple Silicon 的统一内存架构和 SIMD 组进行了深度定制，解决了消费级硬件跑不动大模型的痛点。
*   **理论深度**：不仅提供了工程实现，还从数学角度解释了为什么某些量化方法在特定架构上会失效，为未来的量化研究提供了指导。
*   **开源贡献**：代码、Metal 着色器和 Benchmark 全部开源，具有很高的实用价值。

### 8. 不足与局限
*   **硬件局限性**：内核高度依赖 Metal 和 Apple 芯片特性，迁移到 NVIDIA CUDA 需要重新实现。
*   **端到端开销**：由于 KV 缓存管理的开销（如拼接操作），端到端吞吐量比 mlx-lm 低约 18%，尽管内核本身极快。
*   **质量评估范围**：虽然 Top-1 token 一致，但尚未在 RULER 或 Needle-in-a-Haystack 等长上下文基准上进行详尽的困惑度（Perplexity）分析。
*   **带宽瓶颈**：受限于 70B 模型的权重加载速度，端到端速度仍被限制在 ~10 tok/s 左右，无法仅通过优化注意力内核彻底解决。
