## 摘要
大语言模型（LLM）推理受限于 GPU 显存容量和带宽。分层内存架构通过允许 GPU 将内存卸载到远程层来缓解这一问题。然而，现有的内存卸载框架依赖于将数据预取到本地 GPU HBM 中。这种方法通过引入 HBM 竞争、浪费内存容量并产生流水线气泡，导致系统资源利用不足。我们证明，启用 GPU 直接访问远程内存的效果显著优于预取，能够实现最优的系统聚合带宽。我们提出了 DAK，这是一个端到端的直接访问内存卸载框架，它重新利用张量内存加速器（TMA）将卸载的权重和 KV 缓存从远程内存异步直接获取到 GPU 共享内存（SMEM）中。为了最大化远程访问性能，DAK 引入了一种贪心算法来确定每个操作的最优卸载比例，并结合主动拥塞控制和 TMA 组播来消除互连瓶颈和读取放大。在不同架构上的评估表明，与最先进的内存卸载基准相比，DAK 实现了接近最优的带宽聚合，在 NVLink-C2C 上获得了高达 3 倍的性能提升，在 PCIe 系统上获得了 1.8 倍的性能提升。

## Abstract
LLM inference is constrained by GPU memory capacity and bandwidth. Tiered memory architectures mitigate this by allowing the GPU to offload memory to the remote tier. However, existing memory offloading frameworks rely on prefetching data into local GPU HBM. This approach underutilizes system resources by introducing HBM contention, squandering memory capacity, and creating pipeline bubbles. We show that enabling direct GPU access to remote memory significantly outperforms prefetching, achieving optimal aggregate system bandwidth. We propose DAK, an end-to-end direct-access memory offloading framework that repurposes the Tensor Memory Accelerator (TMA) to asynchronously fetch offloaded weights and KV caches directly from remote memory into GPU shared memory (SMEM). To maximize remote access performance, DAK introduces a greedy algorithm to determine optimal per-operation offloading ratios, alongside active congestion control and TMA multicast to eliminate interconnect bottlenecks and read amplification. Evaluations across diverse architectures show that DAK achieves near-optimal bandwidth aggregation, with up to 3$\times$ performance gains on NVLink-C2C and 1.8$\times$ on PCIe systems compared to state-of-the-art memory offloading baselines.

### DAK: 面向大语言模型推理的具有最优效率的直接访问 GPU 内存卸载

#### 1. 核心问题与背景（研究动机）
随着大语言模型（LLM）参数量和上下文长度的激增，GPU 的本地显存（HBM）容量和带宽已成为推理性能的瓶颈（即“内存墙”）。虽然分层存储架构（如通过 NVLink-C2C 或 PCIe 连接 CPU 内存）提供了扩展空间，但现有的卸载框架（如 FlexGen）主要依赖**“预取（Prefetching）”**机制。
论文指出预取存在三大缺陷：
*   **带宽竞争**：预取数据从远程内存写入 HBM 时，会与计算内核读取 HBM 的流量竞争带宽。
*   **容量浪费**：需要在 HBM 中开辟静态缓冲区（Bounce Buffers），减少了可用于 KV Cache 的有效空间。
*   **流水线气泡**：粗粒度的层级预取难以实现计算与通信的完美重叠。

#### 2. 方法论
DAK 的核心思想是**“直接访问（Direct Access）”**，即让 GPU 的流式多处理器（SM）绕过 HBM，直接从远程内存读取数据到共享内存（SMEM）。

*   **核心技术细节**：
    *   **TMA 重新利用**：首次将 NVIDIA Hopper 架构中的张量内存加速器（TMA）从原本的“HBM 到 SMEM”传输，重新设计为“远程内存到 SMEM”的异步直接传输。
    *   **贪心卸载算法**：针对推理中的内存受限（Memory-bound）和计算受限（Compute-bound）算子，建立性能模型。算法分为三个阶段：优先分配预算给内存受限算子以达到带宽聚合峰值，随后填充计算受限算子，最后处理剩余预算。
    *   **主动拥塞控制**：通过限制在途（In-flight）的 TMA 请求数量和分配给远程访问的 SM 数量，防止远程流量阻塞 GPU 内部互连，确保本地 HBM 访问不受干扰。
    *   **TMA 组播与主机感知调度**：针对 CPU 内存不可被 GPU L2 缓存的特性，利用 TMA Multicast 技术实现“一次读取，多 SM 广播”，消除读取放大效应。
    *   **算子实现**：开发了 `SplitK_GEMM` 和 `SplitK_FlashAttn` 内核，支持在 HBM 和远程内存之间切分权重和 KV 缓存。
