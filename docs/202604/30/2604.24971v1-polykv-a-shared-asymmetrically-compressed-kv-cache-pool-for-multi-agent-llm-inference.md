## Abstract
We present PolyKV, a system in which multiple concurrent inference agents share a single, asymmetrically compressed KV cache pool. Rather than allocating a separate KV cache per agent -- the standard paradigm -- PolyKV writes a compressed cache once and injects it into N independent agent contexts via HuggingFace DynamicCache objects. Compression is asymmetric: Keys are quantized at int8 (q8_0) to preserve softmax stability, while Values are compressed using TurboQuant MSE -- a Fast Walsh-Hadamard Transform (FWHT) rotation followed by 3-bit Lloyd-Max quantization with centroids tuned to N(0,1). We evaluate across two model scales (SmolLM2-1.7B-Instruct and Llama-3-8B-Instruct), three context lengths (600-7,194 tokens), and up to 15 concurrent agents. PolyKV achieves a stable 2.91x compression ratio across all configurations. On Llama-3-8B with 15 agents sharing a 4K-token context, PolyKV reduces KV cache memory from 19.8 GB to 0.45 GB -- a 97.7% reduction -- while maintaining only +0.57% perplexity degradation and a mean BERTScore F1 of 0.928. PPL delta does not grow with agent count and improves as context length increases, inverting to -0.26% at 1,851 coherent tokens. To our knowledge, no prior work combines a single shared, lossy-compressed KV pool with multi-reader concurrent agent access.

### 1. 核心问题与整体含义（研究动机和背景）
在多智能体推理场景中，多个智能体往往需要处理相同的长文档上下文。传统的做法是为每个智能体分配独立的 KV 缓存，这导致内存需求随智能体数量 $N$ 线性增长，成为系统扩展的核心瓶颈。
*   **研究动机**：现有的研究要么只关注 KV 缓存压缩（减少单个缓存大小），要么只关注 KV 共享（通过前缀缓存减少冗余），但从未有人将“有损压缩”与“多读者并发共享池”结合。
*   **核心目标**：创建一个“一次写入、多次读取”的压缩 KV 缓存池，让 $N$ 个并发智能体共享同一份内存，从而实现内存占用的极大幅度削减。

### 2. 论文提出的方法论
PolyKV 引入了 **SharedKVPool（共享 KV 池）** 和 **PooledAgent（池化智能体）** 两个核心抽象：

*   **核心思想**：对共享上下文进行一次预填充（Prefill），生成 KV 状态并进行非对称压缩，存储在单一对象中。
*   **关键技术细节**：
    *   **非对称压缩方案**：
        *   **Key (K)**：采用 **int8 (q8_0)** 量化。由于 Key 参与 Softmax 计算，对精度较敏感，使用 8 位量化以保持注意力机制的稳定性。
        *   **Value (V)**：采用 **TurboQuant MSE** 方案。首先进行快速沃尔什-哈达玛变换（FWHT）旋转，使数据分布趋于正态分布，然后使用针对 $N(0,1)$ 优化的 **3-bit Lloyd-Max 量化**。
    *   **注入机制**：通过修改 HuggingFace 的 `DynamicCache` 对象，将解压后的张量直接注入到每个智能体的推理层中，绕过了标准的逐层累积过程。
    *   **压缩比**：理论压缩比稳定在 **2.91倍**（相对于 16-bit 基准）。

### 3. 实验设计
*   **模型**：SmolLM2-1.7B-Instruct（用于概念验证）和 Llama-3-8B-Instruct（主要验证对象，支持 GQA）。
*   **数据集/场景**：
    *   自定义相干文档（阿波罗11号任务、ARPANET 历史）。
    *   WikiText-2（2K 和 4K 窗口）。
*   **Benchmark（基准）**：全精度（bfloat16）的独立 Per-agent DynamicCache。
*   **对比维度**：不同智能体数量（3, 5, 10, 15 个）、不同上下文长度（600 到 7,194 tokens）。
*   **评估指标**：困惑度（PPL）变化、BERTScore F1（语义相似度）、Token 重合率、内存占用。

### 4. 资源与算力
*   **硬件使用**：
    *   Llama-3-8B 实验在 **Kaggle T4 ×2** GPU 上运行。
    *   SmolLM2-1.7B 实验在 CPU 上运行。

### 5. 实验数量与充分性
*   **实验规模**：共设计了 10 组核心测试配置（Phase 0 到 Test 9）。
*   **充分性**：实验涵盖了从轻量级到主流中型模型（8B），并测试了极端的并发智能体数量（15个）。
*   **客观性**：通过 PPL 和 BERTScore 双重指标评估，既关注了预测概率的准确性，也关注了生成内容的语义一致性。实验结果在不同模型架构（MHA 和 GQA）上表现一致，具有较好的说服力。

### 6. 主要结论与发现
*   **内存大幅削减**：在 15 个智能体共享 4K 上下文的场景下，KV 缓存内存从 19.8 GB 降至 0.45 GB，**缩减率达 97.7%**。
*   **质量稳定性**：PPL 的增加（退化）不随智能体数量增加而增长。在 Llama-3-8B 上，4K 上下文的 PPL 仅增加 0.57%。
*   **PPL 逆转现象**：在长且高度相干的文档中，压缩后的 PPL 甚至低于全精度基准（-0.26%）。
*   **正则化假设**：作者提出，TurboQuant 的 FWHT 噪声可能在长文本推理中起到了类似“隐式正则化”的作用，抑制了冗余 token 的干扰。

### 7. 优点
*   **架构创新**：首次实现了共享池与有损压缩的结合，内存复杂度从 $O(N)$ 降至 $O(1)$（相对于智能体数量）。
*   **非对称设计的合理性**：针对 K 和 V 的不同敏感度采用不同位宽，平衡了压缩率与模型表现。
*   **即插即用**：兼容 HuggingFace 生态，无需对模型进行微调（Tuning-free）。

### 8. 不足与局限
*   **模型规模覆盖**：尚未在 70B 或更大规模的模型上验证，虽然理论上 head dimension 越大效果越好，但仍需实测。
*   **评估协议**：WikiText-2 的评估采用了固定窗口而非标准的滑动窗口（stride-based），可能导致与其他论文的 PPL 数据无法直接横向对比。
*   **系统性能指标缺失**：未报告首字延迟（TTFT）、吞吐量或端到端延迟的具体数据。
*   **因果验证**：关于“噪声正则化”导致 PPL 逆转的假设尚未进行严格的消融实验验证。
