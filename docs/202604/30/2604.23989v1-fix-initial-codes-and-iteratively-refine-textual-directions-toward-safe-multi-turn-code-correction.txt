# Abstract 

Recent work on large language models (LLMs) has emphasized the im-portance of scaling inference compute. From this perspective, the state-of-the-art method S CATTERED FOREST SEARCH (SFS) has been proposed, employing Monte Carlo Tree Search with carefully crafted initial seeds and textual optimization for multi-turn code correction. However, its complex-ity makes it unclear what factors contribute to improvements in inference performance. To address this problem, we analyze SFS and propose a sim-pler method, I TERATIVE REFINEMENT OF TEXTUAL DIRECTIONS (IRTD), which fixes initial codes and iteratively refines textual directions. Because of the simplicity of IRTD, we theoretically establish the safety of IRTD using Oracle-Guided Inductive Synthesis (OGIS). Experiments on several code generation benchmarks suggest that IRTD achieves inference performance comparable to state-of-the-art methods. These results indicate that, even without complex search structures, refining initial codes with high-quality textual directions alone can effectively improve inference performance. 

# 1 Introduction 

Recent work on large language models (LLMs) has increasingly focused on scaling inference-time compute to improve inference performance (Brown et al., 2024; Snell et al., 2024). In such approaches, the goal is to obtain higher-quality outputs by expanding the computa-tional process at inference time, without modifying the model’s parameters or the training data. Thus, this paradigm is collectively referred to as inference scaling. Several represen-tative approaches have been proposed under this paradigm, including best-of-N (BoN) sampling (Cobbe et al., 2021; Lightman et al., 2023), linear self-refinement methods (Madaan et al., 2023; Shinn et al., 2023), and tree-search-based methods that integrate multiple reason-ing trajectories (Tian et al., 2024; Zhou et al., 2024). A representative task in the study of inference scaling is code generation. In this context, multi-turn code correction, in which an initial program is refined through multiple rounds of feedback, has been widely studied as a practical approach to improving output quality. Previous methods such as BoN and iterative refinement have been shown to improve performance (Chen et al., 2022; Wang et al., 2024). One reason for this effectiveness is that providing enough trials increases the probability that an LLM will sample a correct answer (Brown et al., 2024; Snell et al., 2024). However, Light et al. (2025) have shown that existing methods often generate highly similar outputs, resulting in insufficient exploration of the solution space. Light et al. (2025) have proposed S CATTERED FOREST SEARCH (SFS) that builds on Monte Carlo Tree Search (MCTS) and incorporates techniques such as careful seed initialization and textual optimization. They have empirically reported that SFS achieves state-of-the-art performance in multi-turn code correction (Light et al., 2025). However, since SFS tightly integrates multiple components, it is unclear what factors drive improvements in inference performance. This ambiguity hinders the development of simpler and more 1

To address the problem, we analyze SFS to identify the essential factors that contribute to performance improvements. In these analyses, we reveal that breadth-wise exploration toward high-quality textual directions plays a crucial role in the self-refinement process. From this finding, we propose I TERATIVE REFINEMENT OF TEXTUAL DIRECTIONS that fixes initial codes and iteratively refines textual directions. IRTD is a simpler method than SFS for multi-turn code correction and is capable of achieving comparable inference performance. Moreover, because of its simplicity, we theoretically establish the safety of IRTD using the framework of Oracle-Guided Inductive Synthesis (OGIS) (Jha & Seshia, 2017a;b). This theoretical analysis ensures that IRTD does not lose the true solution and can reach the correct answer. In summary, our main contributions include: • We analyze SFS and reveal that breadth-wise exploration is more important than depth-wise exploration for multi-turn code correction. • We propose a simpler method, IRTD, which iteratively revises textual directions while fixing initial codes as the target codes. • We theoretically analyze IRTD based on the framework of OGIS and show the safety of IRTD as a multi-turn code correction method. This guarantees that IRTD does not lose the true solution in the refinement process. • We empirically evaluate IRTD on several code generation benchmarks and confirm that IRTD achieves performance comparable to SFS. 

# 2 Related work 

Recent studies have increasingly focused on improving model performance by allocating more computation at inference time (Brown et al., 2024; Snell et al., 2024). One of the simplest and most common methods is best-of-N (BoN) sampling (Cobbe et al., 2021; Lightman et al., 2023). In this approach, multiple samples are generated from the same prompt to ensure diversity among responses and improve output quality. Prior work has also proposed self-correction methods in which an initial response is generated and then iteratively verified and revised (Madaan et al., 2023; Shinn et al., 2023). These methods improve the model’s responses by incorporating internal or external feedback as additional information. Yang et al. (2025) has proposed a probabilistic theory of self-correction to explain the performance improvements observed in these methods. Furthermore, incorporating tree search methods such as DFS, BFS, and MCTS has been shown to be effective for inference scaling. Prior work has applied tree search to multi-step tasks, including selecting the next reasoning step in reasoning processes (Yao et al., 2023; Besta et al., 2024; Tian et al., 2024; Zhou et al., 2024) and decoding the next token during generation (Feng et al., 2023; Zhang et al., 2023). Tree search has also been applied to optimization problems, where it functions as a black-box method for exploring a solution space (Light et al., 2025). Overall, we build upon previous research (Light et al., 2025) and aim to identify the factors that truly contribute to performance improvements in increasingly complex inference scaling methods. Motivated by these observations, our study proposes a simpler method and demonstrates that performance comparable to state-of-the-art methods can be achieved without relying on complex search strategies. 

# 3 Understanding S CATTERED FOREST SEARCH (SFS) 

This section provides an overview of SFS, which is a state-of-the-art method for multi-turn code correction, and presents analyses of SFS. 2Preprint. Under review. 

3.1 Problem formulation 

Following Light et al. (2025), we formulate code generation as a black-box optimization problem (Golovin et al., 2017). A code generation task is represented as x = ⟨p, H⟩, where p

denotes a prompt written in natural language or code, and H represents a set of hidden test cases used for evaluation. The objective is to provide an LLM with a prompt p and generate a code c ∼ LLM (p) that passes all hidden tests in H. Although the model cannot access the hidden tests, it can generate a set of validation tests V. Using these validation tests 

V and the program execution environment, the model obtains feedback on its generated code c. This feedback is then used to determine whether the code should be refined. In the evaluation phase, generated codes are evaluated against a hidden test set H and are deemed correct only when all tests are passed. For each task, an LLM is allowed to generate codes up to k times, and a task is considered solved if at least one of them is correct. Following these rules, for a task set X, the evaluation metric Pass@k Rate (Chen et al., 2021) is calculated as the proportion of tasks x ∈ X that are solved with at most k generated codes. An overview of this problem formulation is illustrated in Appendix A. 

3.2 SCATTERED FOREST SEARCH (SFS) 

Several inference-time methods have been explored in prior work to improve LLM perfor-mance on code generation tasks. However, Light et al. (2025) have shown that these methods often generate highly similar candidate codes, resulting in insufficient exploration of the solution space. To address this challenge, they have also proposed S CATTERED FOREST 

SEARCH (SFS) (Light et al., 2025). SFS is a MCTS-based method that combines careful seed initialization with textual optimization and consists of the following three components. • SCATTERING : Dynamically varies input prompts with different textual directions when sampling revised codes. • F ORESTING : Generates multiple initial codes that act as starting points for MCTS. • SCOUTING : Provides feedback on textual directions employed during the refine-ment process and distills general insights that can be shared across search branches. Their study has confirmed that SFS outperforms existing methods in terms of inference scaling performance (Light et al., 2025). Details of SFS are provided in Appendix B.

# 4 ITERATIVE REFINEMENT OF TEXTUAL DIRECTIONS (IRTD) 

From the findings in Section 3.3, we hypothesize that if textual directions are of high quality, refining initial codes may become effective in multi-turn code correction. Accordingly, we propose I TERATIVE REFINEMENT OF TEXTUAL DIRECTIONS (IRTD), which refines initial codes through iterative feedback on textual directions. • Section 4.1 presents the framework of IRTD, which becomes simpler than SFS. • Section 4.2 theoretically analyzes IRTD based on the framework of Oracle-Guided Inductive Synthesis (OGIS) (Jha & Seshia, 2017a;b). 

4.1 Proposed framework 

Given an input prompt, IRTD first generates one or multiple initial codes. It then generates one or more textual directions for the selected initial code and uses one of them to refine the code. After this refinement step, it evaluates the refined code on the validation tests and provides feedback on the used textual direction, which is stored as shared information. This shared information is then used as an additional context for an LLM when generating textual directions in subsequent refinement steps. IRTD repeats this refinement process iteratively until a desired condition is met. In contrast to existing self-refinement methods, IRTD fixes the initial codes as the refinement targets throughout the process. Moreover, by giving feedback on textual directions , IRTD enhances the probability of successful refinements. An overview of IRTD is illustrated in Appendix D. 4.2 Theoretical analysis using Oracle-Guided Inductive Synthesis (OGIS) 

We show the safety of IRTD using the framework of OGIS (Jha & Seshia, 2017a;b). In OGIS, program safety is defined by the absence of counterexamples found by an oracle (a formal verifier). Building on this, the following three properties regarding the version space 

are required to guarantee that an inductive algorithm interacting with the oracle reliably converges to a safe program. • The version space shrinks monotonically with each step by eliminating incorrect hypotheses that contradict the counterexamples provided by the oracle. • As a premise for the search, the target hypothesis is assumed to exist within the initial hypothesis space. • Since the target hypothesis does not contradict any counterexample, it is never eliminated from the version space .By satisfying these properties, it is shown that the algorithm can reliably synthesize a safe program in a finite number of steps. Therefore, it is important to theoretically demonstrate the safety of the method in multi-turn code correction.

# 5 Experiments 

We numerically evaluated IRTD on several code generation benchmarks. • Section 5.1 describes the experimental setup. • Section 5.2 reports the inference performance of IRTD compared to baseline methods. Although IRTD is a simpler method than SFS, it achieves comparable performance. • Section 5.3 reports the diversity of textual directions in IRTD. # 6 Conclusion 

We have analyzed the state-of-the-art method SFS and found that breadth-wise exploration toward high-quality textual directions plays an important role in the self-refinement process. From this analysis, we propose IRTD as a simpler method for multi-turn code correction. The simplicity of IRTD enables theoretical analysis, allowing us to provide guarantees on the safety of the method. Experimental results demonstrate that IRTD achieves competitive performance compared to SFS without requiring complex search structures. These findings provide new insights into the mechanisms underlying improved inference performance in multi-turn code correction and suggest promising directions for future research. 9Preprint. # B Details of S CATTERED FOREST SEARCH (SFS) 

To enable diverse exploration of the solution space in code generation, Light et al. (2025) have proposed S CATTERED FOREST SEARCH (SFS). SFS builds on MCTS and incorporates techniques such as careful seed initialization and textual optimization. This method encour-ages broader exploration of the solution space by combining LLM-driven textual directions, the introduction of multiple initial codes, and the utilization of feedback and past search experience. Specifically, SFS consists of three main components: S CATTERING , F ORESTING ,and S COUTING , as illustrated in Figure 4. 

SCATTERING : This component dynamically varies input prompts with different textual directions when sampling revised codes. More concretely, given a code selected by MCTS 11 Preprint. Under review. 

Figure 3: Overview of the problem formulation. Adapted from Light et al. (2025). Given a prompt p, an LLM iteratively generates and refines codes c on the basis of feedback from validation tests V. The objective is to generate a code ctrue that passes all hidden tests H.along with its associated feedback, an LLM generates multiple textual directions. Then, one of these textual directions is selected and incorporated into the prompt for generating the revised code. 

FORESTING : This component generates multiple initial codes that act as starting points for MCTS. In this process, initial codes are generated using either the same base prompt or diversified prompts obtained by randomly appending instruction statements. 

SCOUTING : This component provides feedback on the textual directions used during the refinement process and distills general insights that can be shared across search branches. These insights are incorporated into the prompt as auxiliary information when an LLM generates reflections and textual directions. This enables the promotion of effective textual directions and the suppression of unproductive ones.

E.1 Conceptual origin of version space 

The notion of version space originates as a fundamental concept in learning theory, where it denotes the set of hypotheses that are consistent with the observed data. In learning theory, let H denotes the hypothesis space, and let D = {(x1, y1) , . . . , (xn, yn)} denotes the observed (training) data. Then, the version space is defined as 

VS H(D) := {h ∈ H | h (xi) = yi for all i} . (26) From the perspective of version space , learning can be viewed as a process of progressively eliminating hypotheses based on the training data.
