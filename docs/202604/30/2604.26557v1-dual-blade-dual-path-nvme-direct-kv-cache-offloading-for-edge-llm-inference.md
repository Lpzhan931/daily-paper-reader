## 摘要
在边缘 AI 系统上部署大语言模型（LLM）推理的需求日益增长，这要求在紧张的内存预算下实现高效执行。一个关键挑战源于键值（KV）缓存，它通常会超出可用的设备内存。尽管基于 NVMe 的卸载提供了可扩展的容量，但现有的基于文件的设计严重依赖内核页面缓存（page cache），导致在内存压力下出现缓存抖动、不可预测的延迟以及高昂的软件开销。我们提出了 DUAL-BLADE，这是一个双路径 KV 驻留框架，它根据运行时内存可用性，动态地将 KV 张量分配到页面缓存路径或 NVMe 直连路径。NVMe 直连路径通过将 KV 张量映射到连续的逻辑块地址（LBA）区域来绕过文件系统，从而实现低开销的直接存储访问。DUAL-BLADE 进一步结合了自适应流水线并行，以使存储 I/O 与 GPU DMA 重叠，从而提高推理吞吐量。我们的评估表明，DUAL-BLADE 显著缓解了 I/O 瓶颈，在不同内存预算下，分别将预填充（prefill）和解码（decode）延迟降低了高达 33.1% 和 42.4%，同时将 SSD 利用率提高了 2.2 倍。

## Abstract
The increasing deployment of Large Language Model (LLM) inference on edge AI systems demands efficient execution under tight memory budgets. A key challenge arises from Key-Value (KV) caches, which often exceed available device memory. Although NVMe-based offloading offers scalable capacity, existing file-based designs rely heavily on the kernel page cache, leading to cache thrashing, unpredictable latency, and high software overhead under memory pressure. We present DUAL-BLADE, a dual-path KV residency framework that dynamically assigns KV tensors to either a page-cache path or an NVMe-direct path based on runtime memory availability. The NVMe-direct path bypasses the filesystem by mapping KV tensors to contiguous logical block address (LBA) regions, enabling low-overhead direct storage access. DUAL-BLADE further incorporates adaptive pipeline parallelism to overlap storage I/O with GPU DMA, improving inference throughput. Our evaluation shows that DUAL-BLADE substantially mitigates I/O bottlenecks, reducing prefill and decode latency by up to 33.1% and 42.4%, respectively, while improving SSD utilization by 2.2x across diverse memory budgets.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）向边缘 AI 系统（如本地服务器、嵌入式设备）迁移，**内存受限**成为了核心瓶颈。LLM 推理过程中产生的 **KV 缓存（KV-Cache）** 随上下文长度增加而剧增，往往超出 GPU 显存甚至主机 DRAM。
*   **现有方案缺陷**：目前的磁盘卸载（Offloading）方案（如 FlexLLMGen）依赖操作系统的页面缓存（Page Cache）和文件系统接口。在内存压力下，这会导致严重的**页面缓存抖动（Thrashing）**、高昂的内核软件栈开销以及 SSD 顺序访问特性的丧失，从而大幅增加推理延迟。
*   **研究目标**：设计一种高效的存储卸载架构，在极低内存预算下实现高性能的 KV 缓存读写。

#### 2. 方法论：核心思想与关键技术
DUAL-BLADE 提出了三项核心设计：
*   **双路径 KV 驻留（Dual-Path KV Residency）**：
    *   根据运行时可用的 DRAM 预算，将 KV 张量动态划分为两组：**Group 1（页面缓存路径）**利用剩余 DRAM 提高命中率；**Group 2（NVMe 直连路径）**直接绕过内核缓存，防止大批量 I/O 挤占有限的内存资源。
*   **NVMe 直连与顺序 LBA 放置（NVMe-direct with Sequential-LBA Placement）**：
    *   **内核绕过**：利用 `io_uring_cmd` 接口绕过虚拟文件系统（VFS）和块设备层，直接发送原生 NVMe 命令。
    *   **地址映射**：将 KV 张量映射到连续的逻辑块地址（LBA），确保在 SSD 控制器层面维持完美的顺序访问模式，降低闪存转换层（FTL）的开销。
*   **自适应流水线并行（Adaptive Pipeline Parallelism）**：
    *   在多线程环境下，动态选择“重叠策略”。通过“交错启动（Staggered Start）”技术，使存储 I/O 操作与 GPU DMA 传输在时间上重叠，从而隐藏 I/O 延迟。

#### 6. 主要结论与发现
*   **性能提升**：相比 Baseline，DUAL-BLADE 将 **Prefill（预填充）延迟降低了高达 33.1%**，将 **Decode（解码）延迟降低了高达 42.4%**。

*   **精准击中痛点**：首次在 LLM 卸载领域识别并解决了 LRU 页面缓存与自回归解码模式之间的冲突（缓存抖动问题）。

*   **应用限制**：主要针对单 GPU 边缘系统。在多 GPU 或分布式推理场景下的适用性尚未讨论。
