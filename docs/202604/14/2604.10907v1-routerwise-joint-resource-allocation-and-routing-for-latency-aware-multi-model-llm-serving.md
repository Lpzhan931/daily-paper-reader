## 摘要
多模型 LLM 路由已成为一种有效的方法，通过将每个提示词（prompt）分配给合适的模型，在保持输出质量的同时降低推理成本和延迟。然而，先前的路由方法通常假设每个模型具有固定的延迟。在实际部署中，这一假设并不准确：多个模型通常共享有限的 GPU 资源，且模型的延迟在很大程度上取决于其分配的资源以及由路由策略产生的请求负载。因此，路由与资源分配是紧密耦合的。在这项工作中，我们研究了 GPU 集群中面向延迟感知的多模型 LLM 推理服务的联合资源分配与路由问题。给定一组已部署的模型和延迟服务水平目标（SLO），我们寻求一种系统配置和路由策略，在满足延迟目标的同时最大化整体输出质量。我们将该问题形式化为针对部署配置和路由比例的有约束联合优化问题，并提出了 RouterWise。它结合了用于得分最大化路由的双重价格（dual-price）公式，以及通过系统性能分析得出的特定配置下的延迟模型。RouterWise 在可行的系统配置中进行搜索，并针对每个固定配置计算延迟目标下的最佳路由策略。我们的结果表明，即使在相同的 GPU 集群上，不同配置下可实现的输出质量得分差异可达 87%，这突显了资源分配是决定路由性能的关键因素。

## Abstract
Multi-model LLM routing has emerged as an effective approach for reducing serving cost and latency while maintaining output quality by assigning each prompt to an appropriate model. However, prior routing methods typically assume that each model has a fixed latency. In real deployments, this assumption is inaccurate: multiple models often share limited GPU resources, and a model's latency depends strongly on both its allocated resources and the request load induced by the routing policy. Consequently, routing and resource allocation are tightly coupled.   In this work, we study joint resource allocation and routing for latency-aware multi-model LLM serving in GPU clusters. Given a set of deployed models and a latency service-level objective (SLO), we seek a system setup and routing policy that maximize overall output quality while satisfying the latency target. We formalize this problem as a constrained joint optimization over deployment setup and routing fractions, and propose RouterWise, which combines a dual-price formulation for score-maximizing routing with setup-specific latency models derived from system profiling. RouterWise searches over feasible system setups and, for each fixed setup, computes the best routing policy under the latency target. Our results show that even on the same GPU cluster, achievable output-quality score can vary by up to 87% across retained setups, highlighting that resource allocation is a key determinant of routing performance.

### 2. 方法论：RouterWise 框架
RouterWise 将问题建模为一个受约束的联合优化问题，分为三个核心阶段：
*   **阶段 1：基于双重价格（Dual-Price）的路由优化**
    *   引入“模型价格”概念：为每个模型分配一个动态价格（对偶变量），反映其资源消耗。
    *   决策逻辑：对于每个提示词，路由器选择“预测质量得分减去模型价格”最大的模型。这通过子梯度下降法高效求解，将复杂的全局约束分配问题转化为简单的逐提示词决策。
*   **阶段 2：基于系统剖析（Profiling）的延迟建模**
    *   针对不同的部署配置（TP 等级、GPU 线程百分比），预先测量模型在不同负载下的延迟。
    *   使用分段线性模型（Piecewise-linear）来近似延迟与负载的关系，从而准确捕捉非线性行为。
*   **阶段 3：联合优化流水线**
    *   **搜索与剪枝**：在离散的配置空间（TP $\in \{1,2,4\}$，线程比例 $\in \{10\%-100\%\}$）中搜索。通过显存容量检查和计算利用率阈值过滤掉不可行或低效的配置。
    *   **双层优化**：对于每个保留的配置，使用二分查找法寻找最优的拉格朗日乘子（控制得分与延迟的权衡），最终选出得分最高的配置-路由组合。

### 3. 实验设计
*   **数据集**：使用 **RouterBench**，包含 36,000 个提示词及其在不同模型上的质量得分。
*   **模型池**：包含三个不同规模的模型：Mistral-7B、Vicuna-13B 和 Yi-34B。
*   **对比基准（Baselines）**：
    *   **Equal-Split**：平均分配 GPU 计算资源。
    *   **Size-Proportional**：按模型参数规模比例分配资源。
    *   **Isolated**：禁用资源共享，每个模型独占 GPU（传统部署方式）。
*   **场景设置**：分别在 4 个和 8 个 GPU 的集群规模下，测试不同的请求到达率（RPS）和延迟目标（SLO）。

*   **硬件**：实验基于 **NVIDIA A100 80GB PCIe GPU**。
*   **软件栈**：使用 **vLLM** 作为推理引擎，利用 **NVIDIA MPS**（多进程服务）实现 GPU 线程份额的精细化控制。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 4 GPU 和 8 GPU 两种集群规模，测试了从 50 到 80 RPS 的多种负载压力，以及从 400ms 到 800ms 的不同延迟目标。

*   **客观性**：对比了多种启发式分配策略，并考虑了显存和计算力的物理约束，实验设计较为严谨。

### 6. 主要结论与发现
*   **资源分配至关重要**：在相同的 GPU 集群上，仅通过改变资源分配方案，可实现的最高输出得分差异竟高达 **87%**。
*   **协同部署的优势**：允许模型共享 GPU（Co-location）比隔离部署（Isolated）能更有效地利用资源，尤其是在高负载下。
*   **负载敏感性**：最优的资源分配方案会随请求负载（RPS）的变化而变化，静态分配无法达到最优性能。

### 7. 优点与亮点
*   **首创性**：这是首个明确研究多模型 LLM 服务中“资源分配”与“路由”联合优化的工作。
*   **系统与算法结合**：不仅停留在数学优化层面，还结合了 NVIDIA MPS 等底层系统特性，具有很强的工程落地价值。
*   **高效性**：通过双重价格公式简化了路由决策，使得在线推理时的开销极低。

### 8. 不足与局限
*   **扩展性挑战**：随着模型数量增加，配置搜索空间呈指数级增长。虽然论文展示了 4 个模型的扩展性，但对于拥有数十个模型的超大规模模型池，搜索效率可能成为瓶颈。

*   **冷启动开销**：新模型加入系统时，需要进行详尽的性能剖析，这增加了部署初期的准备工作量。
