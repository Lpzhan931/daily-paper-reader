## 摘要
键值（KV）缓存通过存储中间注意力状态并避免自回归生成过程中的冗余计算，在加速大语言模型（LLM）推理中起着至关重要的作用。然而，其内存占用随序列长度线性增长，在资源受限的硬件上往往会导致严重的内存瓶颈。先前的研究探索了将 KV 缓存卸载到 CPU，同时仅在 GPU 上保留子集的方法，但这些方法通常依赖于不精确的 token 选择，并在思维链推理等长生成任务中面临性能下降。在本文中，我们提出了一种新颖的 KV 缓存管理策略 IceCache，它将语义 token 聚类与 PagedAttention 相结合。通过将语义相关的 token 组织到由分层、动态可更新数据结构管理的连续内存区域中，我们的方法实现了更高效的 token 选择，并在 CPU-GPU 传输过程中更好地利用了内存带宽。LongBench 上的实验结果表明，在 256 个 token 的预算下，IceCache 保持了完整 KV 缓存模型 99% 的原始准确率。此外，与其他基于卸载的方法相比，IceCache 在仅使用 25% 的 KV 缓存 token 预算的情况下，实现了具有竞争力甚至更优的延迟和准确率，证明了其在长序列场景中的有效性。代码可在我们的项目网站 https://yuzhenmao.github.io/IceCache/ 获取。

## Abstract
Key-Value (KV) cache plays a crucial role in accelerating inference in large language models (LLMs) by storing intermediate attention states and avoiding redundant computation during autoregressive generation. However, its memory footprint scales linearly with sequence length, often leading to severe memory bottlenecks on resource-constrained hardware. Prior work has explored offloading KV cache to the CPU while retaining only a subset on the GPU, but these approaches often rely on imprecise token selection and suffer performance degradation in long-generation tasks such as chain-of-thought reasoning. In this paper, we propose a novel KV cache management strategy, IceCache, which integrates semantic token clustering with PagedAttention. By organizing semantically related tokens into contiguous memory regions managed by a hierarchical, dynamically updatable data structure, our method enables more efficient token selection and better utilization of memory bandwidth during CPU-GPU transfers. Experimental results on LongBench show that, with a 256-token budget, IceCache maintains 99% of the original accuracy achieved by the full KV cache model. Moreover, compared to other offloading-based methods, IceCache attains competitive or even superior latency and accuracy while using only 25% of the KV cache token budget, demonstrating its effectiveness in long-sequence scenarios. The code is available on our project website at https://yuzhenmao.github.io/IceCache/.

### 1. 核心问题与研究动机
*   **核心问题**：在自回归生成过程中，KV 缓存的大小随序列长度线性增长。在处理长文本（如长文档摘要、多步推理）时，巨大的内存占用会导致显存溢出（OOM）或严重的性能下降。
*   **研究动机**：
    *   现有的**丢弃策略**（Eviction-based）虽然快，但往往基于静态规则，容易丢失关键信息。
    *   现有的**卸载策略**（Offloading-based）将缓存存至 CPU，但由于缺乏精确的 Token 选择机制，导致 CPU-GPU 传输效率低下，且在复杂任务（如思维链推理）中准确率下降明显。
    *   现有的 **PagedAttention** 机制按 Token 原始顺序存储，导致语义相关的 Token 散落在不同页面，检索效率低。

### 2. 方法论
IceCache 的核心思想是将**语义 Token 聚类**与 **PagedAttention** 集成，通过层级化数据结构实现精准检索。
*   **核心技术细节**：
    *   **DCI-tree（层级索引树）**：基于多级动态连续索引（M-DCI）算法，将 Key 向量映射到高维空间并进行聚类。语义相似的 Token 被组织在同一个逻辑节点中。
    *   **语义感知分页**：不同于传统按顺序分页，IceCache 将 DCI-tree 的叶子节点直接映射到物理内存页。这意味着语义相关的 Token 在物理内存上是连续的。
    *   **三阶段流程**：
        1.  **索引（Indexing）**：在 Prefill 阶段或卸载时，构建/更新 DCI-tree。
        2.  **页面选择（Page Selection）**：推理时，利用当前 Query 在 DCI-tree 中进行快速近似最近邻搜索（ANN），识别最相关的 Top-k 个页面。
        3.  **批量加载（Bulk Loading）**：通过预设的 CPU/GPU 缓冲区，将选中的非连续页面打包，利用 PCIe 高带宽一次性传输。
    *   **流水线优化（Pipelining）**：将 CPU 端的索引构建与 GPU 端的计算重叠（Overlap），隐藏索引延迟。

### 7. 优点（亮点）
*   **语义局部性利用**：首次将语义聚类深度集成到分页内存管理中，解决了长文本中关键信息散落的问题。
*   **动态更新能力**：DCI-tree 支持增量更新，能够适应长生成任务中不断变化的上下文。
*   **系统级优化**：不仅有算法创新，还通过批量传输和流水线并行解决了 PCIe 带宽瓶颈和 CPU 计算开销问题。

### 8. 不足与局限
*   **硬件依赖**：该方法依赖于较强的 CPU 多线程能力（实验使用了 64 线程）来处理 DCI 索引，在 CPU 资源受限的设备上可能存在瓶颈。

*   **超参数敏感性**：DCI-tree 的层级数、提升比例（Promotion ratio）等超参数可能需要针对不同模型进行微调。
