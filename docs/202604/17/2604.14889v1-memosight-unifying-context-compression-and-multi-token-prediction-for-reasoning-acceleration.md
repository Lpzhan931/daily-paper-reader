## 摘要
虽然思维链（CoT）推理使大语言模型（LLMs）能够解决具有挑战性的推理问题，但由于 KV 缓存随生成标记的数量线性增长，CoT 推理在速度和内存使用方面面临扩展性问题。在这项工作中，我们提出了 MemoSight（基于记忆-预见的推理），这是一个集成了上下文压缩和多标记预测的统一框架，旨在缓解效率问题，同时保持 CoT 推理性能。我们的框架通过特殊标记及其针对每种标记类型定制的相应位置布局，为上下文压缩和多标记预测采用了相同的极简设计。在四个推理基准测试上的全面实验表明，MemoSight 将 KV 缓存占用减少了高达 66%，并将推理速度提升了 1.56 倍，同时性能优于现有的 CoT 压缩方法。

## Abstract
While Chain-of-thought (CoT) reasoning enables LLMs to solve challenging reasoning problems, as KV cache grows linearly with the number of generated tokens, CoT reasoning faces scaling issues in terms of speed and memory usage. In this work, we propose MemoSight (Memory-Foresight-based reasoning), a unified framework that integrates both context compression and multi-token prediction to mitigate the efficiency issues while maintaining CoT reasoning performance. Our framework adopts the same minimalist design for both context compression and multi-token prediction via special tokens and their corresponding position layout tailored to each token type. Comprehensive experiments on four reasoning benchmarks demonstrate that MemoSight reduces the KV cache footprint by up to 66% and accelerates inference by 1.56x, while outperforming existing CoT compression methods.

**研究动机与背景：**
大语言模型（LLM）通过思维链（CoT）推理显著提升了解决复杂问题的能力。然而，基于 Transformer 的架构在推理时面临严重的效率瓶颈：**KV 缓存（Key-Value Cache）随生成标记数量线性增长**，导致内存占用巨大且推理延迟增加。

目前存在两种互补的优化方向：
*   **上下文压缩：** 将过去的推理步骤浓缩为少量的“记忆标记”，以减少内存占用。
*   **多标记预测（MTP）：** 训练模型一次预测多个标记，结合投机采样（Speculative Sampling）来加速生成。

**核心挑战：** 简单地将两者结合往往会导致性能下降。论文认为这是因为两者的训练范式和架构要求存在冲突（压缩通常在微调阶段，而 MTP 常需修改架构并在预训练阶段引入）。

### 2. 方法论
**核心思想：**
提出 **MemoSight** 框架，通过引入特殊标记和定制化的**位置感知对齐（Position-Aware Alignment, PAA）**，在不修改模型架构的前提下，将上下文压缩与多标记预测统一在一个数据驱动的微调框架中。

**关键技术细节：**
*   **特殊标记载体：**
    *   **记忆标记 $\langle m \rangle$：** 用于压缩已完成的推理步骤。
    *   **预见标记 $\langle f \rangle$：** 用于触发多标记预测，实现并行规划。
*   **位置感知对齐 (PAA)：**
    *   **步内预见投影：** 给 $\langle f \rangle$ 分配未来 $d$ 个位置的 Position ID，使其学习预测 $d$ 步后的标记。
    *   **步间记忆冷凝：** 给 $\langle m \rangle$ 分配当前推理步内的插值 Position ID，使其学习总结该步内容。
*   **统一注意力掩码：**
    *   **预见隔离：** $\langle f \rangle$ 标记作为独立预测分支，不干扰主推理链。
    *   **步间压缩：** 强制后续标记仅关注压缩后的记忆标记 $\langle m \rangle$，从而允许丢弃原始推理标记的 KV 缓存。
*   **推理流程（加速-压缩循环）：**
    1.  **加速阶段：** 利用 $\langle f \rangle$ 生成多个候选标记，通过投机采样并行验证并接受。
    2.  **压缩阶段：** 推理步结束时，生成 $\langle m \rangle$ 总结上下文，随后从 KV 缓存中剔除冗余的原始推理标记。
