## 摘要
自投机解码是一种针对大语言模型的推理技术，旨在不牺牲输出质量的前提下加速生成。它将使用模型紧凑版本作为草稿模型的快速近似解码，与完整目标模型的选择性重新评估相结合。一些现有方法通过在推理过程中动态学习要跳过的层来构建草稿模型，从而有效地创建一个较小的子网络以加速计算。然而，使用基于启发式的方法来选择要跳过的层通常更简单且更有效。在本文中，我们提出了 ConfLayers，这是一种通过基于置信度的中间层跳过来构建自投机解码中草稿模型的动态即插即用方法。该过程迭代地计算所有层的置信度分数，根据自适应阈值选择要跳过的层，评估所得集合的性能，并更新最佳选择，直到不再有改进或达到最大迭代次数。该框架避免了训练层跳过策略的开销和复杂性，在保持草稿模型对不同任务和数据集的自适应性的同时，能够提供更一致的速度与质量权衡。对 ConfLayers 在不同模型和数据集上的性能评估表明，我们的新方法相比原始大语言模型生成提供了高达 1.4 倍的加速。

## Abstract
Self-speculative decoding is an inference technique for large language models designed to speed up generation without sacrificing output quality. It combines fast, approximate decoding using a compact version of the model as a draft model with selective re-evaluation by the full target model. Some existing methods form the draft model by dynamically learning which layers to skip during inference, effectively creating a smaller subnetwork to speed up computation. However, using heuristic-based approaches to select layers to skip can often be simpler and more effective. In this paper, we propose ConfLayers, a dynamic plug-and-play approach to forming the draft model in self-speculative decoding via confidence-based intermediate layer skipping. The process iteratively computes confidence scores for all layers, selects layers to skip based on an adaptive threshold, evaluates the performance of the resulting set, and updates the best selection until no further improvement is achieved or a maximum number of iterations is reached. This framework avoids the overhead and complexity of training a layer skipping policy and can provide more consistent speed-quality trade-offs while preserving the adaptivity of the draft model to diverse tasks and datasets. The performance evaluation of ConfLayers across different models and datasets shows that our novel approach offers up to 1.4x speedup over vanilla LLM generation.

### 1. 核心问题与背景（研究动机）
*   **核心问题**：大语言模型（LLM）的自回归解码过程由于逐个生成 Token，计算成本高且延迟大。
*   **研究背景**：
    *   **投机解码**通常需要一个额外的、经过训练的小型草稿模型，这增加了部署复杂性和内存开销。
    *   **自投机解码（SSD）**通过跳过原模型的某些层来构建“子网络”作为草稿模型，无需额外模型。
    *   **现有痛点**：现有的 SSD 方法要么需要复杂的策略训练（Policy-based），要么使用静态的启发式规则，缺乏对不同任务和输入的自适应性，导致速度与质量的权衡不够理想。

### 2. 方法论
ConfLayers 提出了一种**即插即用、无需训练**的动态层选择机制，核心步骤包括：
*   **置信度计算**：在推理过程中，提取中间层的隐藏状态，通过共享的 LM Head 投影得到 Logits。利用 **Shannon 熵**衡量预测的不确定性，并将其转化为归一化的置信度分数（0 到 1 之间）。
*   **自适应窗口过滤**：
    *   不使用全局固定阈值，而是计算每一层相对于其邻近层的局部置信度均值。
    *   **关键技术**：窗口大小是自适应的。根据置信度变化的二阶梯度（波动程度）和层深度动态调整窗口。早期层使用小窗口进行精细比较，后期层使用大窗口进行上下文聚合。
*   **层跳过决策**：如果某层的置信度显著低于局部基准（由超参数 $\lambda$ 控制），则该层在草稿阶段被跳过。
*   **周期性迭代搜索**：框架在生成过程中定期评估当前“跳层集合”的性能（以接受的 Token 数量为指标），并不断更新最佳配置，直到性能达标或达到最大迭代次数。

### 6. 主要结论与发现
*   **加速效果**：ConfLayers 在各种模型和任务中始终优于 Vanilla 生成，最高实现 **1.4 倍**的加速。
*   **优于基准**：在大型模型（如 70B）上，ConfLayers 的加速效果显著超过 SWIFT 和 DEL。
*   **质量保持**：通过 Rouge-2 等指标验证，该方法在加速的同时几乎不损失生成质量。
*   **目标函数优势**：ConfLayers 优先考虑最大化“被接受的 Token 数量”而非单纯的“接受率”，这使其在实际推理中比 SWIFT 更高效。

### 7. 优点
*   **无需训练**：完全即插即用，不需要对原模型进行任何微调或策略训练。
*   **细粒度自适应**：通过自适应窗口处理不同层之间的置信度波动，比粗粒度的跳层方法更精准。
*   **低开销搜索**：搜索过程在线进行，置信度计算开销极低，且能快速收敛到最优跳层配置。
*   **泛化性强**：实验证明在一个任务上找到的跳层集合在其他任务上依然能保持较好的加速效果。

### 8. 不足与局限
*   **超参数依赖**：虽然是自适应的，但仍需要预设一些超参数（如 $\lambda$ 和最小跳层比例），这些参数在不同模型族之间可能需要微调。
*   **搜索初期波动**：在生成的前几十个 Token 中，由于处于搜索阶段，加速效果可能不如稳定期。
*   **采样限制**：实验主要基于贪婪采样（Greedy Sampling），虽然这是投机解码的常见用法，但在需要高随机性的创造性写作任务中表现尚未充分验证。
