## 摘要
智能体工作流（Agentic workflows）通过编排多个大语言模型（LLMs）和工具来执行复杂任务。在保证低延迟的同时以目标吞吐量提供此类工作流服务具有挑战性，因为它们可以使用任意智能体框架定义，且表现出不可预测的执行时间：执行过程可能会根据数据产生分支、扇出或递归。由于工作流中的 LLM 数量通常超过可用 GPU 的数量，其执行还会导致 GPU 超额订阅。我们介绍了 Scepsy，这是一种新型智能体服务系统，能够高效地将任意多 LLM 智能体工作流调度到 GPU 集群上。Scepsy 利用了这样一个洞察：虽然智能体工作流具有不可预测的端到端延迟，但每个 LLM 在总执行时间中所占的比例在不同执行过程中相对稳定。Scepsy 根据这些聚合比例决定 GPU 分配：首先，它在不同的并行度下对 LLM 进行性能分析。然后，它利用这些统计数据构建聚合 LLM 流水线（Aggregate LLM Pipeline），这是一种用于分配方案的轻量级延迟/吞吐量预测器。为了在达到目标吞吐量的同时找到最小化延迟的 GPU 分配方案，Scepsy 使用聚合 LLM 流水线在分数级 GPU 份额、张量并行度和副本数量构成的搜索空间中进行探索。它采用分层启发式算法将最佳分配方案部署到 GPU 集群上，在遵守网络拓扑约束的同时最小化碎片化。我们在真实智能体工作流上的评估表明，与独立优化 LLM 或依赖用户指定分配的系统相比，Scepsy 实现了高达 2.4 倍的吞吐量提升和 27 倍的延迟降低。

## Abstract
Agentic workflows carry out complex tasks by orchestrating multiple large language models (LLMs) and tools. Serving such workflows at a target throughput with low latency is challenging because they can be defined using arbitrary agentic frameworks and exhibit unpredictable execution times: execution may branch, fan-out, or recur in data-dependent ways. Since LLMs in workflows often outnumber available GPUs, their execution also leads to GPU oversubscription.   We describe Scepsy, a new agentic serving system that efficiently schedules arbitrary multi-LLM agentic workflows onto a GPU cluster. Scepsy exploits the insight that, while agentic workflows have unpredictable end-to-end latencies, the shares of each LLM's total execution times are comparatively stable across executions. Scepsy decides on GPU allocations based on these aggregate shares: first, it profiles the LLMs under different parallelism degrees. It then uses these statistics to construct an Aggregate LLM Pipeline, which is a lightweight latency/throughput predictor for allocations. To find a GPU allocation that minimizes latency while achieving a target throughput, Scepsy uses the Aggregate LLM Pipeline to explore a search space over fractional GPU shares, tensor parallelism degrees, and replica counts. It uses a hierarchical heuristic to place the best allocation onto the GPU cluster, minimizing fragmentation, while respecting network topology constraints. Our evaluation on realistic agentic workflows shows that Scepsy achieves up to 2.4x higher throughput and 27x lower latency compared to systems that optimize LLMs independently or rely on user-specified allocations.

随着大语言模型（LLM）应用从单一对话转向**智能体工作流（Agentic Workflows）**，系统面临严峻的推理挑战。这些工作流通常编排多个不同规模的 LLM（如生成器、验证器、重排序器）和外部工具。
*   **核心挑战**：
    *   **执行不可预测**：工作流包含数据驱动的分支、循环和扇出，导致端到端延迟波动巨大。
    *   **资源超额订阅**：工作流中的模型数量往往超过物理 GPU 数量，导致资源竞争。
    *   **现有方案不足**：传统系统要么孤立地优化单个 LLM（忽视了模型间的依赖），要么依赖用户手动分配 GPU（效率低下且难以扩展）。
*   **研究动机**：开发一个能够自动、高效地将任意多 LLM 智能体工作流调度到 GPU 集群上的服务系统。

### 2. 方法论：核心思想与关键技术
Scepsy 的核心洞察是：**虽然工作流的绝对延迟不可预测，但每个 LLM 在总执行时间中所占的比例（Aggregate Shares）在统计上是稳定的。**

*   **聚合 LLM 流水线（Aggregate LLM Pipeline, ALP）**：
    1.  **工作流追踪**：通过 HTTP 代理捕获 LLM 调用，提取平均调用次数（$n_m$）和平均并行度（$p_m$）。
    2.  **性能分析（Profiling）**：通过回放真实追踪数据，在不同张量并行（TP）和到达率下测量每个 LLM 的性能。
    3.  **流水线合成**：将工作流抽象为由 LLM 组成的流水线，通过公式 $L_w = \sum (L_m \cdot n_m / p_m)$ 预测端到端延迟和吞吐量。
*   **联合 GPU 调度**：
    *   在三个维度搜索最优分配：**分数级 GPU 份额（Fractional GPU shares）**、**张量并行度（TP）**和**副本数量（Replica counts）**。
    *   利用延迟比例进行剪枝，优先为延迟贡献大的模型分配更多资源。
*   **拓扑感知放置**：
    *   采用分层启发式算法，优先放置大尺寸或高 TP 要求的模型，确保其处于 NVLink 域内，同时利用 NVIDIA MPS 实现 GPU 内部的资源隔离。

### 6. 主要结论与发现
*   **性能提升**：相比 Kubernetes HPA，Scepsy 在相同 GPU 预算下实现了高达 **2.4 倍的吞吐量提升**和 **27 倍的延迟降低**。

### 7. 优点
*   **框架无关性**：不依赖特定的智能体框架（如 LangChain 或 AutoGen），通过底层追踪即可工作。
*   **精细化资源管理**：引入分数级 GPU 分配，极大地提高了小模型（如 Embedding）与大模型共存时的 GPU 利用率。
*   **联合优化**：同时考虑了数据并行（副本）和模型并行（TP），在吞吐量和延迟之间找到了更好的平衡点。

### 8. 不足与局限
*   **关键路径建模简化**：目前的 ALP 模型将所有 LLM 视为串行流水线，对于复杂的跨 LLM 并发（Fan-out 到不同模型）可能建模不够精确。
*   **双峰负载分布**：如果同一个 LLM 在工作流中承担完全不同的角色，单一的聚合统计可能会失效。
*   **工具执行时间**：目前假设外部工具执行时间可忽略不计，若工作流包含耗时极长的外部操作（如科学实验模拟），系统预测准确度会下降。
