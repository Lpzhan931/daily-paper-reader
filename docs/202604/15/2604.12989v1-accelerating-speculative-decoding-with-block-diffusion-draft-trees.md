## 摘要
投机解码通过使用轻量级草稿模型预测多个未来 token，并由目标模型进行并行验证，从而加速自回归语言模型。DFlash 表明，块扩散草稿模型可以在单次前向传播中生成整个草稿块，并实现最先进的投机解码性能，优于 EAGLE-3 等强力自回归草稿模型。然而，原始 DFlash 每轮仍仅验证单条草稿轨迹，这可能限制了其接受长度。我们提出了 DDTree（扩散草稿树），这是一种直接从块扩散草稿模型的逐位置分布中构建草稿树的方法。在固定节点预算下，DDTree 使用简单的最佳优先堆算法，根据草稿模型输出定义的代理指标，选择最有可能与目标模型匹配的续写。生成的树通过使用仅祖先注意力掩码，在目标模型的单次前向传播中得到高效验证。由于 DDTree 构建在领先的投机解码草稿模型 DFlash 之上，这些提升使 DDTree 跻身投机解码的领先方法之列。

## Abstract
Speculative decoding accelerates autoregressive language models by using a lightweight drafter to propose multiple future tokens, which the target model then verifies in parallel. DFlash shows that a block diffusion drafter can generate an entire draft block in a single forward pass and achieve state-of-the-art speculative decoding performance, outperforming strong autoregressive drafters such as EAGLE-3. Vanilla DFlash, however, still verifies only a single drafted trajectory per round, potentially limiting its acceptance length. We introduce DDTree (Diffusion Draft Tree), a method that constructs a draft tree directly from the per-position distributions of a block diffusion drafter. Under a fixed node budget, DDTree uses a simple best-first heap algorithm to select the continuations that are most likely to match the target model according to a surrogate defined by the draft model's output. The resulting tree is verified efficiently in a single target model forward pass using an ancestor-only attention mask. Because DDTree builds on DFlash, a leading draft model for speculative decoding, these gains place DDTree among the leading approaches to speculative decoding.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：如何更充分地利用“块扩散（Block Diffusion）”草稿模型的输出信息，以进一步提升大语言模型（LLM）的推理速度。
*   **背景**：投机解码通过轻量级草稿模型预测 token，再由目标模型并行验证。**DFlash** 是目前最先进的草稿模型之一，它利用块扩散技术在单次前向传播中生成整个 token 块。
*   **动机**：原始 DFlash 虽然生成速度快，但每轮仅验证一条单一的路径（Trajectory），忽略了扩散模型输出中包含的其他高概率候选 token。如果能将这些候选 token 组织成“草稿树（Draft Tree）”进行并行验证，理论上可以显著提高接受长度（Acceptance Length），从而获得更高的加速比。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：从块扩散模型产生的逐位置边缘分布中，构建一个在给定节点预算 $B$ 下能最大化“预期接受长度”的草稿树。
*   **关键技术细节**：
    *   **代理目标函数**：由于目标模型的条件概率不可知，DDTree 使用草稿模型的因子化分布（各位置概率的乘积）作为代理指标，证明了在该指标下，选择概率最高的 $B$ 个前缀构成的集合即为最优树。
    *   **最佳优先堆算法（Best-first Heap Algorithm）**：为了避免穷举指数级的路径，论文提出了一种基于最大堆的搜索算法。该算法以 $O(B \log B)$ 的复杂度，通过不断扩展当前最高概率的前缀（子节点或兄弟节点），高效地提取出最优的 $B$ 个节点。
    *   **树状验证**：将选定的节点编译成树状结构，利用“仅祖先（Ancestor-only）”注意力掩码，在目标模型的一次前向传播中完成整棵树的验证。
    *   **KV 缓存更新**：验证后根据匹配路径压缩 KV 缓存，并将第一个未匹配的 token 作为下一轮的“奖励 token”。

*   **目标模型**：Qwen3-4B、Qwen3-8B、Qwen3-Coder-30B-MoE。
*   **数据集（Benchmark）**：涵盖 10 个主流评测集，包括数学（MATH-500, GSM8K, AIME）、代码（HumanEval, MBPP, LiveCodeBench, SWE-bench Lite）和通用指令/对话（MT-Bench, Alpaca）。
*   **对比方法**：
    *   **Autoregressive Decoding**：标准的自回归解码（基准）。
    *   **Vanilla DFlash**：原始的单路径块扩散投机解码。
*   **评估指标**：相对于自回归解码的加速比（Speedup）、平均接受长度（$\tau$）。

### 4. 资源与算力

### 5. 实验数量与充分性
*   **实验规模**：论文在 **3 种模型规模**、**10 个数据集**以及 **2 种温度设置**（0.0 和 1.0）下进行了全面测试，总计 60 组核心对比实验。
*   **消融与分析**：进行了节点预算（Node Budget）从 16 到 1024 的权衡分析，并绘制了接受长度的分布直方图。
*   **充分性评价**：实验覆盖了从 4B 到 30B 不同规模的模型以及从逻辑推理到代码生成的多种任务，实验设计客观、对比公平（基准测试中甚至给予了对比方法更优的注意力算子选择权）。

### 6. 主要结论与发现
*   **显著加速**：DDTree 在所有测试场景下均优于原始 DFlash。在 Qwen3-30B 模型上，相对于自回归解码实现了高达 **8.2倍** 的加速。
*   **接受长度提升**：通过树状结构，DDTree 显著增加了每轮被接受的 token 数量，尤其是在数学和代码等逻辑性强的任务中。
*   **预算权衡**：实验发现节点预算并非越大越好，通常在 256 到 512 之间达到加速比峰值，过大的预算会导致验证开销超过接受长度带来的收益。

### 7. 优点
*   **理论完备性**：为草稿树的构建提供了数学证明，确保在给定代理目标下算法的最优性。
*   **极低开销**：利用块扩散模型单次前向传播的特性，避免了自回归草稿模型构建树时需要多次前向传播的开销。
*   **通用性**：方法不依赖于特定的任务领域，在多种类型的 LLM 任务中表现稳定。

### 8. 不足与局限
*   **代理指标偏差**：DDTree 假设草稿模型的边缘分布乘积能代表目标模型的路径概率，但在草稿模型与目标模型分布差异较大时，这种代理可能失效。
*   **硬件实现依赖**：加速效果高度依赖于树状注意力（Tree Attention）的实现效率。如果硬件或算子对非连续内存访问支持不佳，树状验证的开销可能会抵消算法优势。
*   **草稿模型限制**：该方法专门针对块扩散类草稿模型设计，无法直接应用于不输出并行分布的其他类型草稿模型。
