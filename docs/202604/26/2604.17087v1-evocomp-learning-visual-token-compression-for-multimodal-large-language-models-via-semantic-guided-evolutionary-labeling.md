## 摘要
近期，多模态大语言模型（MLLMs）在视觉-语言理解任务中展现出强大的性能，但其推理效率往往受限于大量的视觉 Token，特别是在高分辨率或多图场景下。为了解决这一问题，我们提出了 EvoComp，这是一个视觉 Token 压缩框架，能够在显著减少 Token 数量的同时保持任务准确率。EvoComp 引入了一个轻量级的、基于仅编码器 Transformer 的压缩器，通过联合考虑视觉和文本上下文来选择最具信息量且非冗余的视觉 Token。一个核心挑战在于如何为训练压缩器提供有效的监督。为此，我们设计了一种进化标注策略，通过搜索使 MLLM 输出损失最小化的 Token 子集，同时通过基于词表的 Token 分组来强制实现语义多样性。我们进一步使用定制的损失函数训练压缩器，该函数结合了 GHM 损失以缓解类别和难度不平衡，以及余弦相似度正则化以促进保留 Token 与舍弃 Token 之间的语义分离。在多个视觉-语言基准测试上的广泛实验表明，EvoComp 优于现有的基于注意力或相似度启发式的方法。值得注意的是，在 3 倍 Token 压缩下，它保留了 99.3% 的原始准确率，并在移动设备上实现了高达 1.6 倍的加速。

## Abstract
Recent Multimodal Large Language Models (MLLMs) have demonstrated strong performance on vision-language understanding tasks, yet their inference efficiency is often hampered by the large number of visual tokens, particularly in high-resolution or multi-image scenarios. To address this issue, we propose EvoComp, a visual token compression framework that significantly reduces token count while preserving task accuracy. EvoComp introduces a lightweight encoder-only transformer-based compressor that selects the most informative and non-redundant visual tokens by jointly considering visual and textual contexts. A core challenge lies in providing effective supervision for training the compressor. To this end, we design an evolutionary labeling strategy that searches for token subsets minimizing the MLLM's output loss, while enforcing semantic diversity through vocabulary-based token grouping. We further train the compressor using a tailored loss function combining the GHM loss to mitigate class and difficulty imbalance, and a cosine similarity regularization to encourage semantic separation between retained and discarded tokens. Extensive experiments across multiple vision-language benchmarks show that EvoComp outperforms existing methods based on attention or similarity heuristics. Notably, it retains 99.3% of the original accuracy under 3x token compression and delivers up to 1.6x speedup on mobile devices.

多模态大语言模型（MLLM）在处理高分辨率图像或多图任务时，会产生数以千计的视觉 Token，导致推理时的内存消耗和延迟呈二次方增长，尤其在移动端等资源受限设备上难以部署。
**核心问题：** 现有的视觉 Token 压缩方法多依赖于注意力分数（存在位置偏差）或相似度度量（忽略语义重要性），且缺乏直接针对模型输出准确性进行优化的监督信号。
**研究目标：** 开发一种能够兼顾任务相关性、语义多样性且易于部署的轻量级视觉 Token 压缩框架。

### 2. 方法论
EvoComp 框架由一个轻量级的 **Transformer 压缩器** 和一套 **进化标注策略** 组成：
*   **进化标注（Evolutionary Labeling）：** 
    *   **核心思想：** 将 Token 选择建模为一个搜索问题。利用进化算法在现有训练数据上搜索最优的二进制掩码（Binary Mask），目标是使保留 Token 后的 MLLM 输出损失（Task Loss）最小化。
    *   **语义分组策略：** 为了缩小搜索空间并保证多样性，利用 LLM 的词表嵌入作为锚点，将视觉 Token 按语义相似度分组，强制每组只选一个 Token，从而消除冗余。
*   **压缩器训练：**
    *   **模型结构：** 采用单层仅编码器（Encoder-only）Transformer，接收视觉和文本 Token，输出每个视觉 Token 的保留概率。
    *   **定制损失函数：** 
        1.  **GHM 损失（Gradient Harmonized Mechanism）：** 解决正负样本（保留 vs 舍弃）及难易样本之间的极度不平衡问题。
        2.  **余弦相似度正则化：** 惩罚保留 Token 与舍弃 Token 之间的表示相似度，增强分类边界的清晰度。
*   **推理阶段：** 压缩器作为插件置于视觉编码器与 LLM 之间，根据预测概率保留 Top-K 个 Token。

*   **任务导向：** 不同于传统的启发式方法，EvoComp 通过进化算法直接优化任务损失，确保了 Token 选择与最终输出质量挂钩。
*   **即插即用：** 压缩器是独立的轻量级模块，无需微调原始 MLLM 的参数，部署灵活。
*   **解决不平衡问题：** 引入 GHM 损失有效解决了视觉 Token 中“背景/冗余信息”远多于“关键信息”导致的训练偏差。
*   **端侧友好：** 提供了详尽的移动端 NPU 部署方案和实测数据，具有很强的实用价值。
