## 摘要
大语言模型（LLM）的长文本推理面临着注意力机制的二次方复杂度和极高的 KV 缓存（KV cache）内存占用这两大挑战。虽然 Token 级稀疏注意力提供了卓越的准确度，但其索引开销巨大；而块级方法虽然提高了效率，却牺牲了精度。我们提出了 AsyncTLS，这是一种分层稀疏注意力系统，它结合了粗粒度的块过滤与细粒度的 Token 选择，以平衡准确度与效率，并配备了一个异步卸载引擎，通过利用时间局部性使 KV 缓存传输与计算实现重叠。在 Qwen3 和 GLM-4.7-Flash 上针对 GQA 和 MLA 架构进行的评估表明，AsyncTLS 在 48k - 96k 上下文长度下，实现了与全量注意力相当的准确度，同时提供了 1.2 倍至 10.0 倍的算子加速以及 1.3 倍至 4.7 倍的端到端吞吐量提升。

## Abstract
Long-context inference in LLMs faces the dual challenges of quadratic attention complexity and prohibitive KV cache memory. While token-level sparse attention offers superior accuracy, its indexing overhead is costly; block-level methods improve efficiency but sacrifice precision. We propose AsyncTLS, a hierarchical sparse attention system that combines coarse-grained block filtering with fine-grained token selection to balance accuracy and efficiency, coupled with an asynchronous offloading engine that overlaps KV cache transfers with computation via temporal locality exploitation. Evaluated on Qwen3 and GLM-4.7-Flash across GQA, and MLA architectures, AsyncTLS achieves accuracy comparable to full attention while delivering 1.2x - 10.0x operator speedups and 1.3x - 4.7x end-to-end throughput improvements on 48k - 96k contexts.

*   **核心问题**：长文本推理中，自注意力机制具有**二次方计算复杂度**，且 **KV Cache（键值缓存）** 随序列长度线性增长，极易超出 GPU 显存限制。
*   **研究动机**：
    *   **Token 级稀疏方法**（如 H2O, Double-Sparsity）：精度高，但每步解码都要扫描全局 Token，索引开销（Indexing Overhead）巨大。
    *   **块级（Block-level）稀疏方法**（如 Quest）：效率高，但粗粒度的选择会引入噪声或丢失关键信息，导致精度下降。
    *   **内存瓶颈**：当 KV Cache 卸载（Offloading）到 CPU 时，PCIe 带宽成为瓶颈，导致推理停顿。
*   **整体含义**：AsyncTLS 通过分层架构结合了块级的效率和 Token 级的精度，并利用异步机制隐藏了内存传输延迟。

### 2. 论文提出的方法论
AsyncTLS 的核心由两个主要组件构成：
*   **两级稀疏注意力（Two-level Sparse Attention）**：
    *   **第一级（块级过滤）**：将 KV Cache 划分为固定大小的块。通过将 Quest 的评分机制重构为 **GEMM（通用矩阵乘法）算子**，快速筛选出包含重要信息的 Top-Kb 个块。
    *   **第二级（Token 级精选）**：在筛选出的块内，利用 **Double-Sparsity** 技术（基于通道重要性压缩）进行细粒度的 Token 选择，仅保留最相关的 Top-Kt 个 Token 进行最终计算。
*   **异步卸载引擎（Asynchronous Offloading Engine）**：
    *   **预取与时间重叠**：利用注意力模式在相邻时间步之间的**时间局部性（稳定性）**。在当前步计算注意力时，异步预取下一步可能需要的 KV 块。
    *   **增量块传输**：仅传输当前步与上一步选择结果之间的差异块（Incremental Difference），极大降低了 PCIe 带宽占用。
    *   **错位执行策略**：使用当前步的块过滤结果来预测下一步的 Token 选择需求，实现计算与传输的完全重叠。

### 7. 优点
*   **分层设计巧妙**：通过“先粗后细”的策略，解决了 Token 级稀疏索引开销过大的痛点。
*   **硬件友好**：将块选择重构为 GEMM 算子，能更好地利用 Tensor Cores。
*   **异步优化**：通过时间局部性隐藏了卸载场景下的传输延迟，使长文本推理在显存受限的设备上更具可行性。
*   **架构通用性**：同时支持 GQA 和最新的 MLA 架构。

### 8. 不足与局限
*   **时间局部性假设**：如果模型生成的文本发生剧烈的语义转折（导致注意力模式突变），异步预取的准确率可能会下降，从而影响性能。
*   **校准依赖**：虽然免训练，但仍需在特定数据集上进行通道校准，不同分布的任务可能需要重新校准以维持最优精度。
*   **硬件依赖**：性能提升高度依赖于 TileLang 等底层算子优化工具，普通用户直接实现的难度较大。
