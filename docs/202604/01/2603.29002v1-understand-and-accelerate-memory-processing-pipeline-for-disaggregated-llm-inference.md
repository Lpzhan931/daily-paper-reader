## Abstract
Modern large language models (LLMs) increasingly depends on efficient long-context processing and generation mechanisms, including sparse attention, retrieval-augmented generation (RAG), and compressed contextual memory, to support complex reasoning. We show that these optimizations can be unified into a four-step memory processing pipeline: Prepare Memory, Compute Relevancy, Retrieval, and Apply to Inference. Through systematic profiling, we identify a 22%-97% memory processing overhead in LLM inference and strong heterogeneity in its computational characteristics. Motivated by this insight, we argue that \textbf{heterogeneous systems} are well-suited to accelerate memory processing and thus end-to-end inference. We demonstrate this approach on a GPU-FPGA system by offloading sparse, irregular, and memory-bounded operations to FPGAs while retaining compute-intensive operations on GPUs. Evaluated on an AMD MI210 GPU and an Alveo U55C FPGA, our system is $1.04\sim2.2\times$ faster and requires $1.11\sim4.7\times$ less energy across multiple LLM inference optimizations than the GPU baseline (similar results hold on NVIDIA A100). These results establish heterogeneous systems as a practical direction for efficient LLM memory processing and inform future heterogeneous hardware design.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）向长上下文（128K 至 1M+ tokens）演进，KV Cache 的存储和访问成为了性能瓶颈。虽然业界提出了稀疏注意力（Sparse Attention）、检索增强生成（RAG）和压缩上下文内存等优化技术，但这些方法通常被视为孤立的算法。
*   **研究动机**：作者发现这些优化技术在底层逻辑上具有高度一致性，且在传统 GPU 架构上运行时，内存处理开销占总延迟的 **22%–97%**。
*   **核心问题**：GPU 在处理稀疏、不规则且内存受限（Memory-bound）的操作时效率低下，如何通过异构硬件架构加速这一通用的“内存处理流水线”？

#### 2. 论文提出的方法论
论文的核心思想是将多样化的 LLM 优化技术抽象为统一的**四阶段内存处理流水线**，并利用 **GPU-FPGA 异构系统**进行硬件匹配加速。

*   **四阶段流水线模型**：
    1.  **准备内存 (Prepare Memory)**：将原始内存预处理为索引或压缩格式（如线性投影、RoPE）。
    2.  **计算相关性 (Compute Relevancy)**：为内存条目分配重要性得分（如内积计算、BM25 评分）。
    3.  **检索 (Retrieval)**：根据得分选择子集（如 Top-k 选择、阈值过滤）。
    4.  **应用于推理 (Apply to Inference)**：将检索内容整合进解码过程。
*   **关键技术细节**：
    *   **异构映射策略**：将计算密集型、规则访问的操作（如线性层、Prefilling）保留在 **GPU**；将内存受限、不规则访问、数据依赖型操作（如 Top-k、BM25 查找、稀疏索引计算）卸载至 **FPGA**。
    *   **FPGA 内核设计**：采用流式数据流（Streaming Dataflow）架构，利用 FPGA 内部的三级存储层次（BRAM、URAM、HBM）提供极高的片上带宽（比 GPU SRAM 高约 5 倍）。
    *   **通信优化**：通过 PCIe P2P（Peer-to-Peer）技术绕过系统内存，直接在 GPU 和 FPGA 之间交换索引和向量数据。

#### 3. 实验设计
*   **场景与方法**：
    *   **稀疏注意力**：DeepSeek Attention, SeerAttention-R, LServe。
    *   **RAG**：DRAGIN, FLARE, 固定句子 RAG, 两阶段 RAG。
    *   **压缩/合成内存**：Titans/HMT, MemAgent。
*   **数据集**：Wikipedia dump（用于 RAG 检索）、长文本推理数据集。
*   **Benchmark**：以纯 GPU 实现作为基准（使用 vLLM、TileLang 等高度优化的内核）。
*   **对比维度**：端到端延迟、内存处理阶段延迟、能耗（J/request）。

#### 4. 资源与算力
*   **硬件配置**：
    *   **GPU**：1 台 AMD Instinct **MI210** (64GB HBM2e, 1.6 TB/s 峰值带宽)。
    *   **FPGA**：1 块 AMD Alveo **U55C** (16GB HBM2, 460 GB/s 峰值带宽)。
    *   **CPU**：AMD EPYC 7v13 (64 核)。
    *   **补充分析**：论文还通过性能建模评估了在 **NVIDIA A100** 上的表现。
*   **软件栈**：ROCm 6.3 (GPU), Vitis/Vivado 2024.2 (FPGA)。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 3 大类共 9 种主流的长上下文优化算法。
*   **序列长度**：测试范围从 4K 到 **1M tokens**，充分展示了随上下文增长时的性能趋势。
*   **Batch Size**：测试了从 1 到 32 的批处理规模，分析了系统的扩展性。
*   **客观性**：实验对比了相同算法在 GPU 上的最优开源实现（如 vLLM），并考虑了 PCIe 通信开销，实验设计较为公平、客观。

#### 6. 主要结论与发现
*   **加速效果**：相比纯 GPU 基准，异构系统实现了 **1.04× 至 2.2×** 的端到端加速。
*   **能效提升**：能耗降低了 **1.11× 至 4.7×**，这主要归功于 FPGA 在处理不规则计算时较低的运行功耗。
*   **瓶颈转移**：在 1M token 以上时，FPGA 的 HBM 带宽可能成为新瓶颈，系统需动态切换回 GPU。
*   **Batch 影响**：对于稀疏注意力和 RAG，Batch Size 越大，异构系统的加速比越明显；但对于合成内存（MemAgent），大 Batch 下 GPU 的计算密度优势更强，异构系统优势减弱。

#### 7. 优点
*   **理论统一性**：首次将碎片化的 LLM 优化技术统一到“内存处理流水线”框架下，具有很强的启发性。
*   **硬件匹配度高**：精准识别了 GPU 在 Top-k 和不规则内存访问上的短板，利用 FPGA 的可编程逻辑和片上高带宽存储进行针对性补强。
*   **实用性**：在不改变模型权重的前提下，通过系统级优化显著降低了长上下文推理的成本。

#### 8. 不足与局限
*   **硬件门槛**：FPGA 的开发周期长、难度大，且目前主流数据中心仍以纯 GPU 集群为主，异构部署的运维成本较高。
*   **通信瓶颈**：虽然 PCIe P2P 缓解了部分压力，但在极大规模数据交换场景下，PCIe 带宽仍远低于 NVLink 等专用互连。
*   **扩展性验证不足**：实验主要集中在单卡对单卡的异构组合，未深入探讨在多 GPU、多 FPGA 的分布式环境下的负载均衡和同步开销。
*   **模型覆盖**：主要针对 Transformer 架构及其变体，对于完全非 Transformer 的架构（如纯线性 RNN 或 Mamba）的适用性讨论较少。
