## 摘要
长上下文生成日益受到解码阶段键值（KV）缓存传输流量的限制，特别是当 KV 缓存被卸载到 GPU 显存之外时。查询感知检索（如 Top-K 选择）通过仅加载 KV 对的一个子集来减少此类流量，但当注意力权重分散在未检索到的标记（tokens）上时，对该子集重新进行 softmax 归一化会引入偏差。我们提出了一种检索-补全注意力模块，该模块保持骨干网络权重和 KV 缓存格式不变。对于每个查询，我们计算针对 sink/tail 锚点以及与查询相关的 Top-K 检索标记的精确注意力，并使用在预填充（prefill）阶段计算的固定大小特征图摘要来估计剩余中间区域的分子和分母。我们在未归一化域中累加精确贡献和估计贡献，并进行单次归一化，从而在不增加额外注意力侧 KV 读取的情况下恢复缺失的 softmax 权重。在长上下文基准测试中，该方法在相同的等效标记读取预算下优于仅采用选择策略的 Top-K，且在高熵注意力头中提升最为显著。

## Abstract
Long-context generation is increasingly limited by decode-time key-value (KV) cache traffic, particularly when KV is offloaded beyond GPU memory. Query-aware retrieval (e.g., Top-K selection) reduces this traffic by loading only a subset of KV pairs, but renormalizing the softmax over the subset introduces bias when attention mass is spread over unretrieved tokens. We propose a retrieval-completion attention module that keeps backbone weights and the KV-cache format unchanged. For each query, we compute exact attention over sink/tail anchors and the query-dependent retrieved Top-K tokens, and estimate the remaining mid-region numerator and denominator using a fixed-size feature-map summary computed at prefill time. We add the exact and estimated contributions in the unnormalized domain and apply a single normalization, recovering the missing softmax mass without additional attention-side KV reads. Across long-context benchmarks, the proposed method improves over selection-only Top-K at matched token-equivalent read budgets, with the largest gains in high-entropy heads.

#### 1. 核心问题与整体含义（研究动机和背景）
在长上下文大语言模型（LLM）生成任务中，推理速度主要受限于**KV缓存（KV Cache）的内存带宽瓶颈**，尤其是当上下文极长、KV缓存需要从 GPU 显存卸载（Offload）到 CPU 内存或 SSD 时，读取延迟成为主要矛盾。
*   **现有方案的缺陷**：传统的 Top-K 检索方法通过只读取一小部分 KV 对来减少 I/O，但这种“硬截断”会导致 Softmax 归一化偏置。当注意力权重分布较为分散（高熵）时，忽略未检索到的 Token 会导致严重的精度损失。
*   **研究目标**：在不改变模型权重、不改变 KV 缓存格式的前提下，通过一种轻量级的补全机制，在减少 KV 读取的同时恢复 Softmax 的完整性。

#### 2. 方法论：核心思想与关键技术
论文提出了一种**检索-补全（Retrieval-Completion）混合注意力机制**：
*   **Token 分区策略**：将预填充（Prefill）阶段的 Token 分为三部分：
    1.  **锚点（Anchors）**：包括起始的 Sink Token 和最近的 Tail Token，始终进行精确读取。
    2.  **中间区域（Mid-region）**：应用 Top-K 检索，仅精确读取权重最大的 $K$ 个 Token。
    3.  **剩余部分（Remainder）**：使用线性注意力摘要进行估算。
*   **固定大小补全缓存（Completion Cache）**：在预填充阶段，利用学习到的正向特征图（Positive Feature Maps）将中间区域的 KV 信息压缩成一个固定大小的摘要（类似于线性注意力）。
*   **减法补全（Subtractive Completion）**：在解码阶段，从摘要中减去已被 Top-K 检索到的 Token 的特征贡献，确保补全项仅针对未读取的 Token。
*   **单次归一化合并**：在非归一化域（Unnormalized Domain）将精确计算的分子/分母与补全项的估计分子/分母相加，最后进行统一的 Softmax 归一化。
*   **蒸馏训练**：为了使线性特征图能更好地模拟 Softmax 核函数，论文对每个注意力头进行了专门的轻量级蒸馏训练。
