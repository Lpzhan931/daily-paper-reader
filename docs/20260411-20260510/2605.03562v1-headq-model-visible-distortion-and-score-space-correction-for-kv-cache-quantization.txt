# HeadQ: Model-Visible Distortion and Score-Space Correction for KV-Cache Quantization 

## Jorge L. Ruiz Williams Independent Researcher jorgeruizwilliams@gmail.com May 6, 2026 

Abstract 

KV-cache quantizers usually optimize storage-space reconstruction, even though attention reads keys through logits and values through attention-weighted readout. We argue that persistent cache error should be measured in model-visible coordinates. For keys, the visible object is score error modulo constant shifts; this yields HeadQ, a key-side method that stores a low-rank residual side code in a calibration-learned query ba-sis and applies it as an additive logit correction. For values, fixed-attention readout gives an A2-weighted token-distortion surrogate. Across six mod-els, Fisher/score-space error predicts attention KL far better than raw key MSE; same-budget counterexamples, null-space interventions, query-PCA controls, and wrong-sign HeadQ falsify storage-MSE alternatives. Matched Pythia checkpoints localize the main anomaly to a small-model low-entropy route-flip boundary. In K-only WikiText-103 decode exper-iments with dense values, HeadQ removes roughly 84 –94% of the excess perplexity on the strongest 2-bit rows; in an auxiliary full-KV 2-bit com-position, HeadQ plus an A2 value policy improves all six models. 

# 1 Introduction 

KV caches are the persistent memory of autoregressive transformers [1, 2]. Dur-ing decoding, every new token reads previous keys and values, so cache size and bandwidth grow with context length [9, 10, 11]. Recent systems papers already exploit practical K/V asymmetries [9, 15, 16, 17], but the dominant optimiza-tion targets are still mostly storage-centered: mean-squared error, distortion, outlier suppression, or rotation quality [26, 14, 13, 22]. That is convenient, but it is not what attention reads. This paper has one central claim: persistent cache error should be mea-sured in model-visible coordinates. # 4 HeadQ: Key-Side Method 

HeadQ implements the score-space correction suggested by the fixed-query de-composition. It is a key-side side-code method layered on top of an ordinary low-bit key quantizer. At cache write time, HeadQ stores the base quantized key and a low-rank code for the base residual in a fixed per-head query basis learned from calibration queries. At attention read time, the current query con-verts that code into an additive correction to the logits. Values are unchanged in the K-only experiments. The main downstream evidence is concentrated on the rows where the baseline actually breaks. With dense values and a KIVI-like per-channel affine key base-line [9], HeadQ is strongest on Qwen2, Pythia, GPT-2, and Qwen2.5, where rank-4 HeadQ removes roughly 84 –94% of the excess PPL. # 6 Discussion 

The design lesson is operator matching. Keys are score objects, so good key compression avoids softmax-null range waste, protects query-visible residuals, and applies corrections where scores are computed. Values are readout objects, so the natural first surrogate is A2-weighted token error. This view helps explain why KIVI-like per-channel keys, KVQuant-style centering, key-favored precision schedules, and value readout weighting can all help for different reasons [9, 10, 15]. The tests cover different failure modes of storage MSE. Fisher-versus-MSE predicts attention KL; the paired counterexample reverses the storage rank-ing; the softmax-null intervention changes quantized behavior without chang-ing dense attention; query-PCA makes the active residual deployable; and the wrong-sign control verifies the direction of the deployed patch. The Pythia anomaly then becomes a boundary case: ultra-sharp attention rows under ro-tated/product 3-bit perturbations enter a finite route-flip regime, while matched 410M and 1B checkpoints recover the local ranking. 

# 7 Relation to KV-Cache Quantization Work 

KV-cache quantization methods such as KIVI [9], KVQuant [10], GEAR [11], QuaRot-style rotations [13], AXELRAM [21], RotateKV [22], PolyKV [23], LOOKAT [24], and SpinQuant [25] reduce cache memory through practical serv-ing designs. Token-importance methods such as H2O [12] choose which cache entries to retain or protect. Generic vector quantization and rotation meth-12 ods, including TurboQuant-style approaches [14] and product quantization [26], optimize storage-centered objectives such as MSE or product-code distortion. The closest papers are asymmetry-oriented. Quantize What Counts [15] ar-gues for key-favored precision using spectral and Frobenius structure; AsymKV [16] and KVTuner [17] use layer-wise sensitivity. Our claim is narrower: key sensitivity should be measured in query-conditioned behavioral geometry, not only by layer-level or norm-level statistics. KIVI [9] is conceptually important because it discovers a granularity asym-metry: per-channel keys and per-token values. Our fixed-query view gives one key-side reason such asymmetries can arise. TriAttention [18] is especially rele-vant geometrically: it uses pre-RoPE Q/K centers to estimate key importance for long-reasoning KV compression, whereas our object is a post-RoPE score-space mean-key component whose logit contribution is constant across positions and therefore softmax-inert. KIVI-like per-channel K repairs bad key coordi-nates; KVQuant-style centering avoids softmax-null range waste; TurboQuant-style MSE can be strong for value-side readout reconstruction while missing key-side quotient geometry. The central distinction is storage-coordinate versus model-visible distortion. 

# 8 Limitations 

The paper is mechanistic, not a serving-system claim. Oracle active-axis probes are diagnostic upper bounds; the main downstream PPL rows quantize keys and keep values dense; the auxiliary full-KV composition is unoptimized and includes side-code overhead before any packed metadata design. We do not study packed kernels, latency, or throughput. The model panel uses short real-corpus windows and six models up to Mistral 7B, below the 7B–70B frontier of recent systems papers [15, 16, 17]. The six-model KL-ranking table is a point-estimate summary rather than a confidence-interval result, and not every auxiliary probe spans exactly the same model set. The matched Pythia probes explain one verified family-specific route-flip boundary rather than a universal scaling law. The deployed wrong-sign control tests correction direction but does not ex-haust all same-budget side-code alternatives. Random side bases can be model-and window-dependent, so we use them only as attention-level controls. Query quantization also remains out of scope: queries are not persistent cache entries, and we do not show that the key-side chart transfers cleanly to Q. 

# 9 Conclusion 

KV-cache distortion should be defined in the coordinates the model reads. For keys, that is score error modulo constant shifts; for values, it is attention-weighted readout error with an A2 diagonal surrogate. HeadQ is the key-side consequence: store a small residual code in a calibration-learned query basis 13 and apply it directly as a logit correction. The evidence supports that operator-matched view. Score-space metrics beat raw MSE, paired controls expose storage-coordinate failures, query-PCA makes the useful residual deployable, and full-KV 2-bit composition shows the key correction can coexist with the value objective. The next step is to complete that same story as an optimized full-KV codec rather than treating storage MSE as the default target. 14 Appendix A Formal Statements and Proofs
