## 摘要
基于 Transformer 的大语言模型 (LLM) 依赖键值 (KV) 缓存，以避免自回归推理过程中的冗余计算。虽然这种机制极大地提高了效率，但缓存大小随输入序列长度线性增长，迅速成为长上下文任务的瓶颈。现有的解决方案通过在估计的重要性得分指导下，逐出被认为不重要的 Prompt KV 来缓解这一问题。值得注意的是，近期的一系列工作提出通过“窥见未来”来提高逐出质量，即利用草稿生成器产生一个逼近目标模型真实响应的代理未来响应，随后利用该代理更准确地估计缓存 KV 的重要性。然而，这些方法依赖于计算成本高昂的草稿生成，这引入了大量的预填充开销，并限制了它们在实际部署中的实用性。为了应对这一挑战，我们提出了 LookaheadKV，这是一个轻量级的逐出框架，它利用了代理未来响应的优势，而无需显式的草稿生成。LookaheadKV 通过参数高效的模块增强了 Transformer 层，这些模块经过训练可以高精度地预测真实的重要性得分。我们的设计确保了与现有廉价启发式方法相当的、可忽略不计的运行时开销，同时实现了优于更昂贵近似方法的准确性。在各种模型上进行的广泛长上下文理解基准测试实验表明，我们的方法不仅在各种长上下文理解任务中优于近期具有竞争力的基准模型，而且将逐出成本降低了高达 14.5 倍，从而显著缩短了首字响应时间 (Time-to-First-Token)。我们的代码可在 https://github.com/SamsungLabs/LookaheadKV 获取。

## Abstract
Transformer-based large language models (LLMs) rely on key-value (KV) caching to avoid redundant computation during autoregressive inference. While this mechanism greatly improves efficiency, the cache size grows linearly with the input sequence length, quickly becoming a bottleneck for long-context tasks. Existing solutions mitigate this problem by evicting prompt KV that are deemed unimportant, guided by estimated importance scores. Notably, a recent line of work proposes to improve eviction quality by "glimpsing into the future", in which a draft generator produces a surrogate future response approximating the target model's true response, and this surrogate is subsequently used to estimate the importance of cached KV more accurately. However, these approaches rely on computationally expensive draft generation, which introduces substantial prefilling overhead and limits their practicality in real-world deployment. To address this challenge, we propose LookaheadKV, a lightweight eviction framework that leverages the strength of surrogate future response without requiring explicit draft generation. LookaheadKV augments transformer layers with parameter-efficient modules trained to predict true importance scores with high accuracy. Our design ensures negligible runtime overhead comparable to existing inexpensive heuristics, while achieving accuracy superior to more costly approximation methods. Extensive experiments on long-context understanding benchmarks, across a wide range of models, demonstrate that our method not only outperforms recent competitive baselines in various long-context understanding tasks, but also reduces the eviction cost by up to 14.5x, leading to significantly faster time-to-first-token. Our code is available at https://github.com/SamsungLabs/LookaheadKV.

### LookaheadKV：通过无需生成的“窥见未来”实现快速且准确的 KV 缓存逐出

#### 1. 核心问题与整体含义（研究动机和背景）
在处理长文本任务时，Transformer 架构的键值（KV）缓存会随序列长度线性增长，导致巨大的显存压力。现有的 KV 缓存逐出（Eviction）方法存在“精度-延迟”的权衡：
*   **启发式方法（如 SnapKV）：** 速度快，但仅基于输入 Prompt 的后缀来估计重要性，在低预算（压缩率高）时准确率大幅下降。
*   **基于草稿的方法（如 SpecKV, LAQ）：** 通过“窥见未来”提高精度，即先用小模型或快速生成一段“草稿响应”，再根据草稿的注意力模式来决定保留哪些 KV。虽然精度高，但显式生成草稿的过程带来了极高的首字响应延迟（TTFT），限制了实际应用。
**研究动机：** 能否在不进行显式生成的情况下，获得“窥见未来”带来的高精度重要性预测？

#### 2. 方法论
LookaheadKV 的核心思想是**将“预测未来注意力模式”的任务转化为一个轻量级的表征学习问题**。
*   **核心组件：**
    *   **可学习的 Lookahead Tokens：** 在预填充（Prefill）阶段，在输入序列后附加一组可学习的软标记（Soft Tokens）。这些标记的 Query 被训练用于模拟模型未来真实响应的注意力分布。
    *   **Lookahead LoRA：** 引入一种选择性激活的低秩适配器。它仅在处理 Lookahead Tokens 时激活，而不改变原始 Prompt 标记的计算，从而保证了原始模型权重的完整性和行为。
*   **算法流程与公式：**
    1.  **训练阶段：** 使用离线生成的真实响应作为“地面真值”（Ground Truth）。计算真实响应对 Prompt 的注意力得分 $s_{GT}$。
    2.  **损失函数：** 通过最小化 Lookahead Tokens 产生的预测得分 $s_{LKV}$ 与 $s_{GT}$ 之间的 **KL 散度**（等价于 ListNet 排序损失），训练 Lookahead Tokens 和 LoRA 模块。
    3.  **推理阶段：** 在 Prefill 过程中，直接利用训练好的模块产生重要性得分，根据得分保留 Top-K 个 KV 对，随后直接进入解码阶段，无需生成任何中间草稿。

    *   **训练：** 混合了 ChatQA2（长指令）、Tulu（指令遵循）、The Stack（代码）以及 MetaMath 等数据集，共约 86K 样本，最大训练长度 16K。

*   **对比方法：**

#### 5. 实验数量与充分性
*   **实验规模：** 论文在 6 个不同规模和家族的模型上进行了测试，涵盖了从 1B 到 8B 的参数区间。

#### 6. 主要结论与发现
*   **性能卓越：** LookaheadKV 在所有测试的预算（Cache Budget 从 64 到 2048）下均优于现有方法，尤其在极低预算下优势更明显。
*   **极低开销：** 相比于基于草稿的方法（LAQ），LookaheadKV 将逐出开销降低了高达 **14.5 倍**。在 32K 上下文下，其带来的额外延迟仅为 2.16% 左右，与廉价的 SnapKV 相当。

*   **长文本生成优势：** 在 LongProc 任务中表现优异，说明其预测的未来注意力比仅看局部草稿的方法更具全局性。

#### 7. 优点
*   **打破权衡：** 成功解决了“高精度必须依赖高延迟草稿生成”的问题，实现了“既要快又要准”。
*   **即插即用：** 采用 LoRA 和附加 Token 形式，不改变原模型权重，可以根据需要随时开启或关闭。
*   **训练高效：** 相比全量微调，这种参数高效的训练方式成本极低，且对训练数据的依赖具有一定的弹性（使用原始数据响应替代模型生成响应也能维持较高性能）。

*   **模型规模限制：** 实验最高仅覆盖到 8B 模型，尚未在 70B 或更大规模的模型上验证（主要受限于计算资源）。

*   **数据依赖：** 训练过程仍需要一定量的长文本数据及其对应的模型响应，对于完全闭源或无法获取响应的模型，训练实施难度较大。
