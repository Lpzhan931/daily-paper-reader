## 摘要
高效的大语言模型（LLM）推理调度对于提升用户体验至关重要。然而，LLM 推理表现出显著的需求不确定性（输出长度预先未知）和混合特性（同时具有计算密集型和内存密集型特征）。现有的 LLM 调度器依赖于简单的启发式方法或仅关注计算资源，导致性能欠佳。在这项工作中，我们提出了 SageSched，这是一种能够妥善处理推理负载需求不确定性和混合特性的高效 LLM 调度器。SageSched 将提示词内容与过去的推理结果相结合，以轻量且准确的方式预测输出长度分布。同时，它在建模推理请求的真实服务成本时，综合考虑了计算和内存两个方面。最后，SageSched 采用了一种不确定性感知调度策略，能够在给定请求成本分布的情况下实现最佳的整体效率。在多种设置下的实验台测试证实，SageSched 可以实现超过 28.7% 的效率提升。

## Abstract
Efficient LLM inference scheduling is crucial for user experience.However, LLM inferences exhibit remarkable demand uncertainty (with unknown output length beforehand) and hybridity (being both compute and memory intensive). Existing LLM schedulers rely on simple heuristics or focus purely on compute resource, suffering suboptimal performance.   In this work, we propose SageSched, an efficient LLM scheduler that properly handles demand uncertainty and hybridity of inference workloads.SageSched combines prompt contents with the past inference results to predict output-length distribution in a light-weight and also accurate manner.Meanwhile, it models the true service cost of an inference request with both compute and memory aspects considered.Finally, SageSched employs an uncertainty-aware scheduling policy that can yield the best overall efficiency given the request cost distributions.Testbed experiments over diverse setups confirm that SageSched can attain an efficiency improvement of over 28.7%.

以下是对该论文的结构化深入总结：

### 1. 核心问题与研究动机
*   **研究背景**：随着 LLM 的广泛应用，如何在 GPU 服务器上高效调度并发请求以优化用户体验（尤其是缩短端到端延迟 TTLT）变得至关重要。
*   **核心挑战**：
    *   **需求不确定性 (Uncertainty)**：由于自回归生成机制，输出长度在推理完成前是不可知的，且同一提示词多次运行长度也可能不同。
    *   **需求混合性 (Hybridity)**：LLM 推理既是计算密集型（矩阵运算），又是内存密集型（KVCache 占用）。
*   **现有方案不足**：
    *   **FCFS（先来先服务）**：导致队头阻塞（Head-of-line blocking）。
    *   **预测型调度（如 SSJF, TRAIL）**：依赖重型模型预测单一数值（不准确），且忽略了内存开销对调度的影响。

### 2. 论文提出的方法论
SageSched 集成了三大关键技术：
*   **语义感知且基于历史的预测器 (Semantic-aware History-based Predictor)**：
    *   **核心思想**：利用提示词语义相似性与输出长度分布的相关性。
    *   **实现**：记录近期请求的提示词嵌入（Embedding）和输出长度。新请求到达时，通过向量检索（FAISS）寻找相似度 >0.8 的历史请求，直接获取其输出长度的**概率分布**，而非单一预测值。
*   **基于资源边界的成本建模 (Resource-bound-based Cost Modeling)**：
    *   **公式**：$C = \frac{O^2}{2} + IO$（其中 $I$ 为输入长度，$O$ 为输出长度）。
    *   **逻辑**：该模型统一了计算和内存成本。在内存受限时，它反映了 KVCache 的累积占用；在计算受限时，它反映了注意力机制随序列增长的计算量。
*   **不确定性感知调度策略 (Uncertainty-aware Scheduling)**：
    *   **核心算法**：采用 **Gittins Index（吉廷斯索引）** 策略。该策略源于多臂老虎机理论，能在已知分布但时长未知的情况下实现最优调度。
    *   **动态更新**：将请求成本划分为多个桶（Buckets），在推理过程中定期（如每 200 token）刷新 Gittins 索引，以确保调度计划的时效性。

### 3. 实验设计
*   **数据集**：使用了三个具有代表性的真实 LLM 数据集：SharedGPT（通用对话）、Alpaca-Summarization（摘要）、Document-Write（写作）。
*   **对比方法 (Benchmarks)**：
    *   **FCFS**：vLLM 默认调度。
    *   **FastServe**：基于多级反馈队列（MLFQ）的抢占式调度。
    *   **SSJF / LTR / TRAIL**：最先进的基于预测（单一值或排名）的调度器。
*   **评估指标**：平均 TTLT（Time-to-Last-Token，主要指标）和平均 TTFT（Time-to-First-Token）。

### 4. 资源与算力
*   **硬件平台**：
    *   **Llama3.1-8B**：部署在 A40-48GB GPU 服务器上。
    *   **Qwen3-32B**：部署在 H800-96GB GPU 服务器上。
*   **扩展性模拟**：构建了最高支持 **64 个 GPU 节点** 的模拟器，用于评估大规模集群下的调度开销。
*   **训练成本**：预测器无需离线训练（Training-free），仅需维护一个 10,000 条记录的历史窗口。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了混合数据集测试、单一数据集深度测试、消融实验（针对预测器、成本模型、调度策略分别验证）以及敏感性分析（针对相似度阈值、桶大小）。
*   **充分性评价**：实验设计较为全面，不仅在真实硬件上验证了性能提升，还通过模拟器证明了在大规模集群下的低开销（延迟约 100ms，相对于数秒的推理时间可忽略不计），结果具有较强的说服力和客观性。

### 6. 主要结论与发现
*   **性能提升**：SageSched 在平均 TTLT 指标上比现有最先进方案（如 TRAIL）提升了 **28.7%** 以上。
*   **多指标平衡**：在优化 TTLT 的同时，SageSched 也能保持优秀的 TTFT 表现，有效缓解了队头阻塞。
*   **分布预测的优势**：相比预测单一数值，保留概率分布信息能显著提升调度决策的质量。
*   **成本模型的有效性**：综合考虑 $I$ 和 $O$ 的二次方关系比单纯考虑输出长度更符合硬件运行的真实成本。

### 7. 优点
*   **轻量化**：无需为每个模型训练专门的预测模型，部署成本极低。
*   **理论严谨**：引入 Gittins Index，为不确定性下的作业调度提供了坚实的数学基础。
*   **通用性强**：适用于不同规模的模型（8B 到 32B+）和不同的硬件环境。

### 8. 不足与局限
*   **冷启动问题**：在系统运行初期或面对全新领域的提示词时，由于缺乏相似历史记录，预测准确度可能受限（虽然文中提到可用公共数据集补充）。
*   **语义依赖性**：该方法假设提示词语义相似则输出长度分布相似，但在某些特定任务（如逻辑推理或随机性极强的生成）中，这一假设的强度可能会减弱。
