## 摘要
高效的大语言模型（LLM）推理调度对于用户体验至关重要。然而，LLM 推理表现出显著的需求不确定性（输出长度预先未知）和混合特性（同时具有计算密集型和内存密集型特征）。现有的 LLM 调度器依赖简单的启发式方法或仅关注计算资源，导致性能不佳。在本研究中，我们提出了 SageSched，这是一种能够妥善处理推理工作负载需求不确定性和混合特性的高效 LLM 调度器。SageSched 将提示词内容与过去的推理结果相结合，以轻量且准确的方式预测输出长度分布。同时，它综合考虑计算和内存两个方面，对推理请求的真实服务成本进行建模。最后，SageSched 采用了一种不确定性感知调度策略，在给定请求成本分布的情况下，能够实现最佳的整体效率。在多种设置下的实验台实验证实，SageSched 可以实现超过 28.7% 的效率提升。

## Abstract
Efficient LLM inference scheduling is crucial for user experience. However, LLM inferences exhibit remarkable demand uncertainty (with unknown output length beforehand) and hybridity (being both compute and memory intensive). Existing LLM schedulers rely on simple heuristics or focus purely on compute resource, suffering suboptimal performance.   In this work, we propose SageSched, an efficient LLM scheduler that properly handles demand uncertainty and hybridity of inference workloads. SageSched combines prompt contents with the past inference results to predict output-length distribution in a light-weight and also accurate manner. Meanwhile, it models the true service cost of an inference request with both compute and memory aspects considered. Finally, SageSched employs an uncertainty-aware scheduling policy that can yield the best overall efficiency given the request cost distributions. Testbed experiments over diverse setups confirm that SageSched can attain an efficiency improvement of over 28.7%.

这篇论文介绍了一种名为 **SageSched** 的新型大语言模型（LLM）推理调度器，旨在解决现有调度方案在面对需求不确定性和资源需求混合性时的低效问题。以下是对该论文的结构化总结：

### 1. 核心问题与研究动机
*   **核心问题**：LLM 推理调度面临两大挑战：
    *   **需求不确定性（Uncertainty）**：由于自回归生成的特性，输出 Token 长度在推理完成前是未知的，导致难以准确预估任务时长。
    *   **需求混合性（Hybridity）**：LLM 推理既是计算密集型（矩阵运算）又是内存密集型（KVCache 占用），而现有调度器往往只关注其中之一。
*   **研究动机**：现有的调度器（如 vLLM 的 FCFS 或基于模型预测长度的 SSJF）存在缺陷：要么因“队头阻塞”导致延迟高，要么预测不准（仅预测单一数值而非分布），且忽略了内存开销对整体效率的影响。

### 2. 方法论
SageSched 提出了三个关键技术组件：
*   **语义感知且基于历史的预测器（Semantic-aware History-based Predictor）**：
    *   **核心思想**：利用 Prompt 的语义相似性来预测输出长度。
    *   **流程**：记录近期推理的 Prompt 嵌入（Embedding）和输出长度。对于新请求，通过向量检索（FAISS）寻找相似的历史请求，直接提取其输出长度的**概率分布**。这种方法无需针对每个模型进行微调，且比单一数值预测更准确。
*   **基于资源边界的成本建模（Resource-bound-based Cost Modeling）**：
    *   **核心思想**：统一建模计算和内存开销。
    *   **公式**：提出成本模型 $C = \frac{O^2}{2} + IO$（其中 $I$ 为输入长度，$O$ 为输出长度）。该模型通过分析计算受限和内存受限两种场景得出，反映了 KVCache 随时间增长对资源的累积占用。
*   **不确定性感知调度策略（Uncertainty-aware Scheduling）**：
    *   **核心思想**：采用 **Gittins Index（吉廷斯索引）** 策略。
    *   **流程**：在已知成本分布而非确切值的情况下，计算每个请求的 Gittins 索引（代表最小化平均延迟的最优优先级）。调度器会定期（按 Bucket 边界）刷新正在运行任务的索引，以动态调整优先级并支持抢占。

### 3. 实验设计
*   **数据集/场景**：使用了三个代表性数据集：SharedGPT（通用对话）、Alpaca-Summarization（摘要生成）、Document-Write（长文写作）。
*   **Benchmark（基准方法）**：
    *   **FCFS**：先来先服务（vLLM 默认）。
    *   **FastServe**：基于多级反馈队列（MLFQ）的调度。

*   **评价指标**：主要关注 **TTLT**（Time-to-Last-Token，端到端延迟），兼顾 **TTFT**（Time-to-First-Token，首字延迟）。

### 6. 主要结论与发现
*   **效率提升**：SageSched 在平均 TTLT 指标上比现有最优方案（如 TRAIL）提升了 **28.7%** 以上。
*   **综合表现**：在优化 TTLT 的同时，SageSched 也能有效缓解队头阻塞，保持较低的 TTFT。
*   **鲁棒性**：即使在预测分布存在噪声的情况下，基于 Gittins 索引的策略依然比基于均值的策略更稳健。
*   **低开销**：在大规模集群（64 节点）中，调度和预测带来的额外延迟仅约 100ms，相对于数秒的推理时间可忽略不计。

### 7. 优点
*   **分布感知**：意识到 LLM 输出的不确定性，使用分布而非单一值进行调度，理论依据（Gittins Index）扎实。
*   **轻量化**：基于历史记录和语义检索，无需昂贵的模型微调，具有很强的通用性。
*   **混合建模**：首次深入探讨了 KVCache 增长带来的内存开销与计算开销在调度中的统一表达。

### 8. 不足与局限
*   **冷启动问题**：在系统运行初期或面对全新领域的 Prompt 时，历史记录可能不足，虽然论文提到可以用公共数据集补充，但初期预测精度可能受限。
*   **相似度阈值依赖**：语义相似度阈值（如 0.8）需要根据经验设定，不同模型或任务的最优阈值可能存在差异。
*   **预填充阶段优化较少**：论文主要关注解码（Decoding）阶段的调度，对于长文本预填充（Prefill）带来的瞬时资源冲击讨论相对较少。
