## 摘要
后缀越狱攻击是针对大语言模型（LLMs）进行红队测试的一种系统化方法，但由于在识别出越狱后缀之前需要评估大量的候选后缀，因此面临着极高的计算成本。本文提出了前缀共享 KV 缓存（PSKV），这是一种专为越狱后缀生成而设计的即插即用推理优化技术。我们的方法源于一个关键观察：在进行后缀越狱时，虽然需要评估大量的候选提示词，但它们共享相同的目标有害指令作为前缀。因此，PSKV 不再对重复的前缀执行冗余推理，而是为该前缀维护一个单一的 KV 缓存，并将其与每个候选提示词共享，从而以极低的显存开销实现多种后缀的并行推理。这种设计允许采用更激进的批处理策略，而这些策略在其他情况下会受到显存限制。在五种广泛部署的 LLM 上针对六种常用后缀攻击进行的广泛实验表明，PSKV 在保持原始攻击成功率（ASR）的同时，将推理时间减少了 40%，峰值显存占用减少了 50%。代码已提交并将公开发布。

## Abstract
Suffix jailbreak attacks serve as a systematic method for red-teaming Large Language Models (LLMs) but suffer from prohibitive computational costs, as a large number of candidate suffixes need to be evaluated before identifying a jailbreak suffix. This paper presents Prefix-Shared KV Cache (PSKV), a plug-and-play inference optimization technique tailored for jailbreak suffix generation. Our method is motivated by a key observation that when performing suffix jailbreaking, while a large number of candidate prompts need to be evaluated, they share the same targeted harmful instruction as the prefix. Therefore, instead of performing redundant inference on the duplicated prefix, PSKV maintains a single KV cache for this prefix and shares it with every candidate prompt, enabling the parallel inference of diverse suffixes with minimal memory overhead. This design enables more aggressive batching strategies that would otherwise be limited by memory constraints. Extensive experiments on six widely used suffix attacks across five widely deployed LLMs demonstrate that PSKV reduces inference time by 40\% and peak memory usage by 50\%, while maintaining the original Attack Success Rate (ASR). The code has been submitted and will be released publicly.

*   **核心问题**：后缀越狱攻击（如 GCG, BEAST 等）通过迭代搜索最优后缀来诱导模型输出有害内容。这一过程需要对成百上千个候选后缀进行频繁的推理评估。
*   **研究动机**：
    *   **计算冗余**：在每一轮迭代中，所有候选提示词都共享相同的有害指令（前缀），但标准推理流程会为每个候选后缀重复计算或复制该前缀的 KV 缓存。
    *   **显存瓶颈**：标准 KV 缓存实现需要将长前缀的缓存复制 $N$ 份（$N$ 为批处理大小），导致显存占用随 Batch Size 线性增长，限制了搜索宽度。
    *   **现有工具局限**：高性能推理引擎（如 vLLM, SGLang）虽然支持前缀共享，但通常不支持反向传播（梯度计算），无法直接用于需要梯度的白盒攻击（如 GCG）。

### 2. 方法论：核心思想与技术细节
PSKV 是一种轻量级、即插即用的推理优化技术，其核心思想是**“一次计算，全量共享”**：
*   **虚拟缓存广播（Virtual Cache Broadcasting）**：在攻击开始前，仅计算一次有害指令前缀的 KV 缓存。在推理过程中，PSKV 维护该单份缓存，并在每一层注意力机制计算时，将其动态“广播”给所有候选后缀，避免了物理上的多份拷贝。
*   **逐层延迟扩展策略（Layer-wise Lazy Expansion）**：为了进一步降低峰值显存，PSKV 不在初始化时分配全量显存，而是在推理到特定层时才临时扩展该层的 KV 向量，计算完成后立即释放，确保显存占用不随模型深度累积。
*   **后缀中心对齐（Suffix-Centric Alignment）**：针对多指令批处理场景，通过对指令进行左填充（Left-padding）和对目标响应进行右填充（Right-padding），确保所有后缀在张量维度上对齐，从而实现高效的向量化并行计算。

### 6. 主要结论与发现
*   **显著提效**：相比无缓存基准，PSKV 平均减少了 **40% 的推理时间**。
*   **显存优化**：相比标准 KV 缓存，PSKV 平均降低了 **50% 的峰值显存占用**，且显存占用对搜索宽度的增加不敏感。
*   **无损攻击力**：PSKV 仅优化推理过程，不改变攻击逻辑，因此**完全保留了原始的攻击成功率（ASR）**。
*   **通用性**：PSKV 能够无缝集成到需要梯度的白盒攻击中，填补了高性能推理引擎在对抗性攻击领域的空白。

### 7. 优点（亮点）
*   **即插即用**：不需要修改模型架构，兼容 PyTorch 原生环境。
*   **支持梯度**：解决了 vLLM 等引擎无法进行反向传播的问题，对白盒攻击极度友好。
*   **工作负载感知**：专门针对“一前缀多后缀”的攻击模式进行了深度定制，比通用推理引擎更高效。

### 8. 不足与局限
*   **攻击类型限制**：该方法主要针对“后缀式”攻击，对于非前缀共享的攻击（如提示词注入或多轮对话攻击）优化效果有限。

*   **实验覆盖**：虽然涵盖了主流开源模型，但未涉及超大规模参数模型（如 70B 以上）或闭源模型的代理攻击场景。
