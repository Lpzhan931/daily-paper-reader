## 摘要
我们提出了一种针对语音感知 LLM 的自投机解码方法，通过将 CTC 编码器作为草稿模型来加速自回归 (AR) 推理并提高 ASR 准确率。我们的三步流程如下：(1) 如果 CTC 输出分布的帧熵低于阈值，则接受贪心 CTC 假设作为最终结果；(2) 否则，使用基于 token 似然度的宽松接受标准，在单次 LLM 前向传递中验证 CTC 假设；(3) 如果验证失败，则从已接受的 CTC 前缀恢复自回归解码。在九个语料库和五种语言上的实验表明，该方法可以同时加速解码并降低词错误率 (WER)。在 HuggingFace Open ASR 基准测试中，使用 10 亿参数的 LLM 和 4.4 亿参数的 CTC 编码器，我们实现了创纪录的 5.58% WER，并将逆实时因子提高了 4.4 倍，而相比自回归搜索，WER 仅相对增加了 12%。代码和模型权重已在许可协议下公开。

## Abstract
We propose self-speculative decoding for speech-aware LLMs by using the CTC encoder as a draft model to accelerate auto-regressive (AR) inference and improve ASR accuracy. Our three-step procedure works as follows: (1) if the frame entropies of the CTC output distributions are below a threshold, the greedy CTC hypothesis is accepted as final; (2) otherwise, the CTC hypothesis is verified in a single LLM forward pass using a relaxed acceptance criterion based on token likelihoods; (3) if verification fails, AR decoding resumes from the accepted CTC prefix. Experiments on nine corpora and five languages show that this approach can simultaneously accelerate decoding and reduce WER. On the HuggingFace Open ASR benchmark with a 1B parameter LLM and 440M parameter CTC encoder, we achieve a record 5.58% WER and improve the inverse real time factor by a factor of 4.4 with only a 12% relative WER increase over AR search. Code and model weights are publicly available under a permissive license.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：语音感知大语言模型（SLM）虽然在语音识别（ASR）准确率上表现卓越，但其**自回归（AR）推理机制**导致生成速度缓慢，每生成一个 token 都需要经过一次完整的 LLM 前向计算，严重限制了推理效率。
*   **研究背景**：现有的投机解码（Speculative Decoding）通常需要一个额外的、较小的草稿模型。本研究旨在利用 SLM 架构中已有的 **CTC 编码器**作为天然的“草稿模型”，在不增加额外模型负担的情况下，实现推理加速并尝试进一步提升识别精度。

### 2. 方法论：核心思想与关键技术
该论文提出了一种三步走的**自投机解码（SSD）**流程：
1.  **CTC 解码与初步验证**：
    *   利用编码器自带的 CTC 头生成贪心解码假设。
    *   计算 CTC 输出分布的**帧级别熵（Frame-level Entropy）**。如果所有帧的熵都低于设定阈值 $\tau_{CTC}$，说明模型极度自信，直接接受该结果，跳过 LLM。
2.  **LLM 并行验证**：
    *   若第一步未通过，将 CTC 假设输入 LLM。
    *   利用 LLM 的因果掩码特性，通过**单次前向传递**计算所有 token 的似然度。
    *   如果所有 token 的似然度均高于阈值 $\tau_{SLM}$，则接受该 CTC 假设。
3.  **自回归回退（AR Fallback）**：
    *   若验证仍失败，找到 CTC 假设中通过验证的最长前缀，从该位置开始执行标准的自回归解码。
*   **核心公式/逻辑**：通过熵（衡量 CTC 自信度）和似然度（衡量 LLM 对 CTC 结果的认可度）双重把关，平衡速度与精度。

### 6. 主要结论与发现
*   **加速显著**：在 High RTFx 模式下，推理速度提升了 **4.4 倍**，而 WER 仅相对增加 12%。

### 7. 优点
*   **无需额外模型**：巧妙复用 SLM 现有的 CTC 编码器，实现“零成本”草稿生成。

### 8. 不足与局限
*   **架构限制**：该方法要求 SLM 必须包含一个经过 CTC 训练的编码器，对于纯 AED 或 RNN-T 架构的 SLM 不直接适用。
*   **任务局限**：目前仅证明在 ASR 任务有效，对于口语翻译或语音问答等需要大幅改变输出结构的序列任务，CTC 草稿可能无法匹配。
*   **回退开销**：如果验证失败，需要从失败点重新进行 AR 解码，对于长音频或低质量音频，这种回退机制可能会削弱加速效果。
