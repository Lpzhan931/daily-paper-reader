## 摘要
自回归解码仍然是大语言模型（LLM）推理服务中的主要瓶颈，这促使了投机解码方法的发展，该方法通过每步验证多个候选 token 来减少昂贵的教师模型调用。树状结构的投机进一步增加了并行性，但在跨异构后端和加速器栈移植时往往表现脆弱，因为注意力掩码、KV 缓存布局和索引语义并不通用。我们提出了 EAGLE-Pangu，这是一个可复现的系统，它将 EAGLE-3 风格的树状投机解码移植到昇腾 NPU 上的 Pangu 教师后端。EAGLE-Pangu 的贡献包括：(i) 一个基于 Cache API 构建的显式分支/提交缓存管理器；(ii) 加速器安全的树状张量化，通过构造消除了未定义的负索引并验证了结构不变性；以及 (iii) 一个与算子融合兼容的教师验证路径，并带有可调试的 eager 回退机制。在来自 MT-Bench 和 HumanEval 风格提示词的 240 轮对话中，在算子融合性能路径下，EAGLE-Pangu 相比仅使用教师模型的贪婪解码，平均将端到端解码吞吐量提高了 1.27 倍，在 p99 分位数下最高可达 2.46 倍。我们还提供了一个不含算子融合的参考路径，具有结构化追踪和不变性检查，以支持跨执行模式和树预算的可复现调试与消融实验。

## Abstract
Autoregressive decoding remains a primary bottleneck in large language model (LLM) serving, motivating speculative decoding methods that reduce expensive teacher-model invocations by verifying multiple candidate tokens per step. Tree-structured speculation further increases parallelism, but is often brittle when ported across heterogeneous backends and accelerator stacks, where attention masking, KV-cache layouts, and indexing semantics are not interchangeable. We present EAGLE-Pangu, a reproducible system that ports EAGLE-3-style tree speculative decoding to a Pangu teacher backend on Ascend NPUs. EAGLE-Pangu contributes (i) an explicit branch/commit cache manager built on the Cache API, (ii) accelerator-safe tree tensorization that removes undefined negative indices by construction and validates structural invariants, and (iii) a fused-kernel-compatible teacher verification path with a debuggable eager fallback. On 240 turns from MT-Bench and HumanEval-style prompts, EAGLE-Pangu improves end-to-end decoding throughput by 1.27x on average, up to 2.46x at p99, over teacher-only greedy decoding in the fused-kernel performance path. We also provide a fused-kernel-free reference path with structured traces and invariant checks to support reproducible debugging and ablation across execution modes and tree budgets.

### 1. 核心问题与研究动机
*   **研究背景**：大语言模型（LLM）的自回归解码受限于单 token 串行生成的特性，导致推理吞吐量低、延迟高。投机解码通过“草稿模型（Draft Model）提议 + 教师模型（Teacher Model）验证”的模式来加速生成。
*   **核心问题**：树状投机解码（如 EAGLE-3）虽然能显著提升并行度，但在跨异构后端（如从 GPU 移植到昇腾 NPU）时非常脆弱。主要挑战包括：
    1.  **KV 缓存兼容性**：不同后端的 KV 缓存布局和索引语义不统一。
    2.  **算子融合限制**：高性能融合注意力算子对掩码（Mask）形状和对齐有严格要求。
    3.  **索引安全性**：加速器运行时通常不支持负索引（如常用的 -1 填充），容易导致运行时错误或静默计算错误。

### 2. 方法论：核心思想与关键技术
EAGLE-Pangu 并没有提出新的解码算法，而是针对昇腾后端设计了一套**加速器安全（Accelerator-Safe）**的系统实现：
*   **可分支的 KV 缓存抽象**：
    *   引入显式的缓存管理器，区分“已提交前缀状态”和“各分支投机状态”。
    *   通过 `Replicate` 操作实现分支隔离，支持基于长度或路径索引的快速提交（Commit）机制，确保缓存更新与串行解码语义等价。
*   **加速器安全的树张量化（Tree Tensorization）**：
    *   **消除负索引**：引入“哑根节点（Dummy Root）”机制，将索引 0 作为根节点，所有父节点索引偏移 1，从而消除 -1 等非法索引。
    *   **祖先表构建**：预先计算 $O(M \times D_{max})$ 的祖先索引表，确保设备端 Gather 操作始终在合法范围内。
*   **融合算子兼容的树掩码验证**：
    *   将树状结构转化为 4D 注意力掩码（[B, 1, M, M]），确保不同分支间无信息泄露。
    *   支持在昇腾融合算子路径下执行，同时保留一个 Eager 模式的回退路径用于调试。
*   **双模式执行协议**：提供“参考模式（关闭融合算子，用于验证正确性）”和“性能模式（开启融合算子，用于生产加速）”。
