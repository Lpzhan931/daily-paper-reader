## 摘要
语言模型难以泛化到预训练上下文长度之外，这限制了长程推理和检索能力。在长上下文数据上进行持续预训练虽有帮助，但由于注意力机制（Attention）的二次方缩放特性，其成本非常高昂。我们观察到，大多数标记（token）并不需要对整个序列进行（全局）注意力计算，而可以依赖局部上下文。基于此，我们提出了 L2A（Learning To Attend），这是一种通过决定何时调用全局注意力来实现条件化（逐标记）长程记忆访问的层。我们在 Qwen 2.5 和 Qwen 3 模型上评估了 L2A，将其有效上下文长度从 32K 扩展到了 128K 标记。L2A 在跳过约 80% 标记的全局注意力计算的同时，性能与标准长上下文训练相比差距在 3% 以内，优于之前的基准方法。我们还设计了定制的 Triton 内核，以在 GPU 上高效实现这种逐标记的条件注意力，与 FlashAttention 相比，训练吞吐量和首字生成时间（time-to-first-token）提升了约 2 倍。此外，L2A 支持对高度稀疏的全局注意力层进行训练后剪枝，在性能损失微乎其微的情况下，将 KV 缓存内存减少了高达 50%。

## Abstract
Language models struggle to generalize beyond pretraining context lengths, limiting long-horizon reasoning and retrieval. Continued pretraining on long-context data can help but is expensive due to the quadratic scaling of Attention. We observe that most tokens do not require (Global) Attention over the entire sequence and can rely on local context. Based on this, we propose L2A (Learning To Attend), a layer that enables conditional (token-wise) long-range memory access by deciding when to invoke global attention. We evaluate L2A on Qwen 2.5 and Qwen 3 models, extending their effective context length from 32K to 128K tokens. L2A matches the performance of standard long-context training to within 3% while skipping Global Attention for $\sim$80% of tokens, outperforming prior baselines. We also design custom Triton kernels to efficiently implement this token-wise conditional Attention on GPUs, achieving up to $\sim$2x improvements in training throughput and time-to-first-token over FlashAttention. Moreover, L2A enables post-training pruning of highly sparse Global Attention layers, reducing KV cache memory by up to 50% with negligible performance loss.

### 论文总结：L2A (Learning When to Attend) —— 长上下文 LLM 的条件记忆访问

#### 1. 核心问题与整体含义
*   **研究背景**：大语言模型（LLM）在处理长文本时面临巨大的计算挑战。标准注意力机制（Attention）的计算复杂度随序列长度呈二次方增长。
*   **核心观察**：并非所有标记（token）都需要全局上下文。例如，处理“Section 2”可能需要回顾数千个 token 之前的内容，但处理普通连接词则仅需局部信息。
*   **核心问题**：能否让模型在 token 级别自主决定“何时”需要访问长程记忆，从而跳过不必要的全局计算？

#### 2. 方法论：L2A (Learning To Attend)
*   **核心思想**：将注意力层改造为“局部注意力 + 路由器 + 条件全局注意力”的组合。
*   **关键技术细节**：
    1.  **局部注意力**：所有 token 首先通过滑动窗口注意力（SWA，窗口通常为 4K）获取局部上下文表示 $s_t$。
    2.  **路由器 (Router)**：利用 $s_t$ 计算一个概率得分，通过 Sigmoid 函数和阈值（0.5）生成离散决策 $d_t \in \{0, 1\}$。
    3.  **条件全局注意力**：仅当 $d_t=1$ 时，该 token 才会执行昂贵的全局注意力计算；否则直接跳过，仅保留局部表示。
    4.  **训练目标**：采用“下一标记预测（NTP）损失”加上“稀疏正则化项”，鼓励模型在保证性能的前提下尽量减少全局注意力的调用。
    5.  **优化策略**：使用直通估计器（STE）解决离散决策不可导的问题；引入 Bernoulli 采样（训练时以 10% 概率强制开启全局访问）以防止路由器崩溃（即所有 token 都趋向于只看局部）。
    6.  **定制内核**：开发了基于 Triton 的稀疏注意力内核，通过索引映射（Index Mapping）在 GPU 上实现真正的计算加速。

#### 3. 实验设计
*   **基座模型**：Qwen 2.5 (1.5B, 7B) 和 Qwen 3 (8B)，预训练上下文为 32K。
*   **评估场景**：将上下文扩展至 128K，涵盖以下 Benchmark：
    *   **HELMET**：合成召回（大海捞针）、RAG（检索增强生成）、ICL（长文本原位学习）、引用生成。
    *   **BabiLong**：事实链与关系推理。
    *   **MRCR**：多轮指代消解。
    *   **短文本测试**：BoolQ, MMLU 等，确保长文本训练不损害基础能力。
*   **对比方法**：
    *   **CLP**：标准持续长文本预训练（全量注意力，性能上限）。
    *   **训练稀疏法**：NSA (Native Sparse Attention), $S^2$-Attn。
    *   **免训练法**：SelfExtend, DCA (Dual Chunk Attention)。
    *   **推理加速法**：FlexPrefill。

#### 4. 资源与算力
*   **硬件**：使用 **NVIDIA H200 GPU**。
*   **训练规模**：
    *   Qwen 2.5 7B / Qwen 3 8B：Batch Size 为 600 万 token，总训练量为 **250 亿 (25B) token**。
    *   Qwen 2.5 1.5B：Batch Size 为 400 万 token，总训练量为 **167 亿 (16.7B) token**。
*   **精度**：采用 BF16 混合精度训练。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1.5B 到 8B 不同参数规模的模型，以及从 8K 到 128K 的多种上下文长度。
*   **消融实验**：非常充分。包括了对正则化系数 $\lambda$、SWA 窗口大小（0 到 4K）、路由器决策机制（对比随机路由）的详细分析。
*   **客观性**：实验不仅关注准确率，还深入分析了不同任务下的稀疏度模式（如 ICL 任务最不稀疏，而召回任务极度稀疏），并提供了实际的训练/推理吞吐量数据，具有较强的说服力。

#### 6. 主要结论与发现
*   **性能接近无损**：L2A 在 128K 长度下的表现与全量注意力的 CLP 相比，差距在 **1.5% - 3%** 以内，显著优于其他稀疏注意力基准。
*   **高效稀疏**：模型平均跳过了约 **80%** 的全局注意力计算。
*   **速度翻倍**：相比 FlashAttention-2，L2A 实现了约 **2 倍** 的训练吞吐量提升和首字生成时间（TTFT）加速。
*   **KV 缓存压缩**：通过对稀疏度极高（>95%）的层进行训练后剪枝，可在几乎不损失性能的情况下减少 **50%** 的 KV Cache 内存占用。

#### 7. 优点
*   **Token 级自适应**：不同于固定模式的稀疏（如固定步长或块），L2A 能根据输入内容的复杂程度动态分配计算资源。
*   **软硬件协同优化**：通过定制 Triton 内核将算法稀疏性转化为实际的硬件加速，解决了“稀疏算子在 GPU 上变慢”的常见痛点。
*   **推理灵活性**：测试时可以通过调整 Sigmoid 阈值，在推理速度和模型精度之间进行动态权衡（Test-time compute scaling）。

#### 8. 不足与局限
*   **KV Cache 存储瓶颈**：虽然计算量减少了，但为了保证后续 token 可能的全局访问，默认仍需存储完整的 KV Cache（除非应用文中提到的层剪枝策略）。
*   **训练复杂性**：引入了额外的路由器模块和正则化超参数，训练稳定性依赖于 Bernoulli 采样等技巧。
*   **架构依赖**：目前主要针对 Transformer 架构，对于 SSM（状态空间模型）等非注意力架构的适用性尚未探讨。
