## 摘要
由于注意力稀释和分布外退化，长上下文推理对于大语言模型而言仍具挑战性。上下文选择通过关注键值缓存（KV cache）条目的子集来缓解这一限制，但尽管 token 级别的上下文需求高度不均匀，大多数方法在整个解码过程中仍分配固定的上下文预算。为了解决这一问题，我们提出了不确定性触发的自适应上下文分配（UT-ACA），这是一个根据逐 token 不确定性动态调整上下文窗口的推理时框架。UT-ACA 学习了一个不确定性检测器，该检测器结合了语义嵌入与基于 logit 的置信度，同时考虑了跨解码步骤的不确定性累积。当显示证据不足时，UT-ACA 会选择性地回退，扩大上下文窗口，并在额外的支持下重新生成 token。实验表明，在长上下文设置下，UT-ACA 在保持生成质量的同时，大幅减少了平均上下文使用量。

## Abstract
Long-context inference remains challenging for large language models due to attention dilution and out-of-distribution degradation. Context selection mitigates this limitation by attending to a subset of key-value cache entries, yet most methods allocate a fixed context budget throughout decoding despite highly non-uniform token-level contextual demands. To address this issue, we propose Uncertainty-Triggered Adaptive Context Allocation (UT-ACA), an inference-time framework that dynamically adjusts the context window based on token-wise uncertainty. UT-ACA learns an uncertainty detector that combines semantic embeddings with logit-based confidence while accounting for uncertainty accumulation across decoding steps. When insufficient evidence is indicated, UT-ACA selectively rolls back, expands the context window, and regenerates the token with additional support. Experiments show that UT-ACA substantially reduces average context usage while preserving generation quality in long-context settings.

#### 1. 核心问题与整体含义
大语言模型（LLM）在处理长文本时面临两大挑战：**注意力稀释**（无关信息干扰关键证据检索）和**分布外（OOD）退化**（长文本模式偏离预训练分布）。现有的上下文管理方法（如 KV Cache 剪枝或选择）通常在解码过程中分配**固定**的上下文预算。然而，不同 Token 的生成难度并不一致：有些 Token 仅依赖局部信息，而有些则需要全局证据。
**研究动机：** 论文旨在打破固定预算的限制，提出一种根据生成难度动态调整上下文窗口的机制，以在提升推理效率的同时保证生成质量。

#### 2. 方法论
UT-ACA 提出了一种“先尝试、再检测、必要时回退重来”的推理框架：
*   **核心思想：** 初始使用极小的上下文窗口进行尝试性生成，通过一个轻量级检测器监控 Token 的不确定性。若检测到高风险（如幻觉或证据不足），则执行回退（Rollback），扩大上下文窗口并重新生成该 Token。
*   **关键技术细节：**
    *   **不确定性检测器（Uncertainty Detector）：** 结合了两种信号：
        1.  **Logit Margin：** 最高概率与次高概率之间的差值（置信度信号）。
        2.  **语义嵌入（Semantic Embedding）：** 从 LLM 最后一层提取的隐藏状态（语义复杂度信号）。
    *   **时序建模：** 使用 **LSTM** 捕捉解码过程中不确定性的累积效应。
    *   **三分类指标（GDM）：** 将生成状态分为“正确（Correct）”、“未知（Unknown）”和“幻觉（Hallucination）”。当后两者概率之和超过前者时，触发上下文扩展。
    *   **自适应分配逻辑：** 采用基于块（Block）的检索机制。若触发扩展，则增加检索的 KV 块数量；若生成稳定，则逐步缩小窗口以节省资源。

*   **数据集：**
    *   **合成传记数据集：** 专门构建的 17.1万至40万 Token 的长文本，用于训练检测器和评估生成质量。

*   **实验规模：** 涵盖了从合成数据到多个权威公开榜单的广泛测试。
*   **消融实验：** 验证了 Logit Margin、语义嵌入和 LSTM 模块的必要性，并对比了“二分类”与“三分类”指标的影响。

#### 6. 主要结论与发现
*   **效率与质量的双赢：** UT-ACA 在显著减少平均上下文 Token 使用量（例如在某些任务中减少 25%-50%）的同时，保持甚至超过了固定大预算方法的生成准确率。
*   **不确定性作为信号：** 语义特征和时序累积对于准确判断“何时需要更多上下文”至关重要，单纯依靠 Logit 概率并不准确。
*   **回退机制的可行性：** 虽然回退增加了单 Token 延迟，但由于它仅在必要时触发，整体计算开销在长文本场景下依然具有竞争力。

#### 7. 优点
*   **动态性：** 首次实现了 Token 级别的上下文预算动态分配，符合自然语言生成的非均匀特性。
*   **即插即用：** 无需对原始 LLM 进行微调，仅需训练一个极小的外挂检测器。
*   **评估创新：** 提出的 CAS 指标比传统的字符串匹配更适合评估长文本生成的语义正确性。

#### 8. 不足与局限
*   **推理延迟波动：** 回退机制会导致推理速度的不稳定，在不确定性频繁触发的场景下，每 Token 平均延迟会增加。
*   **检测器依赖性：** 检测器需要针对不同的模型家族（如 Llama 和 Qwen）分别训练，虽然训练开销小，但增加了部署流程。
