## 摘要
分布式前缀缓存已成为高效大语言模型（LLM）服务的核心技术。然而，对于具有高缓存命中率的长上下文请求，从远程服务器检索可重用的 KVCache 块已成为新的性能瓶颈。随着智能体 AI（agentic AI）工作负载的持续增长，这种网络密集型 LLM 推理预计将变得越来越普遍。然而，现有的 LLM 推理引擎在很大程度上仍以计算为中心：它们将 KVCache 加载视为 GPU 执行的附属阶段，且在调度过程中往往未能明确考虑其延迟。我们提出了 CALVO，这是一种将 KVCache 加载视为首要关注点的 LLM 服务引擎。CALVO 将 KVCache 加载和 GPU 计算解耦为独立管理、异步进行的阶段，从而能够更好地利用网络、PCIe 和计算资源。此外，CALVO 将 KVCache 加载延迟作为单次请求服务成本的明确组成部分，从而实现了更准确的调度决策。在具有多种长上下文工作负载的真实测试平台上的实验表明，CALVO 显著提高了网络密集型 LLM 推理的效率，与基准方案相比，其服务水平目标（SLO）达成率提高了多达 61.67%。

## Abstract
Distributed prefix caching has become a core technique for efficient LLM serving. However, for long-context requests with high cache hit ratios, retrieving reusable KVCache blocks from remote servers has emerged as a new performance bottleneck. Such network-intensive LLM inference is expected to become increasingly common as agentic AI workloads continue to grow. However, existing LLM inference engines remain largely compute-centric: they treat KVCache loading as a subordinate phase to GPU execution and often fail to account for its delay explicitly during scheduling.   We present CALVO, an LLM serving engine that treats KVCache loading as a first-class concern. CALVO decouples KVCache loading and GPU computation into independently managed, asynchronously progressing stages, enabling better utilization of network, PCIe, and computation resources. In addition, CALVO incorporates KVCache loading delay as an explicit component of per-request service cost, leading to more accurate scheduling decisions. Experiments on a real testbed with diverse long-context workloads show that CALVO substantially improves the efficiency of network-intensive LLM inference, achieving up to 61.67% higher SLO attainment than the baseline.

*   **背景**：随着长上下文（Long-context）LLM应用的普及，分布式前缀缓存（Distributed Prefix Caching）成为减少重复计算、提升首字响应时间（TTFT）的关键技术。
*   **核心问题**：在缓存命中率高、上下文极长的场景下（如文档QA、代码补全），从远程服务器检索KVCache的网络传输延迟已取代GPU计算，成为新的性能瓶颈。作者称此类负载为**网络密集型LLM推理（Network-Intensive LLM Inference）**。
*   **现有缺陷**：主流引擎（如vLLM）采用“以计算为中心”的设计，将KVCache加载视为计算的附属步骤。这导致：
    1.  **资源利用率低**：各阶段（网络传输、PCIe传输、计算）串行执行，无法充分流水化。
    2.  **调度不准确**：调度器（如FIFO或仅考虑计算量的SJF）忽视了巨大的网络加载成本，导致调度决策次优。

### 2. 方法论
CALVO的核心思想是将**KVCache加载提升为与计算同等地位的一等公民**，具体包含两大关键技术：
*   **跨阶段工作流优化（解耦与流水线）**：
    *   将KVCache处理分为L3→L2（远程到本地内存）、L2→L1（内存到显存）和GPU计算三个独立阶段。
    *   每个阶段由独立的**调度器-执行器（Dispatcher-Executor）**对管理，实现异步并行。
    *   **主动空间分配**：低层调度器在启动传输时，会主动触发高层存储空间的预分配，确保数据到达后能立即开始下一阶段传输，实现细粒度的流水线重叠。
*   **跨请求调度优化（成本感知调度）**：
    *   **二元线性成本模型**：通过系统分析，将推理成本建模为“加载延迟（与上下文长度线性相关）+ 计算延迟（与Query长度线性相关）”的函数。
    *   **调度算法**：
        *   针对平均TTFT优化：采用基于总成本的**最短作业优先（SJF）**。
        *   针对SLO达成率优化：采用**最小松弛时间优先（LSTF）**，即优先处理截止日期与预估总成本之差最小的请求。
