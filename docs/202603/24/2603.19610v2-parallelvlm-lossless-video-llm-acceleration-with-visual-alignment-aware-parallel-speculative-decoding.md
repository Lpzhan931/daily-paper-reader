## 摘要
尽管目前的视频大语言模型（Video-LLMs）在视频理解任务中取得了显著的性能，但其自回归解码效率仍受限于海量的视频 token 数量。视觉 token 剪枝可以部分缓解这一瓶颈，但现有方法仍面临信息丢失的问题，且解码加速效果有限。在本文中，我们提出了 ParallelVLM，这是一种无需训练的“先草拟后验证”投机解码框架，旨在解决长视频场景下草拟模型与目标模型之间相互等待以及加速比受限的问题。ParallelVLM 包含两个并行化阶段，可最大限度地提高硬件利用率，并引入了一种无偏验证器引导剪枝（Unbiased Verifier-Guided Pruning）策略，通过消除注意力引导剪枝中的位置偏差，使草拟模型与目标模型更好地对齐。大量实验表明，ParallelVLM 在保持高接受长度的同时，有效地将草拟窗口扩大了 1.6 至 1.8 倍；与原生自回归解码相比，它在 LLaVA-Onevision-72B 上将各种视频理解基准测试加速了 3.36 倍，在 Qwen2.5-VL-32B 上加速了 2.42 倍。

## Abstract
Although current Video-LLMs achieve impressive performance in video understanding tasks, their autoregressive decoding efficiency remains constrained by the massive number of video tokens. Visual token pruning can partially ease this bottleneck, yet existing approaches still suffer from information loss and yield only modest acceleration in decoding. In this paper, we propose ParallelVLM, a training-free draft-then-verify speculative decoding framework that overcomes both mutual waiting and limited speedup-ratio problems between draft and target models in long-video settings. ParallelVLM features two parallelized stages that maximize hardware utilization and incorporate an Unbiased Verifier-Guided Pruning strategy to better align the draft and target models by eliminating the positional bias in attention-guided pruning. Extensive experiments demonstrate that ParallelVLM effectively expands the draft window by $1.6\sim1.8\times$ with high accepted lengths, and accelerates various video understanding benchmarks by 3.36$\times$ on LLaVA-Onevision-72B and 2.42$\times$ on Qwen2.5-VL-32B compared with vanilla autoregressive decoding.

### ParallelVLM：通过视觉对齐感知的并行投机解码实现无损视频大语言模型加速

#### 1. 核心问题与整体含义（研究动机和背景）
随着视频大语言模型（Video-LLMs）的发展，处理长视频时产生的海量视觉 Token（可达数万甚至百万级）导致了严重的推理延迟。现有的加速方案存在两大瓶颈：
*   **顺序执行瓶颈：** 传统的投机解码（Speculative Decoding, SD）要求草稿模型（Draft Model）和目标模型（Target Model）交替运行，导致硬件在对方执行时处于闲置状态。
*   **对齐与速度的矛盾：** 为了提高速度比，通常需要对草稿模型的视觉 Token 进行剪枝，但现有的注意力引导剪枝存在“位置偏差”（倾向于保留开头、结尾或靠近查询的 Token），导致草稿模型与目标模型输出不一致，降低了接受率。

**ParallelVLM** 的研究动机在于通过并行化架构和无偏剪枝策略，在保证输出无损的前提下，最大化视频 LLM 的推理效率。

#### 2. 论文提出的方法论
ParallelVLM 采用了一种无需训练的协同设计方案，核心包含以下三个关键技术：

*   **并行预填充（Parallel Prefilling, PP）：**
    在预填充阶段，目标模型处理全量视频 Token，而草稿模型同步处理剪枝后的 Token。目标模型在计算早期层时，将语义信号传递给草稿模型以指导剪枝，从而将草稿模型的预填充耗时完全隐藏在目标模型的延迟之下。
*   **并行解码（Parallel Decoding, PD）：**
    采用流水线并行机制：当目标模型在验证第 $i-1$ 组草稿 Token 时，草稿模型同步生成第 $i$ 组候选 Token。这种重叠执行消除了相互等待的时间。
*   **无偏验证器引导剪枝（UV-Prune）：**
    *   **核心思想：** 不再依赖容易产生偏差的注意力权重，而是观察视觉 Token 在目标模型不同层之间与文本查询（Query）的**相似度变化**。
    *   **算法流程：** 计算视觉 Token $V_i$ 与文本 Token $X_j$ 在目标模型浅层（如前 20 层）的余弦相似度。计算相邻层间的相似度增量 $\Delta S_i$，增量越大表示该 Token 随层数深入与任务的相关性越强。根据此增量选择 Top-K 个 Token 供草稿模型使用。

*   **即插即用：** 无需任何微调或训练，降低了部署成本。

*   **多卡需求：** 为了实现真正的并行执行，通常需要多 GPU 环境来分别承载草稿和目标模型的计算任务。
