## 摘要
多智能体应用通常将复杂任务执行为多阶段工作流，其中每个阶段都是一次大语言模型（LLM）调用，其输出会成为后续步骤上下文的一部分。现有的 LLM 推理服务系统在很大程度上假设集群是同构的，即使用相同的模型副本。这种设计忽视了异构部署的潜力，而不同规模和能力的模型可以在延迟和性能之间实现更精细的权衡。然而，异构性在跨具有不同吞吐量和性能的模型进行调度时引入了新的挑战。我们提出了 Chimera，这是一个用于异构 LLM 集群上多智能体工作流推理服务的预测性调度系统，可同时优化端到端延迟和任务性能。Chimera 应用语义路由来估计每个请求针对各模型的置信度分数，预测工作流的总剩余输出长度，并利用处理中的预测 token 量来估计各模型的拥塞情况，从而实现负载均衡。我们在代码生成和数学推理的代表性智能体工作流上，使用多种异构 LLM 配置对 Chimera 进行了评估。在可比设置下，Chimera 描绘了最佳的延迟-性能前沿，与包括 vLLM 在内的竞争基准相比，端到端延迟降低了 1.2-2.4 倍，任务性能平均提高了 8.0-9.5 个百分点。

## Abstract
Multi-agent applications often execute complex tasks as multi-stage workflows, where each stage is an LLM call whose output becomes part of context for subsequent steps. Existing LLM serving systems largely assume homogeneous clusters with identical model replicas. This design overlooks the potential of heterogeneous deployments, where models of different sizes and capabilities enable finer trade-offs between latency and performance. However, heterogeneity introduces new challenges in scheduling across models with diverse throughput and performance. We present Chimera, a predictive scheduling system for multi-agent workflow serving on heterogeneous LLM clusters that jointly improves end-to-end latency and task performance. Chimera applies semantic routing to estimate per-model confidence scores for each request, predicts the total remaining output length of the workflow, and estimates per-model congestion using in-flight predicted token volumes for load balancing. We evaluate Chimera on representative agentic workflows for code generation and math reasoning using multiple heterogeneous LLM configurations. Across comparable settings, Chimera traces the best latency-performance frontier, reducing end-to-end latency by 1.2--2.4$\times$ and improving task performance by 8.0-9.5 percentage points on average over competitive baselines including vLLM.

### 1. 核心问题与研究动机
*   **研究背景**：大语言模型（LLM）正从独立对话向多智能体系统（Multi-agent Systems）演进，任务通常表现为多阶段工作流（Workflow），前一阶段的输出是后一阶段的上下文。
*   **核心问题**：现有的 LLM 推理服务系统（如 vLLM）大多假设集群是同质的（部署相同模型）。然而，实际部署中往往存在异构模型（不同参数规模、不同能力），这在延迟（Latency）和性能（Performance）之间提供了权衡空间。
*   **挑战**：在异构集群中，如何根据任务难度、模型能力和实时负载，动态地为多阶段工作流选择最优模型并进行请求调度，以同时优化端到端延迟和任务准确率。

### 2. 方法论：Chimera 系统架构
Chimera 是一个位于应用层和推理引擎之间的中间件，包含四个核心模块：
*   **语义路由器 (Semantic Router)**：基于 ModernBERT-large 的变体，为每个请求预测不同候选模型的“置信度分数”（即解决该任务的概率）。
*   **长度预测器 (Length Predictor)**：使用轻量级的分位数随机森林（QRF）算法，预测整个工作流剩余阶段的总输出 Token 长度。
*   **活动监控器 (Activity Monitor)**：实时跟踪每个推理引擎中正在处理的预测 Token 总量，用于估算负载引起的延迟。
*   **负载均衡器 (Load Balancer)**：
    *   **模型选择策略**：在满足“延迟松弛度（Latency Slack）”限制的候选模型中，选择置信度最高的模型。如果模型负载过高导致延迟超过最快模型的特定比例（$\tau$），则回退到较小模型。
    *   **调度优先级**：采用**最短总任务优先（STJF）**策略，即优先处理预测总长度较短的工作流。
    *   **抗饥饿机制**：引入基于老化（Aging）的提升机制，防止长任务被无限期阻塞。

### 3. 实验设计

*   **对比基准 (Baselines)**：

    *   **MLFQ**：多级反馈队列调度。
    *   **LTR (Learning to Rank)**：基于排序预测的调度方法。

*   **实验规模**：涵盖了 2 种核心任务场景、4 种不同的异构模型组合、3 种不同的请求到达率（RPS=8, 12, 16）。
*   **消融实验**：
    *   验证了语义路由器的有效性（对比有无路由器的 Pareto 前沿）。

### 6. 主要结论与发现
*   **性能提升**：相比 vLLM，Chimera 将端到端延迟降低了 **1.2–2.4 倍**，同时将任务准确率平均提升了 **8.0–9.5 个百分点**。
*   **调度优势**：STJF（最短总任务优先）比传统的 SJF（最短单次任务优先）更适合多智能体工作流，能显著减少排队时间。
*   **权衡能力**：通过调整“延迟松弛度”参数，系统管理员可以灵活地在“极速响应”和“高质量输出”之间切换。

### 7. 优点与亮点
*   **联合优化**：首次在异构 LLM 推理服务中实现了路由（选模型）与调度（排优先级）的深度耦合。
*   **工作流感知**：不同于以往只关注单次请求的系统，Chimera 能够感知多阶段任务的总体工作量。
*   **轻量化设计**：利用 CPU 端的随机森林和异步批处理路由，确保了复杂的调度逻辑不会成为推理瓶颈。

### 8. 不足与局限
*   **冷启动与分布偏移**：语义路由器和长度预测器依赖于特定任务的离线训练数据，对于全新的、未见过的任务分布，预测准确率可能会下降。

*   **模型规模限制**：实验主要集中在 1.5B 到 14B 的模型，对于超大规模模型（如 70B+）在极度资源受限情况下的表现尚待验证。
