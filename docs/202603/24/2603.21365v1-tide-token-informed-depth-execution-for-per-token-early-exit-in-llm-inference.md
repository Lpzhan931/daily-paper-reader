## 摘要
大语言模型无论任务难度如何，都会让每个 token 经过每一层。我们提出了 TIDE，这是一个训练后系统，它在周期性的检查点层附加微小的学习型路由，并在推理时为每个 token 选择其隐藏状态已收敛的最早层。TIDE 无需模型重训练，适用于任何 HuggingFace 因果语言模型（causal LM），可自动检测 GPU 架构，并通过融合 CUDA 算子支持 float32、float16 和 bfloat16。在配备 NVIDIA A100 的 DeepSeek R1 Distill 8B 上，TIDE 实现了 100% 的预填充（prefill）退出率（5% 的 token 在第 11 层退出，其余在第 31 层退出），预填充延迟降低了 7.2%，单批次吞吐量提升了 6.6%。在自回归解码过程中，98-99% 的 token 实现了早期退出，同时模型能够正确解决包含 95 个唯一输出 token 的多步数学问题。在 Qwen3 8B（36 层）上，当批大小为 8 时，吞吐量提升了 8.1%。在 2,000 个 WikiText 样本上的校准耗时不到 3 分钟，并生成约 4 MB 的路由检查点。该系统包含 1,308 行 Python 代码和 1,081 行 CUDA/C++ 代码，并通过了 74 项测试。代码：https://github.com/RightNow-AI/TIDE

## Abstract
Large language models run every token through every layer, regardless of difficulty. We present TIDE, a post-training system that attaches tiny learned routers at periodic checkpoint layers and, at inference time, selects the earliest layer whose hidden state has converged for each token. TIDE requires no model retraining, works with any HuggingFace causal LM, auto-detects GPU architecture, and supports float32, float16, and bfloat16 through fused CUDA kernels. On an NVIDIA A100 with DeepSeek R1 Distill 8B, TIDE achieves 100% prefill exit rate (5% of tokens exit at layer 11, the remaining at layer 31), reduces prefill latency by 7.2%, and increases single-batch throughput by 6.6%. During autoregressive decoding, 98-99% of tokens exit early while the model correctly solves a multi-step math problem with 95 unique output tokens. On Qwen3 8B (36 layers), throughput improves by 8.1% at batch size 8. Calibration on 2,000 WikiText samples takes under 3 minutes and produces a ~4 MB router checkpoint. The system comprises 1,308 lines of Python and 1,081 lines of CUDA/C++ with 74 passing tests. Code: https://github.com/RightNow-AI/TIDE

*   **核心问题**：当前的 Transformer 模型对所有 token（无论是简单的虚词如“the”还是复杂的数学推理步骤）都分配相同的计算量（经过所有层），这在计算上是极大的浪费。
*   **研究动机**：
    *   **计算冗余**：研究表明许多 token 的中间隐藏状态在到达最后一层前就已经与最终状态高度相似。
    *   **现有方法缺陷**：现有的早期退出（Early Exit）方法要么需要昂贵的预训练/微调（如 LayerSkip），要么仅适用于 Encoder 模型，或者在处理自回归生成的 KV 缓存（KV Cache）时存在兼容性问题。
    *   **目标**：开发一种无需重训练、支持通用模型、且能保持 KV 缓存完整性的轻量级后训练（Post-training）加速系统。

### 2. 方法论
*   **核心思想**：在预训练模型的特定层（检查点层）附加微小的学习型二分类器（路由器），预测当前 token 的隐藏状态是否已“收敛”到足以代表最终输出。
*   **关键技术细节**：
    *   **离线校准（Calibration）**：在少量数据（如 2,000 条 WikiText）上运行模型，计算中间层与最后一层隐藏状态的余弦相似度。若相似度 > 0.98，则标记为可退出。
    *   **微型路由器**：每个路由器是一个两层 MLP（瓶颈维度 128），参数量极小（约 0.5M），使用二元交叉熵损失训练。
    *   **通用适配器（Universal Adapter）**：通过自动探测 17 条属性路径，支持 LLaMA、Qwen、Phi 等几乎所有 HuggingFace 架构。
    *   **后验退出策略（Post-hoc Exit）**：在自回归解码时，模型依然运行完整的前向传递以保持 KV 缓存的连续性，但路由器会选择最早收敛的层来计算 Logits，从而绕过后续层的 Norm 和 Head 计算。
    *   **融合 CUDA 算子**：开发了融合 RMSNorm + 路由评估的 CUDA 内核，减少内核启动开销，支持 fp16/bf16。
