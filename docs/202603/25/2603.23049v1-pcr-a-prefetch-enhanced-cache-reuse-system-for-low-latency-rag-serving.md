## 摘要
检索增强生成（RAG）系统通过引入补充的检索文档来增强大语言模型（LLM）的性能，从而实现更准确且具备上下文感知能力的响应。然而，集成这些外部文档通常会导致极长的输入序列，这显著增加了预填充（prefill）阶段的计算开销，在该阶段会为所有输入 token 生成键值（KV）表示。这种延迟瓶颈在高吞吐量服务场景下变得尤为突出。KV 缓存复用通过存储共享输入前缀先前计算出的 KV 状态，从而避免了包含重叠上下文的请求之间的冗余计算，提供了一个极具前景的解决方案。然而，缓存复用的有效性通常受到三个实际挑战的限制：由于简单的淘汰策略导致的低缓存命中率、高昂的 CPU-GPU 数据传输开销，以及当缓存溢出到存储设备时缓慢的 SSD I/O。为了解决这些问题，我们提出了 PCR，这是一个旨在通过智能预取和流水线数据移动来最大化 KV 缓存复用效率的系统。具体而言，PCR 引入了三项关键技术：（1）一种带有前瞻性 LRU 替换策略的前缀树缓存结构，该策略利用调度器队列中的待处理请求来提高缓存命中率；（2）层级重叠技术，通过 CUDA 流将 KV 缓存加载与 GPU 计算进行流水线化，以隐藏通信延迟；（3）基于队列的预取，在需要之前主动将相关的 KV 缓存从 SSD 加载到 DRAM 中。大量实验表明，PCR 优于现有的 KV 缓存复用方法，在平均首字延迟（TTFT）方面实现了高达 2.47 倍的加速。

## Abstract
Retrieval-Augmented Generation (RAG) systems enhance the performance of large language models (LLMs) by incorporating supplementary retrieved documents, enabling more accurate and context-aware responses. However, integrating these external documents often results in very long input sequences, which significantly increases computation costs during the prefill stage, where key-value (KV) representations for all input tokens are generated. This latency bottleneck becomes especially pronounced under high-throughput serving scenarios. KV-cache reuse offers a promising solution by storing previously computed KV states for shared input prefixes, thereby avoiding redundant computation across requests that contain overlapping context. Yet, the effectiveness of cache reuse is often limited by three practical challenges: low cache hit rates due to naive eviction policies, high CPU-GPU data transfer overhead, and slow SSD I/O when caches spill to storage. To address these issues, we propose PCR, a system designed to maximize KV-cache reuse efficiency through intelligent prefetching and pipelined data movement. Specifically, PCR introduces three key techniques: (1) a prefix-tree caching structure with a look-ahead LRU replacement policy that uses pending requests in the scheduler queue to improve cache hit ratios; (2) layer-wise overlapping that pipelines KV-cache loading and GPU computation across CUDA streams to hide communication latency; and (3) queue-based prefetching that proactively loads relevant KV caches from SSD into DRAM before they are needed. Extensive experiments show that PCR outperforms existing KV-cache reuse methods, achieving up to a 2.47x speedup in terms of average TTFT.

#### 1. 核心问题与研究动机
检索增强生成（RAG）通过引入外部文档提升了 LLM 的准确性，但导致输入序列极长，使得推理的**预填充（Prefill）阶段**（即生成 KV Cache 的过程）成为严重的延迟瓶颈。虽然 KV 缓存复用（KV-cache reuse）可以减少重复计算，但现有方案面临三大挑战：
*   **缓存命中率低**：传统的淘汰策略（如 LRU）未考虑 RAG 特有的请求模式。
*   **传输开销大**：在 CPU 内存与 GPU 显存之间频繁换入换出 KV Cache 产生显著延迟。
*   **存储瓶颈**：当缓存溢出到 SSD 时，缓慢的磁盘 I/O 会阻塞计算流水线。

#### 2. 方法论：核心思想与关键技术
PCR 系统旨在通过智能预取和流水线化数据移动，最大化多级存储（GPU-DRAM-SSD）下的缓存复用效率。其核心技术包括：
*   **前缀树缓存与前瞻性 LRU（Look-ahead LRU）**：
    *   将文档划分为固定大小的块（Chunks），利用前缀树管理。
    *   **核心创新**：通过观察调度器队列中的待处理请求，提前提升即将被复用块的优先级，防止其被错误淘汰，从而提高命中率。
*   **层级重叠技术（Layer-wise Overlapping）**：
    *   利用 LLM 的层级结构，将 KV 缓存的加载（CPU→GPU）、当前层的计算、以及新生成缓存的写回（GPU→CPU）在三个独立的 CUDA 流中并行执行。
    *   这种流水线设计将传输开销隐藏在计算时间之后，使有效延迟缩减至仅约一层楼的传输时间。
*   **基于队列的预取机制（Queue-based Prefetching）**：
    *   当请求在队列中等待时，后台线程根据前缀匹配结果，主动将所需的 KV 缓存从 SSD 加载到 DRAM。
    *   结合异步写回（Write-back）策略，确保 SSD 的慢速 I/O 不会直接阻塞推理过程。

#### 3. 实验设计
*   **数据集与场景**：使用 Wikipedia 作为检索语料库，SQuAD 作为查询数据集。模拟 RAG 工作流，每个请求包含 2 个检索文档和 1 个查询，平均长度约 6.8k tokens。
*   **Benchmark 与对比方法**：
    *   **vLLM**：工业级标准推理框架（含 PagedAttention）。
    *   **LMCache**：最先进的分布式 KV 缓存层。
    *   **CCache/SCCache**：作者实现的基准方案，分别代表仅扩展 CPU 内存和扩展 SSD 的简单缓存方案。
*   **评估指标**：首字生成时间（TTFT）、端到端延迟（E2EL）、以及 P95/P99 尾部延迟。

#### 5. 实验数量与充分性
作者进行了多维度的详尽实验：
*   **总体性能对比**：在不同请求速率（0.5 到 1.0 req/s）下对比了 6 种模型。
*   **消融实验**：分别验证了层级重叠（仅上传、仅下载、双向）和预取窗口大小对性能的影响。
*   **性能分解**：量化了各技术点在不同模型（如 MHA 架构的 Llama 2 vs GQA 架构的 Qwen 2.5）上的收益差异。
*   **充分性评价**：实验覆盖了主流开源模型架构和不同显存容量的硬件，对比基准具有代表性，实验设计客观且考虑了高负载下的系统稳定性。

#### 6. 主要结论与发现
*   **显著加速**：PCR 在 TTFT 上相比 vLLM 最高实现 **2.47 倍**加速，相比 LMCache 也有明显提升。
*   **尾部延迟优化**：在 P99 延迟指标上，PCR 相比竞品降低了超过 30%，证明了其在处理突发流量时的稳定性。
*   **架构敏感性**：KV 缓存较大的模型（如 Llama 2）从层级重叠和预取中获益更多；而 KV 缓存较小的模型（如 Qwen 2.5）受传输开销影响较小，但 PCR 依然能通过提高命中率带来收益。
*   **SSD 的双刃剑效应**：简单引入 SSD 可能因 I/O 慢导致性能下降，但 PCR 的异步预取机制成功将其转化为容量优势。

#### 7. 优点
*   **系统级协同优化**：不改变模型权重或牺牲精度（Exact Matching），通过纯工程手段压榨硬件潜力。
*   **延迟隐藏能力强**：层级重叠技术巧妙利用了 Transformer 的计算特性，极大地缓解了 PCIe 带宽瓶颈。
*   **前瞻性调度**：Look-ahead 策略将调度信息与缓存管理深度耦合，比传统通用缓存算法更契合 LLM 服务场景。

#### 8. 不足与局限
*   **应用限制**：仅适用于具有**精确前缀重叠**的场景。如果 RAG 检索到的文档顺序发生变化或内容有微小差异，前缀树将无法有效匹配。
*   **硬件依赖**：性能高度依赖于 PCIe 带宽和 SSD 随机读取性能，在带宽受限的云环境或移动端设备上效果可能打折扣。
*   **预取窗口调优**：实验表明最佳预取窗口大小随模型和负载变化，系统尚缺乏自动调整该超参数的动态机制。
