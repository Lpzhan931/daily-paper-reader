# LLM Router: Prefill is All You Need 

Tanay Varshney ∗ Annie Surla ∗ Michelle Xu Gomathy Venkata Krishnan Maximilian Jeblick David Austin Neal Vaidya Davide Onofrio NVIDIA ∗

Abstract 

LLMs often share comparable benchmark accuracies, but their complemen-tary performance across task subsets suggests that an Oracle router—a theoretical selector with perfect foresight—can significantly surpass stan-dalone model accuracy by navigating model-specific strengths. While current routers rely on fragile semantic signals, we propose using internal prefill activations via Encoder-Target Decoupling —a functional separation between the model providing the predictive signal (the Encoder ) and the model whose performance is being estimated (the Target ). This allows optimized heterogeneous pairing between unique encoders and target models. We utilize Fisher Separability ( J) and Effective Dimensionality (deff ) as mathematical probes to isolate optimal layer-wise signals, pro-viding the predictive foundation for our SharedTrunkNet architecture. SharedTrunkNet captures up to 45.58% of the accuracy gap between the strongest standalone model and the Oracle while achieving 74.31% cost savings relative to the highest-cost model. 

1 Introduction 

Current model routing is primarily predicated on semantic or intent-based signals, leverag-ing query features and domain descriptions to map tasks to specialized LLMs (e.g., KNN on embeddings). While effective for domain specialization, this paradigm suffers from two foundational flaws: 1. Lack of mechanistic interpretation: Routers ignore the intrinsic generative limits of the underlying models. 2. Semantic-complexity gap: Semantic similarity does not correlate with task diffi-culty; routers fine-tuned for representation often fail to capture signals indicative of model behavior. Furthermore, traditional evaluations are often inflated by “agreement” cases where all models share the same output. Without a signal reflecting intrinsic difficulty, routing performance degrades under strict cost-based Service Level Agreements (SLAs). Our primary contributions include: • Encoder-Target Decoupling: We demonstrate that “foreign” open-source encoders (e.g., Qwen) can predict the performance of closed-source targets (e.g., Claude) more effectively than the targets’ own states. • Fisher-based layer selection: We move beyond “final layer” assumptions by using 

deff and Fisher Separability ( J) to mathematically identify the most separable layer. • SharedTrunkNet: A multi-output architecture providing cross-model context to assess relative difficulty by comparing simultaneous encoder responses. • Comprehensive evaluation: A framework quantifying “Router Efficacy” by mea-suring the area between the router’s accuracy-cost curve and the theoretical Pareto frontier.

2 Literature survey 

The landscape of LLM orchestration has transitioned from static selection to dynamic, data-driven routing, categorized by decision-making signals and architectural complexity. 

Non-predictive frameworks. Initial approaches relied on heuristics or cascading strategies. Chen et al. (2023) sequentially direct queries through increasingly complex models. While cost-effective for trivial tasks, these suffer from prohibitive cumulative latency on complex queries. 

Semantic and intent-based routing. Predictive “black-box” routers select LLMs before generation using query embeddings. Ong et al. (2024) utilize human preference data, while Shao et al. (2024) and Lin et al. (2024) leverage retrieval-based patterns. Similarly, Zhu et al. (2024) and Xu et al. (2025) employ GNNs or shallow encoders (e.g., DeBERTa) to map embeddings to model choices. These are fundamentally limited by the representational bottleneck of semantic embeddings, which fail to capture intrinsic task complexity. 

Mechanistic routing. Recent research shifts toward internal hidden states and attention patterns as high-fidelity signals. Cencerrado et al. (2025) demonstrate that LLMs encode accuracy within the residual stream, with “correctness directions” crystallizing during prefill. Furthermore, Chen et al. (2025) extract activation statistics for zero-cost intent classification. Our work extends this via Encoder-Target Decoupling, using small open-source models to predict closed-source performance. 

Strategy and multi-level orchestration. Frameworks like Route to Reason (Pan et al., 2025) dynamically allocate both models and reasoning strategies (e.g., Chain-of-Thought), reducing token usage by 60%. Xu et al. (2025) further operationalize routing across query, thought, and model levels. 

3 Architecture 

From first principles, a router is a function of two signals: per-LLM confidence and per-model cost of inference. These signals, collected for all models in a pool, are fed into a routing policy to select the optimal model. 

Per-LLM confidence. A scalar in [0, 1 ] representing confidence in an LLM’s ability to answer the given query correctly. 

Per-LLM cost. The cost in USD required to fulfill one LLM inference request. Cost factors differ by deployment scenario: • API endpoints: cost is a function of input tokens, output tokens, and per-token pricing. • Local engine deployment: cost is a function of input tokens, output tokens, inter-token latency (ITL), time to first token (TTFT), and GPU rental cost per unit time. In both scenarios, all signals except output token count are observable or estimable prior to generation, yielding: best model = ROUTER (confidence, COST (isl, osl, ITL, TTFT, GPU cost )) 

Therefore, the two signals imperative to predict are per-LLM confidence and per-LLM output token count. This work focuses on confidence prediction; output token prediction is deferred to future work. We use each model’s median training output tokens as a cost proxy. Architecturally, we have a two-step process: 1. Signal extraction: model(s) that extract signals from inputs available at routing time. 2. Signal capture: model(s) that map those signals to a per-LLM confidence score. 2Preprint

4 Methodology 

4.1 LLM evaluation 

In addition to data from LLMRouterBench, we design a lightweight evaluation harness that queries each model across three benchmarks: MMLU-Pro, Humanity’s Last Exam (HLE), and LiveCodeBench (LCB). Models are queried via OpenAI-compatible streaming endpoints with per-provider adapters. All models use a maximum of 128,000 tokens and their highest available reasoning effort— reasoning effort: high for Claude and GPT-OSS, and enable thinking with provider-recommended sampling for Qwen and Nemotron. For each (model, question) pair we record the response, reasoning trace, and wall-clock latency. Scoring follows standard protocols: exact-match extraction for MMLU-Pro, GPT-4o-as-a-judge for HLE, and sandboxed pass@1 execution for LCB. All evaluators produce a unified binary correctness label per (model, question) pair, which serves as the ground-truth signal for router training and evaluation. No intentional prompt engineering or harness tuning was applied. 

4.2 Semantic router 

Semantic routers predict per-model correctness P(model k correct | q) from features derived exclusively from the input query text, without executing any candidate LLM. We organize all data into pre-split train, calibration, and test bundles and extract the following feature families: • Question embeddings: 2048-d dense vectors from NV-Embed-v2 (Lee et al., 2025), optionally reduced via PCA to 128 or 256 dimensions. • kNN statistics (50-d per model): Using FAISS (Johnson et al., 2019) flat-L2 indices over training embeddings, we retrieve K ∈ { 5, 10, 25, 50, 100 } neighbors. For each 

K, 10 statistics are computed—including neighbor-correct ratio, distance-weighted correctness, and class-conditional distance means. • Handcrafted text complexity (∼30-d): Readability indices (Flesch-Kincaid, Coleman-Liau, ARI, SMOG), lexical diversity (type-token ratio, hapax legomenon ratio, Yule’s K), structural signals (negation count, math/code presence, parentheti-cal nesting depth, number density), and information-theoretic measures (character entropy, zlib compression ratio). • GPT-2 perplexity (4-d): Mean log-probability, perplexity, max surprise, and token-level entropy from a frozen GPT-2 model (Radford et al., 2019), capturing language-model-perceived complexity. 3Preprint These feature families were evaluated across a wide array of architectures and classifiers. Standard training used multi-seed initialization, Adam/AdamW optimizers, and early stopping (patience 20–25), with the best model selected by mean calibration-set AUC. Core architectures included: • Rule-based kNN routers: kNN Majority Vote and kNN Score ( ∑i wi · yi, inverse distance-weighted) (Shao et al., 2024). • Feature-based classifiers: kNN, raw embedding, or complexity features fed into Logistic Regression (LR), XGBoost (Chen & Guestrin, 2016), plain MLP, or a Multi-Task Neural Network (MT) with dual heads for correctness (BCE) and difficulty (CE). • Unified Multi-Task Network: Following Stripelis et al. (2024), a single shared backbone predicting correctness and difficulty for all models jointly in one forward pass. • End-to-End text models: LoRA-finetuned (Hu et al., 2021) DeBERTa-v3-base (He et al., 2021), in “Shared” (shared LoRA, per-model heads) and “Per-model” (inde-pendent encoders) configurations. Raw predictions were passed through a calibration suite: Platt scaling, isotonic regression, percentile mapping, and z-score normalization. We also swept kNN distance-weighting ker-nels, FAISS index sizes, and multi-task loss λ weights. Over 400 non-ceiling configurations were evaluated in total. Additional advanced architectures were explored but did not yield improvements dis-tinguishable from noise: (i) IRT & MIRT-2PL (Embretson & Reise, 2000), mapping fea-tures to a latent difficulty vector θq ∈ RD and per-model ability vectors am, computing

4.3 Prefill router 

Following Chen et al. (2025), we propose leveraging an LLM’s internal activations during prefill as a predictive signal for model correctness—a fundamentally different approach from coarse query embeddings. We distinguish between the Encoder LLM , which provides hidden-state signals, and the Target LLM , whose performance is predicted. This decoupling lets us approximate closed-source model capabilities using open-weights encoders. We evaluate three operational modes: 

per model (fixed assignments), single (one encoder for all targets), and auto (optimal encoder per target). For a query q, we extract hidden states from the upper half of the encoder’s transformer layers (Lugoloobi et al., 2026) ( L/ 2 to L, where L is the total number of layers), compare last-token vs. mean-pooling, and apply PCA to reduce dimensionality ( d ∈ { 50, . . . , 300 }). Layer selection is determined by a grid search validated with 5-fold stratified cross-validation using L2-regularized logistic regression. 5 Evaluation paradigm 

5.1 Metrics 

5.1.1 ROC-AUC 

ROC-AUC measures classifier quality across all thresholds—it equals the probability that a model ranks a random positive above a random negative. Because routing requires judging model capability across varying difficulty levels, threshold-dependent metrics such as F1 are unsuitable as primary drivers.

5.1.3 Accuracy 

We track three accuracy paradigms: 1. Individual LLM accuracy: baseline performance of each model. 2. Routed pool accuracy: aggregate accuracy with the router active. 3. Conditional accuracy: accuracy of each model on the subset of queries routed to it. 

5.1.4 Cost 

Cost in USD arises from two sources: (1) LLM inference , computed from total input/output tokens and per-million rates; and (2) router overhead , which in a production vLLM setting can be negated by caching prefill activations. 

5.2 Framework 

We partition the dataset by model consensus to isolate where routing adds value: • All correct: queries answered correctly by every model. • All incorrect: queries failed by every model. • Model disagreement: queries where at least one model succeeds while others fail—the primary regime where routing matters.

7 Foundations for prefill signals 

A significant empirical finding is that “foreign” encoders (e.g., Qwen-35B/122B) consistently outperform a target model’s own internal states in predicting its correctness. 8 Conclusion 

This work establishes a mechanistic paradigm for LLM routing, moving beyond fragile semantic signals to leverage transformer activation geometry. By introducing Encoder-Target Decoupling, we provide a scalable pathway for routing to proprietary APIs using only the prefill activations of lightweight local models. We ground this approach mathematically using Fisher Separability ( J) and Effective Dimen-sionality ( deff ) as geometric probes, enabling principled identification of optimal, separable transformer layers. Our SharedTrunkNet architecture leverages this geometry to deliver cross-model context and calibrated simultaneous correctness probabilities. Our evaluation spans three model tiers: • Frontier pool: 10 high-parameter models (e.g., GPT-5, Claude 4.6) across 10 bench-marks. • Small pool: 20 lightweight models (7B–9B) across 18 benchmarks. • Mixed pool: A tier spanning frontier and small models designed to test routing’s ability to offload easy tasks to cheaper architectures. SharedTrunkNet closed 45.58% of the accuracy gap between the strongest standalone model and the theoretical Oracle, while achieving 74.31% cost savings relative to the highest-cost model. Our approach also yielded a 53.62% improvement in Router Efficacy and a 14.67% increase in P-AUCCC over the model-only Pareto frontier, confirming that mechanistic rout-ing provides a robust, cost-effective, and theoretically grounded foundation for collaborative LLM systems. 11 Preprint
