## 摘要
扩展自回归大语言模型（LLM）推动了前所未有的进步，但也带来了巨大的计算成本。在这项工作中，我们通过利用 LLM 前馈层中的非结构化稀疏性来解决这些成本问题，前馈层是占据模型大部分参数和执行 FLOPs 的核心组件。为实现这一目标，我们引入了一种新的稀疏打包格式和一组 CUDA 算子，旨在与现代 GPU 的优化执行流水线无缝集成，从而在 LLM 推理和训练期间实现高效的稀疏计算。为了证实我们的收益，我们对 LLM 稀疏性进行了定量研究，证明简单的 L1 正则化可以诱导超过 99% 的稀疏性，且对下游性能的影响微乎其微。当与我们的算子结合使用时，这些稀疏水平可以转化为显著的吞吐量、能效和内存使用优势，且这些优势随模型规模的扩大而增加。我们将以开源许可证发布所有代码和算子，以促进采用并加速相关研究，旨在将稀疏性确立为提高现代基础模型效率和可扩展性的实用维度。

## Abstract
Scaling autoregressive large language models (LLMs) has driven unprecedented progress but comes with vast computational costs. In this work, we tackle these costs by leveraging unstructured sparsity within an LLM's feedforward layers, the components accounting for most of the model parameters and execution FLOPs. To achieve this, we introduce a new sparse packing format and a set of CUDA kernels designed to seamlessly integrate with the optimized execution pipelines of modern GPUs, enabling efficient sparse computation during LLM inference and training. To substantiate our gains, we provide a quantitative study of LLM sparsity, demonstrating that simple L1 regularization can induce over 99% sparsity with negligible impact on downstream performance. When paired with our kernels, we show that these sparsity levels translate into substantial throughput, energy efficiency, and memory usage benefits that increase with model scale. We will release all code and kernels under an open-source license to promote adoption and accelerate research toward establishing sparsity as a practical axis for improving the efficiency and scalability of modern foundation models.

### 1. 核心问题与整体含义
*   **研究动机**：随着大语言模型（LLM）规模的扩展，计算成本、能耗和内存压力剧增。前馈网络（FFN）层占据了模型约 2/3 的参数和 80% 的计算量。
*   **核心矛盾**：虽然 FFN 层在使用 ReLU 激活时具有天然的稀疏性，但由于非结构化稀疏与现代 GPU 架构（针对稠密计算优化）不匹配，导致现有的稀疏算子往往比稠密算子更慢。
*   **整体含义**：本文旨在通过底层算子创新和简单的正则化手段，打破“稀疏不如稠密快”的僵局，实现 LLM 在训练和推理全生命周期的实质性加速。

### 2. 方法论
*   **稀疏诱导机制**：采用 **ReLU** 激活函数，并在标准交叉熵损失基础上增加 **L1 正则化** 损失，强制模型在训练过程中产生极高的激活稀疏度。
*   **TwELL 格式（Tile-wise ELLPACK）**：提出一种新型稀疏数据打包格式。它将矩阵划分为 1D Tile，在 Tile 内部进行局部 ELLPACK 压缩。其关键优势在于可以无缝集成在稠密矩阵乘法内核的收尾阶段（Epilogue），避免了传统稀疏格式转换带来的高昂同步开销。
*   **融合推理内核**：设计了单内核流水线，将 Gate、Up 和 Down 三个投影层融合。利用 TwELL 格式，内核仅加载非零激活对应的权重行/列，大幅减少 DRAM 访问。
*   **混合训练格式（Hybrid Format）**：针对训练中激活值分布不均的问题，将大部分稀疏行存入紧凑的 ELL 矩阵，而将少数稠密行（溢出行）存入稠密备份矩阵，从而在保证精度的同时最小化中间激活值的内存占用。

### 6. 主要结论与发现
*   **性能无损**：通过轻微的 L1 正则化，可以诱导超过 **99% 的稀疏度**，而下游任务的准确率几乎没有下降。

*   **规模效应**：模型规模越大，稀疏化的收益越明显，这表明稀疏性是未来超大规模模型扩展的有效路径。
*   **可解释性**：发现模型在处理高信息量 Token（如特定名词、动词）和序列起始位置时会激活更多神经元。

### 7. 优点
*   **底层突破**：TwELL 格式解决了稀疏计算中长期存在的格式转换瓶颈。
*   **全栈优化**：方案覆盖了从算法（L1 正则）到数据格式（TwELL）再到硬件执行（CUDA Kernels）的全链路。
*   **实用性强**：不需要改变主流的 Transformer 架构，易于集成到现有的训练框架中。

### 8. 不足与局限
*   **模型规模上限**：实验目前最高仅验证到 2B 参数，在百亿或千亿级参数模型上的表现尚需进一步验证。

*   **硬件绑定**：部分优化（如 TMA 异步加载）针对 NVIDIA Hopper 架构设计，在旧款 GPU 或非 NVIDIA 硬件上的加速效果可能受限。
