## 摘要
长上下文大语言模型（LLM）推理的瓶颈不在于计算，而在于每个解码步骤中扫描键值（KV）缓存所带来的 O(n) 内存带宽开销——这是一个无论如何扩展算力都无法打破的障碍。最近的光子加速器在稠密注意力计算方面展示了极高的吞吐量；然而，当应用于长上下文时，这些方法继承了与电子注意力相同的 O(n) 内存缩放特性。我们观察到，真正的突破点在于粗粒度的块选择（block-selection）步骤：这是一种决定提取哪些 KV 块的受内存限制的相似性搜索。我们首次发现，该任务在结构上与光子“广播与加权”（broadcast-and-weight）范式高度匹配——查询通过被动分束扇出到所有候选块，特征签名是准静态的（匹配电光微环谐振器 MRR 编程），且仅排序顺序重要（从而将精度要求放宽至 4-6 位）。至关重要的是，光子优势随上下文长度增加而增长：随着 N 的增加，电子扫描成本线性上升，而光子评估保持为 O(1)。我们将这一见解实现在 PRISM（通过微环权重的内积相似度进行光子排序）中，这是一种薄膜铌酸锂（TFLN）相似性引擎。在 Qwen2.5-7B 上进行的考虑硬件损伤的“大海捞针”评估证实，在 k=32 时，从 4K 到 64K 标记均实现了 100% 的准确率，并在 64K 上下文下减少了 16 倍的数据传输。在实际上下文长度（n >= 4K）下，PRISM 相比 GPU 基准实现了四个数量级的能效优势。

## Abstract
Long-context LLM inference is bottlenecked not by compute but by the O(n) memory bandwidth cost of scanning the KV cache at every decode step -- a wall that no amount of arithmetic scaling can break. Recent photonic accelerators have demonstrated impressive throughput for dense attention computation; however, these approaches inherit the same O(n) memory scaling as electronic attention when applied to long contexts. We observe that the real leverage point is the coarse block-selection step: a memory-bound similarity search that determines which KV blocks to fetch. We identify, for the first time, that this task is structurally matched to the photonic broadcast-and-weight paradigm -- the query fans out to all candidates via passive splitting, signatures are quasi-static (matching electro-optic MRR programming), and only rank order matters (relaxing precision to 4-6 bits). Crucially, the photonic advantage grows with context length: as N increases, the electronic scan cost rises linearly while the photonic evaluation remains O(1). We instantiate this insight in PRISM (Photonic Ranking via Inner-product Similarity with Microring weights), a thin-film lithium niobate (TFLN) similarity engine. Hardware-impaired needle-in-a-haystack evaluation on Qwen2.5-7B confirms 100% accuracy from 4K through 64K tokens at k=32, with 16x traffic reduction at 64K context. PRISM achieves a four-order-of-magnitude energy advantage over GPU baselines at practical context lengths (n >= 4K).

#### 1. 核心问题与整体含义
长上下文大语言模型（LLM）推理的主要瓶颈已从计算转向**内存带宽**。在自回归解码过程中，每生成一个新 Token 都需要扫描完整的键值（KV）缓存，其成本随上下文长度 $n$ 呈线性增长（$O(n)$）。现有的电子架构（如 GPU）受限于 HBM 带宽，导致长文本推理效率极低。
论文提出了 **PRISM**，这是一种基于光子计算的相似性引擎。其核心含义在于：利用光子技术的并行性，将 KV 缓存的“粗粒度块选择”步骤从 $O(n)$ 的电子扫描转变为 **$O(1)$ 的光子评估**，从而在物理层面突破内存墙。

#### 2. 方法论：核心思想与技术细节
PRISM 采用了光子“广播与加权”（Broadcast-and-Weight）范式，具体流程如下：
*   **核心思想**：将查询向量（Query）编码为多波长光信号，并行广播到存储了块特征签名（Signatures）的光子权重库中，通过物理光学反应直接输出相似度得分。
*   **关键技术步骤**：
    1.  **查询编码**：将 $d$ 维查询向量通过数模转换（DAC）和调制器（MZM）编码到 $d$ 个波长的 WDM 光束上。
    2.  **无源广播**：利用 $1 \times N$ 光分路器将信号无损（仅有分光损耗）地发送到 $N$ 个并行通道。
    3.  **特征加权**：每个通道包含一组微环谐振器（MRR），其权重通过电光效应（Pockels effect）预先编程，代表 KV 块的特征签名。
    4.  **光电检测**：宽带光电探测器（PD）对所有波长的加权功率进行积分，直接输出代表内积结果的电流信号。
    5.  **Top-k 选择**：电子比较器选出得分最高的 $k$ 个块，GPU 仅从内存中提取这些块进行精确计算。
*   **硬件平台**：采用**薄膜铌酸锂（TFLN）**平台，利用其超低静态功耗和高速电光响应。

#### 6. 主要结论与发现
*   **性能飞跃**：PRISM 在 $n \ge 4K$ 时展现出优势，在 64K 上下文下实现 **100% 的检索准确率**，同时减少了 **16 倍**（64K）至 **32 倍**（128K）的内存流量。
*   **能效优势**：相比 GPU 基准，PRISM 的能效高出 **4 个数量级**（约 10,000 倍），这主要归功于 TFLN 的电容性调谐（静态功耗接近零）。
*   **复杂度突破**：成功将块选择的延迟从随长度增长的线性时间缩减为固定的 **~9ns** 光学延迟。

#### 7. 优点
*   **架构匹配性**：敏锐地发现“块选择”任务与光子“广播与加权”范式在结构上完美契合（查询广播、准静态权重、低精度要求）。
*   **物理平台先进**：选用 TFLN 而非传统的硅光（SOI），解决了热稳定性带来的高功耗问题。
*   **端到端验证**：不仅停留在物理指标，还通过 NIAH 证明了该方案对 LLM 实际推理能力的保护。

#### 8. 不足与局限
*   **缺乏物理原型**：所有结果均基于物理仿真和 FDTD 模拟，尚未有实际流片硬件的实测数据。
*   **集成规模挑战**：支持 128K 上下文需要约 6.5 万个 MRR，这对当前光子集成电路（PIC）的良率和封装提出了极高要求。
*   **接口瓶颈**：虽然光学计算仅需 9ns，但如果通过 PCIe 接口与 GPU 通信，系统级延迟仍会受到电子总线的限制（需协同封装技术解决）。
*   **模型依赖性**：实验主要基于 Qwen 模型，对于不同架构（如非 Transformer 结构）的适用性有待进一步验证。
