## 摘要
KV 缓存压缩对于高效的长上下文大语言模型（LLM）推理至关重要。减少每个 KV 对占用空间的方法（如量化和低秩分解）与减少缓存序列长度的方法是正交的。在序列长度维度上，现有方法涵盖了从纯粹的剔除（选择保留哪些 KV 对）到合并（将相似的 KV 对组合成更少的对）。这两类方法都受限于原始缓存条目。我们提出了 KVSculpt，它转向了这一频谱的另一端：我们不再选择或组合原始 KV 对，而是在连续嵌入空间中优化一组更小的、不受约束的 KV 对，以保持每一层的注意力行为。Key 通过 L-BFGS 进行优化，Value 通过最小二乘法以闭式解求解，每隔几步交替进行。在此基础上，我们引入了自适应预算分配，利用低成本的初步压缩运行，根据每个组件的难度在各层和 KV 头之间重新分配压缩预算。在具有 2048 个 token 上下文的 Qwen2.5-1.5B-Instruct 上，在压缩率 r ∈ {0.3, 0.5, 0.7} 的情况下，与 Select+Fit（基于注意力分数的剔除结合最小二乘 Value 拟合）相比，KVSculpt 将 KL 散度降低了 3.5-4.1 倍。自适应分配在不增加额外推理成本的情况下，使 KL 散度进一步降低了 1.3 倍。分析表明，压缩难度是高度非均匀的：各层之间的初步均方误差（MSE）差异可达 100 倍，而同一层内的两个 KV 头之间的差异可达 467 倍，这证明了细粒度的预算分配是必不可少的。

## Abstract
KV cache compression is critical for efficient long-context LLM inference. Approaches that reduce the per-pair footprint -- quantization and low-rank decomposition -- are orthogonal to those that reduce the sequence length of the cache. Along the sequence-length dimension, existing methods range from pure eviction -- selecting which KV pairs to keep -- to merging, which combines similar pairs into fewer ones. Both remain anchored to the original cache entries. We propose KVSculpt, which moves to the other end of this spectrum: instead of selecting or combining original pairs, we optimize a smaller set of unconstrained KV pairs in continuous embedding space to preserve each layer's attention behavior. Keys are optimized via L-BFGS and values are solved in closed form via least squares, alternating every few steps. On top of this, we introduce adaptive budget allocation, which uses a cheap pilot compression run to redistribute the compression budget across layers and KV heads based on per-component difficulty. On Qwen2.5-1.5B-Instruct with 2048-token contexts, KVSculpt reduces KL divergence by 3.5-4.1x compared to Select+Fit -- attention-score eviction with least-squares value fitting -- across compression ratios r in {0.3, 0.5, 0.7}. Adaptive allocation provides an additional 1.3x KL reduction at no extra inference cost. Analysis reveals that compression difficulty is highly non-uniform: per-layer pilot MSE varies by up to 100x across layers, and the two KV heads within a single layer can differ by up to 467x -- demonstrating that fine-grained budget allocation is essential.

### 1. 核心问题与研究背景
在长文本大语言模型（LLM）推理中，KV Cache（键值缓存）随序列长度线性增长，成为显存瓶颈。现有的压缩方法主要分为两类：**剔除（Eviction）**（选择性保留原始 Token）和**合并（Merging）**（将相似 Token 加权组合）。
*   **研究动机**：作者认为现有方法都受限于“原始缓存条目”的离散框架。由于 RoPE 编码已将位置信息嵌入向量，压缩后的 KV 对不一定非要是原始 Token 的子集。
*   **核心目标**：将 KV 压缩重新定义为**连续嵌入空间中的蒸馏问题**，寻找一组不受限的“虚拟”KV 对，以最大程度还原原始注意力的输出行为。

### 2. 方法论
KVSculpt 的核心是将压缩视为对原始缓存的蒸馏优化：
*   **核心思想**：给定 $N$ 个原始 KV 对，优化 $k$ 个全新的 KV 对（$k < N$），使压缩后的缓存产生的注意力输出（Output）和 Log-Sum-Exp（LSE）值与原始缓存一致。
*   **关键技术细节**：
    *   **交替优化算法**：
        1.  **Key 优化**：使用 **L-BFGS** 算法（拟牛顿法）在非凸的 Softmax 景观中寻找最优 Key 向量。
        2.  **Value 求解**：在 Key 固定时，利用**岭回归（最小二乘法）**以闭式解形式高效求解最优 Value 向量。
    *   **合成查询（Synthetic Queries）**：由于压缩时无法预知未来查询，作者通过“去位置编码（De-RoPE）”提取内容向量，并重新应用未来位置的 RoPE 编码来构造合成查询，增强泛化性。
    *   **自适应预算分配（Adaptive Budget Allocation）**：通过一个简短的“试点运行（Pilot Run）”测量各层和各 KV 头的压缩难度（MSE），将有限的 KV 预算向难度大的组件倾斜。

### 3. 实验设计
*   **模型与数据**：使用 **Qwen2.5-1.5B-Instruct** 模型；测试集采用 **PG19**（长文档数据集），上下文长度 $N=2048$，评估后续 128 个 token 的生成质量。
*   **基准方法（Benchmarks）**：
    *   **Random**：随机保留。
    *   **Attention Score**：基于注意力分数累积的剔除（类似 H2O）。
    *   **Select+Fit**：先按分数筛选，再用最小二乘法拟合 Value（目前最强的剔除变体）。
    *   **Joint Opt**：使用 Hard Concrete 门控进行离散选择的联合优化。
*   **评估指标**：主要使用 **KL 散度**（衡量压缩模型与原始模型输出分布的差异）。

### 4. 资源与算力
*   **硬件**：实验在单张 **NVIDIA A100 GPU** 上运行。

*   **算力特性**：该方法属于“离线压缩”或“摊销压缩”，即在处理长系统提示词或静态文档时进行一次性高强度压缩，随后在多次推理中受益。

### 5. 实验数量与充分性
*   **实验规模**：主实验涵盖了 20 个序列，详细分析和消融实验使用了 5 个序列的子集。

    *   测试了多种压缩率（$r \in \{0.1, 0.3, 0.5, 0.7\}$）。
    *   进行了详尽的消融实验，包括优化器对比（L-BFGS vs Adam）、查询构造策略对比、分配权重（$\alpha$ 因子）对比。
    *   **客观性**：通过 Oracle（100 次随机重启）验证了 L-BFGS 的近乎最优性，实验设计较为严谨。

### 6. 主要结论与发现
*   **性能飞跃**：在 $r=0.3$ 时，KVSculpt 的 KL 散度比最强基准（Select+Fit）低 **4.1 倍**。
*   **连续空间的优势**：实验证明，允许 Key 在连续空间自由移动（而非限制在原始位置）是性能提升的关键，仅优化 Value 效果有限。
*   **非均匀难度**：发现不同层之间的压缩难度差异可达 100 倍，同一层内不同 KV 头的难度差异甚至高达 467 倍。

### 7. 优点
*   **范式创新**：打破了“KV 压缩必须是原始 Token 子集”的思维定式，将离散选择问题转化为连续优化问题。
*   **高保真度**：在极高压缩比下仍能保持极低的 KL 散度，部分序列甚至达到近乎无损（KL < 0.01）。
*   **细粒度控制**：引入了跨层和跨头的自适应分配，充分利用了 Transformer 内部结构的异构性。

### 8. 不足与局限
*   **模型与长度限制**：目前仅在 Qwen2.5-1.5B 上验证，且上下文长度仅为 2048，尚未在 7B+ 模型或 100K+ 极长文本上测试。
*   **压缩开销**：170 秒的压缩时间对于实时单次对话过长，仅适用于“一次压缩、多次查询”的场景（如长文档问答）。
*   **评估维度**：主要依赖 KL 散度，缺乏下游任务（如 MMLU、摘要生成）的直接准确率评测。
*   **误差累积**：分析显示，尽管单层优化已接近极限，但层与层之间的误差累积仍是导致“困难序列”表现不佳的主要原因。
