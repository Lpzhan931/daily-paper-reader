## 摘要
注意力机制一直是现代 Transformer 架构的核心组件。然而，标准全注意力的计算量随序列长度呈二次方增长，成为长上下文语言建模的主要瓶颈。滑动窗口注意力通过限制上下文长度来提高效率，但代价是感受野变窄。虽然现有研究尝试通过构建混合模型来兼顾两者的优点，但它们通常采用静态的、基于启发式设计的交替模式，这限制了在各种场景下计算资源的有效分配。在本文中，我们提出了 Switch Attention (SwiAttn)，这是一种新型混合 Transformer，能够在全注意力和滑动窗口注意力之间实现动态且细粒度的路由。对于每个 Transformer 层中的每个 token，SwiAttn 动态地将计算路由到用于全局信息聚合的全注意力分支，或用于高效局部模式匹配的滑动窗口分支。我们设计了一个自适应正则化目标，以鼓励模型提高效率。此外，我们采用持续预训练来优化模型，将全注意力架构转换为混合架构。我们在 23 个基准数据集上进行了广泛的实验，涵盖了常规（4K）和长（32K）上下文长度，证明了所提方法的有效性。

## Abstract
The attention mechanism has been the core component in modern transformer architectures. However, the computation of standard full attention scales quadratically with the sequence length, serving as a major bottleneck in long-context language modeling. Sliding window attention restricts the context length for better efficiency at the cost of narrower receptive fields. While existing efforts attempt to take the benefits from both sides by building hybrid models, they often resort to static, heuristically designed alternating patterns that limit efficient allocation of computation in various scenarios. In this paper, we propose Switch Attention (SwiAttn), a novel hybrid transformer that enables dynamic and fine-grained routing between full attention and sliding window attention. For each token at each transformer layer, SwiAttn dynamically routes the computation to either a full-attention branch for global information aggregation or a sliding-window branch for efficient local pattern matching. An adaptive regularization objective is designed to encourage the model towards efficiency. Moreover, we adopt continual pretraining to optimize the model, transferring the full attention architecture to the hybrid one. Extensive experiments are conducted on twenty-three benchmark datasets across both regular (4K) and long (32K) context lengths, demonstrating the effectiveness of the proposed method.

### 1. 核心问题与研究动机
*   **核心问题**：标准 Transformer 的全注意力（Full Attention）计算复杂度随序列长度呈二次方增长，是长文本建模的主要瓶颈。
*   **研究背景**：现有的优化方案如滑动窗口注意力（SWA）虽然高效但感受野受限；混合模型（如一层全注意力交替三层 SWA）多采用**静态、启发式**的设计，无法根据输入内容动态调整计算资源。
*   **研究动机**：并非所有 Token 在所有层都需要全局信息。论文旨在实现一种**动态且细粒度**的路由机制，让模型自主决定每个 Token 在每一层是使用昂贵的全注意力还是高效的滑动窗口注意力。

### 2. 方法论
*   **核心思想**：提出 **Switch Attention (SwiAttn)** 架构，在每个 Transformer 层引入一个轻量级路由（Router），实现全注意力分支与 SWA 分支的动态切换。
*   **关键技术细节**：
    *   **双分支路由（Dual-branch Routing）**：路由函数根据输入隐藏表示生成门控值（Gate），通过 Sigmoid 激活和 0.5 阈值二值化。使用直通估计器（STE）解决不可导问题。
    *   **自适应正则化（Adaptive Regularization）**：为了鼓励模型多使用 SWA 以提升效率，设计了正则化损失。其权重 $\gamma$ 是自适应的：当模型预测不确定（负对数似然 NLL 高）或两个分支输出差异大（MSE 高）时，降低正则化强度，允许模型回退到全注意力。
    *   **持续预训练（Continual Pretraining, CPT）**：无需从头训练。SwiAttn 从现有的全注意力模型初始化，将全注意力权重同时赋予两个分支，通过少量 Token 的持续训练完成架构转换。
*   **算法流程**：输入 -> 路由决策 -> 分支计算（共享 QKV）-> 根据门控值合并输出 -> FFN 层。

### 6. 主要结论与发现
*   **性能优异**：SwiAttn 在长文本任务上超越了所有基准模型，平均得分高于 FullAttn，且显著优于静态混合模型。
*   **效率大幅提升**：在 32K 上下文下，解码阶段实现 **4 倍（标准数据）至 12 倍（简单数据）** 的加速。
*   **路由规律**：全注意力的平均使用比例仅为 **0.13**（低于静态混合模型的 0.25）。模型在中间层更倾向于使用全注意力，而底层和顶层倾向于 SWA。
*   **动态适应性**：在“大海捞针”实验中，当目标信息在窗口外时，全注意力比例自动升高；在窗口内时则降低，证明了路由的智能性。

### 7. 优点
*   **细粒度分配**：将计算资源分配精确到 Token 级别，比层级切换更灵活。
*   **自适应平衡**：通过 NLL 和 MSE 动态调节正则化，巧妙平衡了模型精度与推理速度。
*   **实用性强**：支持从现有模型持续预训练，大幅降低了采纳新架构的算力门槛。

### 8. 不足与局限
*   **硬件实现**：目前在 Prefill（预填充）阶段缺乏针对双分支并行计算的底层硬件优化内核（如定制的 CUDA/Triton Kernel），实际加速效果受限。
*   **模型规模验证**：实验仅在 1.5B 规模上验证，尚未在百亿或千亿级参数模型上测试其扩展性（Scaling Law）。
*   **训练开销**：虽然 CPT 节省了时间，但双分支结构在训练时仍需计算两个路径，增加了训练时的显存和计算负担。
