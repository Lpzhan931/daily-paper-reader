## 摘要
近期的多模态大语言模型计算开销巨大，因为 Transformer 必须处理大量的视觉 Token。我们提出了 ReDiPrune，这是一种在视觉-语言投影器之前应用的无需训练的 Token 剪枝方法，此时视觉特征仍保持着丰富的判别性。与在压缩表示上运行的投影后剪枝方法不同，ReDiPrune 直接从视觉编码器输出中选择信息丰富的 Token，从而保留了细粒度的空间和语义线索。每个 Token 都由一个轻量级规则进行评分，该规则共同考虑了文本条件相关性和最大最小多样性，确保所选 Token 既与查询相关又不冗余。ReDiPrune 完全是即插即用的，不需要重新训练或修改架构，并且可以无缝插入到编码器和投影器之间。在四个视频和五个图像基准测试中，它一致地改善了准确率与效率之间的权衡。例如，在 EgoSchema 基准上使用 LLaVA-NeXT-Video-7B 时，仅保留 15% 的视觉 Token 即可获得 +2.0% 的绝对准确率提升，同时将 TFLOPs 计算量减少 6 倍以上。代码可在 https://github.com/UA-CVML/ReDiPrune 获取。

## Abstract
Recent multimodal large language models are computationally expensive because Transformers must process a large number of visual tokens. We present \textbf{ReDiPrune}, a training-free token pruning method applied before the vision-language projector, where visual features remain rich and discriminative. Unlike post-projection pruning methods that operate on compressed representations, ReDiPrune selects informative tokens directly from vision encoder outputs, preserving fine-grained spatial and semantic cues. Each token is scored by a lightweight rule that jointly consider text-conditioned relevance and max-min diversity, ensuring the selected tokens are both query-relevant and non-redundant. ReDiPrune is fully plug-and-play, requiring no retraining or architectural modifications, and can be seamlessly inserted between the encoder and projector. Across four video and five image benchmarks, it consistently improves the accuracy-efficiency trade-off. For example, on EgoSchema with LLaVA-NeXT-Video-7B, retaining only 15\% of visual tokens yields a +2.0\% absolute accuracy gain while reducing computation by more than $6\times$ in TFLOPs. Code is available at https://github.com/UA-CVML/ReDiPrune.

#### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLM）在处理高分辨率图像或长视频时，会产生数以千计的视觉 Token。由于 Transformer 注意力机制的计算复杂度随序列长度呈二次方增长，这些冗余的视觉 Token 导致了巨大的内存消耗和推理延迟。
*   **现有方法的局限：** 多数现有的剪枝方法（如 FastV, CDPruner）是在“投影后”（Post-projection）进行的，即在视觉特征进入 LLM 空间后进行筛选。此时特征已被压缩或混合，容易丢失细粒度的空间和语义细节。
*   **本文动机：** 提出一种在“投影前”（Pre-projection）进行的剪枝策略，在视觉编码器输出的原始特征空间中直接筛选，以保留更丰富的判别性信息。

#### 2. 方法论：核心思想与关键技术
ReDiPrune 是一种**无需训练（Training-free）**、**即插即用**的模块，其核心思想是结合“文本相关性”与“视觉多样性”来筛选 Token。
*   **核心流程：**
    1.  **查询构建（Query Construction）：** 从用户提示词（Prompt）中提取文本特征，通过加权聚合（如指数加权）生成一个归一化的查询向量 $\hat{q}$。
    2.  **相关性评分（Relevance）：** 计算每个视觉 Token 与查询向量 $\hat{q}$ 的余弦相似度，识别出对回答问题最有用的区域。
    3.  **多样性评分（Diversity）：** 计算 Token 之间的余弦不相似度，采用“最大最小”（Max-Min）策略，确保选出的 Token 集合能够覆盖尽可能多的视觉信息，减少冗余。
    4.  **统一目标函数：** 使用贪婪算法求解一个联合优化目标，在满足预设保留率（Retention Ratio）的情况下，选择既与文本高度相关又彼此互补的 Token 子集。
    5.  **投影与输入：** 仅将选中的 Token 送入投影器（Projector）并输入 LLM。

#### 6. 主要结论与发现
*   **效率大幅提升：** 在 ActivityNet-QA 上，将 Token 从 2056 个减至 206 个，计算量（TFLOPs）从 11.61 降至 1.17，推理延迟从 0.447s 降至 0.146s。

*   **投影前剪枝的优势：** 实验证明在投影器之前进行筛选能更好地保留细粒度语义，比投影后剪枝更有效。

*   **即插即用：** 无需对现有模型进行任何微调或架构修改，适配性极强。
