## 摘要
大语言模型（LLMs）表现出强大的性能，但在实际服务中会产生高昂的推理成本，特别是在跨用户和会话存在重复或近乎重复查询的工作负载下。在这项工作中，我们提出了 MemBoost，这是一个内存增强的大语言模型服务框架，它使轻量级模型能够重用先前生成的答案并检索相关的支持信息以进行低成本推理，同时有选择地将困难或不确定的查询升级到更强大的模型。与主要针对单次响应进行基准化的标准检索增强生成（RAG）不同，MemBoost 专为交互式场景设计，支持答案重用、持续内存增长和成本感知路由。在模拟工作负载下对多个模型进行的实验表明，MemBoost 显著减少了昂贵的大模型调用次数和整体推理成本，同时保持了与强模型基准相当的高回答质量。

## Abstract
Large Language Models (LLMs) deliver strong performance but incur high inference cost in real-world services, especially under workloads with repeated or near-duplicate queries across users and sessions. In this work, we propose MemBoost, a memory-boosted LLM serving framework that enables a lightweight model to reuse previously generated answers and retrieve relevant supporting information for cheap inference, while selectively escalating difficult or uncertain queries to a stronger model. Unlike standard retrieval-augmented generation, which primarily grounds a single response, MemBoost is designed for interactive settings by supporting answer reuse, continual memory growth, and cost-aware routing. Experiments across multiple models under simulated workloads show that MemBoost substantially reduces expensive large-model invocations and overall inference cost, while maintaining high answer quality comparable to the strong model baseline.

    *   **冗余计算**：在真实服务场景中，大量用户查询是重复或语义高度相似的，重复调用大模型造成了严重的计算资源浪费。

### 2. 方法论
MemBoost 提出了一个“检索-决策-升级-回写”的闭环框架，包含三个核心组件：
*   **关联记忆引擎 (AME)**：
    *   维护一个外部存储库，包含历史查询、对应答案及元数据。
    *   支持快速语义检索（基于向量嵌入和 FAISS 索引）。
    *   支持“回写”机制，将新生成的高质量答案存入记忆。
*   **大模型先知 (Oracle)**：
    *   高性能但高成本的基准模型（如 Qwen3-14B），作为系统在记忆失效时的最终保障。
*   **元控制器 (Meta Controller, MC)**：
    *   一个轻量级 LLM（如 2B-4B 规模），负责编排流程。
    *   **流程细节**：对于新查询 $x_t$，MC 首先从 AME 检索相似条目。MC 判断检索内容是否足以回答问题：若足够，则直接复用或合成答案；若不足（不确定或无匹配），则将查询“升级”路由至 Oracle。
    *   **成本模型**：系统目标是最小化总成本 $C_T$，公式表达为：$C_T = \sum (c_{MC} + c_{Retrieval}) + \sum (1 - I_t) c_{Oracle}$，其中 $I_t$ 是记忆复用指示函数。

### 3. 实验设计
*   **数据集与场景**：使用 **MMLU-Pro** 评估集（涵盖多学科的挑战性基准）。
*   **工作负载模拟**：为了模拟真实世界的重复查询，研究者采用 **Zipf 分布**（齐夫定律）对查询进行采样，设置了不同的偏斜参数 $\alpha$（0.8, 1.1, 1.4）来代表不同的重复率。
*   **Benchmark 与对比方法**：
    *   **基准模型**：独立运行的小模型（Qwen3.5-2B, Ministral-3B, Qwen3-4B）和独立运行的 Oracle（Qwen3-14B）。
    *   **对比维度**：准确率（Accuracy）、内存使用率（代表成本节省程度）、响应延迟（Latency）。

### 4. 资源与算力
*   **硬件配置**：实验使用了 **NVIDIA A100 80GB GPU**。
*   **部署细节**：
    *   元控制器（MC）、记忆引擎（AME）和大模型先知（Oracle）分别部署在独立的 GPU 上以确保推理时间的公平比较。
    *   使用 **vLLM** 库优化吞吐量和延迟。
*   **模型参数**：MC 选用了 2B 到 4B 规模的模型，Oracle 选用 14B 规模。

### 5. 实验数量与充分性
*   **实验规模**：针对每个 Zipf 分布进行了 5,000 步（即 5,000 次连续查询）的模拟。
*   **覆盖范围**：测试了 3 种不同的轻量级模型作为 MC，以及 3 种不同重复率的工作负载。
*   **充分性评价**：实验设计较为客观，通过动态展示 5,000 步中的内存使用率变化，清晰地证明了系统具有“持续学习”和“记忆增长”的特性。对比了准确率与成本两个关键维度，论证了其在不同偏斜度负载下的鲁棒性。

### 6. 主要结论与发现
*   **性能持平甚至超越**：MemBoost 在所有设置下均显著优于小模型基准，且准确率与 Oracle 持平。在某些情况下（如使用 Qwen3.5-2B 作为 MC 时），其准确率甚至略高于 Oracle，因为系统能复用之前正确的历史答案，减少了随机生成错误。
*   **显著降本增效**：随着时间推移，AME 逐渐积累知识，内存复用率（$I_t$）稳步上升，导致对 Oracle 的调用次数大幅减少。
*   **负载敏感性**：查询重复率越高（Zipf $\alpha$ 越大），MemBoost 的优势越明显，延迟和成本下降越快。

### 7. 优点与亮点
*   **统一框架**：将语义缓存、RAG 和模型路由集成在一个自动化循环中，解决了 LLM 服务中的冗余计算问题。
*   **动态增长**：支持“回写”机制，使系统能够随着服务时间的增加而变得更聪明、更廉价。
*   **成本感知**：通过轻量级 MC 进行路由决策，实现了质量与成本的动态平衡。

### 8. 不足与局限
*   **任务类型局限**：目前仅在 MMLU-Pro（多项选择题）上进行了评估，尚未在长文本生成、代码编写或开放式对话等更复杂的任务上验证。
*   **语义检索风险**：在处理“语义相似但答案不同”的微妙查询时，可能存在误匹配（False Hit）风险，导致返回错误答案。
*   **模拟环境**：实验基于固定基准集的重复采样，真实世界中的查询变体（如改写、同义词）可能比 Zipf 模拟更复杂，对 AME 的鲁棒性提出了更高要求。
