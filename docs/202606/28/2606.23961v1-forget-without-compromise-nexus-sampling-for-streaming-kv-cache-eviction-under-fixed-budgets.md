## 摘要
长上下文和智能体LLM工作负载将KV缓存推至任何固定内存预算之外，迫使推理堆栈在连续推理流的每一步永久驱逐令牌。现有方法都遵循相同的模板：每一步的直接注意力分数后接确定性top-K选择，这会将单次低于阈值的步骤转化为不可逆的判定，并永久擦除任何直接注意力无法从噪声中分辨出的微妙重要令牌。为解决这一挑战，我们提出Nexus采样，一种免训练的驱逐方法，它将Nexus评分（一种迭代遍历直接注意力以揭示桥梁令牌的方法）与加权蓄水池采样（以包含概率保留令牌代替确定性top-K）配对。理论上，我们证明Nexus采样在微妙重要令牌的长期存活上优于确定性top-K。实验上，在80% KV缓存驱逐下，Nexus采样在LongBench上匹配密集注意力（误差1%以内），同时在检索密集型任务上优于top-K基线，每序列缓存内存最多减少10倍。

## Abstract
Long-context and agentic LLM workloads push the KV cache past any fixed memory budget, forcing the inference stack to permanently evict tokens at every step of a continuous-inference stream. Existing methods all share the same template, a per-step direct-attention score followed by deterministic top-$K$ selection, which converts a single below-cutoff step into an irreversible verdict and permanently erases any subtly important token that direct attention cannot single out from noise. To address this challenge, we propose Nexus Sampling, a training-free eviction method that pairs Nexus scoring, an iterative walk over direct attention that surfaces bridge tokens, with weighted reservoir sampling, which retains tokens with inclusion probability in place of deterministic top-$K$. Theoretically, we show that Nexus Sampling dominates deterministic top-$K$ in long-run survival of subtly important tokens. Empirically, at 80% KV cache eviction, Nexus Sampling matches dense attention within 1% on LongBench while outperforming top-$K$ baselines on retrieval-heavy tasks, with up to 10x smaller per-sequence cache memory.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：大语言模型（LLM）的长上下文推理中，Key-Value（KV）缓存随序列长度线性增长，GPU内存有限，迫使推理堆栈在每个步骤永久驱逐令牌（token）。现有的 KV 缓存驱逐方法（如 H2O、SnapKV、PyramidKV、MorphKV 等）均采用相同的模板：**单步直接注意力分数 + 确定性 top-K 选择**。这种设计存在根本缺陷：它将单次低于阈值的步骤转化为不可逆的判决，导致那些重要性波动、但在长期平均上重要的令牌（称为“微妙重要令牌”）在首次掉出阈值时就被永久擦除，即**边缘侵蚀**。
- **核心挑战**：在流式、固定预算场景下，驱逐与稀疏注意力不同——驱逐是不可逆的，且需对未知的未来查询做出保留决策。现有方法将稀疏注意力的一次性读取逻辑直接用于驱逐，混淆了暂时边缘性与永久无足轻重的区别。
- **任务意义**：LG 工作负载（多轮代理、持久推理、代码智能体）使有效上下文无限增长，高效的永久驱逐策略对于将大模型部署到实际推理系统中至关重要。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 整体框架：Nexus Sampling

Nexus Sampling 是一个**免训练**的 KV 缓存驱逐方法，在每个驱逐步骤（预算超出时）顺序执行两个组件：

1. **Nexus Scoring（上游：权重构建）**：通过迭代遍历直接注意力，发现“桥梁令牌”（bridge tokens），生成每个键块的采样权重。
2. **加权蓄水池选择（下游：选择规则）**：用加权无放回的蓄水池采样替代确定性 top-K，以包含概率保留令牌。

### 2.2 Nexus Scoring

- 对于每个键块 j，最终 Nexus 分数为：
  \[
  s_j = a_j + \lambda \tilde{c}_j
  \]
  - \(a_j\)：直接重要性——近 W 个查询向量的注意力分布的行平均。
  - \(\tilde{c}_j\)：间接重要性（桥接分数）——通过迭代行走从同样的查询行提取，衡量该块在互相关注簇中的锚定作用。
  - \(\lambda \ge 0\)：平衡权重。
- **桥接分数计算**：从 C=0 开始，对 H 步迭代更新：
  \[
  \gamma_q = 1 + \langle a^{(q)}, C \rangle, \quad C \leftarrow C + \gamma_q a^{(q)}
  \]
  其中 \(a^{(q)}\) 是窗口查询 q 的行注意力分布。最终 \(\tilde{c} = C / \|C\|_1\)。
- 加入小的时间戳 tie-break 后得到采样权重 \(w_j = s_j + \varepsilon_{\text{tie}} r_j\)。

### 2.3 加权蓄水池块选择

- 无条件保留注意力沉块和最近几块（\(B_{\text{forced}}\)），剩余预算 \(K = B - B_{\text{forced}}\) 用于蓄水池。
- 对每个候选块 j，独立采样 n 个均匀随机数 \(u_j^{(i)} \sim U(0,1)\)，计算 n 平均优先级：
  \[
  \pi_j = \frac{1}{n} \sum_{i=1}^n (u_j^{(i)})^{1/w_j}
  \]
- 保留 top K 个最高 \(\pi_j\) 的块。\(n=1\) 时为经典加权蓄水池；\(n \to \infty\) 时退化为确定性 top-K。

## 6. 主要结论与发现

1. **理论发现**：
   - 确定性 top-K 的长期存活概率是各步骤指示函数的**最小值乘积**（若有一步低于阈值则立刻取 0），而蓄水池的存活概率是各步骤正包含概率的**代数乘积**（为正值），结构上更适合流式场景。
   - Nexus 行走（迭代自注意力）在期望上将“枢纽块”的分数提升超越直接平均，放大其包含概率（定理 4.2）。

2. **实验发现**：
   - 在 LongBench 上，Nexus Sampling 在 80% 驱逐率下与全注意力精度差**≤1 个点**，在所有模型中平均超越或持平 SnapKV/PyramidKV 等 top-K 基线。
   - 在 RULER 上，优势尤其明显：Llama-3.1-8B 在预填充+解码设置下，Nexus 得分 90.24% vs 次佳 H2O 71.63%（高出 18.6 点），且随上下文长度增长优势保持。
   - 在代理编码（SWE-bench）的“真驱逐”最高难度设置下，Nexus 保持稳定（16% Pass@1），而 PyramidKV 等从 14% 暴跌至 2%。
   - 解码吞吐量比全注意力高 1.2–1.3×，每序列内存节省最高 10×（解码长度 32K 时：0.5 GiB vs 5.0 GiB）。

3. **关键洞察**：当 LLM 从单次提示转向长期代理时，**选择原始（如蓄水池）比评分函数更重要**——已有的评分（直接注意力）足以产生合理权重，但只有能保留边缘质量的原始才匹配流式预算场景。

## 7. 优点

1. **理论贡献深刻**：首次将加权蓄水池采样（流式算法标准）引入 LLM 缓存驱逐，并形式化证明其生存概率乘积 vs top-K 最小值坍缩，为驱逐规则提供了算法理论基础。

3. **桥接令牌发现机制**：Nexus 评分通过短迭代行走捕获跨查询共识，识别出传统直接注意力评分无法看到的簇锚定令牌，这是对“评分”层面的实质性改进。
4. **正交可组合**：提出的评分改进（行走）和选择改进（蓄水池）均可独立与其他方法（如 Ada-KV）组合，实验证明组合后仍领先。

6. **实验设置多样化**：涵盖学术基准（LongBench / RULER）和实用代理基准（SWE-bench），更具说服力。
