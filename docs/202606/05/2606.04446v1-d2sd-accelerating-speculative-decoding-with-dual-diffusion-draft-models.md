## 摘要
推测解码通过草拟多个令牌并在单个目标模型前向传播中验证它们，从而加速自回归大语言模型的推理。最近的基于扩散的草稿模型并行生成整个令牌块，但通常在每次验证中只提交一个草稿序列：一旦出现第一个不匹配，所有后续草稿令牌被丢弃，导致接受率有限。简单地批处理更多草稿候选序列只会带来边际改进，因为冗余或放置不当的分支会增加草稿和验证的成本，而不会按比例增加接受的令牌数量。我们提出D^2SD，一个双扩散草稿推测解码框架，将候选组织成置信度引导的前缀树，其中第一个扩散草稿生成一个令牌块以及每个位置的置信度分数，用于识别最可能的拒绝边界并选择前K个前缀范围进行恢复；第二个可变前缀扩散草稿在每个选定前缀处重新锚定，并在一次批处理传递中提出替代延续；得到的共享前缀候选通过级联注意力共同验证。实验表明，D^2SD在基础扩散方法和强自回归推测解码基线上均有明显改进。

## Abstract
Speculative decoding accelerates autoregressive large language model inference by drafting multiple tokens and verifying them in a single target-model forward pass. Recent diffusion-based drafters generate an entire block of tokens in parallel but usually commit to a single draft sequence per verification: once the first mismatch occurs, all subsequent draft tokens are discarded, resulting in a limited acceptance rate. Naively batching more draft candidate sequences only introduces a marginal improvement, as redundant or poorly placed branches increase the cost of drafting and verification without proportionally increasing the number of accepted tokens.   We propose D^2SD, a dual diffusion draft speculative decoding framework that organizes candidates into a confidence-guided prefix tree, where the first diffusion drafter generates a block along with per-position confidence scores that are used to identify the most likely rejection boundary and select the top-K prefix ranges for recovery; the second variable-prefix diffusion drafter re-anchors at each selected prefix and proposes alternative continuations in one batched pass; the resulting shared-prefix candidates are jointly verified via cascade attention. Empirically, D^2SD shows clear improvements over both the underlying diffusion approach and strong autoregressive speculative decoding baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：推测解码（Speculative Decoding）通过轻量草稿模型快速生成多个候选令牌，再由目标LLM并行验证，以加速自回归推理。但现有扩散草稿模型（如DFlash）一次性生成整个令牌块，一旦早期出现不匹配，后续所有草稿令牌均被丢弃，导致接受率受限；简单地增加块大小或批量生成更多候选（如独立重采样）收效甚微，因为冗余分支增加开销而不成比例提升接受令牌数。
- **动机**：如何将扩散草稿的并行生成优势与树状推测解码的结构化候选组织结合起来，以更高效地利用验证预算，避免早期拒绝造成的浪费。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：构建“双扩散草稿”框架（D²SD）：第一阶段用DFlash生成一个令牌块及每个位置的置信度；基于置信度预测最可能的拒绝边界，选出前K个前缀长度；第二阶段用专门训练的“可变前缀扩散草稿器”（VP-Drafter）在每个选定前缀处重新锚定，并行生成替代后缀；最终共用前缀的候选集通过级联注意力联合验证。
- **关键技术细节**：
  - **置信度估计与拒绝边界预测**：利用草稿器每个位置的概率最大值 \(c_k = \max_v p_k(v)\) 作为置信度，其校准性良好。定义 \(r(i) = \prod_{k=1}^i c_k \cdot (1 - c_{i+1})\) 为恰好接受i个令牌的概率，作为拒绝边界的后验分布。
  - **Top-K前缀选择**：从 \(\{0,...,\gamma-2\}\) 中选择 \(r(i)\) 最大的K个前缀长度，每个选中的前缀保留已生成的令牌并重新屏蔽后续位置，作为VP-Drafter的输入。
  - **VP-Drafter训练**：共享DFlash架构但使用不同的掩码调度——训练时从截断几何分布采样前缀长度 \(l\)，使模型适应任意前缀长度；损失函数采用指数衰减交叉熵，对靠近锚点的位置赋予更高权重（\(w_k = \exp(-(k-l-1)/\tau)\)），强调早期错误的严重性。
  - **端到端推理流程**：① DFlash第一草稿；② Top-K unmask 得到K个变量前缀批次；③ VP-Drafter批量并行生成第二草稿；④ 目标模型通过级联注意力联合验证K+1个候选，提交最长正确前缀。
- **公式与算法**：见上述 \(c_k\)、\(r(i)\) 定义及损失函数 \(L_{VP}\)。具体流程在论文 Section 3 及 Fig. 1 中说明。

## 6. 主要结论与发现

- **核心结论**：置信度引导的共享前缀恢复——而非单纯扩大块大小或均匀重采样——是将并行块扩散草稿与最长正确前缀验证规则结合的正确方法。D²SD在贪心解码下平均加速比达4.98×（Qwen3-8B）和6.15×（GPT-OSS-20B），接受长度分别达7.05和8.02，显著优于DFlash和EAGLE-3。
- **关键发现**：
  - 块大小γ从16增至32时DFlash接受长度反而下降（“缩放墙”），而D²SD通过重新锚定避免此问题。
  - 简单重采样K个分支只能获得微小增益（表5：+0.37 token），而D²SD通过不同归纳偏好的VP-Drafter提升2.25 token。
  - 第三级级联虽提升接受长度（+8%），但壁钟加速比下降（-4%），因为额外开销超过边际收益。

## 7. 优点

- **方法创新**：首次将扩散草稿的并行生成与树状候选组织结合，通过置信度引导的拒绝边界预测实现“定点重锚”，避免了均匀重采样的冗余和自回归树构建的串行开销。
- **训练设计合理**：VP-Drafter使用可变的、靠近锚点加权的训练损失，使其能有效地从任意前缀长度开始生成，与推理时需求匹配。

- **实验全面**：在8个数据集、两个目标模型、两种解码模式下验证，消融实验深入剖析设计选择，结论可靠。
