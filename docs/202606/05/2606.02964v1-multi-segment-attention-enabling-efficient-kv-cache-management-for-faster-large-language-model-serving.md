## 摘要
大型语言模型（LLM）推理依赖于键值（KV）缓存以避免冗余的注意力计算。虽然近似KV缓存保留技术通过牺牲模型精度来减少内存使用，但无损方法则从GPU内存中逐出KV缓存块，并在需要时按需重建以保持精确输出。现有的无损KV缓存管理系统主要基于访问频率或位置启发式来做出逐出决策，而未考虑不同KV缓存块如何影响GPU注意力内核的执行效率。在本文中，我们提出了AsymCache，一种面向LLM推理的计算延迟感知KV缓存管理系统，它明确地将缓存驻留决策与GPU注意力内核性能对齐，包括三个关键组件：用于高效非连续KV上下文处理的多段注意力（MSA）、联合优化命中率和位置感知重计算成本的缓存逐出策略，以及用于高硬件利用率的自适应分块调度器。实验表明，与最新基线相比，AsymCache将TTFT减少了最高1.90-2.03倍，每个输出token的时间（TPOT）减少了1.62-1.71倍，证实了该方法在常见工作负载中的有效性，并验证了其在计算效率与缓存命中率之间平衡的设计目标。此外，AsymCache的低层设计允许无缝集成到诸如Continuum等代理服务系统中，进一步将平均作业延迟降低高达18.1%。

## Abstract
Large Language Model (LLM) inference relies on key-value (KV) caches to avoid redundant attention computation. While approximate KV cache retention techniques reduce memory usage by sacrificing model accuracy, lossless approaches instead evict KV cache blocks from GPU memory and reconstruct them on demand to preserve exact outputs. Existing lossless KV cache management systems primarily base eviction decisions on access frequency or positional heuristics, without considering how different KV cache blocks affect the execution efficiency of GPU attention kernels.   In this paper, we propose AsymCache, a computation-latency-aware KV cache management system for LLM inference that explicitly aligns cache residency decisions with GPU attention kernel performance, including three key components: Multi-Segment Attention (MSA) for efficient non-contiguous KV context processing, a cache eviction policy that jointly optimizes hit rate and position-aware recomputation cost, and an adaptive chunking scheduler for high hardware utilization. Experiments show that AsymCache reduces TTFT by up to 1.90-2.03x and time-per-output-token (TPOT) by 1.62-1.71x over latest baselines, confirming the effectiveness of the method in common workloads and validating its design goal of balancing computational efficiency with cache hit rate. Moreover, the low-level design of AsymCache allows seamless integration into agent serving systems such as Continuum, where it further reduces average job latency by up to 18.1%.

## 1. 核心问题与整体含义（研究动机和背景）

大型语言模型（LLM）推理依赖键值（KV）缓存来避免重复的注意力计算。KV缓存占用了大量GPU显存，成为制约并发请求数和序列长度的主要瓶颈。为了缓解显存压力，研究者提出了两类方案：近似方案（丢弃或压缩“不重要”的KV状态）会牺牲模型精度；无损方案则从GPU显存中临时逐出KV缓存块，并在需要时按需重建（通过重计算或从主机内存交换），从而保持精确输出。

现有无损KV缓存管理系统（如vLLM、SGLang、Pensieve）的逐出决策主要基于访问频率（LRU/LFU）或位置启发式（优先保留前缀或后缀），但未考虑不同KV缓存块对GPU注意力内核执行效率的差异化影响——忽略了块级异质性（如内存访问模式、线程块利用率等），导致次优的逐出决策，未能充分挖掘GPU性能潜力。本文首次提出**计算延迟感知**的KV缓存管理思路，显式对齐缓存驻留决策与GPU注意力内核性能。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
AsymCache 通过三个紧密集成的组件实现“期望延迟感知”的KV缓存管理：

1. **多段注意力（Multi-Segment Attention，MSA）**：一种新型注意力内核，将注意力计算分解为多个非连续段，支持对非连续KV上下文的细粒度控制。MSA将不同请求的不同缓存段分派到独立的CTA组，在一个内核调用中并行计算多个段的注意力，避免多次内核启动开销。

2. **计算感知块逐出器（Computational-Aware Block Evictor）**：基于每个缓存块的边际贡献（期望注意力延迟）进行优先级排序。逐出标准为：
   - \( \text{block\_id} \leftarrow \arg\min_B f_B(t) \cdot \Delta T_B \)
   - \( f_B(t) \)：指数衰减的分段函数（高频使用期平缓衰减，之后快速衰减），满足“保序规则”（仅指数函数满足），从而可使用平衡树实现 \(O(\log n)\) 时间复杂度的插入、删除和最小值查找。
   - \( \Delta T_B \)：块的重计算成本，通过线性模型近似（关注块的位置索引），可在常数时间获取。
   - 联合优化了缓存命中率（\( f_B(t) \)）与位置感知重计算成本（\( \Delta T_B \)），自动识别工作负载的复用模式（前缀共享 vs 后缀高成本）。

3. **分块调度器（Chunking Scheduler）**：采用自适应分块预填（chunked-prefill），支持分块跨越多个缓存段和高低负载动态调整块大小，平衡重计算开销和GPU内核效率。

### 关键技术细节
- 线性成本模型：\( T = k_1 l_1 + k_2 q_1 + k_3 l_2 + k_4 q_2 + k_5 (l_1+q_1)^2 + k_6 q_2(l_1+q_1+l_2+q_2) + \beta \)，通过约1100次实际性能剖析拟合，\(R^2\)均>0.999。
- 逐出算法基于两棵平衡树（对应分段函数的两段），通过可在线调整的系数\(\lambda\)适应工作负载寿命特征变化。
- 多段注意力内核基于CUDA和CUTLASS，扩展FlashAttention/PagedAttention，支持任意数量非连续缓存段。

## 6. 主要结论与发现

1. AsymCache在几乎所有配置下均优于基线：TTFT降低最高1.90–2.03×，TPOT降低1.62–1.71×。
2. 缓存命中率并非唯一决定因素：AsymCache在保持相似或略低命中率时，因保留高重计算成本的后缀块而获得显著延迟优势。
3. 分段指数频率函数和\(O(\log n)\)算法在实际中至关重要：线性时间算法在高缓存容量下会成为瓶颈（控制平面时间约500秒 vs 对数算法约5秒）。
4. MSA内核消除了多段注意力中多次内核调用的开销，使后缀缓存甚至适用于短序列场景。
5. 超参数敏感性低：在合理范围内（寿命、复用概率、斜率变化比）性能稳定。
6. AsymCache可无缝集成到代理系统Continuum中，平均作业延迟额外降低4.4%–18.1%，P90降低6.8%–16.6%，表明其正交性和广泛适用性。
