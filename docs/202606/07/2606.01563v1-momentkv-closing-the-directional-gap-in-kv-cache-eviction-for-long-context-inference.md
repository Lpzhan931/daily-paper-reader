## 摘要
自动回归解码在基于Transformer的语言模型中依赖于KV缓存，其内存占用随序列长度线性增长，成为长上下文推理的主要瓶颈。KV缓存驱逐通过保留固定大小的键值对子集并丢弃其余部分来解决此问题。我们发现输出退化的主要来源不是被驱逐令牌上的残余注意力质量（现有方法已最小化此问题），而是保留令牌集与驱逐令牌集之间的方向性不匹配。具体而言，实践中被驱逐的令牌通常与保留的令牌近乎正交。因此，即使少量的被驱逐质量也可能对最终的方向分布产生超大影响，并放大为显著的输出误差。这揭示了现有策略的根本局限性。为解决此问题，我们提出了MomentKV，它在被驱逐令牌集上维护紧凑的小尺寸矩统计量，包括计数、键均值、值均值以及值-键协方差。在驱逐过程中，利用矩统计量来识别已经与累积摘要良好对齐并被捕获的令牌，使得被驱逐集在几何上保持规则。在推理过程中，它们产生了被驱逐注意力输出的闭式一阶近似，在选择性驱逐和精确校正之间形成相互增强的循环。在LongBench和RULER上使用LLaMA-3.1-8B-Instruct和Qwen3-4B-Instruct进行的实验中，MomentKV在每个缓存预算下都优于所有基线，在激进压缩下取得了最大增益。

## Abstract
Autoregressive decoding in Transformer-based language models relies on the KV cache, whose memory footprint grows linearly with sequence length and becomes the primary bottleneck for long-context inference. KV cache eviction addresses this by retaining a fixed-size subset of key-value pairs and discarding the rest. We identify that a primary source of output degradation is not the residual attention mass on evicted tokens, which existing methods already minimize, but a directional mismatch between the retained and evicted token sets. Specifically, the evicted tokens in practice are often near-orthogonal to the retained ones. Thus, even a small evicted mass could have an oversized impact on the resulting direction distribution and amplify into substantial output error. This reveals a fundamental limit in existing strategies. To address this, we propose MomentKV, which maintains compact, small-size moment statistics over the evicted token set, including a count, key mean, value mean, and value-key covariance. During eviction, the moment statistics is leveraged to identify tokens already well aligned with and captured by the accumulated summary, keeping the evicted set geometrically regular. During inference, they yield a closed-form first-order approximation of the evicted attention output, forming a mutually reinforcing loop between selective eviction and accurate correction. On LongBench and RULER with LLaMA-3.1-8B-Instruct and Qwen3-4B-Instruct, MomentKV outperforms all baselines at every cache budget, with the largest gains under aggressive compression.

# 论文总结：MomentKV：弥合长上下文推理中KV缓存驱逐的方向性差距

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：Transformer 语言模型使用 KV 缓存存储已计算的键值对，序列长度增加导致内存爆炸；KV 缓存驱逐（保留固定大小子集）是主流压缩手段。现有方法重点优化 token 选择（如累积注意力权重、预填充观察窗口、逐层预算分配等），使得保留集能抓住大部分注意力质量。
- **核心问题**：作者发现输出退化的首要原因不是被驱逐 token 的残差注意力质量（已最小化），而是**保留集与驱逐集之间的方向性不匹配**——实践中两者几乎正交（余弦相似度≈0.1，夹角约84°）。这导致即使驱逐质量很小（<8%），也会通过正交放大（ γ(θ) ~ O(1) ）产生超过20%的相对误差。传统重归一化无法恢复正交方向的损失。
- **整体含义**：揭示了现有挑选策略的固有限制——更严格的挑选虽降低驱逐质量，却加剧方向性差距（保留集更集中，驱逐集更互补），导致收益递减。因此改进路径应从仅优化选择转向恢复丢失的方向信息。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：维护被驱逐 token 集的紧凑矩统计量（计数、键均值、值均值、值-键协方差），用于两处：①在驱逐时识别与累积摘要高度对齐的 token（残差小），优先驱逐它们，使驱逐集保持几何规则；②在推理时提供闭式一阶近似，校正重归一化后的输出。
- **关键技术细节**：
  - **矩统计量**：对驱逐集 E，维护 \(n_e, s_k = \sum k_i, s_v = \sum v_i, S = \sum v_i k_i^\top\)。查询时恢复 \(\bar{k} = s_k / n_e\)，\(\bar{v} = s_v / n_e\)，\(\tilde{S} = S - s_v s_k^\top / n_e\)（即协方差）。
  - **一阶近似**：利用 softmax 平移不变性和泰勒展开 \(e^{\delta_i} \approx 1+\delta_i\)，导出 \(\hat{f}_E(q) = \bar{v} + \frac{1}{n_e\sqrt{d}} \tilde{S} q\)。该近似误差为 \(O(\sigma^2)\)，\(\sigma = \max |\delta_i|\)。零均值性质使分母精确为 \(n_e\)。
  - **矩感知驱逐**：对保留 token j，计算残差 \(r_j = v_j - \bar{v} - \frac{1}{n_e\sqrt{d}} \tilde{S} k_j\)，驱逐分数 = \(\alpha_j \cdot \|r_j\|\)（α为注意力权重）。低分 token 被优先驱逐（高注意力且小残差→安全；低注意力且大残差→保留）。
  - **校正推理**：用 Jensen 下界 \(\hat{Z}_E = n_e \exp(q^\top \bar{k}/\sqrt{d})\) 近似驱逐集配分函数，混合权重 \(\hat{w}_R = Z_R / (Z_R + \hat{Z}_E)\)，最终输出 \(\hat{f} = \hat{w}_R f_R + (1-\hat{w}_R) \hat{f}_E\)。该机制自调节：当 σ 大时 \(\hat{Z}_E\) 低估严重，自动压缩校正影响，退化为标准驱逐。
  - **存储开销**：每头 \(O(d^2)\)，与上下文长度无关。d=128时约32KB/头，总开销约6-7%的缓存容量（L=128时），且随 L 增大占比下降。

- MomentKV 在所有缓存预算下均取得最高平均分数，在激进压缩（L=128）时优势最大：在 LLaMA-3.1-8B 上超出最强基线 Ada-KV 达 +1.35，恢复至全缓存性能的 94.1%。

- 误差归因：方向性差距而非驱逐质量是主要误差源，且随着预算增大，MomentKV 的恢复百分比（Recovery）从 31.7% 上升到 59.6%，证实了累积校正的有效性。

## 8. 不足与局限

- **近似误差条件**：一阶近似质量依赖于驱逐集键值相对紧凑（σ 小），虽然在矩感知驱逐下得到保证，但极端分布（如巨大键值偏移）可能降低有效性。
- **固定 d 假设**：推导假设头维度 d 在标准范围内（如128），对于非常小或非常大的 d 可能需调整泰勒展开项的权重；实验仅验证两种模型，未测试非标准架构。
- **数值稳定性**：半精度累积协方差可能发生灾难性抵消，论文虽提出裁剪阈值和日志域计算，但未在更低位宽（如4-bit）下验证。
- **未提及扩展性**：没有探讨与量化、低秩分解等其他压缩技术的联合优化。
- **实验覆盖**：RULER 仅测试了 L=128，未展示更大预算下的 RULER 表现；未评估极端长上下文（超过模型原生长度）或跨语言场景。
- **理论界不够紧**：虽然给出 O(σ²) 误差界，但常数项依赖于值向量最大范数和键偏离，在极端情形下可能较大；缺乏更精细的 Lipschitz 型分析。
