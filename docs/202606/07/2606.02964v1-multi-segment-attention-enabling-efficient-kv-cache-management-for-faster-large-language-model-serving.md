## 摘要
大型语言模型（LLM）推理依赖键值（KV）缓存来避免冗余的注意力计算。虽然近似的KV缓存保留技术通过牺牲模型精度来减少内存使用，但无损方法则从GPU内存中驱逐KV缓存块，并在需要时重新构建它们以保留精确输出。现有的无损KV缓存管理系统主要基于访问频率或位置启发式进行驱逐决策，而未考虑不同KV缓存块如何影响GPU注意力内核的执行效率。在本文中，我们提出AsymCache，一种面向LLM推理的计算延迟感知KV缓存管理系统，它明确地将缓存驻留决策与GPU注意力内核性能对齐，包括三个关键组件：用于高效非连续KV上下文处理的多段注意力（MSA）、联合优化命中率和位置感知重计算成本的缓存驱逐策略，以及用于高硬件利用率的自适应分块调度器。实验表明，与最新基线相比，AsymCache将TTFT降低高达1.90-2.03倍，每输出令牌时间（TPOT）降低1.62-1.71倍，证实了该方法在常见工作负载中的有效性，并验证了其平衡计算效率与缓存命中率的设计目标。此外，AsymCache的低级设计允许无缝集成到诸如Continuum等代理服务系统中，进一步将平均作业延迟降低高达18.1%。

## Abstract
Large Language Model (LLM) inference relies on key-value (KV) caches to avoid redundant attention computation. While approximate KV cache retention techniques reduce memory usage by sacrificing model accuracy, lossless approaches instead evict KV cache blocks from GPU memory and reconstruct them on demand to preserve exact outputs. Existing lossless KV cache management systems primarily base eviction decisions on access frequency or positional heuristics, without considering how different KV cache blocks affect the execution efficiency of GPU attention kernels.   In this paper, we propose AsymCache, a computation-latency-aware KV cache management system for LLM inference that explicitly aligns cache residency decisions with GPU attention kernel performance, including three key components: Multi-Segment Attention (MSA) for efficient non-contiguous KV context processing, a cache eviction policy that jointly optimizes hit rate and position-aware recomputation cost, and an adaptive chunking scheduler for high hardware utilization. Experiments show that AsymCache reduces TTFT by up to 1.90-2.03x and time-per-output-token (TPOT) by 1.62-1.71x over latest baselines, confirming the effectiveness of the method in common workloads and validating its design goal of balancing computational efficiency with cache hit rate. Moreover, the low-level design of AsymCache allows seamless integration into agent serving systems such as Continuum, where it further reduces average job latency by up to 18.1%.

## 1. 论文的核心问题与整体含义（研究动机和背景）
大型语言模型（LLM）推理过程中需要维护键值（KV）缓存来存储历史注意力状态，避免冗余计算。然而，KV 缓存占用巨大的 GPU 内存（例如，70B 模型、32K 上下文需超过 40 GB），成为限制并发请求数和吞吐量的主要瓶颈。现有无损 KV 缓存管理方案（如 vLLM、SGLang、Pensieve）主要基于访问频率或位置启发式进行驱逐，但忽略了不同 KV 缓存块对 GPU 注意力核执行效率的异构影响，导致次优的驱逐决策。本文提出 AsymCache，首次将 KV 缓存驻留决策显式对齐 GPU 注意力核性能，实现计算延迟感知的缓存管理。

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想
利用 KV 缓存块在 GPU 注意力计算中的不对称性（位置靠后的块重计算代价更高），在驱逐决策中同时考虑未来命中概率和位置感知的重计算成本，从而选择“期望计算延迟最小”的块进行驱逐。

### 关键技术细节
1. **Multi-Segment Attention (MSA)**  
   - 支持非连续 KV 上下文的高效注意力计算。通过单个 CUDA 核融合处理多个不连续段（如一个前缀缓存段加一个后缀缓存段），避免多个核调用开销。
   - 每个请求的多个段被分配到不同的 CTA 组并行计算，通过预计算的段长度数组获取每个块对应的实际序列长度。

2. **计算感知的块驱逐策略**  
   - 提出基于期望延迟的驱逐准则：`block_id = argmin_B f_B(t) * ΔT_B`，其中：
     - `f_B(t)`：块 B 在时间 t 的历史访问频率（带指数衰减），反映未来命中概率。
     - `ΔT_B`：若该块被驱逐，后续预填充时的重计算代价增量（通过线性成本模型近似，主要取决于块的位置序号，即前面已有令牌数）。
   - **成本模型**：线性模型估计预填充延迟，包括 MLP/LayerNorm 的线性部分和注意力模块的二次部分；通过约 1.1K 次实际性能样本拟合，R²>0.999。
   - **频率函数设计**：为避免线性时间复杂度，采用分段指数函数：`f_B(t) = min(exp(-τ_B(t)/α), exp(-(τ_B(t)-τ₀)/β))`。该函数满足“顺序保持规则”（仅指数函数满足），允许使用两个平衡树实现 O(log n) 的插入、删除、最小查找。
   - 在线调整参数 λ 可使分段函数的转向点（lifespan）自适应工作负载变化。

3. **自适应分块调度器**  
   - 采用 chunked-prefill 思路，将长预填充请求切分为小块，避免头部阻塞。MSA 内核原生支持跨多个计算/缓存段的分块，调度器可动态调整块大小以平衡重计算开销和 GPU 利用率。

### 算法流程（文字说明）
- 每个缓存块维护频率值（基于分段指数函数）和重计算代价。
- 当新请求到来且 GPU 内存不足时，Block Evictor 从两个平衡树中分别获取当前最小权重的块，按加权比较决定驱逐对象（算法1）。
- 驱逐后，被驱逐块若后续被命中，则需通过 MSA 内核（可能涉及多个缓存段）重新计算，但输出与不驱逐时完全一致（无损）。

## 3. 实验设计：数据集、场景、基准、对比方法
### 数据集

### 场景

### 对比方法
- **vLLM-LRU**：经典 LRU 驱逐。

- 所有系统统一集成 POD-Attention 核作为后端。

### 主要指标
- 平均 TTFT（Time-to-First-Token）、平均 TPOT（Time-Per-Output-Token）、缓存命中率（请求级和块级）。

6. 与现有代理系统 Continuum 集成后，平均作业延迟降低 18.1%，P90 降低 16.6%，表明 AsymCache 的块级优化正交且互补。
