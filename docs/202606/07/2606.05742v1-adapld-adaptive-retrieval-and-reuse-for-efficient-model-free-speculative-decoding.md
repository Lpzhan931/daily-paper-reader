## 摘要
推测解码通过在单次目标模型前向传播中验证多个草稿token来加速生成，减少顺序解码迭代次数。无模型变体通过复用生成过程中已有的文本和模型状态来避免辅助草稿模型，但其加速效果取决于所构建草稿的可靠性。我们指出现有基于复用的方法存在两个局限：基于词汇锚定的检索在表面形式变化下召回率有限，且当检索到的上下文无法唯一确定后续内容时，确定性跨度复制可能变得脆弱。我们提出AdaPLD，一种无需训练的方法，能够自适应地改进检索和草稿构建。AdaPLD在保留高精度词汇复用的同时，利用语义相似度在词汇匹配失败时恢复额外的复用机会。它进一步构建分支复用假设来应对后续内容的不确定性，而非依赖单一复制跨度。在多个基准测试中，AdaPLD减少了目标模型前向传播次数，并实现了高达3.10倍的解码加速。

## Abstract
Speculative decoding accelerates generation by verifying multiple drafted tokens in a single target-model forward pass, reducing sequential decoding iterations. Model-free variants avoid auxiliary draft models by reusing text and model states already available during generation, but their speedup depends on the reliability of the constructed drafts. We identify two limitations of existing reuse-based methods: lexically anchored retrieval has limited recall under surface-form variation, and deterministic span copying can be brittle when the retrieved context does not uniquely determine the continuation. We propose \emph{AdaPLD}, a training-free method that adaptively improves both retrieval and draft construction. AdaPLD preserves high-precision lexical reuse while using semantic similarity to recover additional reuse opportunities when lexical matching fails. It further constructs branched reuse hypotheses to account for continuation uncertainty, rather than relying on a single copied span. Across diverse benchmarks, AdaPLD reduces target-model forward passes and achieves up to $3.10\times$ decoding speedup.

# AdaPLD：高效无模型推测解码的自适应检索与复用 —— 论文详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大型语言模型（LLM）的自回归解码存在严格的顺序瓶颈，每生成一个 token 都需要一次前向传播。推测解码（Speculative Decoding）通过并行验证多个草稿 token 来减少串行迭代次数。其中，**无模型推测解码**（Model-Free SD）无需额外的辅助草稿模型，而是复用生成过程中已有的文本和模型状态来构建草稿，因此更具实用性。
- **核心问题**：现有基于复用的方法（如 PLD、PLD+）存在两大局限性：
  1. **检索召回率低**：依赖精确的词汇匹配（如 n-gram）检索可复用的上下文位置，当表面形式（词汇）不同但语义相关时，会漏掉有用的复用候选，导致“无命中”失败模式。
  2. **草稿构建脆弱**：找到复用锚点后，往往机械地复制该锚点之后的一段连续文本作为唯一草稿。当上下文存在多种合理续接时，这种确定性复制会错过其他可行路径。
- **研究意义**：克服上述瓶颈，在不损失精度的情况下提升复用覆盖率，并应对续接不确定性，从而进一步提升无模型推测解码的效率。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：AdaPLD 是一种**无需训练**的自适应方法，同时改进检索和草稿构建两个阶段。
  - **自适应检索**：默认使用精确词汇匹配（高精度），当词汇匹配无候选时，**语义回退**激活，利用 token 嵌入余弦相似度检索语义相近的位置，扩大候选可达性。
  - **自适应草稿构建**：选定锚点后，不再只复制单一跨度，而是构建一个**草稿树**，包含：
    - **主复制路径**（仅当词汇匹配成功时）：从锚点后复制一段连续 token。
    - **分支路径**：从锚点条件分布中取 top-K 个 token 作为替代即时续接。
    - **后继草拟**（Successor Drafting）：对每个分支 token，再次执行词汇优先检索，找到其后继 token，扩展分支长度（最多额外一个 token）。

- **关键技术细节**：
  1. **检索阶段**：
     - 查询锚点 = 上一生成的 token `a_i = y_{i-1}`。
     - **精确匹配**：`C_lex = {j | M_{i,j} = a_i}`。
     - **语义回退**：若 `C_lex` 为空，则计算 `a_i` 与内存中各位置的 token 嵌入余弦相似度，高于阈值 `τ` 的加入 `C_sem`。
     - 最终候选集 `C_i`：优先使用 `C_lex`，否则使用 `C_sem`。
  2. **候选选择**：通过**隐藏状态重排序**，比较每个候选锚点前缀与当前前缀的隐藏表示（某层）的余弦相似度，选择最高分锚点。
  3. **草稿构建**：
     - 若来自精确匹配（`C_lex ≠ ∅`），则构建主路径 `ŷ_main`（长度不超过 `ℓ_m`），并生成分支：从锚点条件分布取 top-K 后排除主路径第一个 token。
     - 若来自语义命中，则省略主路径，直接从锚点条件分布取 top-K 分支。
     - 对每个分支 token `b`，进行后继草拟：以 `b` 为新查询锚点，执行词汇优先检索，选出最佳后继位置，复制其后一个 token 作为扩展。
  4. **验证**：将所有草稿路径组织成草稿树，使用**树注意力**在单次前向传播中并行验证，采用标准推测解码接受-拒绝规则，保持无损性。
