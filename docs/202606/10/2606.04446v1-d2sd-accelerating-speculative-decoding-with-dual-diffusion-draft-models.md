## 摘要
推测解码通过草拟多个标记并在一次目标模型前向传播中验证它们，从而加速自回归大型语言模型的推理。最近的基于扩散的草稿器可以并行生成整个标记块，但通常每次验证只提交一个草稿序列：一旦出现第一个不匹配，所有后续草稿标记都会被丢弃，导致接受率有限。简单地将更多草稿候选序列批处理只会带来微小的改进，因为冗余或位置不当的分支会增加草拟和验证的成本，而不会成比例地增加接受的标记数量。我们提出了D^2SD，一种双扩散草稿推测解码框架，将候选组织成一个置信度引导的前缀树，其中第一个扩散草稿器生成一个块以及每个位置的置信度分数，这些分数用于识别最可能的拒绝边界，并选择前K个前缀范围进行恢复；第二个可变前缀扩散草稿器在每个选定的前缀处重新锚定，并在一次批处理中提出替代延续；生成的共享前缀候选通过级联注意力进行联合验证。实验表明，D^2SD在基础扩散方法和强自回归推测解码基线上都有明显改进。

## Abstract
Speculative decoding accelerates autoregressive large language model inference by drafting multiple tokens and verifying them in a single target-model forward pass. Recent diffusion-based drafters generate an entire block of tokens in parallel but usually commit to a single draft sequence per verification: once the first mismatch occurs, all subsequent draft tokens are discarded, resulting in a limited acceptance rate. Naively batching more draft candidate sequences only introduces a marginal improvement, as redundant or poorly placed branches increase the cost of drafting and verification without proportionally increasing the number of accepted tokens.   We propose D^2SD, a dual diffusion draft speculative decoding framework that organizes candidates into a confidence-guided prefix tree, where the first diffusion drafter generates a block along with per-position confidence scores that are used to identify the most likely rejection boundary and select the top-K prefix ranges for recovery; the second variable-prefix diffusion drafter re-anchors at each selected prefix and proposes alternative continuations in one batched pass; the resulting shared-prefix candidates are jointly verified via cascade attention. Empirically, D^2SD shows clear improvements over both the underlying diffusion approach and strong autoregressive speculative decoding baselines.

- **研究问题**：推测解码（Speculative Decoding）通过一个轻量级草稿模型快速生成多个候选标记，再由目标模型并行验证，从而加速自回归大语言模型（LLM）推理。然而，现有的基于扩散的草稿器（如 DFlash）虽然能一次性并行生成整个标记块，但每次验证只提交一个草稿序列。按照“最长正确前缀”规则，一旦序列中较早位置出现不匹配，后续所有令牌都会被丢弃，导致接受率受限，形成“扩展墙”（scaling wall）：增大块大小 γ 带来的收益在超过一定阈值后递减甚至变为负值。
- **瓶颈**：简单地增加候选分支数量（如从同一扩散模型重新采样 K 条序列）效果甚微，因为所有候选共享相同的概率分布，容易在相同困难边界上集体失败（误差同质性）。
- **整体含义**：本文旨在通过双扩散草稿框架（D²SD），将单次扩散草稿转化为结构化的共享前缀候选集，在保持并行草稿低延迟的同时，显著提高接受长度 α，从而获得更大的端到端加速。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：利用第一个扩散草稿器（DFlash）生成的块令牌及其逐位置置信度分数，估计最可能的拒绝边界，然后由第二个专门训练的可变前缀扩散草稿器（VP-Drafter）在这些边界处重新锚定并生成多种延续，最终通过级联注意力联合验证所有候选。
- **关键技术细节**：
  1. **第一阶段：DFlash 草稿**  
     与标准 DFlash 相同，一次并行生成 γ 个令牌（位置 0 为锚点，其余为 [MASK]），同时输出每个位置 k 的置信度 \( c_k = \max_v p_k(v) \)，即贪婪采样令牌的预测概率。
  2. **估计拒绝边界**  
     利用置信度计算每个前缀长度 i 的似然：  
     \[ r(i) = \left(\prod_{k=1}^{i} c_k\right) \cdot (1 - c_{i+1}) \]  
     表示恰好接受前 i 个令牌、位置 i+1 被拒绝的概率。
  3. **Top-K 共享前缀分支选择**  
     选取 \( r(i) \) 最大的前 K 个前缀长度 \( S \)，每个 i ∈ S 保留锚点和前 i 个 DFlash 令牌作为可见前缀，其余位置重新掩码，构成 K 个变长前缀输入。
  4. **第二阶段：VP-Drafter 批量生成**  
     该草稿器与 DFlash 共享架构但经过特殊训练，支持任意前缀长度。它将所有 K 个输入组成一个批次，一次前向传播同时生成 K 条替代延续（第二个草稿）。
  5. **联合验证与接受**  
     目标模型采用级联注意力（cascade attention），同时验证原 DFlash 分支和 K 个 VP-Drafter 分支（共 K+1 条候选），选择最长接受前缀，并将目标模型在拒绝位置的预测作为下一轮的锚点。
- **训练 VP-Drafter**：
  - 在训练时，从截断几何分布 \( \Pr(l=j) \propto \beta^j \) 采样前缀长度 l（偏向短前缀），用真实令牌填充前 l+1 个位置，其余掩码。
  - 损失函数为指数衰减交叉熵：  
    \[ L_{VP} = -\frac{\sum_{k=l+1}^{\gamma-1} w_k \log p_k(t_k^\star)}{\sum_{k=l+1}^{\gamma-1} w_k}, \quad w_k = \exp\left(-\frac{k-l-1}{\tau}\right) \]  
    权重随距离锚点增加而指数衰减，强调靠近锚点的位置更关键。

### 6. 论文的主要结论与发现

- D²SD 在所有数据集上均一致优于 DFlash 和 EAGLE-3，特别是在数学和代码任务上提升显著（接受长度增加 1–2 个令牌，加速比提升约 20%–50%）。
- GPT-OSS-20B 上的改进幅度更大（例如 GSM8K 上加速比从 DFlash 的 2.83× 提升至 6.20×），因为更大模型更受益于接受长度的增加。
- 简单的重采样或复用 DFlash 作为第二草稿收益有限，证明了专用 VP-Drafter 和置信度引导分支选择的必要性。
- 增加第三阶段草稿虽然能小幅提高接受长度，但额外开销导致加速比下降，说明两级级联是平衡点。
- 置信度引导的共享前缀恢复比块扩展或均匀重采样更有效。

### 7. 优点
