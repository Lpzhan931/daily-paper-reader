## 摘要
大语言模型（LLM）推理依赖键值（KV）缓存以避免冗余的注意力计算。近似KV缓存保留技术通过牺牲模型精度来减少内存使用，而无损方法则从GPU内存中驱逐KV缓存块并按需重建以保留精确输出。现有的无损KV缓存管理系统主要基于访问频率或位置启发式做出驱逐决策，而未考虑不同KV缓存块如何影响GPU注意力核的执行效率。本文提出AsymCache，一种面向LLM推理的计算延迟感知KV缓存管理系统，它明确地将缓存驻留决策与GPU注意力核性能对齐，包括三个关键组件：用于高效非连续KV上下文处理的多段注意力（MSA）、联合优化命中率和位置感知重计算成本的缓存驱逐策略、以及用于高硬件利用率的自适应分块调度器。实验表明，与最新基线相比，AsymCache将TTFT最高提升1.90-2.03倍，每输出令牌时间（TPOT）提升1.62-1.71倍，验证了该方法在常见工作负载下的有效性及其在平衡计算效率与缓存命中率方面的设计目标。此外，AsymCache的低层设计允许无缝集成到诸如Continuum之类的智能体服务系统中，进一步将平均作业延迟降低高达18.1%。

## Abstract
Large Language Model (LLM) inference relies on key-value (KV) caches to avoid redundant attention computation. While approximate KV cache retention techniques reduce memory usage by sacrificing model accuracy, lossless approaches instead evict KV cache blocks from GPU memory and reconstruct them on demand to preserve exact outputs. Existing lossless KV cache management systems primarily base eviction decisions on access frequency or positional heuristics, without considering how different KV cache blocks affect the execution efficiency of GPU attention kernels.   In this paper, we propose AsymCache, a computation-latency-aware KV cache management system for LLM inference that explicitly aligns cache residency decisions with GPU attention kernel performance, including three key components: Multi-Segment Attention (MSA) for efficient non-contiguous KV context processing, a cache eviction policy that jointly optimizes hit rate and position-aware recomputation cost, and an adaptive chunking scheduler for high hardware utilization. Experiments show that AsymCache reduces TTFT by up to 1.90-2.03x and time-per-output-token (TPOT) by 1.62-1.71x over latest baselines, confirming the effectiveness of the method in common workloads and validating its design goal of balancing computational efficiency with cache hit rate. Moreover, the low-level design of AsymCache allows seamless integration into agent serving systems such as Continuum, where it further reduces average job latency by up to 18.1%.

## 一、核心问题与整体含义（研究动机和背景）

- **问题**：大语言模型（LLM）推理中，KV缓存是加速自回归生成的关键，但其内存消耗巨大（例如70B模型单次32K上下文需超40GB）。无损KV缓存管理通过驱逐部分块并按需重计算来节约GPU内存，但现有方法仅基于访问频率（如LRU）或位置启发式（如Pensieve优先保留尾部）做出驱逐决策，**忽略了不同KV块对GPU注意力核执行效率的差异性**（即重计算成本不对称：尾部块的重计算成本远高于前缀块）。
- **核心挑战**：如何联合优化缓存命中率与重计算成本，使驱逐决策真正对齐GPU计算延迟。
- **论文目标**：提出**AsymCache**，一个计算延迟感知的KV缓存管理系统，显式地将块驻留决策与GPU注意力核性能对齐，在保证无损输出的前提下提升推理速度。

## 二、方法论：核心思想、关键技术细节

### 核心思想

AsymCache基于“块对预期注意力延迟的边际贡献”来确定驱逐优先级，平衡块的未来重用概率与重计算成本。

### 三大组件

1. **多段注意力（Multi-Segment Attention, MSA）**
   - 支持非连续KV上下文（如同时缓存前缀和后缀），将多个段的注意力计算融合到单个GPU核内，通过CTA（线程束）级分组并行处理不同请求/段，避免多次核启动开销。

2. **计算感知块驱逐器（Computational-Aware Block Evictor）**
   - **驱逐准则**：选择`f_B(t) · ΔT_B`最小的块驱逐，其中`f_B(t)`为频率值（指数衰减历史访问），`ΔT_B`为重计算成本（通过线性模型近似，仅依赖于块位置 `l_1+q_1`）。
   - **频率函数设计**：满足“顺序保持规则”（两个块在任何时刻权重序不变），只有指数函数满足该性质。采用**分段指数函数**：第一段保持高重用概率（寿命内），第二段快速衰减（寿命后）。使用两个平衡树分别维护两段权重，实现**O(log n)** 的插入、删除和最小值查询。
   - **重计算成本模型**：近似为`ΔT_B ≈ 2k_5·(l_1+q_1) + (k_2-k_3+k_5)`，仅需常数时间获取前导块数，经1.1K profiling数据拟合，R²>0.999。

3. **自适应分块调度器（Chunking Scheduler）**
   - 采用分块预填（chunked prefill），但允许一个分块跨越计算段和缓存段（即包含非连续上下文），MSA原生支持这种混合分块。当解码请求数量多时，减小预填分块大小以降低TPOT，同时设置下界避免GPU低利用率。

### 算法流程（文字说明）

1. 新请求到达，API服务器拼接历史上下文等。
2. 调度器检查缓存命中状态，可能触发驱逐：调用Evict函数，从两个平衡树中取最小权重块，比较两段结果（考虑可调系数λ），驱逐一方。
3. 请求与运行中请求组成batch，Chunking Scheduler为每个预填请求分配分块大小，分块可能跨非连续段。
4. 使用MSA核执行batch，每个请求的不同段在单个核内并行计算。
5. 推进到下一调度步。

1. AsymCache在所有配置下显著优于vLLM-LRU、Max-Score、Pensieve+MSA：
