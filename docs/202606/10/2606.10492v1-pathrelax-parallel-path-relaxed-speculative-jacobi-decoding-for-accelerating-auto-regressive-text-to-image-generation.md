## 摘要
自回归文本到图像模型中对高分辨率图像生成的需求日益增长，导致令牌序列延长，显著增加了计算成本和推理时间。然而，现有加速自回归文本到图像模型的最先进方法依赖于链式结构的草稿令牌序列，导致草稿令牌搜索效率低下且接受长度有限。为了解决这一问题，我们提出并行路径交叉松弛的推测雅可比解码（PathSpec），一种通过多序列草稿树结构提升效率的新框架。我们的并行路径推测雅可比解码（PathExplore）扩展了令牌搜索空间，在不牺牲图像质量的情况下实现了更高的加速比。此外，我们引入了跨路径松弛验证（PathRelax），利用序列间的语义相似性进一步提高令牌接受率。在Parti-Prompts、MSCOCO2017和T2ICompBench数据集上的评估表明，我们的方法分别实现了4.14倍、3.95倍和4.18倍的加速比。值得注意的是，PathExplore没有任何松弛采样，在加速比上优于GSD和LANTERN等松弛采样方法。此外，PathRelax的松弛机制可以与其他松弛技术无缝集成，实现进一步加速，为实时文本到图像生成提供高效解决方案。我们的代码可在https://github.com/Haodong-Lei-Ray/PathSpec获取。

## Abstract
The growing need for high-resolution image generation in autoregressive text-to-image models has resulted in extended token sequences, significantly increasing computational costs and inference times. However, existing state-of-the-art methods for accelerating autoregressive text-to-image models rely on chain-structured draft token sequences, leading to inefficient draft token search and limited acceptance lengths. To address this, we propose parallel-path cross-relaxed speculative Jacobi decoding (\textbf{PathSpec}), a novel framework that enhances efficiency through a multi-sequence draft tree structure. Our parallel-path speculative Jacobi decoding (\textbf{PathExplore}) expands the token search space, achieving a higher speedup ratio without sacrificing image quality. Additionally, we introduce cross-path relaxed verification (\textbf{PathRelax}) that exploits semantic similarities across sequences to further boost token acceptance rates. Evaluated on the Parti-Prompts, MSCOCO2017, and T2ICompBench datasets, our method achieves a speedup ratio of 4.14 $\times$, 3.95$\times$, and 4.18$\times$, respectively. Remarkably, PathExplore, without any relaxed sampling, outperforms relaxed sampling methods in the speedup ratio, such as GSD and LANTERN. Moreover, PathRelax's relaxation mechanism can be seamlessly integrated with other relaxation techniques, enabling further acceleration and providing an efficient solution for real-time text-to-image generation. Our code is available at https://github.com/Haodong-Lei-Ray/PathSpec.

- **整体含义**：本文旨在通过并行多路径草稿树和跨路径松弛验证，大幅提升 AR 图像生成的推理速度，同时保持或接近无损的生成质量。

## 2. 论文方法论：核心思想、关键技术细节

**核心思想**：提出 **PathSpec** 框架，包含两个关键组件：
- **PathExplore**：将传统的单链草稿扩展为多序列草稿树，多个候选路径并行生成并联合验证，提高接受长度，无需额外草稿模型。
- **PathRelax**：利用多路径下同父节点子 Token 的语义相似性，进行跨路径概率聚合，在验证阶段松绑，进一步提高接受率。

**关键技术细节**：
- **PathExplore 草稿树构造**：每轮迭代初始化草稿树（随机采样自码本），在相同前缀下并行生成多个候选 Token（通过自回归预测），树节点最多 72 个（深度 W=6，分支 B=4）。验证时采用树掩码机制，单次前向传播即可校验所有分支。
- **跨路径概率聚合**：对于同一父节点的 K 个子 Token，聚合后的目标概率为：  
  `ptarget = self-path + λ * (cross-path average)`，其中 λ 控制松弛强度（默认 0.01）。  
  然后使用松弛的拒绝采样规则决定是否接受。
- **验证与更新**：每个分支独立验证，接受节点按最长公共前缀优先保留，未被接受的节点部分传播到下一轮，不足部分用 ISP 初始化补充。

**公式与算法流程**（文字说明）：
- 草稿分布：`q(x_i) = p_θ(x_i | 前一次迭代前缀)`（SJD 方式）。
- 接受准则：`min(1, ptarget / p_θ) > r, r~U[0,1]`。
- 迭代过程：构造草稿树 → 并行前向传播获得目标分布 → 跨路径概率聚合 → 松弛拒绝采样 → 选择最长接受前缀 → 更新草稿树进入下一轮。

## 6. 论文主要结论与发现

- **PathSpec 大幅加速**：在 Parti-Prompts 上 SR=4.14×，MSCOCO2017 上 3.95×，T2ICompBench 上 4.18×，推理时间减少超 70%。
- **PathExplore 即使无松弛也优于松弛方法**：PathExplore（λ=0）加速比 3.06×，超过 GSD（2.58×）和 LANTERN（2.41×）等松弛方法。
- **跨路径松弛进一步提升**：PathRelax（λ=0.01）使 τ 从 4.20 提升至 5.64，SR 达 4.14×，且可与其他松弛技术叠加（如 PathSpec+GSD）。
- **图像质量有所下降**：加速后 CLIP-score、HPSv2、IS 等指标有轻微下降（如 CLIP 从 0.3225 降至 0.3028），但视觉上仍可接受；FID 略有升高（38.14 vs 36.25）。
- **树结构参数敏感**：深度 W=6、分支 B=4、总节点≤72 为最佳；λ=0.01 在速度与质量间平衡。

## 7. 优点

- **无需额外草稿模型**：采用自推测方式，利用目标模型自身输出构造草稿树，避免训练代价。
- **树结构扩大搜索空间**：多路径联合验证，单路失败不影响其他，显著提高接受长度。
- **跨路径松弛创新**：利用同父节点下子 Token 的语义相似性进行概率聚合，比单位置松弛更有效。
- **可集成性**：PathRelax 可与其他松弛技术（如 GSD）无缝结合，进一步提升加速。
- **理论无损性**：证明当 λ=0 时 PathExplore 为精确采样（附录 A）；λ>0 时引入轻微偏差，但实际质量可控。
- **效率-质量权衡可控**：通过调节 λ 可在速度与质量之间灵活选择。

## 8. 不足与局限

- **图像质量轻微退化**：加速后的 CLIP 等指标下降 5%~10%，虽不影响视觉感知，但在高保真需求场景（如精细人像）可能受限。
- **松弛超参数敏感**：λ 的选取对质量影响较大（λ=0.05 时 IS 明显下降），需要针对不同模型或任务手动调节。
- **实验局限性**：
  - 仅测试了 Lumina-GPT-7B-768 和 Emu3，未覆盖更大参数规模模型或扩散模型。
  - 未在视频生成或多模态生成场景中验证。
  - 松弛方法（GSD、LANTERN）的超参数可能未充分调优，对比不完全公平。
- **树结构设计依赖经验**：深度和分支数通过手动搜索确定，缺乏自适应机制；节点数超过 72 后开销增长，限制扩展性。
