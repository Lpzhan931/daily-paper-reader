## 摘要
用于推测解码的一次性草稿生成器在单次前向传递中生成完整草稿，通过消除顺序令牌生成实现了高吞吐量。然而，它们仅基于前缀上下文预测每个草稿令牌，与先前生成的草稿令牌无关。这种非自回归条件导致随着草稿深度增加，生成器分布与验证器真实自回归分布产生偏差。在基于树的草稿生成中，不同分支被迫共享后续令牌的相同边缘分布，这一问题更为严重。我们提出TreeFlash，通过引入一个基于生成器隐藏状态和前一个令牌的MLP层来近似自回归分布。TreeFlash采用两阶段近似机制，保留了一次性生成器的O(1)解码时间复杂度。TreeFlash在各种任务和模型上实现了最先进的性能，与边缘树草稿生成相比，块效率提高12%，加速比提高9%。

## Abstract
One-shot block drafters for speculative decoding generate the full draft in a single forward pass, achieving strong throughput by eliminating sequential token generation. However, they predict each draft token conditioned only on the prefix context, with no dependence on previously drafted tokens. This non-autoregressive conditioning causes the drafter's distribution to diverge from the verifier's true autoregressive distribution as draft depth grows. This problem becomes more severe in tree-based drafting, where distinct branches are forced to share the same marginal distribution for subsequent tokens. We propose TreeFlash, which addresses this by incorporating an MLP layer conditioned on the drafter's hidden state and the previous token to approximate an autoregressive distribution. TreeFlash retains the $\mathcal{O}(1)$ decoding time complexity of one-shot drafters by employing a two-stage approximation mechanism. TreeFlash achieves state-of-the-art performance across a variety of tasks and models, improving over marginal tree drafting by $12\%$ higher block efficiency and $9\%$ higher speedup.

# TreeFlash 论文详细总结

## 1. 核心问题与整体含义

- **研究动机**：大型语言模型的推理速度受限于其自回归（AR）生成方式。推测解码（Speculative Decoding）通过轻量级草稿模型生成候选令牌再由大模型并行验证来加速推理。近期的一次性块起草器（如DFlash）可在单次前向传递中生成完整草稿块，但存在根本性缺陷：每个草稿令牌仅依赖前缀上下文，与先前生成的草稿令牌无关（非自回归条件）。随着草稿深度增加，草稿分布与验证器真实AR分布严重偏离；在树形草稿中，不同分支被迫共享相同边缘分布，问题更加严重。
- **整体含义**：现有一次性块起草器的非自回归条件限制了其草稿质量，尤其在树形场景下。作者提出TreeFlash，通过引入轻量级AR近似机制来解决这一分布偏差，同时保持一次性起草的O(1)解码复杂度，从而提升推测解码的块效率和加速比。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：在一次性块起草器（基于DFlash）的输出隐藏状态上增加一个轻量级SwiGLU层，该层根据前一令牌的嵌入来修正当前隐藏状态，从而近似AR条件分布。采用两阶段树构建策略保证并行性，避免顺序计算。
- **关键技术细节**：
  - **AR近似器**：对位置t+i，计算公式为：  
    `h'_{t+i} = h_{t+i} + SwiGLU( [\tilde{h}_{t+i} :: e_{t+i-1}] )`，其中`\tilde{h}_{t+i}`是归一化的隐藏状态，`e_{t+i-1}`是前一令牌的归一化嵌入。然后将修正后的隐藏状态通过验证器的输出嵌入层得到修正的令牌分布`q'(x_{t+i}|x_{\le t}, x_{t+i-1})`。
  - **两阶段树构建**：
    1. 第一阶段：使用原始DFlash分布构建一个**top-M 叉树**（M为超参数），得到每个深度上的M个候选令牌及其分布。
    2. 第二阶段：仅对这些候选令牌应用AR近似器（由于每个深度仅有M个唯一令牌，所有修正分布可在M·γ次并行评估中计算，保持O(1)复杂度），得到修正后的分布，然后用OPT-Tree算法选择最终包含B个节点的草稿树。
  - **训练**：使用前向KL散度替代交叉熵损失，以更好对齐草稿分布与验证器分布；从预训练DFlash检查点初始化，零初始化AR近似器；采用损失缩放（更重视早期令牌）；教师强制使用真实前一令牌嵌入。
- **算法流程**（文字说明）：
  1. 给定前缀，通过DFlash得到各位置隐藏状态`h_{t+1}...h_{t+γ}`。
  2. 对每个位置i，从原始分布取出top-M令牌及其嵌入。
  3. 对第一个位置，利用前缀最后一个令牌嵌入修正隐藏状态；对后续位置，分别用每个候选前一令牌嵌入修正M个隐藏状态（每个深度并行计算M次）。
  4. 使用OPT-Tree算法，根据修正后的分布，从根开始贪婪选择概率乘积最高的路径节点，直到达到预算B。

## 6. 主要结论与发现

- TreeFlash在所有任务、模型、解码模式下**一致超越**DFlash和DDTree。
- 在预算匹配（B=16）时，TreeFlash相比DFlash块效率平均提升+24.8%（τ+1.35），加速比提升+17.1%。其中树形化贡献约63~72%增益，AR近似贡献剩余部分（τ+7.5%，加速比+3.9%）。

- 消融表明：SwiGLU层优于线性层；冻结骨干仍获得主要收益（AR近似器是关键）；KL散度略优于交叉熵；单令牌条件已足够。
