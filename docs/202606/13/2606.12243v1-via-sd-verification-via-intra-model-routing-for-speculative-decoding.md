## 摘要
推测解码（SD）通过让轻量级草稿模型生成候选项，供大型验证器并行验证，从而解决了大型语言模型的高推理成本问题。现有的草稿-验证方法采用二元决策：要么接受，要么完全重新计算。然而，我们发现许多被拒绝的token可以通过从完整验证器通过模型内路由导出的一个精简子模型（而非完整验证器）正确验证。这促使我们采用精简验证器来处理需要中等验证资源的token，从而减少昂贵的大型模型调用。我们提出了用于推测解码的模型内路由验证（VIA-SD），这是一个使用路由精简验证器的多层级框架。草稿token按层次处理：高置信度情况直接接受，中等置信度情况由精简验证器重新生成，不确定情况由完整模型验证。在四个代表性任务和多个模型族上，VIA-SD将拒绝率降低了0.10-0.22，并在强SD基线基础上实现了10-20%的加速，同时相比非草稿解码实现了2.5-3倍的加速。此外，VIA-SD与现有SD框架兼容，无需修改其训练流程。我们的结果表明，多层级SD是一种可扩展且高效的LLM推理的通用范式。项目页面：https://zju-xyc.github.io/VIA-SD-Project-Page/

## Abstract
Speculative decoding (SD) addresses the high inference costs of LLMs by having lightweight drafters generate candidates for large verifiers to validate in parallel. Existing draft-verify methods use binary decisions: accept or fully recompute. Yet we find that many rejected tokens can be verified correctly by a slim submodel derived from the full verifier via intra-model routing, instead of the full verifier. This motivates our slim-verifier to handle tokens requiring moderate verification resources, reducing expensive large-model calls. We propose Verification via Intra-Model Routing for Speculative Decoding (VIA-SD), a multi-tier framework using a routed slim-verifier. Draft tokens are processed hierarchically: direct acceptance for high-confidence cases, slim-verifier regeneration for medium-confidence cases, and full-model verification for uncertain cases. Across four representative tasks and multiple model families, VIA-SD reduces rejection rates by 0.10-0.22 and delivers 10-20% speedups over strong SD baselines, while achieving 2.5-3x acceleration over non-drafting decoding. Moreover, VIA-SD is compatible with existing SD frameworks without modifying their training procedures. Our results suggest multi-tier SD as a general paradigm for scalable and efficient LLM inference. Project page: https://zju-xyc.github.io/VIA-SD-Project-Page/

## 论文提出的方法论：核心思想、关键技术细节

### 核心思想：多层级分层验证（Hierarchical Verification）

将传统的“草稿→验证”两阶段流程，扩展为一个三阶段或多阶段的层次化验证流程。每个阶段对应一个不同计算能力的模型：

1.  **第一层（高置信度）**：草稿模型（$p$）生成的token，如果被**精简验证器（slim-verifier）** 判断为高置信度，则直接接受。
2.  **第二层（中等置信度）**：如果草稿token处于中等置信度，则由**精简验证器（$q'$）** 负责重新生成（替换）该token，而无需惊动完整模型。
3.  **第三层（低置信度，不确定情况）**：只有当精简验证器也无法处理时，才将token提交给**完整大模型验证器（$q$）** 进行验证或生成。

### 关键技术细节

1.  **精简验证器（Slim-Verifier）的构建：动态模型内路由（Dynamic Intra-Model Routing, DIMR）**
    -   **定义**：精简验证器 $q'$ 并非一个独立的模型，而是从**完整验证器 $q$** 中通过“跳层”方式选出的一个**路由子模型**。
    -   **为何选择路由子模型 vs. 独立中间模型**：路由子模型与完整模型共享词汇表和部分嵌入，具有更好的**分布一致性**，因此在KL散度衡量下的验证成本更低。同时，它避免了加载一个全新模型带来的额外内存开销。
    -   **DIMR优化**：为了找到一个最优的、能平衡效率与准确率的跳层掩码（mask），DIMR结合了**随机搜索**和**贝叶斯优化**。它在一个小规模校准数据集上评估不同跳层方案的成本得分 $C(z)$，最终选定一个固定的、最优的路由掩码 $z^*$，用于后续推理。

2.  **理论支撑：基于KL散度的信息几何视角**
    -   **超越总变差距离**：论文指出，传统基于总变差距离（TV）的分析无法解释引入中间验证器的好处。作者转而采用**KL散度**进行分析。
    -   **KL分解与中间分布**：根据广义毕达哥拉斯定理，在KL散度框架下，从草稿分布 $p$ 到最终目标分布 $q$ 的路径可以分解为多个步骤（如 $p \rightarrow q' \rightarrow q$），其累积信息损失**小于等于**直接路径。这从理论上证明了引入一个与 $p$ 和 $q$ 都“对齐”的中间分布 $q'$ 能够降低验证成本。

3.  **VIA-SD推断流程**
    -   **步骤1（草稿生成）**：小型草稿模型 $p$ 生成一个长度为 $\gamma$ 的候选token块。
    -   **步骤2（层次化验证）**：生成的token块通过两个置信度阈值 $(\alpha_1, \alpha_2)$ 进行分级验证：
        -   **门控1（$p \rightarrow q'$）**：若token在 $q'$ 下的置信度很高（大于等于 $(1-\alpha_1)$ 倍的最大概率），则直接接受。
        -   **门控2（$q' \rightarrow q$）**：若token在 $q'$ 下的置信度处于中等水平（介于 $(1-\alpha_2)$ 和 $(1-\alpha_1)$ 之间），则由 $q'$ 修改或重写该token。
        -   **门控3（$q' \rightarrow q$ 失败）**：若 $q'$ 对token的置信度很低（小于 $(1-\alpha_2)$ 倍的最大概率），则将该token转交完整模型 $q$ 处理。
    -   **步骤3（接受与回退）**：接受最长的有效前缀，回退到完整模型处理第一个未被接受的token位置，并丢弃后续草稿。
