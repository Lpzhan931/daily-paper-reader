## 摘要
随着大型多模态模型（LMM）的快速发展，推理时的开销已成为实际部署的关键瓶颈。现有方法通常在预填充阶段剪枝视觉token，假设推理过程中所需的视觉证据保持静态。然而，我们通过实验表明，视觉证据具有强烈的步骤依赖性：每个解码步骤中只有稀疏的视觉token子集是关键的，并且关键集随推理过程演变。此外，我们识别出一个耦合瓶颈：冗余的视觉上下文可能将模型引导至与查询无关的区域，从而延长推理路径。基于这些见解，我们提出了VisionPulse，一种在推理过程中逐步进行视觉token剪枝的框架。VisionPulse利用轻量级视觉注意力质量估计逐步保留预算，该质量与LMM的有效视觉token使用呈强正相关，并在此预算下仅保留最关键的token。通过在推理过程中强制执行视觉稀疏性，VisionPulse在保留相关视觉证据的同时过滤冗余视觉上下文，从而自然缩短推理路径。大量实验表明，VisionPulse每步仅保留5%的视觉token，推理路径缩短11.2%，同时准确率几乎不变。

## Abstract
With the rapid advancement of large multimodal models (LMMs), inference-time overhead has become a key bottleneck for real-world deployment. Existing methods typically prune visual tokens at prefill, assuming the required visual evidence remains static during reasoning. However, we empirically show that visual evidence is strongly step-dependent: only a sparse subset of visual tokens is critical at each decoding step, and the critical set evolves across reasoning. Furthermore, we identify a coupled bottleneck where redundant visual context can steer the model toward query-irrelevant regions, lengthening the reasoning trace. Guided by these insights, we propose VisionPulse, a step-wise visual token pruning framework during reasoning. VisionPulse computes a lightweight visual attention mass to estimate the step-wise retention budget by exploiting its strong positive correlation with LMMs' effective visual token usage and retain only the most critical tokens under this budget. By enforcing visual sparsity during reasoning, VisionPulse filters redundant visual context while preserving relevant visual evidence, shortening reasoning traces naturally. Extensive experiments show that VisionPulse only retains 5% of visual tokens per step with reasoning traces shortened by 11.2%, while keeping accuracy almost unchanged.

**关键洞见**：
- **动态依赖性**：实验发现，LMM对视觉证据的需求在解码步骤间变化显著——文本主导步骤几乎不需要视觉注意力，只有涉及实体定位的步骤才高度依赖视觉；且关键视觉区域随推理状态迁移。
- **耦合瓶颈**：保留全部视觉上下文会引导模型关注与查询无关的区域（如描述无关细节），导致推理轨迹人为延长，甚至引发错误推理（如从车辆行为推断交通灯状态）。

**研究意义**：指出静态剪枝范式与多模态推理过程本质不匹配，推动向 **逐步（step-wise）动态稀疏性** 转变，兼顾效率与准确性。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
VisionPulse 在解码的**每一步**，基于当前查询 token 对视觉 token 的注意力强度，动态计算保留预算，并只保留该步最关键的视觉 token。

### 关键技术细节
1. **逐步重要性估计**：在选定的锚点层（anchor layer, 如第17层），计算当前查询 token 对所有视觉 token 的平均注意力概率作为重要性分数：
   \[
   S_t^i = \frac{1}{H}\sum_{h=1}^H A_{t,h}^{(l_a)}(q_t, v_i)
   \]
2. **动态预算分配（Visual Attention Mass）**：观察到视觉注意力质量（visual attention mass，即分配给 visual token 的注意力概率总和）与有效激活的 token 数量呈强正相关（Pearson r = 0.82–0.95）。因此用轻量的注意力质量作为预算预测器：
   - 使用温度缩放（τ < 1）计算注意力分布，抑制长尾。
   - 取各头部最大值（head-max variant）以保证保守估计。
   - 步骤预算：\( K_t = M_{t,\text{vis}}^{max} \cdot N_v \)。
3. **保留策略**：每步只保留 top-K_t 个视觉 token，其余丢弃。
4. **附加控制**：通过 clamp 将 \( K_t \) 限定在目标平均保留率范围内（如 1%–5%）。

### 算法流程（文字说明）
- 预填充阶段：正常处理完整视觉和文本输入，初始化 KV cache。
- 解码阶段，对每一步 t：
  - 将当前生成 token 输入模型，计算到锚点层的注意力。
  - 从锚点层获取 query 到所有 visual token 的注意力概率。
  - 计算视觉注意力质量 \( M_t \) → 得出保留 budget \( K_t \)。
  - 按重要性分数排序，保留前 \( K_t \) 个视觉 token，其余从该步的注意力计算中移除（KV cache 中掩码或剪枝）。
  - 继续生成下一 token。
- 所有步骤完成后得到最终答案，并记录生成长度。
