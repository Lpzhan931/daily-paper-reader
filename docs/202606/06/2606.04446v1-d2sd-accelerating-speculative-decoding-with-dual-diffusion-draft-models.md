## 摘要
推测解码通过草拟多个词元并在单次目标模型前向传播中验证它们，从而加速自回归大型语言模型推理。最近的基于扩散的草稿生成器并行生成整个词元块，但每次验证通常只提交单个草稿序列：一旦出现第一个不匹配，后续所有草拟词元都被丢弃，导致接受率有限。简单地对更多草稿候选序列进行批处理只能带来边际改进，因为冗余或位置不当的分支在增加草拟和验证成本的同时，并未成比例地增加接受词元的数量。我们提出D^2SD，一种双扩散草稿推测解码框架，它将候选组织成由置信度引导的前缀树，其中第一个扩散草稿生成器生成一个块及其每个位置的置信度分数，这些分数用于识别最可能的拒绝边界并选择前K个前缀范围进行恢复；第二个可变前缀扩散草稿生成器在每个选中的前缀处重新定位，并在一次批处理中提出替代续写；得到的共享前缀候选通过级联注意力联合验证。实验表明，D^2SD在底层扩散方法和强自回归推测解码基线方面均显示出明显改进。

## Abstract
Speculative decoding accelerates autoregressive large language model inference by drafting multiple tokens and verifying them in a single target-model forward pass. Recent diffusion-based drafters generate an entire block of tokens in parallel but usually commit to a single draft sequence per verification: once the first mismatch occurs, all subsequent draft tokens are discarded, resulting in a limited acceptance rate. Naively batching more draft candidate sequences only introduces a marginal improvement, as redundant or poorly placed branches increase the cost of drafting and verification without proportionally increasing the number of accepted tokens.   We propose D^2SD, a dual diffusion draft speculative decoding framework that organizes candidates into a confidence-guided prefix tree, where the first diffusion drafter generates a block along with per-position confidence scores that are used to identify the most likely rejection boundary and select the top-K prefix ranges for recovery; the second variable-prefix diffusion drafter re-anchors at each selected prefix and proposes alternative continuations in one batched pass; the resulting shared-prefix candidates are jointly verified via cascade attention. Empirically, D^2SD shows clear improvements over both the underlying diffusion approach and strong autoregressive speculative decoding baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：推测解码（Speculative Decoding）通过草稿模型生成候选词元并由目标模型验证，以加速大型语言模型推理。然而，现有方法存在两大瓶颈：
  - **单一草案序列限制**：基于扩散的草稿生成器（如DFlash）一次并行生成整个词元块，但只提交单条序列；一旦早期出现不匹配，后续所有草稿被丢弃，导致接受率受限。
  - **扩展块大小收益递减**：单纯增大块大小γ（如从16增至32）会导致接受长度不增反降（“扩展墙”效应），因为早期错误会浪费后续预算。
- **整体含义**：需要一种更高效的候选生成与验证策略，能够在不显著增加草稿和验证成本的前提下，提高每次验证的接受词元数。本文将并行扩散草稿与树结构推测解码的优势相结合，提出双扩散草稿框架。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 使用两个不同功能的扩散草稿模型：
  1. **第一扩散草稿模型（DFlash风格）**：生成初始词元块，同时输出每个位置的置信度分数。
  2. **第二扩散草稿模型（VP-Drafter，Variable-Prefix Drafter）**：基于置信度估计的最可能拒绝边界，在选中前缀处重新生成替代续文。
- 流程：将候选组织成**置信度引导的前缀树**，通过级联注意力联合验证，共享前缀计算。

### 关键技术细节

#### 阶段一：第一草稿（DFlash）
- 输入：长度为γ的掩码块，位置0是前一次验证的奖励词元（anchor），其余为[MASK]。
- 输出：预测的所有掩码位置词元 \( \hat{t}_1, \dots, \hat{t}_{\gamma-1} \) 及其置信度 \( c_k = \max_v p_k(v) \)，其中 \( p_k \) 是位置k的分类分布。
- 置信度校准：实验表明置信度与经验接受率大致沿对角线分布（图2a），可用于可靠估计拒绝边界。

#### 阶段二：拒绝边界估计与Top-K前缀选择
- 计算每个前缀长度i的概率：\( r(i) = \left( \prod_{k=1}^{i} c_k \right) \cdot (1 - c_{i+1}) \)，其中 \( i = 0, \dots, \gamma-2 \)。
- 选择前K个最大 \( r(i) \) 对应的前缀长度集合S（Top-K unmask）。
- 每个选中的前缀i保留anchor和前i个token，重新掩码后续位置，构成K个变长前缀输入。

#### 阶段三：第二草稿（VP-Drafter，批量并行）
- VP-Drafter与DFlash架构相同但训练方式不同（见后）。
- 一次前向传播处理所有K个变长前缀输入，生成K条替代续文。
- 每个分支与前K个前缀共享前i+1个token，形成共享前缀结构。

#### 阶段四：联合验证与接受
- 目标模型一次性验证K+1个候选（原始草案+第二草稿分支），使用级联注意力（cascade attention）重用共享前缀的KV缓存。
- 找出所有分支中最长的正确前缀，接受该前缀，并将目标模型在拒绝位置的预测作为下一个奖励词元。

### 公式
- 接受长度函数：\( \alpha = \mathbb{E}[ \text{最长正确前缀长度} ] \)。
- 延迟模型：\( L = (T_{\text{draft}} + T_{\text{verify}}) / \alpha \)，加速比 \( \eta = \alpha \cdot L_{\text{target}} / (T_{\text{draft}} + T_{\text{verify}}) \)。

### VP-Drafter训练
- 训练时从几何先验 \( \Pr(l=j) \propto \beta^j \) 采样前缀长度j（偏向短前缀）。
- 损失函数：指数衰减交叉熵，权重 \( w_k = \exp(-(k-l-1)/\tau) \)，强调靠近anchor的错误。
