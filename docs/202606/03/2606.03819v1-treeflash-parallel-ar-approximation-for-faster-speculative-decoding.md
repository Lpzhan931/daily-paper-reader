## 摘要
用于推测解码的单次块草稿生成器在单次前向传播中生成完整草稿，通过消除顺序令牌生成实现了高吞吐量。然而，它们预测每个草稿令牌时仅基于前缀上下文，而不依赖于先前已草拟的令牌。这种非自回归条件化导致草稿生成器的分布随着草稿深度的增加而偏离验证器的真实自回归分布。在基于树的草稿生成中，这个问题变得更加严重，因为不同的分支被迫共享后续令牌的相同边际分布。我们提出TreeFlash，通过引入一个依赖于草稿生成器隐藏状态和前一个令牌的MLP层来近似自回归分布，从而解决了这一问题。TreeFlash通过采用两阶段近似机制，保留了单次块草稿生成器的O(1)解码时间复杂度。TreeFlash在各种任务和模型上实现了最先进的性能，相比边际树草稿生成，块效率提高了12%，加速比提高了9%。

## Abstract
One-shot block drafters for speculative decoding generate the full draft in a single forward pass, achieving strong throughput by eliminating sequential token generation. However, they predict each draft token conditioned only on the prefix context, with no dependence on previously drafted tokens. This non-autoregressive conditioning causes the drafter's distribution to diverge from the verifier's true autoregressive distribution as draft depth grows. This problem becomes more severe in tree-based drafting, where distinct branches are forced to share the same marginal distribution for subsequent tokens. We propose TreeFlash, which addresses this by incorporating an MLP layer conditioned on the drafter's hidden state and the previous token to approximate an autoregressive distribution. TreeFlash retains the $\mathcal{O}(1)$ decoding time complexity of one-shot drafters by employing a two-stage approximation mechanism. TreeFlash achieves state-of-the-art performance across a variety of tasks and models, improving over marginal tree drafting by $12\%$ higher block efficiency and $9\%$ higher speedup.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：推测解码（Speculative Decoding）通过轻量级草稿器（drafter）并行生成候选令牌，再由大型验证器（verifier）一次性验证，从而加速大语言模型（LLM）推理。近年来，**单次块草稿器**（one-shot block drafter）如 DFlash 能够在单次前向传播中生成整个草稿块，消除了逐令牌生成的顺序性，显著提升了吞吐量。
- **核心问题**：这类草稿器**仅基于前缀上下文**（conditioned only on prefix context），而不依赖于之前已生成的草稿令牌，即**非自回归条件化**。这导致两个严重后果：
  1. 随着草稿深度增加，草稿分布与验证器真实自回归分布的**偏差急剧增大**（总变差距离 TVD 从 0.19 增长到 0.81）。
  2. 在**树状草稿**（tree-based drafting）中，不同分支被迫共享同样深度的相同边际分布，进一步降低了树的质量。
- **整体意义**：本文旨在**保留单次块草稿器并行优势的同时，引入轻量级自回归近似（AR-Approximation）**，以解决分布偏差，从而在保持恒定时间复杂度的前提下提升推测解码性能。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 引入一个**轻量级 SwiGLU 层**，将前一个令牌的输入嵌入（input embedding）与当前草稿位置的隐藏状态结合，产生修正后的隐藏状态，从而近似自回归条件分布。
- 采用**两阶段树构造算法**，避免逐令牌顺序执行 AR-approximation，保持 O(1) 解码复杂度。

### 关键技术细节
1. **AR-approximation 模块**：
   - 输入：归一化隐藏状态 $\tilde{h}_{t+i}$ 和归一化的前令牌嵌入 $e_{t+i-1}$。
   - 输出：修正隐藏状态 $h'_{t+i} = h_{t+i} + \text{SwiGLU}([\tilde{h}_{t+i} :: e_{t+i-1}])$。
   - 通过验证器的输出嵌入（LM Head）计算修正后的令牌分布 $q'(x_{t+i}|x_{\le t}, x_{t+i-1})$。

2. **两阶段树构造算法**（图 8）：
   - **阶段一**：使用原始 DFlash 分布构建一个 **top-M 叉树**（$M$ 为超参数，控制分支因子）。计算每个深度的 top-M 候选令牌及其嵌入。
   - **阶段二**：对 top-M 叉树的每一层，应用 AR-approximation 得到修正分布（共 $M \cdot \gamma$ 次并行评估），然后使用 **OPT-Tree** 算法（贪心选择最大化期望接受令牌数的路径）从修正分布中构造最终草稿树。
   - 关键：所有非叶节点均落在 top-M 树内，叶节点可以超出，保证了两阶段的有效性。

3. **训练**：
   - 从预训练的 DFlash 检查点初始化，AR-approximator 零初始化。
   - 损失函数：使用**前向 KL 散度**（而非交叉熵）对齐草稿分布与验证器分布。
   - 损失缩放：同 DFlash，对早期令牌惩罚更重。
   - 教师强制：训练时使用真实令牌作为前文输入，避免暴露偏差（但推理时使用自己生成的令牌）。

1. **TreeFlash 在所有配置下一致优于 DFlash 和 DDTree**：在 B=16 时块效率提升 +24.8%，加速比提升 +17.1%；在 B=64 时提升更明显（+51.6% 和 +46.2%）。

5. **SwiGLU 层优于线性近似**，单令牌条件已足够（双令牌无额外收益）。
6. **推理超参数 M≤32 提供最佳吞吐量**。
