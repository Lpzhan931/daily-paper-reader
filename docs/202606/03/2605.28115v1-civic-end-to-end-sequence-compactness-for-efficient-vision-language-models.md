## 摘要
视觉语言模型（VLM）因高分辨率视觉标记而面临严重的内存和延迟瓶颈。尽管当前的标记缩减方法理论上节省了FLOPs，但事后剪枝引入了结构性开销，未能实现按比例的实际时钟加速。然而，强制实施连续的紧凑路径可能会导致几何方向迷失和细粒度定位的损失。为克服这些障碍，本文引入了CIVIC，一种路径一致的紧凑视觉推理框架。通过在视觉编码器、投影层、LLM预填充和KV缓存之间无缝保持紧凑序列表示，CIVIC避免了非连续内存访问和局部取消合并开销。在Qwen3-VL架构上的评估表明，CIVIC成功地将序列缩减转化为实际的物理硬件效率，将KV缓存内存缩小至基准的大约三分之一，并减少了端到端推理延迟。通过文本对齐的KL散度蒸馏和自适应空间保留下限，CIVIC在严格的多模态推理和视觉基础基准测试中实现了这些效率里程碑，且未降低准确性。

## Abstract
Vision-Language Models (VLMs) face severe memory and latency bottlenecks due to high-resolution visual tokens. While current token reduction methods theoretically save FLOPs, post-hoc pruning introduces structural overhead, failing to yield proportional wall-clock acceleration. However, enforcing a contiguous compact pathway risks geometric disorientation and loss of fine-grained localization. To overcome these barriers, this paper introduces CIVIC, a path-consistent compact visual inference framework. By maintaining compact sequence representations seamlessly across the vision encoder, projection layer, LLM prefill, and KV-cache, CIVIC avoids non-contiguous memory access and localized unmerging overheads. Evaluated on the Qwen3-VL architecture, CIVIC successfully translates sequence reductions into genuine physical hardware efficiency, shrinking KV-cache memory to approximately one-third of the baseline and reducing end-to-end inference latency. Enabled by text-aligned KL distillation and an adaptive spatial retention floor, CIVIC achieves these efficiency milestones without degrading accuracy across rigorous multimodal reasoning and visual grounding benchmarks.

- **核心问题**：视觉语言模型（VLM）在处理高分辨率图像时产生大量视觉 token，导致内存占用大、推理延迟高。现有 token 缩减方法（如剪枝、合并）虽然理论上减少了 FLOPs，但由于采用“事后剪枝”（post-hoc pruning）策略——即先构建密集表示再压缩——引入了额外的运行时开销（如 token 评分、收集/散射操作、密集恢复等），导致实际推理速度不升反降。这种“压缩-实现差距”（compression-realization gap）是当前高效 VLM 推理的主要瓶颈。
- **研究动机**：作者发现，只有当序列紧凑性在视觉编码器、投影层、LLM 预填充和 KV 缓存整个流水线中保持“路径一致”（path-consistent），理论上的 FLOPs 减少才能转化为真实的硬件加速。现有方法仅在中间阶段压缩，下游仍需处理密集表示，破坏了硬件效率。
- **整体含义**：本文提出 CIVIC（Compact Inference for Vision-Language Integrated Compression），通过强制实现端到端的紧凑表示路径，从根本上解决理论效率与实际加速之间的脱节，为边缘部署和低延迟场景提供实用解决方案。

## 2. 论文提出的方法论

### 2.1 核心思想
CIVIC 的核心是“路径一致性”：在视觉输入的整个处理链中（从视觉编码器前到 LLM 预填充），始终维护紧凑的 latent 序列，避免任何形式的密集恢复或运行时路由。具体包括三个关键技术组件。

### 2.2 关键技术细节

- **紧凑视觉聚合（Compact Visual Aggregation）**：在视觉编码器之前，将密集 patch 嵌入通过**可学习的锚点（learnable anchors）** 聚合成紧凑序列。公式上，给定密集 patch 嵌入 \(X_0 \in \mathbb{R}^{T_e \times D_v}\) 和可学习锚点 \(A \in \mathbb{R}^{M_e \times D_v}\)，通过归一化、交叉注意力生成权重矩阵 \(W_A\)，然后计算 \(Z_0 = W_A X_0\)，得到紧凑序列 \(Z_0 \in \mathbb{R}^{M_e \times D_v}\)。该过程保持了空间映射关系。

- **KV 压缩视觉注意力（KV-Compressed Vision Attention）**：在紧凑视觉编码器内部，对每个注意力层的键（Key）和值（Value）进一步压缩到更少的记忆锚点（\(S\) 个），使注意力分数矩阵大小从 \(M_e \times M_e\) 降为 \(M_e \times S\)，大幅降低计算和内存。同时引入**自适应空间保留下限（Adaptive Spatial Retention Floor）**，防止过度压缩导致细粒度定位信息丢失。

- **紧凑多模态预填充（Compact Multimodal Prefill）**：紧凑视觉编码器的输出通过紧凑投影器 \(\Pi_c\) 映射到语言隐藏空间，直接替换 LLM 预填充中的密集视觉 placeholder。预填充序列长度从 \(L+T_p\) 变为 \(L+M_p\)（\(M_p < T_p\)），KV 缓存也随之缩小。

- **文本对齐的 KL 蒸馏（Text-Aligned KL Distillation）**：由于 CIVIC 改变了视觉 token 接口，且 LLM 参数被冻结，使用蒸馏训练优化 CIVIC 特定参数。但标准蒸馏要求 token 一一对应，而密集与紧凑的视觉位置不对齐。因此，仅在文本位置（非视觉位置）上计算温度缩放后的 KL 散度损失，通过映射函数 \(\pi: I_t \rightarrow I_s\) 对齐教师和学生的文本 logit，从而保证紧凑视觉流与冻结语言空间语义一致。

### 2.3 公式与算法流程（文字说明）
- 原始密集推理成本 \(C_{dense}\) 包括视觉自注意力、LLM 预填充注意力和 KV 缓存存储，均与序列长度平方或线性相关。
- CIVIC 紧凑推理成本 \(C_{model}^c\) 中，视觉注意力从 \(T_e^2\) 降为 \(M_e S\)，LLM 预填充从 \((L+T_p)^2\) 降为 \((L+M_p)^2\)，KV 缓存从 \((L+T_p)\) 降为 \((L+M_p)\)。
- 训练目标：\(\phi^* = \arg\min_\phi \mathbb{E}_{(F,x)}[\mathcal{L}_{KL}]\)，仅优化 CIVIC 参数 \(\phi\)，LLM 参数 \(\theta\) 保持冻结。

## 6. 论文的主要结论与发现

- **主要结论**：CIVIC 成功将理论上的序列压缩转化为实际的硬件加速。具体地：
  - 预填充序列长度从 1122.2 降至 407.9 token（减少约 64%）
  - KV 缓存内存从 122.7 MB 降至 44.61 MB（约减少 64%）
  - 端到端推理延迟从 3543.0 ms 降至 2514.9 ms（降低约 29%）
  - 同时，在大多数 benchmark 上保持甚至提升准确率（尤其在细粒度定位任务上略有提升）。
- **关键发现**：
  - 理论 FLOP 减少不等于实际加速：事后剪枝方法（如 DyMU、ZOO-Prune）引入了高达 18.45 ms 的额外开销，而 CIVIC 仅 0.49 ms 开销。
  - 路径一致性是效率转化的关键：仅仅在视觉端压缩而下游保持密集，会导致 LLM 解码阶段仍是瓶颈（占 >90% 时间）。
  - 紧凑表示经过适当蒸馏后可以替代密集表示，不损失多模态理解能力。

## 7. 优点

- **方法创新性**：首次提出“路径一致性”概念，在视觉编码器前聚合、编码器中用 KV 压缩注意力、直接进入 LLM 预填充，避免了所有非连续内存访问和动态路由，从根本上解决理论加速无法落地的问题。
- **实践导向强**：所有实验在消费级 GPU（RTX 4090）上完成，且有详细的延迟分解，证明方法适用于资源受限环境。

- **效果显著**：在降低延迟和内存的同时，准确率无损甚至微涨，说明紧凑表示并非简单丢弃信息，而是通过学习保留了关键空间语义。

## 8. 不足与局限

- **实验覆盖有限**：仅使用了 Qwen3-VL-2B 一个基座模型，未验证在更大规模 VLM（如 Qwen3-VL-7B/72B）或不同架构（如 LLaVA、InternVL）上的泛化性。
- **静态 token 预算**：当前采用固定的紧凑 token 数 C 和固定的 KV 压缩锚点数 S，未实现实例自适应的动态预算。未来需要研究根据输入复杂度动态调整压缩率。
- **场景限制**：仅测试了单图像输入和短视频片段，未验证在多图像、长视频等复杂场景下的效果。作者在局限性中也承认了这一点。
