## 摘要
大语言模型（LLM）因其深度和参数规模而导致推理成本高昂。深度剪枝可以通过跳过冗余的Transformer模块来降低延迟，但现有方法（i）在用户指定的计算预算下提供有限的控制，以及（ii）通常固定路由路径，无法在解码过程中随着上下文增长而自适应。我们提出Buddy，一种预算驱动的动态深度路由框架。Buddy使用轻量级决策模块根据输入对中间层进行评分，并确定性地执行top-k层以满足给定预算。为了支持解码时的自适应，Buddy复用第一层的KV缓存作为低开销的全局上下文源，并在每次路由决策前将其与最新的token表示合并。当没有明确预算时，可选的预算预测器会估计输入相关的计算级别，以平衡质量与效率。在Llama系列和Qwen模型上的实验表明，Buddy与强大的静态剪枝基线相比具有竞争力，并且通常能改善精度-计算量权衡，同时独特地支持严格的预算控制、解码时重路由以及在单个训练模型中支持多个预算。

## Abstract
Large language models (LLMs) incur high inference cost due to their depth and parameter scale. Depth pruning can reduce latency by skipping redundant Transformer blocks, but existing methods (i) provide limited control under user-specific compute budgets and (ii) typically fix the routing path, failing to adapt as the context grows during decoding. We propose Buddy, a budget-driven dynamic depth routing framework. Buddy uses a lightweight Decision Module to score intermediate layers conditioned on the input and deterministically executes the top-k layers to satisfy a given budget. To support decode-time adaptation, Buddy reuses the first-layer KV cache as a low-overhead global context source and pools it together with the newest token representation before each routing decision. When no explicit budget is provided, an optional Budget Predictor estimates an input-dependent compute level to balance quality and efficiency. Experiments on Llama-family and Qwen models show that Buddy is competitive with strong static pruning baselines and often improves the accuracy-compute trade-off, while uniquely supporting strict budget control, decode-time rerouting, and multiple budgets within a single trained model.

- **主要问题**：大型语言模型（LLM）推理成本高，现有深度剪枝方法（静态或动态）存在两个关键缺陷：  
  - ① 无法在用户指定的计算预算下提供严格、可控的执行层数，实际计算量不可预测；

- **研究背景**：已有工作包括静态剪枝（ShortGPT、SLEB等）和动态剪枝（Early-exit、Token-wise路由、PuDDing等），但都缺乏同时满足 **预算严格性**、**预算灵活性**（单模型服务多预算）、**输入自适应** 和 **解码自适应** 四个属性。  
- **整体含义**：Buddy旨在实现一个单一部署模型，能根据用户预算和输入难度动态选择执行哪些Transformer层，并在解码过程中随上下文演变更新路径，从而在精度、效率和灵活性之间取得更优权衡。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将层选择建模为“排名-选择”问题，通过轻量决策模块对中间层评分，并确定性地执行top-k层，其中k由用户预算b决定（b为执行层数占总层数的比例）。  
- **关键技术细节**：  
  - **决策模块（Decision Module）**：  
    - 输入为当前上下文表示（经过token pooling），输出每个中间层（除首尾固定执行层）的分数s。  
    - 分数可与离线计算的层重要性先验（如ΔPPL、Taylor、余弦相似度）经归一化融合，公式：  
      \( s' = s + \omega \tilde{p} \)，其中\(\tilde{p}\)是归一化先验。  
    - 通过Softmax和Top-k选择确定执行掩码M，使用直通估计器（STE）进行端到端训练。  
  - **KV感知规划器（KV-aware Planner）**：  
    - 在解码阶段，复用第一层的KV缓存以获取全局上下文，并与最新token表示拼接，作为决策模块的输入，从而支持解码时重新路由。  
  - **预算预测器（Budget Predictor）**：  
    - 当用户未提供预算时，通过一个独立的分类器（使用GRPO训练）预测离散的计算级别（执行的中间层层数），平衡质量与效率。  

- **算法流程（文字说明）**：  
  1. 输入prompt和可选预算b。  
  2. 从第一层提取KV缓存，经token pooling获得全局上下文。  
  3. 决策模块为L-2个中间层评分，加上归一化先验（可选），softmax后根据b转换为整数k，执行top-k层，其余跳过（保持残差连接）。  
  4. 解码时，每步更新第一层KV缓存，重新调用决策模块，动态调整执行路径。  
  5. 若未提供b，预算预测器自动输出推荐预算。

### 8. 不足与局限

- **实验覆盖**：  
  - 仅使用英文学科推理和数学推理基准，缺乏多语言、长文档生成、对话等场景评估。  
  - 速度测试仅在单GPU环境下进行，未评估分布式或批处理（batch > 5）下的实际加速。  
  - 预算预测器仅在Alpaca上训练，其泛化性可能受限于训练分布。  
- **偏差风险**：  
  - 层重要性先验基于验证集（WikiText-2, PTB）计算，可能对特定数据分布存在偏差。  
- **应用限制**：  
  - **VRAM无减少**：所有Transformer块仍常驻内存，仅跳过前向计算，无法降低显存占用。  
  - **批处理效率**：不同序列可能选择不同路径，导致KV缓存不连续（跳过层时缓存未命中），当前用零填充处理，可能影响精度，且额外收集/分散开销在低稀疏时较明显。  
  - 对于极稀疏（如50%以上），Buddy性能下降较快，需更深入设计。  
- **未来工作**：论文承认需开发内存感知路由、选择性缓存共享、批量级路径分组等方法以支持更大规模自适应推理。
