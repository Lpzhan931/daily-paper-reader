## 摘要
推测解码通过草拟多个令牌并在单次目标模型前向传播中验证它们，从而加速自回归大语言模型推理。最近的基于扩散的草稿模型并行生成整个令牌块，但每次验证通常只提交一个草稿序列：一旦出现第一个不匹配，所有后续草稿令牌都被丢弃，导致接受率有限。简单地对更多草稿候选序列进行批处理只会带来边际改善，因为冗余或位置不当的分支会增加草拟和验证的成本，而不会成比例地增加接受的令牌数量。我们提出了D^2SD，一种双扩散草稿推测解码框架，它将候选组织成一个由置信度引导的前缀树，其中第一个扩散草稿器生成一个块以及每个位置的置信度分数，用于识别最可能的拒绝边界并选择前K个前缀范围进行恢复；第二个可变前缀扩散草稿器在每个选定的前缀处重新锚定，并在一次批量传递中提出替代续接；生成的共享前缀候选通过级联注意力进行联合验证。实验表明，D^2SD在底层扩散方法和强自回归推测解码基线上都有明显改进。

## Abstract
Speculative decoding accelerates autoregressive large language model inference by drafting multiple tokens and verifying them in a single target-model forward pass. Recent diffusion-based drafters generate an entire block of tokens in parallel but usually commit to a single draft sequence per verification: once the first mismatch occurs, all subsequent draft tokens are discarded, resulting in a limited acceptance rate. Naively batching more draft candidate sequences only introduces a marginal improvement, as redundant or poorly placed branches increase the cost of drafting and verification without proportionally increasing the number of accepted tokens.   We propose D^2SD, a dual diffusion draft speculative decoding framework that organizes candidates into a confidence-guided prefix tree, where the first diffusion drafter generates a block along with per-position confidence scores that are used to identify the most likely rejection boundary and select the top-K prefix ranges for recovery; the second variable-prefix diffusion drafter re-anchors at each selected prefix and proposes alternative continuations in one batched pass; the resulting shared-prefix candidates are jointly verified via cascade attention. Empirically, D^2SD shows clear improvements over both the underlying diffusion approach and strong autoregressive speculative decoding baselines.

# D^2SD：利用双扩散草稿模型加速推测解码——论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：自回归大语言模型（LLM）推理受限于串行解码瓶颈——每步生成一个token需一次完整前向传播。推测解码通过轻量草稿模型并行生成多个候选token，再由目标模型一次性验证，从而加速推理。
- **现有方法及问题**：
  - 基于扩散的草稿模型（如DFlash）能在单次前向中并行生成整个token块，但每次验证只提交一个草稿序列：一旦出现首个不匹配，后续所有草稿token被丢弃，接受率有限。
  - 简单增加候选序列数量（如批处理多个独立样本）效果甚微：由于同源概率分布导致候选路径高度重合，且冗余分支增加草拟和验证开销，不能成比例提升接受token数。
  - 基于前缀树的推测解码（如SpecInfer、Sequoia、EAGLE-3）虽可通过分支探索多个续接，但其草稿阶段依赖自回归生成，导致“草稿税”（drafting tax）随候选预算增大而急剧膨胀，难以在延迟敏感场景中扩展至更大候选批次。
- **核心挑战**：如何在保持扩散草稿并行优势的同时，组织多样化的候选序列，将计算资源集中在最可能发生拒绝的位置，从而提升接受长度（α）和端到端加速比。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：采用双扩散草稿框架（D²SD）：
  1. 第一个扩散草稿器（基于DFlash）生成一个长度为γ的token块，同时输出每个位置的置信度分数（cₖ = max概率）。
  2. 利用置信度估计拒绝边界：通过公式 r(i) = (∏_{k=1}^i cₖ) * (1 - c_{i+1}) 计算恰好接受i个token的概率，据此选择最可能拒绝的Top-K个前缀长度。
  3. 第二个可变前缀扩散草稿器（VP-Drafter）针对每个选定的前缀长度重新锚定，在批量并行前向中生成替代续接，形成共享前缀的候选分支。
  4. 所有K+1个候选（原始草稿+第二草稿分支）通过级联注意力（cascade attention）由目标模型联合验证，选取最长接受前缀，并将目标模型在拒绝位置的预测作为下一轮的bonus token。

- **关键技术细节**：
  - **置信度校准**：实验表明扩散草稿器的置信度与经验接受率接近对角线，可作为可靠性估计。
  - **Top-K选择**：取r(i)最高的K个前缀长度，确保分支覆盖最可能的拒绝点。
  - **VP-Drafter训练**：
    - 使用截断的几何先验 Pr(l=j) ∝ β^j 采样前缀长度l，将前l+1个位置填入真实token，其余掩码。
    - 损失函数为指数衰减的交叉熵：越靠近锚点（已接受token）的位置权重越大，强调早期错误的代价。
  - **级联注意力验证**：复用共享前缀的KV缓存，避免重复计算，提升验证效率。

- **算法流程**（文字描述）：
  1. 输入：上一个循环的bonus token（锚点）。
  2. 第一草稿（DFlash）：构建掩码块（锚点+γ-1个[MASK]），并行预测所有掩码位置，得到候选token及置信度。
  3. 计算r(i)并选择Top-K前缀长度。
  4. 第二草稿（VP-Drafter）：对每个选中的前缀长度构建输入（保留前缀+剩余掩码），批量并行预测续接。
  5. 联合验证：将所有K+1个候选序列输入目标模型，通过级联注意力一次验证，取最长接受前缀；目标模型在拒绝位置的预测作为下一个锚点。

## 6. 主要结论与发现

- **核心结论**：D²SD在多个任务上一致优于DFlash和EAGLE-3。
  - 在Qwen3-8B上（γ=16, K=4, 贪婪解码），平均加速比从DFlash的4.16×提升至4.98×，平均接受长度从5.31提升至7.05。
  - 在GPT-OSS-20B上，加速比从DFlash的3.53×提升至6.15×，接受长度从4.13提升至8.02，提升更为显著（验证成本更高的目标模型收益更大）。
  - 即使在采样解码（T=1）下，D²SD仍保持领先。
- **消融发现**：
  - 简单对DFlash添加K=4重采样分支仅带来微小增益（接受长度+0.33，加速比+0.58×），而D²SD提升巨大（接受长度+1.87，加速比+0.87×），说明分支多样性来自第二个草稿器的不同归纳偏置而非简单增加候选数。
  - 若直接复用DFlash作为第二草稿器（不训练VP-Drafter），接受长度比单分支提高，但远低于全D²SD，验证了可变前缀训练的必要性。
  - 堆叠第三级草稿器进一步增加接受长度（+8%），但墙钟加速比下降（-4%），因为边际增益无法抵消额外草稿和验证开销。

## 7. 优点（方法或实验设计亮点）

- **方法论亮点**：
  - 双扩散草稿协同：第一草稿提供基础候选和置信度信号，第二草稿（VP-Drafter）针对最可能拒绝点提供不同分布的替代续接，有效解决单序列早期错误和同源分支误差同质性问题。
  - 置信度引导的前缀树构建：利用扩散模型自身的校准性质，无需额外模型或训练；计算简单高效。
  - 可变前缀训练：使用几何先验采样前缀长度，损失函数按距离指数衰减，使草稿器适应任意前缀长度且优先保证早期正确性。
  - 级联注意力联合验证：复用共享前缀KV计算，避免重复，提升验证效率。
- **实验设计亮点**：
  - 覆盖多领域（数学、代码、聊天）和多尺度目标模型（8B和20B），验证通用性。
  - 同时报告加速比和接受长度，清晰分离草稿质量和系统开销的影响。
  - 多组消融实验严谨隔离变量（分支数量、草稿器类型、级联深度），结论可靠。
