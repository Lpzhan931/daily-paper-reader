## 摘要
离散扩散语言模型能够实现并行令牌生成，为低延迟解码提供了一条途径。然而，基于边际置信度的独立令牌选择限制了并行效率：单独看可靠的令牌在同时更新多个位置时可能会形成不兼容的配置。我们提出了一种无需训练的解码框架来协调这些并行更新。在每次前向传播中，该方法为每个掩码位置分配一个提交分数，并利用模型预测分布中的成对交互来优化这些分数。变分松弛产生了一个简单的定点更新，可在单次前向传播中抑制冲突的同时提交。该机制允许解码器并行提交更多令牌，同时保持有竞争力的生成质量。该方法轻量级，不需要辅助模型或重新训练，并且可以直接嵌入现有的扩散解码流程中而无需修改。在推理和代码生成基准上的实验表明，在质量-延迟权衡方面取得了持续改进。

## Abstract
Discrete diffusion language models enable parallel token generation, offering a pathway to low-latency decoding. However, selecting tokens independently by marginal confidence limits effective parallelism: tokens that appear reliable in isolation can form incompatible configurations when several positions are updated at once. We introduce a training-free decoding framework that coordinates these parallel updates. At each forward pass, the method assigns a commit score to each masked position and refines these scores using pairwise interactions derived from the model's predictive distributions. A variational relaxation yields a simple fixed-point update that suppresses conflicting simultaneous commitments within a single forward pass. This mechanism allows the decoder to commit more tokens in parallel while maintaining competitive generation quality. The method is lightweight, requires no auxiliary model or retraining, and drops into existing diffusion decoding pipelines without modification. Experiments on reasoning and code-generation benchmarks show consistent improvements in the quality-latency trade-off.

## 1. 核心问题与整体含义（研究动机和背景）

离散扩散语言模型（dLLMs）通过迭代去噪过程实现并行令牌生成，从而有望突破自回归解码的延迟瓶颈。然而，现有dLLMs通常采用因子化反向策略，即独立预测每个掩码位置的token分布。这种独立假设忽略了token之间的依赖关系（如句法一致性、语义连贯性），导致“联合不一致性”（joint inconsistency）：当基于边际置信度同时提交多个token时，单个token看似合理，但组合后却产生不兼容的配置（如重复或冲突）。这种“联合错配”严重限制了并行解码的有效性，使得模型在追求高吞吐量时不得不牺牲生成质量。

**研究动机**：现有方法要么依赖辅助模型（如DAWN、DEMASK），要么需要重新训练，或者仅使用边际统计量而忽略交互。本论文旨在提出一种**无需训练、轻量级、可无缝集成**的机制，在单次前向传播中利用模型自身的预测分布来估计token间的相互作用，并据此协调并行提交决策，从而在保证生成质量的同时最大化并行度。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心思想
将并行提交选择形式化为结构化推理问题：为每个掩码位置i定义一个二元指示变量s_i（1表示提交，0表示不提交），并构建一个**成对马尔可夫随机场（MRF）**能量函数，平衡局部置信度（一元势）与全局兼容性（成对势）。通过变分平均场近似的固定点迭代，得到协调后的提交强度q_i，再用阈值确定提交集。

### 2.2 关键技术细节
**一元势（局部置信度）**：  
c_i = log π_i(v_top1) - log π_i(v_top2)，即前两个最可能token的log概率之差，反映位置i的确定性。

**成对势（预测重叠惩罚）**：  
D_ij = (1 - JSD(π_i, π_j) / ln2) 归一化到[0,1]，其中JSD为Jensen-Shannon散度。高D_ij意味着两位置预测分布高度重叠，同时提交容易导致冗余或不一致，因此通过负项 -D_ij * s_i * s_j 抑制同时提交。

**平均场变分推理**：  
用全因子分布q(S)=∏ q_i^{s_i}(1-q_i)^{1-s_i} 近似真实后验。最大化ELBO得到固定点方程：
q_i = σ( c_i - ∑_{j≠i} D_ij q_j )
其中σ为logistic sigmoid函数。该更新直观地使高置信度位置抑制与其分布重叠的其他位置。

### 2.3 算法流程（Algorithm 1）
1. **局部置信度估计**：对每个掩码位置i，计算softmax后的预测分布π_i，并计算置信度c_i。
2. **成对兼容性构建**：计算所有掩码位置对之间的归一化JSD矩阵D。
3. **平均场迭代**：初始化q^(0)=σ(c)，迭代R次（默认R=2）更新q^(r+1)=σ(c - D q^(r))。
4. **阈值化**：对于i，若q_i^(R) ≥ τ，则将i加入提交集C_t。

该框架完全无训练，仅需单次前向传播的模型输出，额外开销为O(m²|V|)的JSD计算，但可高度并行化。
