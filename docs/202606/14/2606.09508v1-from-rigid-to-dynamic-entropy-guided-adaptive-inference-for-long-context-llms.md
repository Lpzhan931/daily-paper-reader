## 摘要
现有用于长上下文LLM推理的稀疏注意力和KV缓存压缩方法通常对所有注意力头应用固定的稀疏模式或统一预算，忽略了头与上下文之间注意力行为的显著差异。我们观察到注意力头中存在两种不同的熵模式：刚性头（Rigid Heads），其熵在输入片段中接近零；动态头（Dynamic Heads），其熵显著波动。关键在于，这些类型的分布是上下文相关的，无法离线预先确定。因此，我们提出EntropyInfer，一种无需训练框架，在预填充阶段利用注意力熵在单个头和片段粒度上自适应分配计算。对于解码，我们引入了一种潜在的KV缓存压缩方案，利用生成的输出令牌（而非仅预填充令牌）来识别并保留最关键的缓存条目。在Llama、Qwen和openPangu模型系列上的大量实验表明，EntropyInfer始终优于SnapKV、AdaKV和CritiPrefill等基线，在超过100k令牌时实现高达2.39倍的端到端加速，同时与全注意力相比质量下降极小。代码已发布在https://github.com/SHA-4096/EntropyInfer。

## Abstract
Existing sparse attention and KV cache compression methods for long-context LLM inference typically apply fixed sparsity patterns or uniform budgets across all attention heads, overlooking the substantial variation in attention behavior among heads and contexts. We observe two distinct entropy patterns among attention heads: Rigid Heads, whose entropy stays near zero across input segments, and Dynamic Heads, whose entropy fluctuates significantly. Crucially, the distribution of these types is context-dependent and cannot be predetermined offline. We therefore propose EntropyInfer, a training-free framework that uses attention entropy to adaptively allocate compute at the granularity of individual heads and segments during prefilling. For decoding, we introduce a latent KV cache compression scheme that leverages generated output tokens, rather than prefill tokens alone, to identify and retain the most critical cache entries. Extensive experiments on Llama, Qwen and openPangu model series show that EntropyInfer consistently outperforms baselines including SnapKV, AdaKV, and CritiPrefill, achieving up to 2.39$\times$ end-to-end speedup beyond 100k tokens with minimal quality degradation compared to full attention. The code is released in https://github.com/SHA-4096/EntropyInfer.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：长上下文大语言模型（LLM）推理面临两个主要成本：预填充阶段（prefilling）的二次注意力计算开销和解码阶段（decoding）线性增长的 KV 缓存开销。现有稀疏注意力和缓存压缩方法通常对所有注意力头应用固定的稀疏模式或统一的预算，忽略了不同头之间的注意力行为差异以及上下文依赖性。
- **背景**：先前方法如 MInference、FlexPrefill、CritiPrefill 等在不同程度上假设注意力行为在位置、头或上下文之间具有同质性，但这种假设并不成立。同时，解码阶段的注意力模式会随生成过程发生偏移，基于预填充阶段选择的 token 可能并不适用于解码。
- **核心观察**：通过对 Llama-3.1-8B-Instruct 和 Qwen-2.5-7B-Instruct 进行逐头、逐段的熵分析，发现两类注意力头：
  - **刚性头（Rigid Heads）**：行熵始终低于 \(10^{-5}\)，注意力近乎确定。
  - **动态头（Dynamic Heads）**：熵随查询位置显著波动，包含上下文相关的语义结构。
  - 更重要的是，同一头在不同输入下可能属于不同类别，因此无法离线预先确定头类型，必须在线实时判断。
- **整体含义**：注意力熵是廉价且有效的在线信号，可用于自适应分配计算资源，从而在保持生成质量的同时大幅加速推理。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
提出 **EntropyInfer**，一个无需训练的框架，分为两个阶段：
- **预填充阶段**：利用观察注意力矩阵计算每头、每段的熵，区分刚性头和动态头；刚性头分配固定预算，动态头根据相邻段熵波动自适应分配预算。
- **解码阶段**：延迟 KV 缓存压缩，在生成少量输出 token 后，基于这些输出 token 与预填充 token 共同计算重要性得分，重新排序并保留关键缓存条目，解决预填充与解码注意力模式不匹配的问题。

### 关键技术细节

#### 1. 观察注意力矩阵构建（复杂度降低）
- 将输入序列划分为 \(C_S\) 个段，每段长度 \(L_S\)；将注意力矩阵划分为 \(C_B\) 个块，块大小 \(L_B\)。
- 对每个段计算 Q 和 K 的最大/最小表示向量（Algorithm 1），然后构造观察注意力矩阵 \(A_{\text{Obs}}\)（Algorithm 2），仅需四次 softmax 操作，避免全矩阵计算。
- 公式（以行熵计算为例）：  
  \(H(a_{i:}) = -\sum_{j=1}^{N} a_{i,j} \log a_{i,j}\)，其中 \(a_{i,j}\) 为第 i 个查询对第 j 个键的注意力权重。

#### 2. 自适应预算分配（Algorithm 3）
- 阈值 \(e_t = 10^{-5}\)。
- **刚性头**（最大段熵 < \(e_t\)）：所有段分配固定基础预算 \(B_h\)。
- **动态头**（最大段熵 ≥ \(e_t\)）：
  - 第一段分配 \(B_h\)。
  - 后续段根据熵波动调整：\(\Delta_{\text{budget}} = \alpha \cdot B_h \cdot \left( \frac{|e_i - e_{i-1}|}{e_{i-1}} - \Delta_t \right)\)，其中 \(\alpha=0.5\)，\(\Delta_t=0.4\)。
  - 最终预算 \(B_i = B_{i-1} + \Delta_{\text{budget}}\)，并裁剪到 \([B_0, 3B_0]\)。

#### 3. 解码阶段潜在 KV 缓存压缩（Algorithm 4）
- 在生成 \(N_D\) 个 token 后，构建大小为 \(N_W\) 的观察窗口（包含部分生成 token 和输入 token）。
- 计算观察窗口查询与所有键的交互得分矩阵 \(S = Q_W K^\top\)，按列求和得到重要性向量 \(w\)。
- 选择前 \(B_d\) 个最高得分的 token 索引，保留对应 KV 缓存条目。

#### 4. 复杂度分析
- 总复杂度：\(O\left(\left(\frac{1}{L_S L_B} + \frac{3}{2L_S L_B}\log\frac{B_h}{L_B}\right) N^2 + 3B_h N\right)\)。
- 由于 \(N^2\) 项系数极小，实际接近线性复杂度，远优于全注意力的 \(O(N^2)\)。
