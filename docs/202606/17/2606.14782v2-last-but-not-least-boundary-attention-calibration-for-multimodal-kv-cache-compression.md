## 摘要
多模态大语言模型（MLLMs）实现了强大的视觉-语言推理，但长视觉上下文会增大KV缓存并增加解码延迟。现有的压缩方法依赖观察窗口注意力来稳定估计标记重要性，但这种聚合可能稀释稀疏的视觉证据，并在激进压缩下丢弃答案关键标记。因此，我们识别出最后查询注意力作为恢复此类证据的补充来源，但其与答案无关的信号可能误导保留。我们提出BACON，一种即插即用的方法，通过最后查询证据校准观察窗口注意力，并通过层内一致性和层间持久性抑制孤立噪声。在各种基准、模型、预算和压缩方法下，BACON在最激进的预算下将多模态KV压缩平均提升7.5%，最高提升30.9%。我们的项目页面可见于https://ryu1ion.github.io/official_BACON/

## Abstract
Multimodal Large Language Models (MLLMs) achieve strong vision-language reasoning, but long visual contexts enlarge the KV cache and increase decoding latency. Existing compression methods rely on observation window attention for stable token-importance estimation, yet this aggregation can dilute sparse visual evidence and discard answer-critical tokens under aggressive compression. Therefore, we identify last-query attention as a complementary source for recovering such evidence, but its answer-irrelevant signals can mislead retention. We propose BACON, a plug-and-play method that calibrates observation window attention with last-query evidence and suppresses isolated noise via intra-layer coherence and inter-layer persistence. Across diverse benchmarks, models, budgets, and compression methods, BACON improves multimodal KV compression by 7.5% on average under the most aggressive budget, with gains up to 30.9%. Our project page is available at https://ryu1ion.github.io/official_BACON/

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：多模态大语言模型（MLLMs）在视觉-语言推理中表现优异，但视觉输入变长导致 KV 缓存显著增大，进而增加解码延迟。KV 缓存压缩成为降低推理内存与延迟的重要方向。
- **核心问题**：现有基于注意力（如 SnapKV、PyramidKV、AdaKV 等）的压缩方法通常依赖“观察窗口”（observation window）——即 prompt 的末尾一段查询——对每个缓存 token 的注意力值取平均来估计其重要性。这种聚合虽然能平均掉查询特异性噪声、稳定重要性估计，但也会**稀释稀疏的视觉证据**，在激进压缩下容易丢弃对答案至关重要的视觉 token。
- **关键发现**：最后一条 prompt 查询（last-query）的注意力能够揭示观察窗口聚合所弱化的边界证据（boundary evidence），但其高注意力 token 中约 70% 是无关噪声，直接使用会引入误导信号。因此，需要一种校准机制，既能利用最后查询的补充信息，又能抑制噪声。

## 2. 方法论：核心思想、关键技术与流程

- **核心思想**：保持观察窗口注意力作为稳定的基础重要性得分，用最后查询注意力提取边界残差，再通过**层内一致性**和**层间持久性**两种结构线索校准边界证据，最后以方差匹配的方式将校准后的证据嵌入到原有得分中，实现即插即用的 token 保留。
- **关键技术细节**：
  1. **边界证据提取**：定义最后查询注意力 \(Q_i\) 与观察窗口得分 \(B_i\) 的正残差 \(E_i = [Q_i - B_i]_+\)，作为可能被窗口稀释的证据估计。
  2. **结构校准**：
     - 层内一致性（Intra‑layer coherence）：在半径为 \(r=5\) 的邻域内平均边界残差，得到平滑后的得分 \(L_i\)。
     - 层间持久性（Inter‑layer persistence）：取前面 \(m=4\) 层的同一 token 位置边界残差的平均值，得到跨层稳定性得分 \(T_i\)。
  3. **合成与缩放**：将原始残差、层内一致性和层间持久性相加得 \(V_i = E_i + L_i + T_i\)；然后对每个层‑头对，计算缩放系数 \(\lambda\)，使 \(\lambda V\) 的标准差匹配观察窗口得分局部区域标准差的 \(\gamma\) 倍（\(\gamma\) 控制全局校准强度，默认 \(\gamma=1\)），从而保持同尺度注入。
  4. **最终得分**：\(S_i = B_i + \lambda V_i\)，以此选择 top‑K 保留。
- **流程**：预填充阶段计算所有注意力 → 提取窗口得分与最后查询得分 → 校准边界证据 → 对每个头按 S_i 排序并选择 K 个 KV 对 → 其余丢弃。不改变解码流程。
