## 摘要
长上下文和代理型LLM工作负载将KV缓存推至固定内存预算之外，迫使推理栈在连续推理流的每一步永久驱逐令牌。现有方法都遵循相同的模板：每步直接注意力分数后接确定性top-K选择，这会将单个低于阈值的步骤转化为不可撤销的判决，并永久擦除任何直接注意力无法从噪声中区分出的微妙重要令牌。为解决这一挑战，我们提出Nexus采样，一种无需训练的驱逐方法，它结合了Nexus评分（一种在直接注意力上迭代遍历以揭示桥接令牌的方法）和加权蓄水池采样（用包含概率保留令牌，取代确定性top-K）。理论上，我们证明Nexus采样在微妙重要令牌的长期存活上优于确定性top-K。实验上，在80% KV缓存驱逐下，Nexus采样在LongBench上与密集注意力匹配（误差在1%以内），同时在检索密集型任务上优于top-K基线，每个序列缓存内存最多减少10倍。

## Abstract
Long-context and agentic LLM workloads push the KV cache past any fixed memory budget, forcing the inference stack to permanently evict tokens at every step of a continuous-inference stream. Existing methods all share the same template, a per-step direct-attention score followed by deterministic top-$K$ selection, which converts a single below-cutoff step into an irreversible verdict and permanently erases any subtly important token that direct attention cannot single out from noise. To address this challenge, we propose Nexus Sampling, a training-free eviction method that pairs Nexus scoring, an iterative walk over direct attention that surfaces bridge tokens, with weighted reservoir sampling, which retains tokens with inclusion probability in place of deterministic top-$K$. Theoretically, we show that Nexus Sampling dominates deterministic top-$K$ in long-run survival of subtly important tokens. Empirically, at 80% KV cache eviction, Nexus Sampling matches dense attention within 1% on LongBench while outperforming top-$K$ baselines on retrieval-heavy tasks, with up to 10x smaller per-sequence cache memory.

# 论文总结：Forget Without Compromise: Nexus Sampling for Streaming KV-Cache Eviction Under Fixed Budgets

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：长上下文和代理型（agentic）大语言模型（LLM）的推理过程中，Key-Value（KV）缓存随上下文长度线性增长，很快超出固定 GPU 内存预算，迫使推理系统在连续推理流的每一步永久性地驱逐（evict）部分 token。
- **现有方法局限**：当前所有 KV 缓存驱逐方法（如 SnapKV、H2O、MorphKV 等）均采用相同的模板：先计算每步的“直接注意力分数”，再通过**确定性 top-K 选择**保留分数最高的 K 个 token。这种设计将单步低于阈值的分数视为不可逆的判决，导致**暂时性分数波动**（分数仅在低于阈值一步）的 token 被永久删除，而实际上该 token 在长期平均重要性上可能很高。作者将这种现象称为**单调边际侵蚀**（monotone marginal erosion）。
- **整体含义**：论文指出，在流式、固定预算的驱逐场景下，需要从根本上改变选择原语（selection primitive），而非仅改进评分函数。核心在于：**驱逐决策应具有概率性而非确定性**，以避免单步错误导致的不可逆损失。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：提出 **Nexus Sampling**，一种无需训练的 KV 缓存驱逐方法，包含两个创新组件：
  1. **Nexus 评分**（Nexus scoring）：通过短迭代游走（iterative walk）在直接注意力分数上揭示“桥接 token”（bridge token）——那些单独不被任何近期查询强烈关注，但维系着强连接簇的 token。
  2. **加权蓄水池采样**（weighted reservoir sampling）：以与 Nexus 分数成正比的包含概率（inclusion probability）选择保留的 K 个 token，取代传统的确定性 top-K。

- **关键技术细节**：
  - **粒度和窗口**：以 key 块（block size b = 32）为单位；使用长度为 W（默认 16）的近期查询观察窗口计算每块的基准分数 a（窗口内每行注意力分布的平均）。
  - **Nexus 评分**：每块 j 的最终分数 s_j = a_j + λ·c̃_j，其中 a_j 是直接重要性（窗口平均），c̃_j 是桥接分数（通过迭代游走获得）。游走初始状态 C=0，对每个窗口查询行 a^(q)，更新 C ← C + γ_q·a^(q)，其中γ_q = 1 + ⟨a^(q), C⟩。游走 H 步（默认 3）后归一化得到 c̃。
  - **加权蓄水池选择**：给定保留预算 K，对每块 j 生成 n 个独立均匀随机数 u_j^(i) ~ U(0,1)，计算平均优先级π_j = (1/n) Σ_i (u_j^(i))^(1/w_j)，然后按π_j 取 top-K。其中 w_j 是最终权重（附带很小的近期性纠偏）。无条件保留注意力沉没块和近期窗口块。
  - **理论保证**：
    - 确定性 top-K 的长期存活概率是“min over steps”，一旦某步低于阈值则为 0；蓄水池采样的存活概率是“product over steps”，每步包含概率严格为正时，长期存活概率仍正。
    - 游走可理论上放大“集线器块”（hub blocks）的分数，高于直接窗口平均。
    - 端到端驱逐质量界：蓄水池无偏，剩余误差由权重对真实未来效用的逼近程度控制。

## 3. 实验设计：数据集、场景、benchmark、对比方法

- **数据集与任务**：
  - **LongBench**：16 个任务，涵盖单文档/多文档 QA、摘要、少样本任务等。
  - **RULER**：合成诊断基准，测试检索和位置敏感推理，上下文长度 4K–64K。
  - **Agentic Coding**：从 SWE-bench 采样的 50 个任务，使用 DeepSWE-Preview 模型（基于 Qwen3-32B）在 R2E-Gym 环境中评估多轮编码能力。

- **场景设置**：
  - **Prefill-only**：仅在预填充阶段进行驱逐。
  - **Prefill + Decode**：在整个推理流（预填充 + 解码）中持续驱逐。
  - 进一步区分“每个回合全上下文再预填充”（可恢复已驱逐 token）与“真正驱逐”（缓存跨回合持久，驱逐即永久丢失）。

- **对比方法**：

  - 预填充+解码阶段对比：H2O、MorphKV

- **模型**：Llama-3.1-8B-Instruct、Llama-3.2-1B-Instruct、Qwen3-8B（均支持最长 128K 上下文）。

- **实现细节**：基于 HuggingFace Transformers，块大小 b=32，窗口 W=16，游走深度 H=3，混合权重 λ=0.5，蓄水池平均次数 n=5。使用 FlashAttention-2 内核。
