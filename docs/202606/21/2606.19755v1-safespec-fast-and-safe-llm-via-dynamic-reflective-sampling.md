## 摘要
推测推理加速了大语言模型的解码，但本身不提供安全保障。现有的安全防御措施与推测推理大多不兼容：它们要么引入额外计算，要么破坏草稿-验证机制，从而抵消加速效果。这揭示了当前安全方法与推测解码之间的根本不兼容性。我们提出SafeSpec，一个安全感知的推测推理框架，将风险估计直接整合到验证过程中。SafeSpec在目标模型上附加一个轻量级的潜在安全头，以在单次前向传播中联合评估语义有效性和安全性。当检测到不安全生成时，SafeSpec应用回滚和安全引导的反射式多采样来恢复安全的延续，而不是终止生成。我们将越狱攻击建模为生成轨迹上的分布偏移，其中对抗性提示增加了有害延续的概率，但不消除安全的。在此模型下，SafeSpec在推测解码过程中执行风险感知的轨迹恢复。在多个模型和对抗性基准测试中，SafeSpec实现了显著改善的安全性与效率权衡。在Qwen3-32B上，SafeSpec将攻击成功率降低15%，同时在良性工作负载上保持2.06倍的推理加速，表明推测加速和推理时安全性可以联合优化。

## Abstract
Speculative inference accelerates large language model (LLM) decoding but provides no inherent safety guarantees. Existing safety defenses are largely incompatible with speculative inference: they either introduce additional computation or disrupt the draft-verify mechanism, negating acceleration benefits. This reveals a fundamental incompatibility between current safety methods and speculative decoding. We propose SafeSpec, a safety-aware speculative inference framework that integrates risk estimation directly into the verification process. SafeSpec attaches a lightweight latent safety head to the target model to jointly evaluate semantic validity and safety in a single forward pass. When unsafe generations are detected, SafeSpec applies rollback and safety-guided reflective multi-sampling to recover safe continuations rather than terminating generation. We model jailbreak attacks as distributional shifts over generative trajectories, where adversarial prompts increase the probability of harmful continuations without eliminating safe ones. Under this model, SafeSpec performs risk-aware trajectory recovery within the speculative decoding process. Across multiple models and adversarial benchmarks, SafeSpec achieves a substantially improved safety-efficiency trade-off. On Qwen3-32B, SafeSpec reduces attack success rates by 15% while preserving a 2.06x inference speedup on benign workloads, demonstrating that speculative acceleration and inference-time safety can be jointly optimized.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：推测推理（Speculative Inference）通过“草稿模型生成候选 + 目标模型验证”的机制显著加速大语言模型（LLM）解码，已成为现代推理管线默认组件。
- **核心问题**：推测推理本身不提供任何安全保证；现有安全防御（如提示级防御 Self-Reminder、RPO，或解码级干预 SafeDecoding、SecDecoding）与推测推理框架存在**根本性的不兼容**：它们要么引入额外计算（如并行安全模型），要么破坏草稿-验证机制（如干扰 token 概率或额外转发），最终抵消加速收益。
- **整体目标**：设计一种**内生于推测解码流程**的安全机制，在保证加速效果的同时提供有效的推理时安全性，实现安全与效率的联合优化。

## 2. 方法论

### 核心思想
在目标模型的验证过程中同时评估候选片段的质量和安全性，当检测到不安全时，不直接终止生成，而是通过回滚、反射提示和基于安全引导的多采样，从模型自身生成空间中**恢复**安全且连贯的续写。

### 关键技术细节

1. **轻量级安全头（Safety Head）**
   - 架构：一个两层 MLP（d → d/2 → 1），接 Sigmoid 输出安全分数 S_safe ∈ [0,1]。
   - 输入：从目标模型最终层的**边界对齐位置**（候选片段最后一个 token）提取隐藏状态 z，避免质量提示干扰语义。
   - 训练：使用 Qwen3Guard-Gen-8B 自动标注的步骤级安全标签，最小化二分类交叉熵损失，冻结目标模型骨干仅优化安全头参数（约 13M ~ 33M，占骨干 < 0.05%）。

2. **双层验证机制**
   - **安全优先**：若 S_safe < 阈值 τ_s，立即触发安全模式。
   - **质量验证**：若安全通过，进一步使用类似 Specreason 的方法（通过 prompt 让目标模型输出 0-9 整数分）评估质量；若质量分 S_q ≥ τ_q 则接受，否则由目标模型重新生成该步骤。

3. **安全引导的反射式多采样（Safety Mode）**
   - **回滚（Rollback）**：不安全检测后，退回到前一个安全状态，避免轨迹继续恶化。
   - **反射提示（Reflection Prompt）**：插入一段提示，强制模型重新评估意图，破坏对抗性上下文的惯性，将概率质量重新分布到安全轨迹。
   - **多采样（Multi-Sampling）**：草稿模型生成 K 个候选步骤，目标模型重计算每个候选的安全分数，选择最高分的作为最终输出。
   - **概率建模**：越狱攻击仅将概率质量转移到有害轨迹，但安全轨迹仍具有非零概率 p_safe；通过 K 次采样，至少找到一个安全轨迹的概率为 P_safe(K) = 1 - (1-p_safe)^K。反射提示可提高 p_safe，减少所需 K。

## 6. 主要结论与发现

1. **安全性优势**：SafeSpec 在两个模型上均取得最低平均 ASR（Qwen3-32B 为 7%，DeepSeek-70B 为 7%），显著优于最强基线 SecDecoding（22% 和 53%）。

3. **保持推理加速**：在良性负载上，Qwen3-32B 实现 2.06× 加速，DeepSeek-70B 实现 1.76× 加速，接近纯加速方法 Specreason（2.14×、1.81×），远优于安全解码方法（0.61×~0.82×）。
4. **通用能力几乎无损**：GSM8K、MATH、GPQA 准确率与无防御基线相当，说明安全干预未破坏模型核心推理能力。
5. **各组件必要性**：消融证明，反射提示和多采样均不可或缺；硬拒绝虽能完全阻止攻击，但导致过拒激增，不可实用。

## 7. 优点

- **架构创新**：将安全头直接嵌入目标模型的前向传播中，利用中间隐藏层的线性可分性，以极低参数开销实现零额外计算的安全检查。
- **软拒绝策略**：用“回滚+反射+多采样”替代传统硬终止，在纠正不安全轨迹的同时保留模型对良性查询的响应能力，实用性强。
- **概率理论支撑**：为多采样的有效性提供显式概率公式，并解释了反射提示如何提高 p_safe。
- **实验全面性**：覆盖两个不同模型家族、7 种多样化越狱攻击、多项消融、超参分析，并且与防护模型等行业方案对比，验证了实际部署价值。
- **开源代码**：论文提供 GitHub 代码仓库，可复现结果。

## 8. 不足与局限

- **模型依赖性**：安全头需针对每个目标模型训练（虽仅需少量数据和轻量头），且依赖其内部表示的线性可分性假设，可能不适用于所有架构。
- **对抗性输入下的效率下降**：在越狱输入上安全模式频繁触发（平均每秒 3.81 次），导致加速比降至 0.87×（Qwen3-32B）。作者辩护为“防御正常工作的代价”，但实际部署中若攻击流量占比高，整体吞吐可能下降。
- **攻击覆盖有限**：仅测试 7 种越狱方法，未覆盖像多轮对话攻击、隐写式攻击、自适应攻击等新兴威胁。
- **超参数敏感**：安全阈值 τ_s 需谨慎选择，过高导致过拒上升（图 4b），过低则安全不足，实际部署中可能需要动态调优。
- **训练数据偏差**：安全头使用 Qwen3Guard-Gen-8B 自动标注，该模型本身可能存在误判，引入的噪音可能影响分类边界；且仅在单一数据集（AdvBench + 部分 XSTest）上训练，未验证对域外场景的泛化性。
- **未在更大规模模型验证**：最大目标模型为 70B，未测试 130B+ 的适用性；安全头的参数开销会随隐藏层维数增大而增加（33M on 70B），但相对骨干仍可忽略。
- **质量验证 prompt 设计**：使用特定评分 prompt 可能引入 prompt 敏感性问题，不同 prompt 模板可能导致质量分数波动。
