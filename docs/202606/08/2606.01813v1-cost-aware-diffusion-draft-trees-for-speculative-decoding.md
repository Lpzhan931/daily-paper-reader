## 摘要
推测解码通过让轻量级草稿器生成候选词，并由目标语言模型并行验证来加速推理。块扩散草稿器（如DFlash）在一次前向传播中生成整个草稿块，得到每个位置的边际分布；DDTree利用这些边际分布构建候选树，在固定节点预算下最大化期望接受长度。然而，我们观察到接受长度随预算单调不减：无论验证成本如何，它总是倾向于更大的树，这为预算选择提供了缺乏原则基础的依据。我们提出CaDDTree（成本感知扩散草稿树），一种通过联合选择树结构和节点预算来直接优化令牌吞吐量（单位时间内期望生成的令牌数）的方法。我们显式建模草稿和验证延迟，证明吞吐量目标可分解为每轮对预算的一维搜索，并证明在凸验证成本下吞吐量函数为单峰函数，从而可实现高效的贪心停止规则。CaDDTree无需离线预算搜索，而是根据当前每轮的位置分布和验证成本自适应调整预算。在Qwen3-4B和Qwen3-8B上，涵盖推理、编程和指令跟随任务的八个基准测试的实验表明，CaDDTree在几乎所有任务上匹配或超越了使用最优预算选择的DDTree。

## Abstract
Speculative decoding accelerates inference by having a lightweight drafter propose tokens verified in parallel by the target language model. Block diffusion drafters such as DFlash generate an entire draft block in one pass, yielding per-position marginals; DDTree uses these to build a candidate tree that maximizes expected acceptance length under a fixed node budget. We observe, however, that acceptance length is non-decreasing in budget: it always favors larger trees regardless of verification cost, offering no principled basis for budget selection. We introduce \textbf{CaDDTree} (Cost-aware Diffusion Draft Tree), a method that directly optimizes token throughput (expected tokens generated per unit time) by jointly selecting the tree structure and node budget. We model draft and verification latencies explicitly, show that the throughput objective decomposes into a per-round one-dimensional search over the budget, and prove that under a convex verification cost the throughput function is \emph{unimodal}, enabling an efficient greedy stopping rule. CaDDTree requires no offline budget search, adapting the budget each round from the current per-position distributions and verification cost. Experiments on Qwen3-4B and Qwen3-8B across eight benchmarks spanning reasoning, coding, and instruction-following tasks show that \caDDTree{} matches or surpasses DDTree with oracle budget selection on nearly all tasks.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：推测解码（Speculative Decoding）中，草稿树（draft tree）的构建需要在增加节点数（提高覆盖率）与验证开销之间权衡。现有方法如 DDTree 在**固定节点预算**下最大化期望接受长度（Expected Acceptance Length），但接受长度随预算单调不减，导致总是倾向于更大的树，忽视验证成本，无法合理选择预算。
- **背景**：块扩散草稿器（如 DFlash）在一次前向传播中生成整个草稿块的每位置边际分布；DDTree 利用这些边际分布构建候选树，但预算需离线网格搜索，且不能适应每轮（per-round）的置信度变化和上下文长度增长。
- **动机**：直接优化**令牌吞吐量**（Tokens per unit time），显式建模草稿和验证延迟，从而自适应选择树结构和节点预算。

## 2. 方法论

### 核心思想
- 将吞吐量代理目标定义为：  
  $$\hat{\theta}(T; \ell) = \frac{1 + \Phi(T)}{C_d + C_v(|T|; \ell)}$$  
  其中 \(\Phi(T)\) 是期望接受长度（基于草稿器联合分布 Q 计算），\(C_d\) 是草稿计算成本，\(C_v(n; \ell)\) 是验证 n 个节点、上下文长度 ℓ 的验证成本。
- 证明该目标可分解为：先按前缀概率排序得到最优树 \(T^*_n\)（固定规模 n 下最优），然后对 n 进行一维搜索。
- **单峰性定理**：在验证成本关于节点数 n 为凸函数（边际成本非递减）的假设下，\(\hat{\theta}^*(n; \ell)\) 是单峰函数。因此，首次遇到吞吐量下降的点即为全局最优预算 \(n^*\)。

### 关键技术细节
- **贪婪停时规则**：使用最大堆按前缀概率弹出节点，每次弹出后计算新吞吐量 \(\hat{\theta}_\text{new}\)。若 \(\hat{\theta}_\text{new} \leq \hat{\theta}_\text{prev}\) 则停止，否则继续扩展节点（包括插入下一个兄弟节点和第一个子节点）。
- **离线成本建模**：对每个（模型，硬件）组合，离线测量不同节点数（1~1024）和不同上下文长度（0~8k tokens）下的验证延迟，用凸回归拟合 \(C_v(n; \ell)\)，保证凸性和单调性。草案成本 \(C_d\) 取一次草稿前向传播的中位延迟。
- **运行时成本插值**：当前上下文长度 ℓ 不在测量网格上时，线性插值相邻网格点，继承凸性。

### 算法流程（文字说明）
1. 输入：每位置边际分布（top-K 令牌及对数概率）、当前上下文长度 ℓ、草稿成本 \(C_d\)、验证成本函数 \(C_v\)。
2. 初始化空树、\(\Phi=0\)、节点数 n=0、吞吐量 \(prev = 1/(C_d + C_v(0; \ell))\)；初始化最大堆，压入第一个节点的前缀。
3. 循环：
   - 弹出前缀概率最大的节点 s，计算 δ。
   - 计算新吞吐量 `new = (1 + Φ + δ) / (Cd + Cv(n+1; ℓ))`。
   - 若 `new <= prev`，停止；否则将节点加入树，更新 Φ、n、prev。
   - 压入 s 的下一个兄弟节点（若存在）和第一个子节点（若深度 < 块大小 L）。
4. 返回构建的树。
