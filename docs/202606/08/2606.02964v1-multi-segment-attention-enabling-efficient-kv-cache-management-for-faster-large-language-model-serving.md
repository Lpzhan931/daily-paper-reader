## 摘要
大型语言模型（LLM）推理依赖键值（KV）缓存来避免冗余注意力计算。近似KV缓存保留技术通过牺牲模型精度来减少内存使用，而无损方法则将KV缓存块从GPU内存中逐出，并在需要时重建以保持精确输出。现有的无损KV缓存管理系统主要基于访问频率或位置启发式做出逐出决策，而不考虑不同KV缓存块如何影响GPU注意力内核的执行效率。本文提出AsymCache，一种面向LLM推理的计算延迟感知KV缓存管理系统，明确将缓存驻留决策与GPU注意力内核性能对齐，包含三个关键组件：用于高效非连续KV上下文处理的多段注意力（MSA）；联合优化命中率和位置感知重计算成本的缓存逐出策略；以及用于高硬件利用率的自适应分块调度器。实验表明，与最新基线相比，AsymCache将TTFT降低高达1.90-2.03倍，每输出token时间（TPOT）降低1.62-1.71倍，证实了该方法在常见工作负载中的有效性，并验证了其平衡计算效率与缓存命中率的设计目标。此外，AsymCache的低级设计允许无缝集成到Continuum等代理服务系统中，进一步将平均作业延迟降低多达18.1%。

## Abstract
Large Language Model (LLM) inference relies on key-value (KV) caches to avoid redundant attention computation. While approximate KV cache retention techniques reduce memory usage by sacrificing model accuracy, lossless approaches instead evict KV cache blocks from GPU memory and reconstruct them on demand to preserve exact outputs. Existing lossless KV cache management systems primarily base eviction decisions on access frequency or positional heuristics, without considering how different KV cache blocks affect the execution efficiency of GPU attention kernels.   In this paper, we propose AsymCache, a computation-latency-aware KV cache management system for LLM inference that explicitly aligns cache residency decisions with GPU attention kernel performance, including three key components: Multi-Segment Attention (MSA) for efficient non-contiguous KV context processing, a cache eviction policy that jointly optimizes hit rate and position-aware recomputation cost, and an adaptive chunking scheduler for high hardware utilization. Experiments show that AsymCache reduces TTFT by up to 1.90-2.03x and time-per-output-token (TPOT) by 1.62-1.71x over latest baselines, confirming the effectiveness of the method in common workloads and validating its design goal of balancing computational efficiency with cache hit rate. Moreover, the low-level design of AsymCache allows seamless integration into agent serving systems such as Continuum, where it further reduces average job latency by up to 18.1%.

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：KV 缓存块在 GPU 注意力核中的计算代价不对称（靠后位置的块重计算代价更高），而不同块的未来复用概率也不同。驱逐时应选择“期望延迟贡献最小”的块，即联合优化复用概率与重计算代价。
- **三大组件**：
  1. **Multi-Segment Attention (MSA) 核**：支持非连续 KV 上下文（例如同时缓存前缀和后缀），将多个请求的多个段在单个 GPU 核内并行计算，避免多核启动开销。通过将不同请求分派到不同 CTA 组实现。
  2. **计算感知块驱逐器（Computational-Aware Block Evictor）**：定义每个块的期望驱逐代价 \( E(B,t) = f_B(t) \cdot \Delta T_B \)，其中 \( f_B(t) \) 是频率值（分段指数函数），\( \Delta T_B \) 是重计算代价增量（通过线性模型近似 \( \Delta T_B \approx 2k_5(l_1+q_1)+(k_2-k_3+k_5) \)，只依赖位置索引）。驱逐时选择 \( \arg\min_B E(B,t) \)。
     - 频率函数采用**分段指数函数** \( f_B(t) = \min(\exp(-\tau_B(t)/\alpha), \exp(-(\tau_B(t)-\tau_0)/\beta)) \)，满足序保持性（仅指数函数满足），从而可用两棵平衡树实现 \( O(\log n) \) 驱逐，而非线性 \( O(n) \)。
  3. **自适应分块调度器（Chunking Scheduler）**：将长预填充请求分裂为小分块（结合 chunked-prefill），且允许分块横跨计算段与缓存段，进一步提升 GPU 利用率和并发度。在线可调节系数 λ 以动态适应工作量寿命分布变化。

- **算法流程**（文字描述）：收到新请求时，块管理器检查缓存命中情况；若需要分配新块但 GPU 内存不足，则调用驱逐器：计算各缓存块的 \( E(B,t) \)，从两个平衡树中分别获取最小权重，取较小者作为驱逐目标。驱逐后分配新块，请求元数据记录多段非连续缓存信息，调度器为该预填充请求分配分块大小，MSA 核执行批量注意力计算。

## 6. 主要结论与发现
- AsymCache 在几乎所有配置下优于基线：TTFT 降低 **1.90–2.03×**，TPOT 降低 **1.62–1.71×**（相比最新基线）。

## 7. 优点
- **方法创新**：首次将 GPU 注意力核性能（期望延迟）纳入缓存驱逐决策，提出位置感知的重计算代价估计。
- **高效算法**：证明只有指数函数满足序保持性，设计分段指数函数使驱逐复杂度从 O(n) 降为 O(log n)，实际控制平面开销仅 ~2ms/请求（0.1% 预填充时间）。
- **通用性**：系统设计不针对特定工作负载，可自动适应前缀/后缀或两端的缓存模式；且 MSA 核兼容 chunked-prefill，可集成到现有系统（如 vLLM、Continuum）。
- **实验扎实**：包含多种场景、模型规模、负载参数，消融和超参敏感性分析验证了各组件及参数鲁棒性。

## 8. 不足与局限
- **数据集局限**：仅使用 LongBench 和 LooGLE（两者结构相似），未在更长的（>100K）上下文或不同领域（如代码、多模态）上评估。请求数量 300 个偏少，可能无法完全反映真实生产环境中的多样缓存模式。
- **偏差风险**：使用的是预先生成的固定输出长度，消除了采样随机性，但可能掩盖解码长度变化对系统的影响。
- **应用限制**：当前仅支持 GPU 内存缓存，不支持分层存储（如 CPU DRAM、磁盘）；如论文讨论，多级存储时驱逐策略需差异化，尚未实现。
- **理论分析**：序保持性证明虽然严谨，但实际中频率值采用分段指数逼近真实访问模式，最优性缺乏严格保证。
- **资源消耗**：虽未提及训练，但推理时线性模型拟合开销（1.1K 实测点）和平衡树维护成本虽低，仍为额外开销。
