## 摘要
长上下文和代理LLM工作负载将KV缓存推至任何固定内存预算之外，迫使推理堆栈在连续推理流的每一步永久驱逐令牌。现有方法均遵循相同模板：每步直接注意力分数后跟确定性top-K选择，这将单步低于阈值的转换变为不可逆的判决，并永久擦除任何直接注意力无法从噪声中区分出的微妙重要令牌。为解决这一挑战，我们提出Nexus采样，一种无需训练的驱逐方法，将Nexus评分（一种迭代遍历直接注意力的方法，揭示桥接令牌）与加权水库采样（以包含概率保留令牌代替确定性top-K）配对。理论上，我们证明Nexus采样在微妙重要令牌的长期生存方面优于确定性top-K。实证上，在80% KV缓存驱逐下，Nexus采样在LongBench上与密集注意力匹配在1%以内，同时在检索密集型任务上优于top-K基线，每序列缓存内存最多减少10倍。

## Abstract
Long-context and agentic LLM workloads push the KV cache past any fixed memory budget, forcing the inference stack to permanently evict tokens at every step of a continuous-inference stream. Existing methods all share the same template, a per-step direct-attention score followed by deterministic top-$K$ selection, which converts a single below-cutoff step into an irreversible verdict and permanently erases any subtly important token that direct attention cannot single out from noise. To address this challenge, we propose Nexus Sampling, a training-free eviction method that pairs Nexus scoring, an iterative walk over direct attention that surfaces bridge tokens, with weighted reservoir sampling, which retains tokens with inclusion probability in place of deterministic top-$K$. Theoretically, we show that Nexus Sampling dominates deterministic top-$K$ in long-run survival of subtly important tokens. Empirically, at 80% KV cache eviction, Nexus Sampling matches dense attention within 1% on LongBench while outperforming top-$K$ baselines on retrieval-heavy tasks, with up to 10x smaller per-sequence cache memory.

### 论文核心分析与总结

#### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在长上下文和大规模语言模型（LLM）的推理过程中，**Key-Value (KV) 缓存**会随着上下文长度线性增长，迅速消耗有限的 GPU 内存。当缓存超过内存预算时，推理系统必须**永久性地驱逐（evict）部分 token**。现有的大多数 KV 缓存驱逐方法都采用一个共同的模式：**每步计算直接注意力分数，然后进行确定性 top-K 选择**。
- **现有方法的根本缺陷**：论文指出，这种“确定性 top-K”方法在处理流式数据时存在两个致命问题：
    1.  **不可逆判决**：它将任何一步中因噪声或查询波动而暂时得分低的 token 永久清除，即使该 token 在其后续步骤中可能变得至关重要。
    2.  **遗漏“桥接令牌”**：那些被多个查询间接关注、在上下文结构中起到连接作用的关键 token（桥接令牌），往往被单步直接注意力所忽视。
- **整体含义**：本文旨在将 KV 缓存驱逐问题从一个简单的“稀疏注意力”问题，重新定义为**流式固定预算下的采样问题**。作者认为，驱逐的决策必须考虑到**长期生存率**和**间接重要性**，而不是只看一步的得分。

#### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出 **Nexus Sampling** 方法，通过两个核心创新来替换传统的确定性 top-K 驱逐范式：
    1.  **从“确定性选择”变为“概率采样”**：用**加权蓄水池采样 (Weighted Reservoir Sampling)** 替代确定性 top-K，使得每个 token 的保留概率与其权重成正比，避免了一刀切的不可逆判决。
    2.  **从“直接重要性”变为“间接重要性”**：通过**Nexus 评分 (Nexus Scoring)** 来发现“桥接令牌”，使得采样权重能反映 token 的结构性角色，而不仅仅是当前查询的直接关注度。

- **关键技术细节**：
    1.  **Nexus 评分**：
        - 首先，计算一个窗口内最近查询的`基础得分( a )`。
        - 然后，通过一个**迭代行走 (Iterative Walk)** 更新一个`桥接得分( ĉ )`。这个行走过程可以理解为在 token 的注意力关系图中传播分数，从而识别出那些连接不同簇的“枢纽 (hub)”或“桥接 (bridge)”token。
        - 最终的**采样权重( w )** 是基础得分、桥接得分和一个小型最近偏置项的线性组合：`w = a + λĉ + ε_tie`。
    2.  **加权蓄水池采样**：
        - 对于每个候选 token，根据其权重 `w` 生成一个随机优先级 `π`。具体公式为：`π_j = (1/n) * Σ (u_j^(i))^(1/w_j)`，其中 `u` 是 (0,1) 均匀分布的随机数，`n` 是平均次数。
        - 具有最高 `π` 值的 `K` 个 token 被保留。这个机制在 `n=1` 时等同于标准的加权蓄水池采样，保证了概率性保留；在 `n→∞` 时趋向于确定性 top-K，为两种策略提供了统一的框架。

- **算法流程**：
    1.  在每个驱逐步骤，系统获取最新的查询窗口。
    2.  计算这些查询对缓存中所有 token 的直接注意力得分。
    3.  **Nexus 评分**：通过迭代行走算法处理注意力得分，得到包含了直接和间接重要性的综合权重 `w`。
    4.  **加权蓄水池采样**：基于权重 `w` 计算每个 token 的优先级 `π`。
    5.  选择优先级最高的 `K` 个 token 保留，其余永久丢弃。
